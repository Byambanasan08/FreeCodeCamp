# SQL-ийн үндсэн өгөгдлийн төрлүүд (Data Types) юу вэ?

SQL-д хүснэгт тодорхойлохын тулд багана бүрийн өгөгдлийн төрлийг зааж өгөх шаардлагатай.
SQL-д хүснэгт үүсгэх үндсэн синтакс (syntax) ийм байдгийг санаарай. Багана бүр өөрийн гэсэн өгөгдлийн төрөлтэй байна.

```sql
CREATE TABLE table_name(
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ... гэх мэт);

```

Хөгжүүлэгчийн хувьд таны зорилго бол эдгээр өгөгдлийн төрлүүдийг аль болох оновчтой сонгох явдал юм.
SQL-д өгөгдлийн төрлийн зургаан нийтлэг ангилал байдаг:

    - Тоон төрөл (Numeric): INTEGER, FLOAT, SERIAL, болон DECIMAL гэх мэт.
    - Огноо ба цаг (Date and time): TIMESTAMP, DATE, болон TIME гэх мэт.
    - Тэмдэгт ба мөр (Character and string): Үүнд CHAR, VARCHAR, болон TEXT багтана.
    - Юникод (Unicode): NTEXT болон NVARCHAR багтана. Эдгээр нь тэмдэгтүүдийн гарал үүслээс үл хамааран текстийг зөв хадгалах, гаргаж авахад ашиглагддаг.
    - Бинар (Binary): Зураг, аудио, видео файл зэрэг текст бус өгөгдлийг хадгалахад ашиглагддаг.
    - Бусад холимог төрлүүд: XML болон TABLE гэх мэт.

Энэ хичээлээр бид `PostgreSQL` дээр хамгийн өргөн хэрэглэгддэг зарим төрлүүдийг авч үзэх болно. `Өгөгдлийн төрлүүд` нь өгөгдлийн сангийн удирдлагын системээсээ хамааран өөр өөр байж болох ч ерөнхийдөө нэлээд төстэй байдаг. Тиймээс та баримт бичиг (documentation) дэх бүрэн жагсаалтыг шалгах шаардлагатай.
Бид тоон утгуудаас эхэлнэ.

1. Эхнийх нь **INTEGER** өгөгдлийн төрөл:

```sql
units_sold INTEGER

```

Энэ жишээнд бид units_sold баганад энэхүү өгөгдлийн төрлийг оноож байна.
`INTEGER` төрөл нь өгөгдлийн санд 4 байт эзэлдэг бөгөөд -2,147,483,648-аас 2,147,483,647 хүртэлх утгыг авч чадна. `PostgreSQL`-ийн албан ёсны баримт бичигт үүнийг **бүхэл тооны ердийн сонголт** гэж тодорхойлсон байдаг.
Мөн `SMALLINT` болон `BIGINT` төрлүүд байдаг бөгөөд эдгээр нь өгөгдлийн санд хуваарилагдсан хэмжээнээсээ хамааран харгалзах бага болон их тооны мужийг агуулдаг гэдгээрээ `INTEGER`-тэй үндсэндээ ижил юм.

2. `PostgreSQL`-д байдаг ашигтай боломж бол **SERIAL** түлхүүр үгийг ашиглан багана үүсгэх явдал юм. Энд id баганыг `SERIAL` ашиглан үүсгэсэн байна:

```sql
id SERIAL

```

Хэдийгээр энэ нь жинхэнэ өгөгдлийн төрөл биш боловч цор ганц таних тэмдэг бүхий (unique identifier) багана үүсгэхэд маш тустай. Учир нь тухайн багана нь `INTEGER` төрөлтэй байх бөгөөд хоосон (NULL) утгыг зөвшөөрөхгүй, мөр нэмэх үед автоматаар өсөх болно.
Жишээлбэл, эхний мөр автоматаар 1 гэсэн id-тай байх ба хоёр дахь мөр 2 гэсэн id-тай байх гэх мэтээр бичлэг бүрт давтагдашгүй ID үүсгэнэ.
**MySQL**-д `SERIAL`-тай дүйцэхүйц зүйл нь `AUTO_INCREMENT` атрибут бөгөөд дараалсан бүхэл тоонуудыг автоматаар үүсгэхэд ашиглагддаг:

```sql
id INT AUTO_INCREMENT

```

Өөр өөр өгөгдлийн сангийн удирдлагын системүүд яг ижил функцийг өөр өөр аргаар хэрэгжүүлж болдгийг та эндээс харж болно.
Эдгээр нь хамгийн түгээмэл хэрэглэгддэг тоон өгөгдлийн төрлүүд юм. Гэвч заримдаа та текст эсвэл тэмдэгтүүдийн дарааллыг илэрхийлэх шаардлагатай болдог.

3. Үүнд зориулж **VARCHAR** өгөгдлийн төрөл байдаг. Энэхүү өгөгдлийн төрөл нь хувьсах урттай тэмдэгт мөрөнд ашиглагддаг. Та хаалтан дотор тэмдэгтийн дээд хэмжээг зааж өгч болно:

```sql
name VARCHAR(50)

```

Энэ жишээнд бид name баганыг тодорхойлсон бөгөөд утгууд нь 50 хүртэлх тэмдэгтийн урттай байж болно.
Энэ төрөл нь мөрүүдийн дээд хязгаарыг тогтоодог. Харин ямар ч урттай тэмдэгт мөр хадгалах шаардлагатай бол `TEXT`-ийг ашиглаж болно:

```sql
name TEXT

```

4. Тоо болон тэмдэгт мөрөөс гадна огноо, цагийг хадгалах нь түгээмэл байдаг.
   Жишээлбэл, хэрэв та арга хэмжээ хадгалах хүснэгт үүсгэвэл арга хэмжээ бүрийн огноог тодорхой форматаар хадгалах шаардлага гарч магадгүй. Үүнд зориулж DATE өгөгдлийн төрөл байдаг:

```sql
event_date DATE

```

Цагийг хадгалахын тулд та TIME өгөгдлийн төрлийг ашиглаж болно. Жишээлбэл, та арга хэмжээ эхлэх цагийг хадгалж болох юм:

```sql
start_time TIME

```

`TIMESTAMP` өгөгдлийн төрөл нь энэ хоёрыг нэгтгэдэг. Энэ нь огноо болон цагийг хоёуланг нь агуулдаг. Хэрэв та `TIMESTAMP WITH TIME ZONE` гэж бичвэл цагийн бүсийг (time zone) мөн багтааж болно:

```sql
event_timestamp TIMESTAMP
event_timestamp TIMESTAMP WITH TIME ZONE

```

5. Эцэст нь, хэрэв та логик утга _boolean_ буюу `TRUE` эсвэл `FALSE` хадгалах шаардлагатай бол `BOOLEAN` өгөгдлийн төрлийг ашиглаж болно:

```sql
is_active BOOLEAN

```

Одоо та SQL-ийн хамгийн нийтлэг зарим өгөгдлийн төрлүүдийг мэддэг боллоо. Гэвч үүнээс гадна маш олон төрөл бий. Та өөрийн ашиглаж буй өгөгдлийн сангийн удирдлагын системийн баримт бичгийг шалгаж, ашиглах боломжтой өгөгдлийн төрлүүдийн талаар дэлгэрэнгүй мэдээлэл авах боломжтой.
