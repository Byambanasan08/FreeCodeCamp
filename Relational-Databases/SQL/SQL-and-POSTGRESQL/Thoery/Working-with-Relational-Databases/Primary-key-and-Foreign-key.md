# SQL-ийн анхдагч түлхүүр (Primary Key) болон гадаад түлхүүр (Foreign Key) гэж юу вэ, тэдгээр нь хэрхэн ажилладаг вэ?

SQL-д `анхдагч түлхүүр` болон `гадаад түлхүүрийг` хүснэгтүүдийн хооронд `харилцаа холбоо(relationship)` тогтооход ашигладаг.

1. Анхдагч түлхүүрээс (primary keys) эхэлцгээе.
   Анхдагч түлхүүр нь хүснэгт дэх бичлэг (мөр) бүрийг цор ганц байдлаар тодорхойлдог багана эсвэл багануудын багц юм. Энэхүү хязгаарлалт (constraint) нь хүснэгт дэх ямар ч бичлэг анхдагч түлхүүрийн хувьд ижил утгатай байхгүйг баталгаажуулдаг.
   Ийм учраас хүснэгт нь зөвхөн нэг л анхдагч түлхүүртэй байх боломжтой.
   Анхдагч түлхүүрийн багана дахь утгууд нь хоосон (NULL) байж болохгүй тул тэдгээр нь үргэлж хүчинтэй утгатай байх болно.
   PostgreSQL-д баганыг хүснэгтийнхээ анхдагч түлхүүр болгохын тулд өгөгдлийн төрлийн ард PRIMARY KEY гэж нэмэхэд л хангалттай:

```sql
column_name data_type PRIMARY KEY

```

Энэ бол оюутнуудын хүснэгтийн (students table) жишээ юм:

```sql
CREATE TABLE students (
  student_id SERIAL PRIMARY KEY,
  name VARCHAR(100));

```

Оюутан бүр өөрийн гэсэн student_id-тай байх бөгөөд энэхүү ID нь хүснэгтийн анхдагч түлхүүр байх болно. `SERIAL` төрөл нь энд маш хэрэгтэй, учир нь энэ нь student_id-г үргэлж давтагдашгүй утгатай байхыг баталгаажуулдаг.
Нийлмэл анхдагч түлхүүр (composite primary key) нь хүснэгтэд мөрийг тодорхойлох цор ганц бие даасан багана байхгүй үед ашиглагддаг. Энэ тохиолдолд та хоёр болон түүнээс дээш багануудын хослолыг анхдагч түлхүүр болгон ашиглаж болох бөгөөд тэдгээр нь хамтдаа давтагдашгүй байх болно.
Үүнийг хийхийн тулд хүснэгт үүсгэхдээ аль баганууд нь нийлмэл анхдагч түлхүүр болохыг дараах байдлаар тэмдэглэнэ:

```sql
CREATE TABLE table_name (
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ...
  PRIMARY KEY (column1, column2));

```

Танд оюутнууд ямар хичээлүүдэд бүртгүүлсэн байгааг харуулсан student_id болон course_id багана бүхий хүснэгт байна гэж төсөөлөөд үз дээ.
Оюутан бүр хэд хэдэн хичээлд бүртгүүлэх бөгөөд хичээл бүр хэд хэдэн оюутантай байх болно. Тиймээс аль ч багана нь дангаараа давтагдашгүй биш юм - гэхдээ нэг оюутан нэг хичээлд нэгээс олон удаа бүртгүүлэхгүй. Иймээс та хоёр баганын хослолыг анхдагч түлхүүр болгон ашиглаж болно:

```sql
CREATE TABLE course_enrollments (
  student_id INT,
  course_id INT,
  ...
  PRIMARY KEY (student_id, course_id));

```

Анхдагч түлхүүрүүд нь өгөгдлийн давтагдашгүй байдал болон бүрэн бүтэн байдлыг хангах, өгөгдөл хайлтыг оновчтой болгох, мөн харилцаа хамааралт өгөгдлийн сан дахь бусад хүснэгтүүдтэй холбоо тогтооход зайлшгүй шаардлагатай.
Энэ нь биднийг энэ хичээлээр үзэх дараагийн түлхүүрийн төрөл болох гадаад түлхүүр (foreign keys) рүү хөтөлж байна.

2. Гадаад түлхүүр нь өөр хүснэгтийн анхдагч түлхүүрийг иш татсан (reference) хүснэгтийн багана (эсвэл багануудын багц) юм. Хүснэгт нь олон тооны гадаад түлхүүртэй байж болно.
   Энэ жишээнд бидэнд customers (хэрэглэгчид) болон orders (захиалгууд) гэсэн хоёр хүснэгт байна:

```sql
CREATE TABLE customers (
  customer_id SERIAL PRIMARY KEY,
  first_name VARCHAR(100) NOT NULL,
  ...);

CREATE TABLE orders (
  order_id SERIAL PRIMARY KEY,
  customer_id INTEGER,
  ...
  FOREIGN KEY (customer_id) REFERENCES customers(customer_id));

```

`Customers` хүснэгтийн анхдагч түлхүүр нь customer_id бөгөөд `orders` хүснэгтийн анхдагч түлхүүр нь order_id юм.
Гэхдээ orders хүснэгтэд мөн адил customer_id багана байгааг анзаараарай.
Бид яагаад хэрэглэгчийн мэдээллийг захиалгын хүснэгтэд хадгалж байгаа вэ?
Бид хэрэглэгчид болон захиалгын хүснэгтүүдийн хооронд `харилцаа холбоо` үүсгэхийн тулд ингэж хийдэг. Захиалга бүр тухайн захиалгыг өгсөн хэрэглэгчийн ID-г хадгалах болно.
Хүснэгтүүдийн хооронд харилцаа холбоо үүсгэхийн тулд бид orders хүснэгт дэх customer_id баганыг customers хүснэгтийн customer_id баганыг иш татсан гадаад түлхүүр болгоно:

```sql
FOREIGN KEY (customer_id) REFERENCES customers(customer_id)

```

`Гадаад түлхүүрийн` багана дахь утгууд нь иш татсан хүснэгтийн анхдагч түлхүүрийн утгуудтай тохирч байх эсвэл NULL байх ёстой (хэрэв гадаад түлхүүр NULL утгыг зөвшөөрдөг бол).
Энэ нь яагаад хэрэгтэй вэ?
Хоёр хүснэгтийн эдгээр багануудыг яг таг таарч байгааг баталгаажуулснаар, гадаад болон анхдагч түлхүүрүүд нь "өнчин" бичлэгүүд (өөр хүснэгтэд байхгүй бичлэгийг иш татсан бичлэгүүд) үүсэхээс сэргийлдэг.
Анхдагч болон гадаад түлхүүрүүд нь харилцаа хамааралт өгөгдлийн сангийн үндэс суурь юм. Тэд эдгээр өгөгдлийн сангуудын "харилцаа хамааралтай" талыг бүрдүүлж, бодит ертөнцийн өгөгдлийг үнэн зөвөөр загварчлах боломжийг олгодог.
