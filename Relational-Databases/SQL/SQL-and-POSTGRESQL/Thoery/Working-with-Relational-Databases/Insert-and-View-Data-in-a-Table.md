# Хүснэгтэд өгөгдөл хэрхэн оруулах болон харах вэ?

SQL дээр өгөгдөл `оруулах` болон `харах` үндсэн командуудыг авч үзэцгээе.
Юуны өмнө, өгөгдлийг харахын тулд та өгөгдөл оруулсан байх шаардлагатай. Тиймээс бид өгөгдөл оруулахаас (insertions) эхэлнэ.
Энэ хичээлээр үзэх жишээнүүд нь нохойн бүртгэлийн дараах хүснэгтэд суурилах болно:

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER);

```

Энэхүү хүснэгтэд янз бүрийн нохдын нэр болон насыг хадгалах болно.
Хэрэв бид `PostgreSQL` дээр ажиллаж байна гэж үзвэл id-д зориулж `SERIAL` өгөгдлийн төрлийг ашиглаж болно. id нь SERIAL ашиглан үүсгэгдсэн тул түүний утга нь 1-ээс эхэлсэн бүхэл тоо (INTEGER) байх бөгөөд шинэ бичлэг орох бүрт автоматаар өсөх болно. Тиймээс та бичлэг оруулахдаа түүнд утга оноож өгөх шаардлагагүй.
dogs хүснэгтэд `INSERT INTO` мэдэгдлийг ашиглан бичлэг оруулж эхэлцгээе. Үүнийг хийх олон арга бий. Эхнийх нь нэг мөр оруулах бөгөөд оруулах утгуудын багануудыг зааж өгөх юм.
Та `INSERT INTO` гэж бичээд араас нь хүснэгтийн нэрийг (энэ тохиолдолд dogs), дараа нь хаалтан дотор багануудыг, дараа нь `VALUES`, тэгээд харгалзах утгуудыг хаалтан дотор бичнэ.
Энэ бол жишээ юм, бид 3 настай 'Gino' нэртэй нохойн бичлэгийг оруулж байна, id нь автоматаар оноогдоно гэдгийг санаарай:

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);

```

Та багануудыг тодорхой зааж өгч байгаа тул утгууд нь тэдгээр багануудад дарааллаар нь оноогдох болно, тиймээс энэ нь ихэвчлэн хамгийн аюулгүй сонголт байдаг.
'Gino' нь VARCHAR төрөл тул дан хашилтанд байх ёстойг анхаарна уу.
Өөр нэг хувилбар бол багануудыг заалгүйгээр бичлэгийг оруулах юм, үүнтэй адил:

```sql
INSERT INTO dogs
VALUES ('Gino', 3);

```

Энэ синтакс нь зөв SQL боловч алдаа гарах магадлалтай.
Утгууд нь баганын дарааллаар оноогддог тул PostgreSQL нь 'Gino'-г бүхэл тоо хүлээж байгаа id баганад оруулахыг оролдох болно. Энэ нь бүхэл тооны төрөлд буруу оролт хийх (invalid input syntax for type integer) гэх мэт алдаанд хүргэдэг.
Ийм учраас өгөгдөл оруулахдаа баганын нэрсийг тодорхой зааж өгөх нь илүү аюулгүй юм.
Та мөн нэг SQL командад таслалаар тусгаарлан олон бичлэг оруулах боломжтой.
Энд бид 'Gino' болон 'Nora' гэсэн хоёр нохойн бичлэгийг оруулж байна.

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);

```

Бичлэг оруулж чаддагтайгаа адил та өгөгдлийн сангаас мэдээлэл асууж (query) авч болно. Өгөгдлийн сангийн хүрээнд асуулга (query) гэдэг нь өгөгдөл авах хүсэлт юм.
SQL-д SELECT мэдэгдлийг нэг болон түүнээс дээш хүснэгтээс өгөгдөл асуухад ашигладаг. Та өөрт хэрэгтэй мэдээллийг яг таг авахын тулд асуулгыг өөрчилж болно.
Хэрэв та dogs хүснэгтээс бүх мэдээллийг асуух шаардлагатай бол SELECT гэж бичээд араас нь од (\*), дараа нь FROM, тэгээд хүснэгтийн нэр болох dogs-ийг бичихэд хангалттай:

```sql
SELECT *
FROM dogs;

```

Энэ командад од нь "бүх багана"-ыг төлөөлөх орлуулагч тэмдэг (wildcard character) юм.
Та хүснэгт дэх бүх бичлэгийн бүх өгөгдлийг авах болно:

```text
 id | name | age
----+------+-----
  1 | Gino |   3
  2 | Nora |   2

```

Тодорхой багануудыг асуухын тулд та SELECT-ийн ард тэдгээр багануудын нэрсийг таслалаар тусгаарлан бичиж болно. Энэ жишээнд бид dogs хүснэгтийн name болон age багануудыг асууж байна:

```sql
SELECT name, age
FROM dogs;

```

Үр дүн нь зөвхөн name болон age багануудыг агуулах болно:

```text
 name | age
------+-----
 Gino |   3
 Nora |   2

```

Заримдаа танд тодорхой нөхцөл дээр үндэслэн өгөгдөл асуух шаардлага гарч магадгүй. Жишээлбэл, 3-аас доош настай бүх нохойг авахын тулд та WHERE түлхүүр үг болон бага ( < ) харьцуулах операторыг ашиглаж болно:

```sql
SELECT *
FROM dogs
WHERE age < 3;

```

Үр дүн нь зөвхөн 3-аас доош настай нохдыг агуулах болно:

```text
 id | name | age
----+------+-----
  2 | Nora |   2

```

Хэрэв та зөвхөн 'Gino'-ийн насыг олохыг хичээж байгаа бол тэнцүү ( = ) харьцуулах операторыг ашиглаж болно:

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';

```

Үр дүн нь энд байна:

```text
 age
-----
  3

```

Эдгээр нь SQL дээр өгөгдөл `оруулах` болон `харах` нийтлэг бөгөөд энгийн аргууд боловч таны сонгож болох олон өөр сонголтууд байдаг бөгөөд тэдгээрийн зарим нь илүү ахисан түвшнийх юм. Та тэдгээрийн талаар ирэх хичээлүүдээс суралцах болно.
