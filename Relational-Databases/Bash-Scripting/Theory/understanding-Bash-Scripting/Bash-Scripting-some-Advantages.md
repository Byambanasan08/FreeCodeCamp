# Bash Scripting гэж юу вэ, мөн үүнийг ашиглахын зарим давуу талууд юу вэ?

Өмнөх хичээлүүд дээр та Bash болон ашиглаж болох түгээмэл командуудын талаар сурсан. Гэхдээ Bash нь бүрэн scripting хэл (scripting language)-ийг дэмждэг бөгөөд үүнийг ашиглан илүү төвөгтэй автоматжуулсан (automated) ажлуудыг гүйцэтгэх боломжтой.

Bash scripting гэдэг нь Bash командуудын дарааллыг файл дотор бичих бөгөөд дараа нь Bash ашиглан тухайн файлын агуулгыг ажиллуулахыг хэлнэ.

Одоо нэг bash script ямар харагддагийг харцгаая:

#!/bin/bash

servers=("prod" "dev")

for server in "${servers[@]}"
do
echo "Pulling $server"
rsync --archive --verbose $server:/etc/nginx/conf.d/server.conf configs/$server.conf
done

Энэ script нь remote серверүүдийн жагсаалтаас NGINX configuration файлуудыг татаж авч, тэдгээрийг local repository-д хадгалах зорилготой бөгөөд энэ нь configuration-уудыг backup хийхэд ашигтай байж болно. NGINX нь түгээмэл web server бөгөөд сервертэй хэрхэн ажиллах талаар та дараагийн модуль дээр илүү ихийг сурах болно.

Эхний мөрийг "shebang" (#!) гэж нэрлэдэг бөгөөд энэ нь системд ямар interpreter ашиглан script-ийг ажиллуулахыг зааж өгдөг. Энд түгээмэл хэрэглэгддэг утгууд нь /usr/bin/bash, /bin/bash, эсвэл /bin/sh байдаг.

servers=("prod" "dev") гэсэн мөр нь server-ийн нэрсийг илэрхийлэх string-үүдийн жагсаалтыг үүсгэж байна. "prod" нь ихэвчлэн production site-ийг илэрхийлдэг бол "dev" нь development site-ийг илэрхийлдэг бөгөөд энэ нь шинэ feature болон өөрчлөлтүүдийг live production environment-д deploy хийхээс өмнө тестлэхэд ашиглагддаг.

Дараа нь, энэ жагсаалтыг for loop ашиглан давтаж байна — servers[@] syntax нь жагсаалтыг тус тусад нь элементүүд болгон задлана. Давталт бүрт элемент бүр server хувьсагчид оноогдоно.

do statement нь loop-ийн логик блокийн эхлэлийг илэрхийлдэг, өөрөөр хэлбэл давталт бүр дээр ажиллах кодын эхлэл юм.

echo "Pulling $server" нь server хувьсагчийн одоогийн утгыг (энэ тохиолдолд prod эсвэл dev) interpolation хийж, terminal дээр хэвлэж харуулна.

Дараа нь rsync командыг ашиглан тухайн серверээс бодит файлыг татаж авч, local дээр хадгалж байна, мөн энэ команд дотор interpolation-ийг ашиглаж байна.

done нь loop-ийн логик блокийн төгсгөлийг илэрхийлнэ.

Bash scripting-ийг ойлгож, ашиглахад олон чухал давуу талууд байдаг.

Эхнийх нь бараг бүх Unix орчин Bash-ийг бэлэн (readily available) байдлаар агуулдаг тул та өөрийн script-ийг Node.js эсвэл Python зэрэг хэлний runtime тохируулахгүйгээр шууд ажиллуулж чадна.

Хоёр дахь нь та систем дээр байгаа бүх binary application-уудыг маш бага хүчин чармайлтаар ашиглах боломжтой. Хэрвээ таны систем дээр doctl суусан байвал таны script тэр binary-г шууд ажиллуулж чадна. Binary гэдэг нь компьютерт шууд ажилладаг compile хийгдсэн програм юм. Bash script-тэй үед та binary-г ажиллуулснаар тухайн програмыг эхлүүлж чадна.

Эцэст нь, таны script дээр ашигласан syntax-ийг terminal дээр шууд ажиллуулах боломжтой байдаг. Энэ нь script-ийн жижиг хэсгийг тестлэхэд маш хялбар болгодог — та тухайн хэсгийг шууд terminal руу copy-paste хийж үр дүнг нь харах боломжтой!

Bash-ийг ойлгох нь зөвхөн таны scripting чадварыг нэмэгдүүлэхгүй, мөн таныг command line дээр жинхэнэ “wizard” болгоно!
