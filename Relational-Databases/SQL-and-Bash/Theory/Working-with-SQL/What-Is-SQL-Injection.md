# SQL тарилга (SQL Injection) гэж юу вэ, түүнийг хэрхэн зогсоох вэ?

`SQL тарилга` гэдэг нь халдагчид вэб програмаас өгөгдлийн сан руу илгээж буй хүсэлт (queries)-ийг өөрчлөх эсвэл давуу тал болгон ашигладаг вэб аюулгүй байдлын эмзэг байдал юм. Жишээлбэл, халдагчид өгөгдлийн сангийн арын систем (back-end) дээр хортой код ажиллуулахын тулд хүсэлт дотор хортой SQL код оруулж чаддаг.
Хэрэглэгчдэд нэвтрэх форм, хайлтын хэсэг, эсвэл URL-ийн параметрүүдээр дамжуулан өгөгдөл оруулах боломж олгож, улмаар тухайн хэрэглэгчийн оруулсан утгыг эцсийн SQL тушаалыг үүсгэхдээ тэмдэгт мөрийн залгаас (string interpolation эсвэл concatenation) хэлбэрээр шууд ашиглах үед энэ нь тохиолдож болно.
Халдагчийн зорилгоос хамааран хортой SQL код нь таны өгөгдлийн сан болон вэб програмд ноцтой саатал учруулж болно. Жишээлбэл, нэвтрэх шалгалтыг тойрч гарах, өгөгдлийн сангаас өгөгдөл хулгайлах, эсвэл өгөгдлийн сангийн бичилтүүдийг өөрчлөх, устгах зэрэг эрсдэлтэй. Тиймээс та энэхүү аюул заналыг маш нухацтай авч үзэх хэрэгтэй.
Жишээлбэл, таны вэб програм хэрэглэгчийг баталгаажуулахын тулд хэрэглэгчийн оруулсан хэрэглэгчийн нэр болон нууц үгийг шууд суулгасан иймэрхүү SQL хүсэлт ашигладаг гэж үзье:

```javascript
"SELECT * FROM users WHERE username = " +
  username_input +
  " AND password = " +
  password_input +
  ";";
```

Хэрэв халдагч хэрэглэгчийн нэрний талбарт дараах зүйлийг оруулбал:

```text
'" " OR "1"="1" --'

```

Мөн нууц үгний талбарт юу ч хамаагүй оруулсан бай, `"1"="1"` нь үргэлж үнэн (true) байх бөгөөд `--` нь хүсэлтийн үлдсэн хэсгийг тайлбар (comment) болгон идэвхгүй болгодог тул WHERE нөхцөл нь дараах байдалтай болно:

```sql
WHERE username = " " OR TRUE

```

Энэ нь үргэлж үнэн байх болно.
Өгөгдлийн сангийн хэрэгжүүлэлтээс хамааран энэ нь өгөгдлийн сан дахь эхний хэрэглэгчийн эсвэл дурын хэрэглэгчийн өгөгдлийг буцааж, халдагчид хүчинтэй итгэмжлэгдсэн мэдээлэлгүйгээр нэвтрэх боломжийг олгож болно.
Тэгвэл та үүнээс хэрхэн сэргийлэх вэ? SQL тарилгаас сэргийлэх хоёр гол арга зам байдаг:

Тэмдэгт мөрийн залгаас (string concatenation) ашиглан динамик хүсэлт бүү бич.
Мөн арын системд ажиллах хүсэлтүүдэд хортой SQL оролт багтахаас сэргийл.
Практикт та параметржүүлсэн тушаалууд (parameterized statements) буюу параметржүүлсэн хүсэлтүүд (parameterized queries)-ийг ашиглаж болно. Тэдгээр нь SQL-ийн бүтцийг хэрэглэгчийн оруулсан өгөгдлөөс тусгаарладаг бөгөөд энэ нь өгөгдлийн санг оруулсан өгөгдлийг гүйцэтгэгдэх код гэж тайлбарлахаас сэргийлдэг.
Өөр нэг чухал арга техник бол оролтын баталгаажуулалт (input validation) юм. Үүнийг бүх тохиолдолд аюулгүй байдлын хоёрдогч арга хэмжээ болгон ашиглахыг зөвлөдөг боловч SQL хүсэлтийн хүснэгтийн нэр, баганы нэр, эрэмбэлэх дараалалтай холбоотой хэсгүүдэд онцгой ач холбогдолтой юм. Хэрэв боломжтой бол хүснэгтийн нэр болон баганы нэрийг хэрэглэгчийн оролтоос биш, харин өөрийн код дотроос шууд оноохыг зөвлөж байна.
Хэрэв урьдчилан сэргийлэх бүх арга хэмжээг авсан ч SQL тарилгын халдлагад өртсөн тохиолдолд та өгөгдлийн сангийн бүртгэл бүрийн эрхийг хязгаарлах замаар өгөгдлийн санд үзүүлэх нөлөөллийг хамгийн бага болгож чадна. Өгөгдлийн сангийн хэрэглэгч бүрт зөвхөн тэдний даалгаварт шаардлагатай эрхийг олгож, түүнээс илүүг бүү олго. Хэрэв бүртгэлд өгөгдлийн сангаас зөвхөн өгөгдөл унших шаардлагатай бол зөвхөн унших эрх (read access)-ийг олго. Мөн урьдчилан сэргийлэх арга хэмжээ болгон өөрийн програмын бүртгэлүүдэд хэзээ ч админ (admin) эрхийг бүү оноо. Ингэх нь маш аюултай бөгөөд учир нь халдагч өгөгдлийн санг бүрэн хянах, бүтэн хандалт хийх эрхийг олж авч болзошгүй.
SQL тарилга нь хөгжүүлэгч таны мэдэж байх ёстой бөгөөд вэб програмаа хэрэгжүүлэхдээ урьдчилан сэргийлэх ёстой маш түгээмэл халдлага юм. Эдгээр аргуудын тусламжтайгаар та өөрийн өгөгдлийн санг хамгаалж, хэрэглэгчийнхээ өгөгдлийг аюулгүй байлгаж чадна.
