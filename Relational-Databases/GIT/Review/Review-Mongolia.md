# 3. Git суулгах болон тохируулах

```
3.1 Git суулгах: Таны компьютер дээр Git аль хэдийн суусан эсэхийг шалгахын тулд terminal дээр дараах командыг ажиллуулж болно:
```

git --version

Хэрвээ та version number харвал Git суусан байна гэсэн үг. Хэрвээ харагдахгүй бол та үүнийг суулгах шаардлагатай болно.

Linux системүүдийн хувьд Git ихэнх distro-ууд дээр урьдчилан суусан байдаг. Хэрвээ танд Git урьдчилан суусан байхгүй бол, та package manager командуудыг ашиглан суулгаж болно, жишээ нь sudo apt-get install git эсвэл sudo pacman -S git.

Mac хэрэглэгчдийн хувьд, та Git-ийг Homebrew ашиглан brew install git командаар суулгаж болно, эсвэл Git-ийн вэбсайтаас executable installer татаж авч болно.

Windows-ийн хувьд, та Git-ийн вэбсайтаас executable installer татаж авч болно. Эсвэл хэрвээ та Chocolatey тохируулсан бол, PowerShell дээр choco install git.install командыг ажиллуулж болно. Windows дээр та мөн Git Bash татаж авахыг хүсэж магадгүй, ингэснээр танд Unix-like shell environment ашиглах боломжтой болно.

Суулгалт амжилттай болсон эсэхийг шалгахын тулд terminal дээр git --version командыг дахин ажиллуулна уу.

```
3.2 Git тохиргоонууд: git config нь Git таны компьютер дээр хэрхэн ажиллахыг тодорхойлох configuration хувьсагчдыг (variables) тохируулахад ашиглагддаг. Одоогийн тохиргооны хувьсагчдыг болон тэдгээр нь таны систем дээр хаана хадгалагдаж байгааг харахын тулд дараах командыг ажиллуулж болно:
```

git config --list --show-origin

Хэрвээ та Git-ийг анх удаа суулгасан бол, одоогоор зөвхөн system-level configuration тохиргоонуудыг харж байх ёстой.

Хэрэглэгчийн нэрээ тохируулахын тулд дараах командыг ажиллуулж болно:

git config --global user.name "Jane Doe"

Эндхүү --global flag нь таны систем дээр Git ашиглаж байгаа бүх төслүүдэд хэрэглэгчийн нэрийг тохируулахад ашиглагддаг. Хэрвээ та тодорхой нэг төслийн хувьд хэрэглэгчийн нэрийг өөрчлөх шаардлагатай бол, тухайн төслийн directory дотор --global flag-гүйгээр командыг ажиллуулж болно.

Хэрэглэгчийн email хаягийг тохируулахын тулд дараах командыг ажиллуулж болно:

git config --global user.email mailto:janedoe@example.com

Мөн та Git ашиглахдаа ямар editor ашиглахыг тохируулж болно. Доорх нь таны preferred editor-ийг Emacs болгож тохируулах жишээ юм:

git config --global core.editor emacs

Хэрвээ та preferred editor тохируулахгүй бол, Git таны системийн default editor-ийг ашиглана.

# 5. GitHub

5.1 Тодорхойлолт: GitHub нь cloud-based шийдэл бөгөөд version-controlled төслүүдийг "repositories" гэж нэрлэгддэг зүйлд хадгалах боломжийг олгож, мөн эдгээр төслүүд дээр хамтран ажиллах боломжуудыг (collaboration features) олгодог.
5.2 GitHub CLI: Энэ хэрэгсэл нь command line-оос гарахгүйгээр GitHub-тэй холбоотой тусгай үйлдлүүдийг хийхэд ашиглагддаг. Хэрвээ танд суусан байхгүй бол GitHub-ийн documentation-оос суулгах зааврыг авч болно — гэхдээ ихэвчлэн таны системийн package manager дээр байх ёстой.
5.3 GitHub Pages: GitHub Pages нь static сайтуудыг deploy хийх, эсвэл back-end server шаардлагагүй аппликейшнуудыг байрлуулах боломжийг олгодог. Өөрөөр хэлбэл, бүрэн client-side дээр ажилладаг, эсвэл хэрэглэгчийн browser дээр ажилладаг аппликейшнуудыг энэ платформ дээр бүрэн deploy хийж болно.
5.4 GitHub Actions: GitHub Actions нь таны GitHub repository дотор шууд workflow-уудыг автоматжуулах боломжийг олгодог feature бөгөөд үүнд таны кодыг build хийх, тестлэх, болон deploy хийх зэрэг орно.

# 6. Түгээмэл Git командууд

```
6.1 git init: Энэ нь хоосон Git repository үүсгэж, Git таны төслийн өөрчлөлтүүдийг хянаж эхлэх боломжийг олгодог. Хоосон Git repository үүсгэх үед .git гэсэн нууц (hidden) directory нэмэгдэнэ. Энэ .git directory нь Git таны төслийг удирдахад шаардлагатай чухал мэдээллийг агуулдаг.
6.2 git status: Энэ команд нь таны working directory-ийн одоогийн төлөвийг харуулахад ашиглагддаг — та энэ командыг workflow-д маш их ашиглах болно.
6.3 git add: Энэ команд нь таны өөрчлөлтүүдийг staging хийхэд ашиглагддаг. Staging area-д байгаа бүх зүйл дараагийн commit-д нэмэгдэнэ. Хэрвээ та бүх unstaged өөрчлөлтүүдийг staging хийхийг хүсвэл git add . командыг ашиглаж болно. Цэг (.) нь таны одоогийн directory-ийн alias юм.
6.4 git commit: Энэ команд нь таны өөрчлөлтүүдийг commit хийхэд ашиглагддаг. Commit гэдэг нь тухайн агшинд таны төслийн төлөвийн snapshot юм. Хэрвээ та git commit командыг ажиллуулбал, Git configuration дээр тохируулсан preferred editor нээгдэнэ. Editor нээгдсэний дараа та өөрчлөлтүүдийн талаар дэлгэрэнгүй message бичиж болно. Мөн та git commit -m командыг ашиглан богино message өгч болно, жишээ нь:
```

git commit -m "short message goes here"

```
6.5 git log: Энэ нь өмнөх бүх commit-уудыг жагсааж, author, commit хийсэн огноо, commit message болон commit hash зэрэг хэрэгтэй мэдээллүүдийг харуулна. Commit hash нь commit-ийг ялгах өвөрмөц урт string юм.
6.6 git remote add: Энэ команд нь таны remote repository-тэй холболтыг тохируулахад ашиглагддаг.
6.7 git push: Энэ команд нь таны өөрчлөлтүүдийг remote repository руу илгээхэд ашиглагддаг.
6.8 git pull: Энэ команд нь remote repository-оос хамгийн сүүлийн өөрчлөлтүүдийг таны local repository руу татаж авахад ашиглагддаг.
6.9 git clone: Энэ команд нь repository-г clone хийхэд ашиглагддаг. Энэ нь repository-ийн хуулбарыг үүсгэнэ гэсэн үг. Энэ хуулбар нь repository-ийн history, бүх файлууд/хавтаснууд болон commit-уудыг таны local төхөөрөмж дээр агуулна.
6.10 git remote -v: Энэ команд нь таны local Git repository-тэй холбогдсон remote repository-уудын жагсаалтыг харуулна.
6.11 git branch: Энэ команд нь таны бүх local branch-уудыг жагсаана.
6.12 git fetch upstream: Энэ команд нь Git-д upstream remote (өөрөөр хэлбэл анхны repo)-оос хамгийн сүүлийн өөрчлөлтүүдийг татаж авахыг заадаг.
6.13 git merge upstream/main: Энэ команд нь upstream remote-ийн main branch-ийн хамгийн сүүлийн өөрчлөлтүүдийг таны одоогийн branch руу merge хийхийг Git-д заадаг.
6.14 git reset: Энэ команд нь branch-ийн одоогийн төлөвийг reset хийх боломжийг олгодог. --hard flag-ийг дамжуулахад Git local файлуудыг branch-ийн төлөвтэй яг адил болтол хүчээр (force) тохируулна. Энэ нь таныг цэвэр орчноос ажиллах боломжийг хангана.
6.15 git rebase: Git дахь rebase нь нэг branch дээрх commit-уудын дарааллыг өөр branch дээр шилжүүлэх эсвэл нэгтгэх арга юм.
```

# 7. Branch-уудтай ажиллах

7.1 **Тодорхойлолт**: Git дахь `branch` гэдэг нь өөрчлөлт хийх тусдаа ажлын орчин (workspace) юм. `Main branch` нь ихэвчлэн бодит аппликейшн дээр үндсэн эсвэл `production branch`-ийг илэрхийлдэг. Developer багууд шинэ feature болон алдааны засваруудын хувьд олон branch үүсгээд, дараа нь эдгээр өөрчлөлтүүдийг main branch руу буцаан merge хийдэг.
7.2 Шинэ Branch үүсгэх: Шинэ branch үүсгэхийн тулд дараах командыг ажиллуулна:

`git branch feature`

Тэр branch руу шилжихийн тулд дараах командыг ажиллуулна:

`git checkout feature`

Ихэнх developer-ууд branch үүсгэж, шууд шилжих shorthand командыг ашигладаг бөгөөд энэ нь:

`git checkout -b new-branch-name`

Шинэ бөгөөд өөр нэг хувилбар команд нь `git switch` юм. Шинэ branch үүсгэж, шилжих жишээ:

`git switch -c new-branch-name`

7.3 `Branching Strategies`: Таны main branch нь default branch бөгөөд ихэвчлэн тогтвортой байдаг. Тиймээс bug fix, шинэ feature, эсвэл бусад ажлуудад зориулж шинэ branch-уудыг түүнээс салаалж (branch off) үүсгэх нь хамгийн зөв.
7.4 `Merge Conflicts`: Энэ нь Git өөр өөр branch-уудын өөрчлөлтүүдийг автоматаар merge хийх гэж оролдох үед, аль өөрчлөлтийг хадгалахаа шийдэж чадахгүй үед үүсдэг. Энэ нь ихэвчлэн нэг файлын ижил хэсэг дээр зөрчилтэй өөрчлөлтүүд орсон үед тохиолддог.

# 8. Git-д хянагддаг (tracked) файлын 5 төлөв

```
8.1 "Untracked": Энэ нь файл repository-д шинээр орж ирсэн бөгөөд Git өмнө нь “харж” байгаагүй гэсэн үг.
8.2 "Modified": Энэ файл өмнөх commit-д байсан бөгөөд commit хийгдээгүй өөрчлөлтүүдтэй болсон байна.
8.3 "Ignored": Та ignored файлуудыг Git дээр харахгүй байх магадлалтай, гэхдээ таны IDE тэдэнд зориулсан тэмдэглэгээтэй байж болно. Ignored файлууд нь Git-ийн үйлдлүүдээс хасагддаг бөгөөд ихэвчлэн .gitignore файлд багтсан байдаг.
8.4 "Deleted": Deleted файл нь untracked файлын эсрэг ойлголт — өмнө нь байсан файл устгагдсан байна.
8.5 "Renamed": Renamed файл гэдэг нь файлын агуулга өөрчлөгдөөгүй боловч нэр эсвэл байрлал нь өөрчлөгдсөн файл юм. Зарим тохиолдолд файл бага зэрэг өөрчлөлттэй байсан ч renamed гэж тооцогдож болно.
```

```

Repository нь таны компьютер дээр local байж болно, эсвэл :contentReference[oaicite:0]{index=0} зэрэг үйлчилгээ дээр remote байж болно.
10.2 Public vs. Private Repository: Public repository-г хэн ч харах, татаж авах боломжтой. Private repository-д зөвхөн та болон таны тусгайлан зөвшөөрөл өгсөн хүмүүс хандах боломжтой.
10.3 GitHub дээр Repository үүсгэх: GitHub дээр шинэ repository үүсгэхийн тулд "New Repository" товчийг дарж, GitHub-ийн UI дээрх алхмуудыг даган repository үүсгэнэ.
10.4 Local Repository-г GitHub руу push хийх: Хэрвээ таны компьютер дээр local төсөл байгаа бол, та түүнийг GitHub руу push хийж болно. Доорх нь энэ процессын алхамчилсан тойм юм:

1. Төслийн directory дотор хоосон git repository үүсгэнэ (git init).
2. Төсөл дээрээ өөрчлөлтүүд хийнэ.
3. git status командыг ажиллуулж Git-ийн хянаж буй бүх өөрчлөлтүүдийг харна.
4. Өөрчлөлтүүдээ staging хийнэ (git add).
5. Өөрчлөлтүүдээ commit хийнэ (git commit).
6. Remote холболтыг тохируулна (git remote add).
7. Өөрчлөлтүүдээ GitHub руу push хийнэ (git push).
```

# 13. SSH болон GPG түлхүүрүүдтэй (Keys) ажиллах

```
13.1 GPG түлхүүрүүд: GPG буюу Gnu Privacy Guard түлхүүрүүд нь ихэвчлэн файл эсвэл commit-уудыг гарын үсэг (sign) хийхэд ашиглагддаг. Дараа нь хэн нэгэн таны public GPG түлхүүрийг ашиглан тухайн файлын гарын үсэг таны түлхүүрээс үүссэн эсэхийг, мөн файлын агуулга өөрчлөгдөөгүй, эсвэл оролдоогүй (tampered) гэдгийг баталгаажуулж чадна.
```

GPG түлхүүр үүсгэхийн тулд та дараах командыг ажиллуулах хэрэгтэй:

gpg --full-generate-key

```
13.2 SSH түлхүүрүүд: SSH буюу Secure SHell түлхүүрүүд нь ихэвчлэн сервер рүү алсын (remote) холболтыг баталгаажуулахад ашиглагддаг — ssh хэрэгслээр дамжуулан. Мөн та SSH түлхүүрийг commit-уудыг гарын үсэгжүүлэхэд ашиглаж болно.
```

SSH түлхүүр үүсгэхийн тулд та дараах командыг ажиллуулна:

ssh-keygen -t ed25519 -C "[your_email@example.com](mailto:your_email@example.com)"

ed25519 нь орчин үеийн public-key signature алгоритм юм.

```
13.3 GPG түлхүүрээр commit-уудыг гарын үсэгжүүлэх: Commit-уудаа GPG түлхүүрээр гарын үсэгжүүлэхийн тулд та өөрийн private түлхүүр биш, харин public түлхүүрийг GitHub account руугаа upload хийх хэрэгтэй. Public түлхүүрүүдээ жагсаахын тулд дараах командыг ажиллуулна:
```

gpg --list-secret-keys --keyid-format=long

Дараа нь public түлхүүрийг авахын тулд дараах командыг ашиглана:

gpg --armor --export "<key id>"

Дараа нь түлхүүрүүдийг жагсаах үед авсан short ID-г ашиглан дараах командыг ажиллуулж үүнийг git-ийн signing түлхүүр болгон тохируулна:

git config --global user.signingkey <your_gpg_key_id>

git config --global commit.gpgsign true
