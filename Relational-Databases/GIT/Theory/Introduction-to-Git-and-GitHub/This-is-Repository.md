# Repository гэж юу вэ, мөн үүнийг хэрхэн үүсгэх вэ?

Бид Git болон GitHub-ийн талаар бага зэрэг ярьсан, одоо эдгээрийг нэгтгэн ойлгож эхэлж болно. Үүнийг хийхийн тулд танд repository хэрэгтэй. Харин repository гэж юу вэ? Та repository-г project-ийн сав (container) гэж ойлгож болно — хэрвээ та нэг app дээр ажиллаж байгаа бол тухайн app-ийн бүх файлуудыг repository дотор хадгална.

Repository нь таны компьютер дээр local байж болно, эсвэл GitHub зэрэг үйлчилгээ дээр remote байж болно. Repository үүсгэхийн тулд та local эсвэл remote-оос аль алинаас нь эхэлж болно — эцсийн үр дүн нь ижил байх тул өөрийн workflow-д хамгийн тохиромжтой аргыг сонгоорой. Remote repository-оос эхлэх workflow-ийн хувьд, та дараах зааврын дагуу GitHub-ийн вэбсайтаар дамжуулан шууд үүсгэж болно:

GitHub-ийн вэбсайтад орж, нэвтэрсэн эсэхээ шалгаад navigation bar-ийн баруун дээд буланд байгаа нэмэх (plus) icon дээр дарна. Та олон сонголтуудыг харах болно, гэхдээ танд хэрэгтэй нь "New Repository" юм. Үүнийг сонгож шинэ repository үүсгэх UI-г нээнэ.

Эхний харагдах сонголт нь template сонгох боломж юм. Та одоогоор ийм зүйл тохируулаагүй байх магадлалтай, гэхдээ template гэдэг нь шинэ repository үүсгэхэд суурь (springboard) болгон ашиглаж болох тусгай төрлийн repository юм. Эдгээр нь бүх project-доо documentation эсвэл issue template-ууд оруулахад хэрэгтэй.

Дараагийн алхам нь owner сонгох бөгөөд та өөрийн хэрэглэгчийн account эсвэл өөрийн эзэмшдэг, эсвэл хандах эрхтэй organization-уудыг сонгож болно. Дараа нь repository-ийн нэрийг өгнө — ерөнхийдөө та хийж буй project-оо товч тодорхой илэрхийлэх нэр сонгох хэрэгтэй. Сайн практик (best practice)-ийн хувьд repo нэрэнд зай (space) ашиглахаас зайлсхийх хэрэгтэй. GitHub автоматаар зайг зураас (hyphen) болгон сольдог.

Дараа нь та optional байдлаар description нэмэж болно, энэ нь repository-ийн homepage дээр харагдана.

Та мөн repository public эсвэл private байхыг сонгож болно. Public repository-г хэн ч харах, татаж авах боломжтой. Private repository-г зөвхөн та болон таны тусгайлан зөвшөөрсөн хүмүүс л ашиглах боломжтой.

Эцэст нь, хэрвээ та template сонгоогүй бол repository дотор файлуудыг автоматаар үүсгэх сонголтууд харагдана. README файл нь repository-ийн homepage дээр гарч ирдэг бөгөөд project-ийн documentation болон тайлбар өгөхөд маш тохиромжтой. .gitignore файл нь Git ямар файлуудыг "ignore" хийхийг зааж өгдөг — энэ нь package dependency, environment файл зэрэг хадгалах шаардлагагүй зүйлсэд тохиромжтой. GitHub энэ файлд зориулсан template-уудыг санал болгодог бөгөөд тухайн хэл эсвэл runtime-д түгээмэл ignore хийх зүйлсийг багтаасан байдаг. Харин license нь таны програмыг хэн ашиглаж болох, юу хийж болохыг тодорхойлсон хууль эрх зүйн баримт бичиг юм.

Та бүх тохиргоогоо хүссэнээрээ хийж дууссаны дараа "Create Repository" товчийг дарж процессыг дуусгана. Ингэснээр та шинэ repository-ийнхээ landing page руу автоматаар шилжинэ!

Эндээс та "Code" товчийг дарж repository-оо өөрийн компьютерт clone хийх сонголтуудыг авах боломжтой.

Энэ хичээлийн хувьд "Codespaces" болон "Copilot" tab-уудыг алгасаж болно. Харин "Local" tab доторх гурван сонголт дээр анхаарлаа төвлөрүүлнэ. Эхний хоёр болох HTTPS болон SSH нь ижил аргаар ашиглагддаг. Та өгөгдсөн URL-ийг ашиглан git clone командыг ажиллуулж remote repository-ийг өөрийн компьютерт clone хийнэ. Эдгээр хоёрын ялгаа нь холболт болон authentication аргачлалд байдаг.

Public repository-г clone хийхийн тулд танд authentication хийх шаардлагагүй. Харин private repository-г clone хийх эсвэл public болон private repository-д push хийхийн тулд та заавал authentication хийх шаардлагатай. HTTPS remote URL нь таныг GitHub username болон password-аар дамжуулан authenticate хийдэг байсан. Гэвч аюулгүй байдлын шалтгаанаар энэ арга одоо дэмжигдэхээ больсон бөгөөд оронд нь username болон access token ашиглах шаардлагатай болсон. Гэсэн хэдий ч, энэ нь SSH-тай харьцуулахад илүү бага аюулгүй арга бөгөөд SSH нь таны private болон public key-үүдийг ашигладаг. Энэ нь мөн нэг давуу талтай — та token-оо бүр дахин дахин оруулах шаардлагагүйгээр автоматаар нэвтрэх боломжтой. SSH болон энэ нь хэрхэн ажилладаг талаар тусгай хичээл дараа нь орно, мөн та GitHub-ийн documentation-оос SSH-ийг хэрхэн тохируулах талаар уншиж болно.

Repository-г clone хийхийн тулд та terminal дээр дараах командыг ажиллуулж болно:

git clone mailto:git@github.com:your-username/your-repository-name

Энэ нь таны компьютерт repository-ийн нэртэй folder үүсгэж, дотор нь repository дахь бүх файлуудыг байрлуулна. Үүний дараа та cd (change directory) командыг ашиглан clone хийсэн repository руугаа орж болно:

cd name-of-cloned-repo

Мөн таны repo-ийн local хувилбар нь GitHub дээрх хувилбар руу заасан remote тохиргоотой байна. Remote-уудаа шалгахын тулд дараах командыг ажиллуулж болно:

git remote -v

Repository-г clone хийх өөр нэг арга бол GitHub CLI ашиглах юм. Энэ хэрэгсэл нь command line-оос гаралгүйгээр GitHub-тэй холбоотой үйлдлүүдийг хийхэд ашиглагддаг. Хэрвээ та үүнийг суулгаагүй бол GitHub-ийн documentation-оос суулгах зааврыг авч болно — мөн таны системийн package manager-д байх магадлалтай.

CLI хэрэгслээр repository-г clone хийхийн тулд дараах командыг ашиглана:

gh repo clone your-username/your-repository-name

Эдгээр командуудын эцсийн үр дүн ижил — таны repository-ийн local хувилбар таны компьютерт бий болно.

Мөн та GitHub CLI ашиглан repository үүсгэж болно:

gh repo create

Хэрвээ та ямар нэг аргумент дамжуулахгүй бол CLI нь interactive wizard харуулж, repository-оо тохируулахад тусална. Энэ wizard нь вэб UI дээрх бүх сонголтуудыг агуулдаг. Энэ хэрэгслийг ашигласнаар та шинэ repository-г шууд clone хийх сонголттой байх бөгөөд өмнө дурдсан гар ажиллагаатай алхмуудыг алгасах боломжтой.

Та одоо repository хэрхэн ажилладаг, мөн өөрийн ажлыг зөв version control-той болгохын тулд хэрхэн үүсгэх талаар үндсэн ойлголттой боллоо. Дараагийн хичээлүүд дээр та local repository-г GitHub руу хэрхэн push хийх, мөн remote repository руу өөрчлөлтүүдийг хэрхэн push хийх талаар сурах болно.
