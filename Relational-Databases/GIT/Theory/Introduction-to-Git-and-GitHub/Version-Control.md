# Version Control гэж юу вэ, мөн яагаад эдгээр системүүд хэрэгтэй вэ?

Хэрвээ та thesis paper шиг зүйл дээр ажиллаж байхдаа томоохон өөрчлөлт бүрийн дараа файлын хуулбар хадгалж, Thesis-Draft, Thesis-Final, Thesis-Final-For-Real, Thesis-Absolute-Final-DO-NOT-CHANGE гэх мэт нэр өгч байсан бол, та version control-ийн маш энгийн (rudimentary) аргыг ашиглаж байсан гэсэн үг. Гэхдээ үүнээс илүү сайн арга байдаг гэдгийг хэлбэл яах вэ?

Та Git, SVN, эсвэл Mercurial зэрэг хэрэгслүүдийн талаар өмнө нь сонсож байсан байж магадгүй. Эдгээр нь version control system-ууд бөгөөд developer бүрийн workflow-ийн чухал хэсэг юм. Version control system нь таны project дээрх өөрчлөлтүүдийг хянах (track) болон удирдах (manage) боломжийг олгодог. Энэ хичээлийн хүрээнд бид Git дээр төвлөрөх болно.

Git нь project-ийн засварын түүхийг (edit history) удирдах хүчирхэг хэрэгслүүдийг санал болгодог. Та дараагийн хичээлүүд дээр Git-ийг хэрхэн ашиглах, ямар боломжуудтай талаар илүү дэлгэрэнгүй сурах болно. Одоогоор, танд мэдэх шаардлагатай гол зүйлсийг харцгаая:

Git нь commit үүсгэх боломжийг олгодог бөгөөд commit гэдэг нь таны кодын тодорхой төлөвийн (state) snapshot юм. Та дараа нь commit-уудыг харьцуулж ямар өөрчлөлт хийснээ харах, буруу өөрчлөлттэй commit-ийг буцаах (revert) зэрэг үйлдлүүдийг хийж болно.

Та мөн "branch" үүсгэж болно, үүнийг та өөр өөр кодын замууд (pathways) гэж ойлгож болно. Branch нь таны алхаж буй зам гэж үзвэл, commit-ууд нь тэр зам дээрх тэмдэглэгээ (landmarks) юм. Branch-ууд нь шинэ feature дээр тусгаарлагдсан байдлаар ажиллах боломж олгодог бөгөөд хэрвээ таны ажил буруу чиглэлд явбал тухайн branch-ийг устгаад үндсэн codebase-д нөлөөлөхгүй байж болно.

Эдгээр боломжууд нь version control-ийг бусад developer-уудтай хамтран ажиллахад зайлшгүй чухал болгодог. Нэг project дээр олон хүн ажиллаж байгаа үед, branch-ууд нь хүн бүр өөрийн код дээр ажиллах боломж олгож, бие биенийхээ өөрчлөлтүүдийг зөрчихөөс сэргийлдэг.

Та ихэвчлэн Git зэрэг version control system-ийг GitHub эсвэл GitLab зэрэг version control provider-уудтай хамт ашигладаг бөгөөд эдгээр нь нэмэлт хамтын ажиллагааны хэрэгслүүд болон project-уудаа cloud дээр backup хийх боломжийг олгодог.

Гэхдээ cloud-д суурилсан хадгалалтын шийдэлгүй байсан ч version control system-ууд нь таны workflow-ийг бүрэн өөрчлөх (game changer) чадвартай. Одоо дахиж THIS-IS-THE-FINAL-COPY-FOREVER гэсэн файлын нэрс хэрэггүй болно.

Эдгээр системүүдийг ашиглах талаар маш олон сонирхолтой зүйлс бий, тиймээс дараагийн хэд хэдэн хичээл дээр илүү гүнзгий (deep dive) судалцгаая!
