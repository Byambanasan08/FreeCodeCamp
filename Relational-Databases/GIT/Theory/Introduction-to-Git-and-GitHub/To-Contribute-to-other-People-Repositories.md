# Та Бусдын Repository-д Хэрхэн Хувь Нэмэр Оруулах вэ?

Та өөрийн хяналтанд байдаг repository-д commit үүсгэж, тэдгээрийг push хийх талаар сурсан. Та мөн адил аргыг өөрийнх биш бусад төслүүд дээр ашиглаж болно, гэхдээ нэмэлт хэд хэдэн алхам байдаг.

Юуны өмнө, та ихэнхдээ өөр хүний төслийн "write access" (бичих эрх)-тэй байдаггүй. Энэ нь та commit-уудаа шууд тэдний repository руу push хийж чадахгүй гэсэн үг юм. Үүний оронд та тухайн repository-ийн хуулбарыг үүсгэж, түүн дээр ажиллах хэрэгтэй болно. Энэ процессыг "forking" гэж нэрлэдэг.

Repository-г fork хийхийн тулд GitHub дээр хувь нэмэр оруулахыг хүссэн repository руу орно. Дээд хэсэгт нь "Fork" товч харагдана. Үүнийг дарснаар та өөрийн fork-ийг үүсгэх UI (хэрэглэгчийн интерфейс) рүү шилжинэ.

Та өөрийн repository-г үүсгэж байсан үеийн дэлгэцтэй төстэй боловч илүү цөөн сонголттой дэлгэцийг харах болно. Та owner (эзэмшигч)-ийг сонгож болно, энэ нь таны хувийн аккаунт эсвэл таны хандах эрхтэй байгууллага байж болно, мөн fork-д шинэ нэр болон шинэ тайлбар өгч болно.

Мөн зөвхөн main branch (эсвэл repository-д анхдагчаар тохируулсан branch)-ийг хуулбарлах сонголт байдаг. Энэ нь ерөнхийдөө сайн сонголт байдаг, хэрэв та тодорхой feature branch-д хувь нэмэр оруулах шаардлагагүй бол. Та branch-уудын талаар дараагийн хичээлүүдэд илүү дэлгэрэнгүй сурах болно. Одоохондоо энэ сонголтыг идэвхтэй (tick-тэй) хэвээр үлдээгээрэй.

"Create fork" товчийг дарснаар таны шинэ repository үүсгэгдэнэ, мөн та автоматаар түүн рүү шилжинэ.

Дараа нь үүнийг өөрийн локал компьютер руу clone хийнэ, яг өөрийн repository-г clone хийдэг шиг, SSH эсвэл GitHub CLI ашиглан.

Одоо таны локал clone дээр тохируулагдсан remote-уудыг харцгаая. Локал repository-ийн хавтсанд terminal нээгээд git remote -v командыг ажиллуулж бүрэн жагсаалтыг авна, энэ нь дараахтай төстэй харагдана:

origin mailto:git@github.com:your-username/your-repo-name.git (fetch)
origin mailto:git@github.com:your-username/your-repo-name.git (pull)

GitHub-оос repository clone хийх үед Git автоматаар таны fork-ийг origin remote болгон тохируулсан байдаг. Гэхдээ fork дээр ажиллаж байх үед та ихэвчлэн өөрийн хуулбарыг анхны repository-ийн өөрчлөлтүүдтэй нийцүүлэх шаардлагатай болдог.

Үүнийг хийхийн тулд анхны repository-г өөр нэг remote болгон нэмье. Та repository-г fork хийсэн хуудсандаа буцаж очоод "Code" товчийг дарж, SSH URL-ийг хуулж авна. Дараа нь шинэ upstream remote үүсгэх командыг ажиллуулна:

git remote add upstream mailto:git@github.com:original-repo-username/repo-name.git

upstream нь remote-ийн нэр юм — та өөр нэр өгч болно, гэхдээ upstream нь нийтлэг хэрэглэгддэг стандарт нэр юм. Хэрэв та remote-уудаа дахин шалгавал шинэ upstream remote харагдана:

origin mailto:git@github.com:your-username/your-repo-name.git (fetch)
origin mailto:git@github.com:your-username/your-repo-name.git (pull)
upstream mailto:git@github.com:original-repo-username/your-repo-name.git (fetch)
upstream mailto:git@github.com:original-repo-username/your-repo-name.git (pull)

upstream remote-ийн тусламжтайгаар та git fetch upstream болон git merge upstream/main зэрэг командуудыг ашиглан анхны төслийн хамгийн сүүлийн өөрчлөлтүүдийг өөрийн локал fork-д татаж авч чадна.

Fork нь мөн л repository хэвээр байдаг бөгөөд та commit үүсгэж, өөрчлөлтүүдийг push хийхдээ өмнөхтэй адил аргыг ашиглана. Гэхдээ үүнд орохоос өмнө, та бусдын төсөл дээр өөрчлөлт хийхдээ дагах ёстой тодорхой стратегиуд байдгийг ойлгох хэрэгтэй. Та эдгээр стратегиудын талаар дараагийн хичээлүүдэд суралцах болно.

Эх кодоо олон нийтэд нээлттэй болгож, кодыг өөрчлөх боломж олгож, мөн кодын хувь нэмрийг хүлээн авдаг төслүүдийг "open source" (нээлттэй эхийн) төслүүд гэж нэрлэдэг. Open source-ийн олон давуу талууд байдаг, жишээлбэл олон нийтэд алдаа (bug)-г илрүүлж, засахад туслах боломж олгодог, хүмүүсийн таны ажилд оруулсан хувь нэмрээс суралцах боломж олгодог, мөн хөгжүүлэгчдийн нийгэмлэгийг (community) бий болгож, түүнийг сүлжээ (network) болгон өргөжүүлэх боломжтой.

Та хэн нэгний төсөл дээр өөрчлөлт хийх үед таныг "open source contributor" (нээлттэй эхийн хувь нэмэр оруулагч) гэж үздэг. Open source төслүүдэд хувь нэмэр оруулах нь мэргэжлийн хөгжүүлэгчдийн багт ажиллах ямар байдгийг урьдчилан харах маш сайн боломж болдог. Мөн энд сурсан стратегиуд таны анхны цалинтай ажилд шууд хэрэг болно.

Хэрэв танд хэрхэн хувь нэмэр оруулах талаар асуулт байвал, ихэнх open source төслүүдэд contributing guidelines (хувь нэмэр оруулах заавар) байдаг бөгөөд энэ нь танд хэрэгтэй бүх мэдээллийн тоймыг агуулсан байдаг. Гэхдээ хэрэв та одоо ч тодорхойгүй байвал, repository дээр issue нээж асуултаа асуухад хэзээ ч буруудахгүй — эсвэл хэрэв тэд Discord зэрэг платформ дээр community орчинтой бол тэнд нэгдэж асуултаа асууж болно.
