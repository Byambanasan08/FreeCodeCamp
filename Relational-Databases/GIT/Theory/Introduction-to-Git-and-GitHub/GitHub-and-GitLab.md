# GitHub болон GitLab зэрэг сайтуудын зорилго юу вэ, мөн тэдгээрийг ашиглахын зарим давуу талууд юу вэ?

Өмнөх хичээл дээр бид GitHub-ийг товчхон дурдсан. GitHub болон GitLab хоёулаа version control provider-ууд юм. Энэ нь тэд cloud-д суурилсан шийдлүүд бөгөөд version control-тэй project-уудыг "repositories" гэж нэрлэгдэх орчинд хадгалах, мөн эдгээр project-ууд дээр хамтран ажиллах боломжуудыг санал болгодог гэсэн үг.

Эдгээр платформуудын санал болгодог боломжууд руу гүнзгий орохоос өмнө, project-уудын төрлүүдийн талаар ярилцах хэрэгтэй. Олон project-ууд "open-source" гэж тооцогддог бөгөөд энэ нь хүмүүс таны нийтэлсэн кодыг харах, өөрчлөлт санал болгох, асуудлууд (issues) мэдээлэх, тэр ч байтугай өөрчилсөн хувилбарыг ажиллуулах боломжтой гэсэн үг.

Гэхдээ зарим тохиолдолд та зөвхөн өөрийн багийн гишүүдийг кодыг харах, дээр нь ажиллахыг хүсэж болно. Та мөн project-оо "closed-source" хэлбэрээр нийтэлж болно, энэ нь зөвхөн таны зөвшөөрсөн хүмүүс л project-ийг харах, түүнтэй харилцах боломжтой гэсэн үг.

Таны project аль ч аргыг сонгосон байсан, GitHub болон GitLab зэрэг сайтууд нь танай багийн хамтын ажиллагааг маш ихээр сайжруулж чадна.

Жишээлбэл, нэг багийн гишүүн feature дээр ажиллаж байхдаа гацаж, асуулттай байж болно. Кодыг Discord руу copy-paste хийх эсвэл screen share хийхийн оронд, тэр хүн өөрчлөлтүүдээ шууд GitHub дээр upload хийж болно. Тэндээс та browser-оор өөрчлөлтүүдийг харах эсвэл локал орчинд татаж авч ажиллуулах (run) болон debug хийх боломжтой.

Багийн гишүүн feature-ээ дуусгасны дараа, тэр өөрчлөлтүүдээ эдгээр сайтуудаар дамжуулан санал болгож (propose) болно. Та дараа нь эдгээр өөрчлөлтүүдийг review хийж, тодорхой файл эсвэл мөрүүд дээр comment үлдээж, санал бодол болон асуултаа илэрхийлж болно. Та бүр шууд browser дотор засварлах санал (suggested edits) гаргаж, түүнийг нь шууд хэрэгжүүлэх боломжтой!

Хэрвээ та өөрчлөлтүүдэд сэтгэл хангалуун байвал, тэдгээрийг хүлээн авч (accept) үндсэн branch руу merge хийж болно. Ингэснээр бүх багийн гишүүд хамгийн сүүлийн өөрчлөлтүүдийг татаж авч, дараагийн ажлаа эхлэхдээ хамгийн шинэ кодтой ажиллах боломжтой болно.

Task-уудын тухайд гэвэл, эдгээр сайтууд хүчирхэг issue tracker-уудыг санал болгодог. Та эдгээрийг ашиглан багийн гишүүдэд task оноох, олон нийтээс bug report хүлээн авах, мөн project board зэрэг хэрэгслүүдийг ашиглан бүтэн sprint-ийг төлөвлөх боломжтой.

Эдгээр сайтууд маш олон нэмэлт боломжуудыг санал болгодог. Дараагийн хэдэн хичээл дээр бид гол ойлголтуудыг авч үзэх бөгөөд ингэснээр та шууд ашиглаж эхэлж, хамтран ажиллаж чадна!
