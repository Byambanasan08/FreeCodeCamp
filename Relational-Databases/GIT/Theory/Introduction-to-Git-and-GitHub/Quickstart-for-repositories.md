# Репозиториудыг хурдан ашиглаж эхлэх заавар (Quickstart for repositories)

Шинэ репозитори (repository) үүсгэж, анхны өөрчлөлтөө 5 минутын дотор хэрхэн баталгаажуулах (commit) талаар суралцаарай. Репозитори үүсгэх
GitHub репозиториуд нь төрөл бүрийн төслүүдийг хадгалдаг. Энэхүү заавраар та репозитори үүсгэж, анхны өөрчлөлтөө баталгаажуулах болно.

1. Аль ч хуудасны баруун дээд буланд байрлах нэмэх тэмдэг дээр дарж, дараа нь "New repository" (Шинэ репозитори) дээр дарна уу.
2. Өөрийн репозиторид богинохон, тогтооход хялбар нэр өгнө үү. Жишээлбэл, "hello-world".
3. Сонголтоор, өөрийн репозиторийн тайлбарыг нэмж болно. Жишээлбэл, "GitHub дээрх миний анхны репозитори."
4. Репозиторийн харагдах байдлыг (visibility) сонгоно уу. Дэлгэрэнгүй мэдээллийг "About repositories" хэсгээс үзнэ үү.
5. "Add README" (README нэмэх) хэсгийг "On" (Асаах) болгож идэвхжүүлнэ үү.
6. "Create repository" (Репозитори үүсгэх) дээр дарна уу.

Баяр хүргэе! Та анхны репозиторио амжилттай үүсгэж, түүнийгээ README файлаар эхлүүллээ.

## Анхны өөрчлөлтөө баталгаажуулах (Commit your first change)

"Commit" (баталгаажуулалт) гэдэг нь тухайн цаг үеийн таны төслийн бүх файлуудын агшин зуурын зураг (snapshot) мэт юм.
Та шинэ репозиторио үүсгэхдээ үүнийг README файлаар эхлүүлсэн. README файлууд нь төслийнхөө талаар илүү дэлгэрэнгүй тайлбарлах эсвэл төслийг хэрхэн суулгах, ашиглах талаарх баримт бичгийг нэмэхэд тохиромжтой газар юм. Таны README файлын агуулга таны репозиторийн нүүр хуудсанд автоматаар харагдана.
Одоо README файлдаа өөрчлөлт оруулж баталгаажуулцгаая (commit).

1. Репозиторийнхоо файлын жагсаалтаас `README.md`-г сонгоно уу.
2. Файл харах цонхны баруун дээд буланд байрлах харандааны дүрс дээр дарж файл засварлагчийг нээнэ үү.
3. Текст хайрцагт өөрийнхөө тухай зарим мэдээллийг бичээрэй.
4. Шинэ агуулгын дээр байрлах "Preview" (Урьдчилан харах) дээр дарна уу.
5. Файлд оруулсан өөрчлөлтөө хянана уу. Хэрэв та "Show diff" (Ялгааг харуулах) гэдгийг сонговол шинэ агуулга ногоон өнгөөр харагдах болно.
6. "Commit changes..." (Өөрчлөлтүүдийг баталгаажуулах) дээр дарна уу.
7. "Commit message" (Баталгаажуулах мессеж) талбарт файлд оруулсан өөрчлөлтөө тодорхойлсон богинохон, утга учиртай мессеж бичнэ үү. Та уг баталгаажуулалтыг нэгээс олон зохиогчид хамааруулж болно. Дэлгэрэнгүй мэдээллийг "Creating a commit with multiple authors" хэсгээс үзнэ үү.
8. Баталгаажуулах мессежийн талбаруудын доор өөрийн баталгаажуулалтыг (commit) одоогийн салбар (branch) руу эсвэл шинэ салбар руу нэмэх эсэхээ шийднэ үү. Хэрэв таны одоогийн салбар нь үндсэн салбар (default branch) бол та баталгаажуулалтадаа зориулж шинэ салбар үүсгэж, дараа нь "pull request" (татгалзах хүсэлт) үүсгэхийг сонгох хэрэгтэй. Дэлгэрэнгүй мэдээллийг "Creating a pull request" хэсгээс үзнэ үү.
9. "Commit changes" (Өөрчлөлтүүдийг баталгаажуулах) эсвэл "Propose changes" (Өөрчлөлтүүдийг санал болгох) дээр дарна уу.
