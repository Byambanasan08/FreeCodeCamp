# Та Өөрийн Repository-д Хэрхэн Шинэ Branch Үүсгэх вэ?

Та өмнө нь өөрчлөлтүүдэд commit хийж, тэдгээрийг өөрийн remote repository руу push хийхийг сурсан, гэхдээ энэ бүхнийг та default main branch дээр хийж байсан. Энэ нь жижиг төсөлд ажиллаж болох ч, та бусад open source төслүүдэд хувь нэмэр оруулах үед эсвэл таны төсөл томрох үед branch-ууд маш чухал болдог.

Жишээлбэл, хэрэв та repository-г fork хийгээд өөрийн fork-ийн main branch дээр commit хийвэл, анхны repository хүлээн аваагүй өөрчлөлтүүд таны fork дээр үлдэж болно. Энэ нь таны branch-ийг анхны repository-оос зөвшөөрөгдсөн өөрчлөлтүүдтэй шинэчлэхийг маш хэцүү болгодог. Эсвэл өөрийн төсөл дээр ажиллах үед, хэрэв та branch ашиглахгүй бол өөр өөр ажлуудын хооронд шилжих нь илүү хэцүү болдог. Үүний оронд та аль өөрчлөлт аль feature эсвэл засварт (fix) хамаарахыг гараар хянах шаардлагатай болно.

Тэгвэл яг branch гэж юу вэ? Та өөрийн main branch-ийг нэгэн хурдны зам (highway) гэж төсөөл, commit бүр нь энэ зам руу орж буй машин юм. Та заримдаа шинэ feature үүсгэх гэх мэт ямар нэг зүйлийг шалгаж үзэхийг хүсэж магадгүй, тиймээс та түр хугацаанд замаас гарч, өөр тийш явж үзэх хэрэгтэй болно.

Branch гэдэг нь үндсэндээ таны main түүхээс (history) салсан нэгэн салаа зам бөгөөд та тэнд шинэ feature эсвэл bug fix дээр чөлөөтэй туршилт хийж болно. Хэрэв та өөрчлөлтдөө сэтгэл хангалуун байвал branch-ийг main-д merge хийж, буцаад хурдны зам руу орж болно. Хэрэв сэтгэл хангалуун биш байвал branch-ийг устгаад, өөрчлөлтүүдийг хүлээн зөвшөөрөлгүйгээр main руу буцаж шилжиж болно.

Шинэ branch үүсгэхээс өмнө, та git branch командыг ашиглан өөрийн branch-уудыг харах боломжтой. Гаралт нь дараах байдалтай байж болно:

- main

Одоогоор бидэнд зөвхөн нэг branch байна, энэ нь default main branch юм. Asterisk (\*) тэмдэг нь таны одоогоор "checked out" хийсэн branch-ийг илэрхийлдэг. Та үүний талаар удахгүй илүү дэлгэрэнгүй сурах болно.

Та git branch командыг ашиглан, шинэ branch-ийн нэрийг аргумент болгон өгч шинэ branch үүсгэж болно:

git branch feature

Энэ нь feature нэртэй шинэ branch үүсгэнэ. Одоо git branch командыг дахин ажиллуулж үзье:

feature

- main

Бид шинэ branch-ээ харж байна, гэхдээ одоогоор бид main branch дээрээ хэвээр байна. Бид шинэ branch руугаа шилжихийн тулд түүнийг "check out" хийх хэрэгтэй. Та шинэ branch-ийг check out хийх үед Git тухайн мөчид байсан файлуудын яг төлөвийг ачаална.

Шинэ branch-ээ дараах байдлаар check out хийе:

git checkout feature

эсвэл

git switch feature

Дараа нь git branch командыг дахин ажиллуулбал:

- feature
  main

Та одоо asterisk тэмдэг таны одоогоор "checked out" хийсэн branch руу шилжсэнийг анзаарах болно.

Ихэнх тохиолдолд та шинэ branch үүсгэхдээ шууд түүн рүү шилжихийг хүсдэг. Хоёр өөр команд ашиглан branch үүсгээд дараа нь шилжихийн оронд, та нэг команд ашиглан branch үүсгээд шууд шилжиж болно:

git checkout -b feature

эсвэл

git switch -c feature

Одоо бид шинэ branch үүсгээд түүн рүү шилжлээ. Энэ branch нь main branch-ийн хуулбар бөгөөд branch үүсгэх үед main дээр байсан бүх кодыг агуулж байдаг. Одоо бид энэ branch дээр дурын код нэмэх боломжтой бөгөөд энэ нь main branch-д нөлөөлөхгүй. Бид үндсэндээ шинэ feature үүсгэхийн тулд хурдны замаас гарсан байна. Хэрэв та git status командыг ажиллуулбал дараах байдалтай харагдана:

On branch feature
nothing to commit, working tree clean

Дараах командыг ажиллуулж feature.md нэртэй шинэ файл үүсгэе:

echo "This is our new feature" > feature.md

Одоо статус дараах байдалтай болно:

On branch feature
Untracked files:
(use "git add <file>..." to include in what will be committed)
feature.md

nothing added to commit but untracked files present (use "git add" to track)

Дараа нь шинэ feature-ээ commit хийх цаг боллоо. Эхлээд файлыг staging хэсэгт нэмнэ:

git add feature.md

Дараа нь өөрчлөлтүүдийг commit хийнэ:

git commit -m "my new feature"

Эцэст нь feature branch-аа өөрийн repository руу push хийнэ:

git push -u origin feature

-u flag нь таны локал feature branch-ийг remote branch-тэй холбодог тул дараагийн push-уудыг зөвхөн git push командаар хийх боломжтой болно.

Дараагийн хэсэгт бид шинэ feature-дээ pull request хэрхэн үүсгэхийг сурах болно!
