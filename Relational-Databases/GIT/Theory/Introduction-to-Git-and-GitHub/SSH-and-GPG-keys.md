# SSH болон GPG keys зэрэг аюулгүй байдлын шаардлагуудыг хэрхэн зохицуулах вэ?

Та өөрийн repository-г GitHub руу push хийхийн өмнө authenticate хийх шаардлагатай. SSH болон GPG key-ууд хоёулаа public-private key pair ашигладаг. Энэ нь та private key-ээ өөрийн локал машин дээр хадгалж, өөрийгөө баталгаажуулах (authenticate) үед ашиглана гэсэн үг. Харин public key-ийг таныг баталгаажуулах шаардлагатай хүмүүс эсвэл үйлчилгээтэй хуваалцана. Өөрөөр хэлбэл, private key нь үйлдэл хийхэд ашиглагддаг, харин public key нь тухайн үйлдлийг баталгаажуулахад ашиглагддаг.

GPG буюу Gnu Privacy Guard key-ууд нь ихэвчлэн файл эсвэл commit-уудыг гарын үсэг зурахад (sign) ашиглагддаг. Хэн нэгэн таны public GPG key-ийг ашиглан тухайн файлын гарын үсэг таны key-ээс ирсэн эсэх, мөн файлын агуулга өөрчлөгдөөгүй эсэхийг шалгаж болно. SSH буюу Secure SHell key-ууд нь ихэвчлэн ssh хэрэгслээр дамжуулан сервертэй remote холболт хийхэд authenticate хийхэд ашиглагддаг. Гэхдээ энэ хичээл дээр та SSH key-ийг commit гарын үсэг зурахад мөн ашиглаж болохыг сурах болно.

GPG key үүсгэхийн тулд дараах командыг ажиллуулна:

gpg --full-generate-key

SSH key үүсгэхийн тулд дараах командыг ажиллуулна:

ssh-keygen -t ed25519 -C "mailto:your_email@example.com"

Өөрийн жинхэнэ email-ээ оруулахаа мартуузай.

Эдгээр командууд хоёулаа key-ийг хамгаалахын тулд passphrase асуух болно. Энэ нь заавал биш боловч маш их зөвлөдөг.

SSH key-ийн хувьд хоёр файл үүснэ — нэг нь private key, нөгөө нь public key. Public key-ийн файлын нэр .pub өргөтгөлтэй байдаг бөгөөд ингэснээр private key-ээ санамсаргүйгээр хуваалцахаас сэргийлдэг. Эдгээр файлууд ~/.ssh directory-д байрлана.

Харин GPG key-ууд нь нууцлагдсан (hidden) бөгөөд шифрлэгдсэн directory-д хадгалагддаг. Тиймээс public key-уудаа жагсаахын тулд дараах командыг ажиллуулна:

gpg --list-secret-keys --keyid-format=long

Дараа нь public key-ээ авахын тулд:

gpg --armor --export "<key id>"

Гэхдээ эдгээр шинэ key-үүдээр та юу хийж болох вэ? Бид commit-уудыг гарын үсэг зурах талаар ярьсан бөгөөд энэ нь эдгээр key-үүдийн аль алинаар нь хийж болох зүйл юм.

GPG key ашиглан commit-уудаа гарын үсэг зурахын тулд, та private key биш public key-ээ GitHub account руугаа upload хийх хэрэгтэй. Дараа нь өмнө нь key-үүдээ жагсаахад авсан short ID-гаа ашиглан дараах командыг ажиллуулж git signing key болгон тохируулна:

git config --global user.signingkey <your_gpg_key_id>

Дараа нь тодорхой commit-ийг гарын үсэг зурахын тулд git commit команд дээр -S flag-ийг ашиглана — энэ үед та passphrase-ээ оруулах шаардлагатай. Эсвэл бүх commit-уудыг автоматаар гарын үсэг зуруулахыг хүсвэл autosign тохиргоог true болгоно:

git config --global commit.gpgsign true

SSH key ашиглан гарын үсэг зурах (энэ нь GitHub дээр харьцангуй шинэ боломж) бол эхлээд key-ээ GitHub account руу upload хийх хэрэгтэй. Дараа нь git-ийн signing mode-ийг SSH ашиглахаар тохируулна:

git config --global gpg.format ssh

Дараа нь signing key-ийг тохируулахдаа ID биш, файлын замыг ашиглана:

git config --global user.signingkey <path_to_your_ssh_keys>

Эцэст нь, GPG key-тэй адил auto signing-ийг идэвхжүүлж болно.

Эдгээр аргуудын аль нэгийг тохируулсны дараа, таны commit-ууд баталгаажсан (verified) болох бөгөөд GitHub дээр тусгай тэмдэг (badge)-тэй харагдах болно.
