# GitHub Pages ашиглан төслийг хэрхэн deploy хийх вэ?

GitHub Pages нь static сайтууд, эсвэл логик боловсруулахад back-end сервер шаарддаггүй application-уудыг deploy хийхэд маш сайн сонголт юм. Өөрөөр хэлбэл, бүхэлдээ client-side дээр (эсвэл хэрэглэгчийн browser дотор) ажилладаг application-уудыг энэ платформ дээр бүрэн deploy хийж болно.

Нэмж хэлэхэд, GitHub Pages нь open source төслүүдийн хувьд бүрэн үнэгүй байдаг. Төслөө open source болгох олон шалтгаан байдаг бөгөөд Pages нь тэдгээрт дахин нэг давуу тал нэмдэг.

Гэхдээ GitHub Pages руу яг яаж deploy хийх вэ? Энэ нь таны application хэрхэн ажиллаж байгаагаас хамаарна.

Эхлээд энгийн HTML хуудсаас эхэлье. Хэрвээ хүсвэл дараах кодыг өөрийн одоо байгаа repository-д нэмж болно:

<!DOCTYPE html>

<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>freeCodeCamp-ийн хичээлүүдэд тавтай морилно уу!</h1>
    <p>Git болон GitHub-ийн талаар суралцахдаа хөгжилтэй байцгаая!</p>
  </body>
</html>

Одоо бид repository руугаа орж, "Settings" tab дээр дарж repository-ийн тохиргоо руу орно. Дараа нь sidebar-аас "Pages"-ийг сонгоно.

Эхэндээ цөөн хэдэн сонголт харагдана, гэхдээ тохиргоонуудыг хийж эхлэхэд энэ өөрчлөгдөнө. Эхлээд бид GitHub-д юуг deploy хийхийг зааж өгөх хэрэгтэй.

Бид үүнийг "Deploy from a branch" хэвээр нь үлдээнэ, учир нь бид framework ашиглаагүй бөгөөд ямар нэгэн GitHub actions тохируулаагүй байна. Та эдгээрийн талаар дараагийн хичээлд илүү дэлгэрэнгүй судлах болно.

Дараа нь deploy хийх branch-ийг main болгож, түүний агуулгыг deploy хийнэ.

Үүний дараа та төслийн root folder-оос эсвэл docs folder-оос deploy хийх сонголттой болно. Энэ боломжийг freeCodeCamp зэрэг төслүүд ашигладаг — бид өмнө нь documentation-ийг app-тайгаа хамт байршуулдаг байсан тул зөвхөн documentation-ийг docs folder-оор дамжуулан GitHub Pages дээр deploy хийх боломжтой байсан. Одоохондоо үүнийг / (root) folder хэвээр нь үлдээгээд, энэ drop-down-ийн хажууд байгаа "Save" товчийг дарж өөрчлөлтүүдээ хадгална.

Та хадгалсны дараа custom domain тохируулах шинэ input гарч ирнэ. Хэрвээ танд аль хэдийн domain байгаа бөгөөд ашиглахыг хүсвэл docs руу орох холбоосыг дагаж тохируулж болно. Харин энэ хичээлийн хувьд бид үүнийг хоосон үлдээнэ.

Мөн таны шинэ GitHub Pages сайт deploy хийгдсэн domain-ийг харах боломжтой. Custom domain ашиглаагүй тохиолдолд энэ нь анхдагчаар <username>.github.io байна, энд <username> нь таны account-ийн нэр (эсвэл хэрвээ organization-ийн repository ашиглаж байгаа бол тухайн organization-ийн нэр) юм. Мөн repository нь тухайн repository-ийн нэртэй ижил path дээр байрлана, жишээ нь <username>.github.io/<repo-name> гэх мэт. Та энэ URL руу орвол өөрийн сайтаа харах болно!

Одоо та main branch руу шинэ өөрчлөлтүүдийг push хийхэд GitHub Pages автоматаар таны deploy хийгдсэн сайтыг хамгийн сүүлийн өөрчлөлтүүдээр шинэчилнэ. Энэ процесс нь серверийн ачааллаас хамаарч хэдэн минут үргэлжилж болох бөгөөд cache-ийг цэвэрлэхийн тулд hard refresh хийх шаардлагатай байж магадгүй.

Ингээд бид энгийн вэб application-ийг GitHub Pages дээр амжилттай deploy хийлээ. Та мөн CSS болон JavaScript файлуудыг Pages deployment-д оруулж болно, ингэснээр илүү гоё, интерактив сайтууд бүтээх боломжтой. Мөн React эсвэл Angular зэрэг framework-уудыг сурмагц тэдгээрийг ч GitHub Pages дээр deploy хийх боломжтой!
