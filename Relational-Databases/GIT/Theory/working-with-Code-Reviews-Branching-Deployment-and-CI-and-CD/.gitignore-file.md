# .gitignore файл гэж юу вэ, мөн дотор нь ямар зүйлсийг оруулах ёстой вэ?

Бид өмнөх хичээлд .gitignore файлыг товчхон дурдсан. Гэхдээ энэ файл яг юу вэ, мөн дотор нь юу оруулах ёстой вэ?

.gitignore файл нь Git үйлдлүүдтэй холбоотой тусгай төрлийн файл юм. Нэрнээсээ харахад энэ файл нь Git-д ямар нэг зүйлсийг ignore хийхийг хэлдэг мэт боловч, энэ нь түгээмэл хэрэглээ нь юм. Харин үнэндээ энэ нь Git-д тухайн файлыг tracking хийхээ зогсоохыг хэлдэг. Энэ ялгаа нь маш чухал бөгөөд дараа нь илүү тодорхой болно.

Тэгвэл яг ямар зүйлсийг энэ файлд оруулах ёстой вэ? Та commit хийхийг хүсэхгүй байгаа зүйлсийг жагсаах хэрэгтэй.

Юуны өмнө, таны secrets файлууд орно. Энэ нь жишээлбэл .env файл байж болно, энд таны application ашиглах API key-ууд хадгалагдсан байдаг. Мөн таны application authentication-д ашигладаг хувийн GPG key файл байж болно. Олон нийтэд ил болохыг хүсэхгүй sensitive мэдээлэл агуулсан бүх зүйлсийг .gitignore ашиглан ignore хийх ёстой. Таны repository private байсан ч гэсэн та үүнийг public мэтээр хандах хэрэгтэй — хэн ч харж чадна гэж үзээд secrets-ийг commit хийхээс зайлсхий.

Одоо манай файл ямар харагдаж болохыг харцгаая:

# Secrets

.env
github.key

Бид Git-д .env файлыг ignore хийхийг хэлсэн, энд бид secrets-ээ хадгалж байна. Мөн github.key файлыг ignore хийхийг хэлсэн, энэ нь хувийн GPG key агуулж байна.

Дараагийн анхаарах зүйл бол таны төсөл ашиглаж буй packages буюу dependencies юм. Жишээ нь, та Node.js төсөл дээр ажиллаж байхад npm-ээс packages суулгадаг. Эдгээр packages нь таны төсөл доторх node_modules folder-д ордог бөгөөд таны код тэдгээрийг import хийж ашигладаг.

Гэхдээ node_modules folder нь маш том болдог бөгөөд та эдгээр бүх packages-ийг commit хийж, шинэчлэлт бүрт tracking хийхийг хүсэхгүй. Үүний оронд та package.json файл болон холбогдох lockfile-ийг (жишээ нь package-lock.json, pnpm-lock.yaml) commit хийдэг. Эдгээр файлууд нь npm-д ямар packages, ямар version суулгахыг зааж өгдөг тул бусад contributor-ууд таны ашиглаж буй dependencies-ийг яг ижил байдлаар суулгаж чадна, repository-д өөрсдийг нь commit хийх шаардлагагүй.

Одоо node_modules-ийг .gitignore файлд нэмье:

# Secrets

.env
github.key

# Packages

node_modules

Одоо .gitignore файлын нэг чухал syntax-ийн талаар ярья. Та node_modules шиг entry нэмэхэд Git тухайн pattern-т таарах бүх зүйлийг ignore хийнэ. Энэ нь таны төслийн root дээрх node_modules folder-ийг ignore хийхээс гадна, төслийн client directory доторх node_modules-ийг ч мөн ignore хийнэ гэсэн үг.

Хэрвээ та зөвхөн root дээрх зүйлийг ignore хийхийг хүсвэл мөрийн эхэнд forward slash (/) тавьж болно. Тэгэхээр node_modules нь /node_modules болно. Ингэснээр зөвхөн root дахь node_modules directory-ийг ignore хийнэ, харин client/node_modules directory-ийг ignore хийхгүй. Бидэнд энэ хэрэггүй тул entry-гээ зүгээр node_modules хэвээр нь үлдээнэ.

Дараагийн ignore хийх ёстой зүйл бол таны build outputs юм. Жишээ нь, TypeScript төсөл compile хийхэд JavaScript output бүхий dist folder үүсч болно. Та энэ output-ийг commit хийхийг хүсэхгүй, учир нь энэ нь Git history-г хэт томруулдаг. Үүний оронд та үүнийг ignore хийх хэрэгтэй — contributor-ууд ижил build command ажиллуулж өөрсдөө үүсгэж чадна.

Одоо dist folder-ийг ignore хийе:

# Secrets

.env
github.key

# Packages

node_modules

# Outputs

dist

Заримдаа буруу байрласан файл TypeScript compiler-ийг эвдэж, JavaScript output хаа сайгүй тархахад хүргэдэг. Бид зөвхөн TypeScript бичиж байгаа тул glob syntax ашиглан бүх JavaScript файлуудыг ignore хийе:

# Secrets

.env
github.key

# Packages

node_modules

# Outputs

dist
\*.js

Мөн таны ignore хийж болох нийтлэг output-ууд нь coverage directory (unit test coverage-ийн үр дүн), хэрвээ Angular ашиглаж байвал .angular directory, хэрвээ Next.js ашиглаж байвал .next directory гэх мэт орно.

Одоохондоо бидэнд байгаа энэ тохиргоо хангалттай. Хэрвээ та файл үүсгэдэг framework ашиглаж байгаа бол тухайн framework-ийн documentation-оос .gitignore-д юу нэмэх ёстойг шалгах хэрэгтэй.

Дөрөв дэх ignore хийх зүйлсийн бүлэг бол IDE файлууд юм. Жишээ нь Jetbrains-ийн бүтээгдэхүүнүүд IDE-д зориулсан тохиргоонуудыг хадгалах .idea folder ашигладаг. Харин Visual Studio Code нь .vscode folder ашигладаг.

Эдгээрийг ignore хийх нь ихэвчлэн зөв байдаг, учир нь эдгээр нь таны local орчны тусгай тохиргоонуудыг агуулдаг бөгөөд бусад contributor-уудад хэрэггүй (тэд өөрсдийн тохиргоог ашигладаг). Бидний IDE бол Visual Studio Code тул .vscode directory-ийг нэмье:

# Secrets

.env
github.key

# Packages

node_modules

# Outputs

dist
\*.js

# IDE

.vscode

Гэхдээ манай .vscode folder дотор settings.json файл байгаа бөгөөд энэ нь workspace-д хамаарах зарим тохиргоонуудыг агуулдаг тул contributor-уудад хэрэгтэй байж магадгүй — энэ файл нь тэдэнд extension болон орчноо манайхтай төстэй байдлаар тохируулах боломж олгодог.

Тэгэхээр одоо negation syntax ашиглан тухайн файлыг “unignore” хийе:

# Secrets

.env
github.key

# Packages

node_modules

# Outputs

dist
\*.js

# IDE

.vscode
!.vscode/settings.json

Хэрвээ та MacOS ашигладаг бол .DS_STORE файлыг мэддэг байх. Энэ файл нь Finder-ийн ашигладаг metadata-г агуулдаг бөгөөд directory-г хэрхэн харуулах (жишээ нь thumbnail-тай эсвэл жагсаалт хэлбэрээр) зэргийг тодорхойлдог. Үүнтэй адил Windows хэрэглэгчид Thumbs.db гэсэн нууц (hidden) файл харж магадгүй, энэ нь directory-ийн thumbnail-уудыг cache хийдэг.

Ерөнхийдөө эдгээр файлуудыг ignore хийх нь зөв байдаг, учир нь эдгээр нь хувь хүний тохиргоо бөгөөд төслийн ажиллагаанд ямар ч нөлөө үзүүлэхгүй. Одоо хоёуланг нь нэмье:

# Secrets

.env
github.key

# Packages

node_modules

# Outputs

dist
\*.js

# IDE

.vscode
!.vscode/settings.json

# System Files

.DS_STORE
Thumbs.db

Өмнө нь commit хийсэн файлуудыг ignore хийх тухай товч тэмдэглэл. Жишээ нь, та санамсаргүйгээр өмнө нь dist directory-ийг commit хийсэн байж болох ч одоо үүнийг устгаж, ignore хийхийг хүсэж байна гэж бодъё. Та зүгээр л .gitignore файлд нэмээд болчихгүй — энэ нь өмнө commit хийгдсэн контентыг алга болгохгүй, зөвхөн Git-д тухайн файлыг tracking хийхээ зогсоохыг хэлдэг. Тиймээс аль хэдийн commit хийгдсэн файлыг ignore хийвэл Git шинэ өөрчлөлтүүдийг track хийхгүй, гэхдээ тухайн файлын устгал (deletion)-ийг ч track хийхгүй.

Үүний оронд эхлээд тухайн файлыг устгах хэрэгтэй. Тэр устгасан үйлдлээр commit үүсгэнэ. Дараа нь .gitignore-д тухайн зүйлийг нэмээд, дахин commit хийнэ. Үүнээс хойш тэр файл дахин үүсэх үед Git үүнийг ignore хийх бөгөөд commit-ууд дотор харагдахгүй болно.

Эцэст нь, хэрвээ та Python зэрэг өөр хэл дээр төсөл хийж байгаа бол ignore хийх зүйлс нь энд ярьснаас өөр байна. GitHub нь төрөл бүрийн хэл, framework-д зориулсан .gitignore template-ууд бүхий маш хэрэгтэй repository-г хадгалдаг. Та энэ жагсаалтыг https://github.com/github/gitignore хаягаас олж болно.

Ингээд та repository-дээ хэрэггүй файлуудыг commit хийхээс зайлсхийхэд бэлэн боллоо!
