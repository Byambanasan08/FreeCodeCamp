# Command options болон flags гэж юу вэ?

Command options болон flags-ийн талаар судалцгаая.

Өмнөх хичээл дээр та touch readme.md гэх мэт командуудад аргумент (arguments) дамжуулах талаар сурсан. Options буюу flags нь командын ажиллах хэлбэрт нөлөөлдөг тусгай аргументууд юм. Эдгээр хоёр нэр томьёог ихэвчлэн ижил утгаар ашигладаг боловч "flags" гэдэг нь ихэвчлэн асаах/унтраах (on/off toggle) үүрэгтэй option-уудыг илүү тодорхойлж хэлдэг.

Options нь ихэвчлэн нэг эсвэл хоёр зураас (-) тэмдгээр эхэлдэг бөгөөд энэ нь option болон аргументыг хооронд нь харагдахаар ялгахад тусалдаг. Эхлээд хоёр зураастай буюу "long form" syntax-ийг харцгаая.

Олон application-ууд, жишээлбэл ls, нь --version flag-ийг дэмждэг бөгөөд энэ нь application-ийн командуудыг ажиллуулахын оронд тухайн application-ийн одоогийн хувилбарыг (version) terminal дээр хэвлэж харуулдаг.

Мөн олон application-ууд --help flag-ийг санал болгодог бөгөөд энэ нь application-ийг хэрхэн ашиглах зааврыг хэвлэж харуулдаг.

Нэг зураастай буюу "short form" syntax нь ихэвчлэн нэг үсэг бүхий option-уудыг ашигладаг. Жишээлбэл, ls команд дээрх -a flag нь бүх файлуудыг, тэр дундаа .env гэх мэт цэгээр (.) эхэлдэг hidden file-уудыг жагсааж харуулдаг.

Эдгээр богино option-уудын давуу тал нь та олон flag-ийг хооронд нь холбож (chain) ашиглах боломжтой. ls --all --human-readable --size гэж бичихийн оронд бүх нэг үсгүүдийг нийлүүлж ls -ahs гэж ашиглаж болно.

Зарим option-ууд өөртөө утга (value) дамжуулахыг шаарддаг.

Long form option ашиглах үед, ихэвчлэн тэнцүүгийн тэмдэг (=) ашиглах шаардлагатай байдаг. Энэ syntax-д value нь option-той тэнцүүгийн (=) тэмдгээр шууд холбогддог. Жишээлбэл, ls-ийн үйлдлийг өнгө оруулах эсвэл оруулахгүй байхаар өөрчлөх:

ls --color=never

Short form option ашиглах үед, ихэвчлэн value-ийг зайгаар тусгаарладаг. Жишээлбэл, ls-ийн үр дүнгийн өргөнийг тохируулах long болон short хэлбэрүүд:

ls -w 50

ls --width=50

Энд short form нь -w 50 гэж бичигдэж байхад, long form нь --width=50 гэж бичигдэж байгааг анзаараарай. Энэ ялгааг ойлгох нь чухал бөгөөд ингэснээр option-ийн value-ийг positional argument болгон буруу дамжуулахгүй.

Гэхдээ хэрвээ та эргэлзэж байвал, ихэнхдээ --help flag-ийг ашиглан option-уудын зөв syntax-ийг харах боломжтой гэдгийг санаарай!
