# Dependency Management гэж юу вэ, мөн энэ нь React зэрэг сангууд (libraries)-тай хэрхэн ажилладаг вэ?

Програм хангамжид (software), `dependency` гэдэг нь аппликейшний нэг `component` эсвэл `модуль` өөр нэг дээр тулгуурлан зөв ажиллах шаардлагатай байхыг хэлдэг. Dependency-ууд нь програм хангамжид түгээмэл байдаг, учир нь энэ нь хөгжүүлэгчдэд (developers) бусдын бүтээсэн бэлэн функцүүд эсвэл хэрэгслүүдийг ашиглах боломж олгодог.

Хэрэв та React аппликейшн бүтээхийг хүсвэл, React dependency-уудыг суулгах шаардлагатай. Эдгээр dependency-уудгүйгээр таны код огт зөв ажиллахгүй бөгөөд аппликейшн алдаануудын (errors) жагсаалтыг харуулна.

Илүү төвөгтэй (complex) төслүүд дээр ажиллах үед та олон dependency-д тулгуурлах шаардлагатай болдог. Хэрэв dependency-уудыг сайн удирдаж (managed) эсвэл тодорхойлоогүй (defined) бол `dependency hell` гэж нэрлэгдэх асуудал үүсдэг.

Төсөл дотор програм хангамжийн dependency-уудыг удирдахын тулд та _package manager_ ашиглах хэрэгтэй. `Package manager` гэдэг нь dependency-уудыг суулгах (installation), шинэчлэх (updates), болон устгах (removal)-д ашиглагддаг хэрэгсэл юм. JavaScript, Python, Ruby, Java зэрэг олон алдартай програмчлалын хэлүүд бүгд `package manager` ашигладаг.

Өмнөх хичээлд та `npm` гэж нэрлэгдэх нэг алдартай package manager-тэй товч танилцсан.

Vite болон npm ашиглан шинэ React төсөл үүсгэхийн тулд та дараах командыг ажиллуулж болно:

`npm create vite@latest my-react-app -- --template react`

Өмнөх хичээлүүдээс санаж байгаачлан, энэ нь шинэ React аппликейшн эхлүүлэхэд шаардлагатай бүх `boilerplate` кодыг үүсгэнэ. Аппликейшнийг ажиллуулахын өмнө та dependency-уудыг суулгахын тулд `npm install` эсвэл товчоор `npm i` командыг ажиллуулах хэрэгтэй.

Та бүх dependency-уудыг package.json файл дотор харах боломжтой бөгөөд энэ нь таны төслийн root directory-д байрладаг.

`package.json` файл нь төслийн тухай мета өгөгдлийг (metadata) агуулдаг гол тохиргооны (configuration) файл бөгөөд үүнд нэр (name), хувилбар (version), болон dependency-ууд ордог. Мөн энэ нь script-үүд, лицензийн мэдээлэл (licensing information), болон төслийг болон түүний dependency-уудыг удирдахад туслах бусад тохиргоонуудыг тодорхойлдог.

Dependency-уудыг суулгасны дараа таны төсөлд `node_modules` нэртэй хавтас нэмэгдэнэ.

`node_modules` хавтас нь таны төсөлд шаардлагатай бүх package болон сангууд (libraries)-ыг хадгалдаг газар юм. Энэ хавтас нь package.json файлд жагсаасан dependency-уудын бодит кодыг агуулдаг бөгөөд үүнд таны төслийн шууд dependency-ууд болон тэдгээрийн dependency-ууд (dependencies of dependencies) бүгд орно.

React төсөлд хэрэгтэй хоёр үндсэн dependency нь react болон react-dom package-ууд юм:

"dependencies": {
"react": "^18.3.1",
"react-dom": "^18.3.1"
}

package.json файл нь эдгээр package-уудын таны ашиглаж буй одоогийн хувилбаруудыг (versions) харуулна. Хэрэв та ямар нэг package-ийг локал дээр шинэчлэх шаардлагатай бол `npm update` командыг ажиллуулж болно. Эсвэл бүх package-уудыг глобал (global) байдлаар шинэчлэхийн тулд `npm update -g ` командыг ашиглаж болно.

package.json файлаас гадна танд `package-lock.json` файл мөн байдаг. Энэ файл нь таны төсөл ашиглаж буй бүх package-уудын яг (exact) хувилбаруудыг түгжиж (lock) өгдөг. Хэрэв та ямар нэг package шинэчилбэл, шинэ хувилбарууд lock файлд мөн шинэчлэгдэнэ.

`package.json` файлын өөр нэг чухал хэсэг бол `dev dependencies` юм:

"devDependencies": {
"@eslint/js": "^9.17.0",
"@types/react": "^18.3.18",
"@types/react-dom": "^18.3.5",
"@vitejs/plugin-react": "^4.3.4",
"eslint": "^9.17.0",
"eslint-plugin-react": "^7.37.2",
"eslint-plugin-react-hooks": "^5.0.0",
"eslint-plugin-react-refresh": "^0.4.16",
"globals": "^15.14.0",
"vite": "^6.0.5"
}

`Dev dependencies` нь зөвхөн хөгжүүлэлтийн (development) үед ашиглагддаг бөгөөд production дээр ашиглагддаггүй package-ууд юм. Үүний жишээ нь Jest гэх мэт тестийн сан (testing library) юм. Та Jest-ийг dev dependency байдлаар суулгана, учир нь та үүнийг зөвхөн төслийг локал дээр тестлэхэд ашигладаг бөгөөд `production` дээр аппликейшн ажиллахад шаардлагагүй.

Энэ хичээлийн ихэнх хэсэгт бид `npm `дээр төвлөрсөн. Гэхдээ `yarn` болон `pnpm` зэрэг өөр package manager-ууд бас байдаг. Тэгэхээр аль package manager-ийг төсөлдөө ашиглах ёстой вэ?

Товч хариулт бол: нөхцөл байдлаас хамаарна.

Хэрэв та багтайгаа ажиллаж буй одоо байгаа (existing) төсөлд нэгдэж байгаа бол эдгээр шийдвэрүүд аль хэдийн гарсан байдаг тул та тухайн төслийн ашиглаж буй `package manager`-ийг ашиглана. Харин хэрэв та төслийг эхнээс нь (from scratch) бүтээж байгаа бол manager бүрийн давуу болон сул талуудыг (pros and cons) судалж, аль нь таны хэрэгцээнд илүү тохирохыг шийдэх хэрэгтэй.
