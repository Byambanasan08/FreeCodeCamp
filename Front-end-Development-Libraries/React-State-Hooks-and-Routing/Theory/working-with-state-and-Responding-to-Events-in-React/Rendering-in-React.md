## React дээр Rendering гэж юу вэ, мөн Component-ууд дэлгэц дээр хэрхэн харагддаг вэ?

React-д `rendering` гэдэг нь `component`-ууд хэрэглэгчийн интерфэйс (user interface, UI), ихэвчлэн браузер дээр харагдах процесс юм.

React нь таны бүх `JavaScript`, `JSX`, болон `CSS` кодыг авч, хэрхэн харагдах ёстойг тодорхойлоод дараа нь хэрэглэгчийн интерфэйс дээр харуулдаг.

`Rendering` процесс нь `trigger`, `render`, болон `commit` гэсэн гурван үе шатнаас бүрдэнэ. Эдгээрийг илүү дэлгэрэнгүй харцгаая.

1. `Trigger` үе шат нь `React` ямар нэг зүйл өөрчлөгдсөнийг илрүүлж, хэрэглэгчийн интерфэйсийг шинэчлэх шаардлагатай байж магадгүй гэж үзэх үед үүсдэг. Энэ өөрчлөлт нь ихэвчлэн `state` эсвэл `props`-ийн шинэчлэлтээс болдог.

Жишээлбэл, оройн хоолны цаг болсныг анзаарах нь таныг гал тогоо руу орж хоол хийж эхлэхэд хүргэж болох `trigger` юм.

Доорх Counter жишээнд `increment` эсвэл `decrement` товчийг дарах нь React-д шинэ count утгыг харуулах `trigger` болдог:

```
import { useState } from "react";

function Counter() {
const [count, setCount] = useState(0);

return ( <div> <h1>{count}</h1>
<button onClick={() => setCount(count - 1)}>Decrement</button>
<button onClick={() => setCount(count + 1)}>Increment</button> </div>
);
}
```

export default Counter;

2. Trigger үүссэний дараа React `render` үе шат руу ордог. Энд React таны component-уудыг дахин үнэлж (re-evaluate), юу харуулах ёстойг тодорхойлдог.

Үүнийг хийхийн тулд React нь “жинхэнэ” DOM-ийн хөнгөн хуулбарыг (lightweight copy) ашигладаг. Үүнийг `virtual DOM` гэж нэрлэдэг. `Virtual DOM` ашигласнаар `React component` дотор юу өөрчлөгдөх шаардлагатайг хурдан шалгаж чаддаг.

Энэ үе шатыг та гал тогоонд орж, орцуудаа бэлдээд хоолоо хийж байгаа мөчтэй зүйрлэж болно.

Counter component-ийн хувьд render үе шат нь React шинэ count утгатайгаар функцуудыг дахин ажиллуулж байгаа үе юм. React <h1>{count}</h1> хэсэг шинэ count утгад үндэслэн хэрхэн харагдахыг дахин тооцоолдог боловч дараагийн үе шат болох `commit` хүртэл та дэлгэц дээр ямар нэг өөрчлөлт харахгүй.

3. `Commit` үе шат нь `React virtual DOM` дээр бэлтгэсэн өөрчлөлтүүдийг авч “жинхэнэ” DOM-д хэрэгжүүлэх үе шат юм. Өөрөөр хэлбэл, энэ нь та дэлгэц дээр эцсийн үр дүнг харах үе юм.

Үүнийг хэрэгжүүлэхийн тулд `React virtual DOM`-ийг бодит `DOM`-тэй харьцуулж, зөвхөн шинэчлэлт шаардлагатай хэсгүүдийг тодорхойлоод тэдгээр өөрчлөлтийг бодит DOM-д хэрэгжүүлж хэрэглэгчийн интерфэйсийг шинэчилдэг.

Энэ үе шатыг та хийсэн хоолоо ширээн дээр тавьж, харагдах болгохтой адилтгаж ойлгож болно, яг React бодит DOM-д шинэчлэлт commit хийх үед шиг.

Counter component-ийн хувьд `commit` үе шат нь шинэ count утгыг h1 элементэд хэрэгжүүлж, та хуудсан дээр өөрчлөлтийг харах үе юм.

Эдгээр `гурван процесс` маш хурдан явагддаг, учир нь React эхлээд `virtual DOM `дээр өөрчлөлтүүдийг тооцоолж, дараа нь бодит DOM дээр зөвхөн өөрчлөгдөх шаардлагатай хэсгүүдийг л шинэчилдэг тул шууд DOM-ийг их хэмжээгээр өөрчлөхөөс (direct DOM manipulation) зайлсхийдэг.
