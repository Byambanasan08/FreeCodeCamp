# Гүйцэтгэлийг (Performance) хэмжих гол хэмжүүрүүд (Key Metrics) юу вэ?

Веб гүйцэтгэлийг хэмжих болон сайжруулахын тулд бид First Contentful Paint (FCP), Speed Index, Total Blocking Time (TBT), Bounce Rate, болон Unique Users зэрэг гол гүйцэтгэлийн хэмжүүрүүдийг ашигладаг. Эдгээр хэмжүүрүүдийг тус бүрээр нь авч үзье.

Эхлээд First Contentful Paint буюу FCP. Энэ нь текст эсвэл зураг гэх мэт анхны контент дэлгэц дээр хэр хурдан гарч ирж байгааг хэмждэг. Сайн FCP нь 1.8 секундээс доош байхыг хэлдэг, харин муу FCP нь ихэвчлэн 3 секундээс дээш байдаг. Та FCP-ээ Chrome DevTools ашиглан дараах байдлаар шалгаж болно:

Алхам 1: Chrome DevTools-ийг нээх

```
Google Chrome-ийг нээх.

Шалгахыг хүссэн веб хуудсандаа орох.

"F12" товч дарах эсвэл Right-click хийгээд "Inspect" сонгох.
```

Алхам 2: DevTools дотор "Lighthouse" табыг олж, дарна.

Алхам 3: "Analyze page load" товчийг дарж, шинжилгээ (analyzation) дуусахыг хүлээнэ.

Одоо бид үр дүнг шинжилж болно. Үүнийг хийхдээ "Metrics" хэсэг рүү доош гүйлгээд "First Contentful Paint" гэсэн үзүүлэлтийг хайна. Тэнд яг миллисекунд (milliseconds) эсвэл секундээр хэмжигдсэн хугацааг харж болно.

FCP болон Interaction to Next Paint (INP) — бидний өмнө үзсэн — хоёулаа гүйцэтгэлийн хэмжүүрүүд боловч хэрэглэгчийн туршлагын өөр өөр талыг хэмждэг. Тэдгээрийг андуурахгүй байх нь чухал.

Сануулбал, FCP нь контент хэрэглэгчид хэр хурдан харагдаж эхэлж байгааг хэмждэг, харин INP нь хэрэглэгч дарж эсвэл бичсэний дараа feedback хэр хурдан харагдаж байгааг хэмждэг.

Дараагийнх нь Speed Index.

Speed Index нь хуудсын харагдах хэсгүүд (visible parts) хэр хурдан рэндэрлэгдэж (painted) байгааг хэмждэг. Оноо бага байх тусам гүйцэтгэл сайн байна гэсэн үг! Та Speed Index-ийг Lighthouse Reports дээрээс шалгаж болно.

Мөн Total Blocking Time буюу TBT байдаг. Энэ нь main thread heavy JavaScript task-уудаас болж хэр удаан блоклогдож байгааг харуулдаг. Хэрэв TBT өндөр байвал хэрэглэгчид lag-тай (laggy) харилцан үйлдэл мэдэрдэг. TBT-г сайжруулахын тулд урт task-уудыг хувааж (break up) хийх, мөн чухал биш скриптүүдийг defer хийх хэрэгтэй.

Дараагийнх нь Bounce Rate. Энэ нь ямар ч харилцан үйлдэл хийхгүйгээр гарч буй хэрэглэгчдийн хувь юм. Хэрэв таны сайт өндөр bounce rate-тэй бол, энэ нь таны хуудас хэт удаан байгаатай холбоотой байж болно.

Эцэст нь Unique Users. Энэ хэмжүүр нь танай сайт руу хэдэн өвөрмөц (individual) хэрэглэгч орж ирж байгааг хянадаг. Bounce rate болон unique users-ийг харахын тулд та Google Analytics ашиглаж болно. Энэ нь танд эдгээр хэмжүүрүүдийг хянах болон хэрэглэгчийн оролцоог (engagement) сайжруулахад тусална.
