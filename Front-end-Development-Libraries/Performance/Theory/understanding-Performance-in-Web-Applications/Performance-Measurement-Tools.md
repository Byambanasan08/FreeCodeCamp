# Гүйцэтгэлийг хэмжих түгээмэл хэрэгслүүд (Performance Measurement Tools) юу вэ?

Хэрэв та вебсайтаа илүү хурдан, илүү хариу үйлдэлтэй (responsive) болгохыг хүсвэл гүйцэтгэлийг хэмжих, оновчтой болгоход зөв хэрэгслүүд хэрэгтэй. Түгээмэл гүйцэтгэлийн хэмжилтийн хэрэгслүүдийг болон тэдгээр нь хэрхэн тусалдгийг харцгаая.

Эхлээд Chrome DevTools.

Chrome DevTools нь Google Chrome доторх built-in хэрэгсэл бөгөөд гүйцэтгэлийг бодит цагт (real-time) шинжилж, debug хийх боломж олгодог. DevTools нь ачаалах хугацаа (loading times), CPU ашиглалт (CPU usage), болон рэндэрлэх саатлыг (render delays) харуулдаг. Энэ нь ялангуяа First Contentful Paint буюу FCP-ийг хэмжихэд хэрэгтэй — бидний мэдэж байгаачлан энэ нь хэрэглэгч анхны харагдах контентыг хэр хурдан харж байгааг илэрхийлдэг. Хэрэв таны вебсайт удаан мэт санагдаж байвал DevTools нь bottleneck-уудыг олоход тусална.

Дараагийнх нь Lighthouse.

Lighthouse нь гүйцэтгэл (performance), SEO, болон хүртээмжийг (accessibility) шалгадаг автоматжуулсан (automated) хэрэгсэл юм. Үүнийг ашиглахын тулд Chrome DevTools-ийг нээгээд "Lighthouse" таб руу орно. Дараа нь "Analyze page load" дээр дарж, таны сайт хэр хурдан, хэр оновчтой байгааг оноогоор (score) харуулж, мөн сайжруулах зөвлөмжүүдийг өгнө. Хэрэв та хурдан гүйцэтгэлийн аудит (performance audit) хийхийг хүсвэл Lighthouse хамгийн тохиромжтой хэрэгсэл.

Одоо WebPageTest-ийн тухай ярья.

WebPageTest нь таны сайтыг өөр өөр байршил (locations) болон төхөөрөмжүүдээс (devices) хэрхэн ачаалагдаж байгааг тест хийх боломж олгодог. Та WebPageTest.org руу орж, вебсайтынхаа URL-ийг оруулж, тест хийх байршил болон браузер сонгоод "Start Test" дээр дарна. Энэ нь таны сайтын Speed Index, Total Blocking Time, болон бусад гол гүйцэтгэлийн хэмжүүрүүдийг нарийвчилсан байдлаар задлан харуулна. Хэрэв та дэлхийн хэмжээнд хэрэглэгчид таны сайтыг хэрхэн мэдэрч байгааг ойлгохыг хүсвэл WebPageTest тохиромжтой хэрэгсэл юм.

Өөр нэг гайхалтай хэрэгсэл бол Google PageSpeed Insights.

Энэ хэрэгсэл нь таны вебсайтыг шинжилж, mobile болон desktop аль алинд нь хурдан сайжруулах боломжуудыг санал болгодог. Зүгээр л PageSpeed Insights руу орж URL-ээ оруулаад "Analyze" дээр дарна. Энэ нь таны сайтыг юу удаашруулж байгааг хэлж, тодорхой зөвлөмжүүд өгнө — жишээ нь зургийг оновчтой болгох, render-blocking скриптүүдийг устгах, серверийн хариу өгөх хугацааг багасгах гэх мэт. PageSpeed Insights нь Google таны сайтын гүйцэтгэлийг хэрхэн харж байгааг хурдан, амархан шалгах арга юм.

Эцэст нь Real User Monitoring буюу RUM хэрэгслүүдийн тухай ярья. Lab test-уудаас ялгаатай нь RUM хэрэгслүүд нь бодит хэрэглэгчдийн (actual users) зан төлөвийг (behavior) хянаж, тэд таны сайтыг хэрхэн мэдэрч байгааг харуулдаг. Түгээмэл RUM хэрэгслүүдэд Google Analytics ордог бөгөөд энэ нь page load time болон bounce rate-ийг хянадаг. Мөн New Relic эсвэл Datadog зэрэг хэрэгслүүд нь бодит цагийн (real-time) гүйцэтгэлийн асуудлуудыг (performance issues) хянадаг. Хэрэв та бодит хэрэглэгчдээс өгөгдөл авахыг хүсвэл RUM хэрэгслүүд зайлшгүй хэрэгтэй.

Хэрэгсэл бүр өөр өөр давуу талтай тул хамгийн сайн ойлголт авахын тулд хослуулан ашиглаарай. Эхлээд Chrome DevTools эсвэл Lighthouse ашиглан тест хийж, дараа нь WebPageTest болон RUM хэрэгслүүдээр илүү гүнзгий шинжилгээ хийгээрэй.
