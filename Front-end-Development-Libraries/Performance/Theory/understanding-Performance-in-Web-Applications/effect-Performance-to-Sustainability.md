# Гүйцэтгэл (Performance) тогтвортой байдалд (Sustainability) хэрхэн нөлөөлдөг вэ?

Веб гүйцэтгэлийг сайжруулах нь хурдан бөгөөд үр ашигтай (efficient) хэрэглэгчийн туршлага (user experience) хүргэхэд маш чухал. Гэхдээ та мэдэх үү, вебсайтынхаа гүйцэтгэлийг оновчтой болгох нь зөвхөн хурдыг сайжруулаад зогсохгүй — мөн байгаль орчинд (environment) тусалдаг?

Хуудас ачаалах (page load) бүрд энерги (energy) зарцуулагддаг. Харин сайтууд үр ашиггүй (inefficient) байвал илүү их нөөц (resources) ашиглаж, улмаар нүүрстөрөгчийн ялгаралтыг (carbon emissions) нэмэгдүүлдэг.

Гүйцэтгэл тогтвортой байдалд хэрхэн нөлөөлдгийг задлан тайлбарлаж, таны кодыг илүү “ногоон веб” (greener web) болгоход туслах практик аргуудыг харцгаая.

Интернет нь дэлхийн нийт нүүрстөрөгчийн ялгаралтын ойролцоогоор 2%-ийг эзэлдэг — энэ нь нисэхийн салбартай (airline industry) ижил хэмжээ юм! Дамжуулж буй байт (byte) бүр дата төвүүдээс (data centers) эхлээд хэрэглэгчийн төхөөрөмж хүртэл цахилгаан (electricity) шаарддаг. Том хэмжээтэй файлууд болон үр ашиггүй скриптүүд нь илүү их эрчим хүч (power) зарцуулдаг гэсэн үг. Өндөр гүйцэтгэлтэй вебсайт нь зөвхөн хурдан биш, мөн шаардлагагүй боловсруулалт (processing) болон энергийн хэрэглээг бууруулдаг.

Доорх нь үр ашиггүй JavaScript-ийн жишээ:

// Inefficient JavaScript
function fetchData() {
for (let i = 0; i < 1000000; i++) {
console.log("Processing...");
}
fetch("https://api.example.com/data")
.then(response => response.json())
.then(data => console.log(data));
}
fetchData();

Энэ скрипт хэт их console log ажиллуулж, CPU цикл (CPU cycles) болон энерги дэмий зарцуулж байна.

Одоо үүнийг оновчтой болгоё:

// Optimized JavaScript
async function fetchDataOptimized() {
const response = await fetch("https://api.example.com/data");
const data = await response.json();
console.log(data);
}
fetchDataOptimized();

Шаардлагагүй давталтуудыг (loops) хасаж, async/await ашигласнаар бид боловсруулалтын хугацаа (processing time) болон энергийн хэрэглээг бууруулж байна.

Ирээдүйн төслүүддээ ашиглаж болох зарим практик оновчлолын зөвлөмжүүд:

```
Assets-аа багасгах (Minify Your Assets): Terser эсвэл UglifyJS зэрэг хэрэгслүүдийг ашиглан JavaScript болон CSS файлуудын хэмжээг багасгах. Жижиг файлууд илүү хурдан ачаалагдаж, бага энерги шаарддаг.

Зургийг оновчтой болгох (Optimize Images): PNG эсвэл JPEG-ийн оронд WebP зэрэг next-gen формат ашиглан чанар алдалгүйгээр файлын хэмжээг багасгах.

Сүлжээний хүсэлтүүдийг багасгах (Reduce Network Requests): Хүсэлт (request) бүр нэмэлт ачаалал (overhead) үүсгэдэг. CSS болон JavaScript файлуудыг нэгтгэж, lazy loading ашиглан ачаалах хугацааг багасгах.

Үр ашигтай алгоритм ашиглах (Use Efficient Algorithms): Муу оновчлогдсон давталт CPU ашиглалтыг нэмэгдүүлж болно. Энэ нь өмнөх жишээнд харсан үр ашиггүй JavaScript бичихтэй адил юм.

Caching болон CDN ашиглах (Leverage Caching & CDNs): Caching нь давтан таталтыг (downloads) багасгаж, CDN нь контентыг хэрэглэгчид ойр байрлалаас хүргэж, дамжуулалтын энергийн хэрэглээг бууруулдаг.
```

Хэрэв та өөрийн нөлөөллийг хэмжиж, вебсайт тань хэр “ногоон” байгааг мэдэхийг хүсвэл яах вэ?

Google Lighthouse зэрэг хэрэгслүүдийг ашиглан гүйцэтгэлийг шинжилж, энергийн хэмнэлтийг (energy savings) тооцоолж болно. Эсвэл илүү гүнзгий тогтвортой байдлын шалгалт хийхийг хүсвэл Website Carbon Calculator-г туршаарай — энэ нь таны хуудасны байгаль орчинд үзүүлэх нөлөөг (environmental impact) тооцоолдог.

Дүгнэж хэлбэл, эдгээр гүйцэтгэлийн суурь ойлголтуудыг (performance fundamentals) ойлгож, хэрэгжүүлснээр хөгжүүлэгчид зөвхөн илүү хурдан, илүү хариу үйлдэлтэй веб аппликейшн (web applications) төдийгүй илүү тогтвортой (sustainable) шийдлүүдийг бүтээж чадна.
