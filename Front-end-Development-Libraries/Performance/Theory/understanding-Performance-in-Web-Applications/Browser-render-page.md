# Браузер (Browser) хуудсыг хэрхэн рэндэрлэдэг вэ?

Та хэзээ нэгэн цагт URL оруулаад enter дарахад цаана нь юу болдог талаар бодож байсан уу? Браузер кодыг хэрхэн бидний харж буй интерактив (interactive) хуудсууд болгон хувиргадаг вэ?

Браузер веб хуудсыг хэрхэн рэндэрлэдгийг ойлгох нь гүйцэтгэлийг (performance) оновчтой болгох, хэрэглэгчийн туршлагыг (user experience) сайжруулахыг зорьж буй хөгжүүлэгчдэд маш чухал.

Энэ хичээлд бид рэндэрлэх процессыг (rendering process) ойлгомжтой, хялбар хэсгүүдэд задлан тайлбарлах болно.

Эхлээд браузер HTML-ийг parse хийж, DOM үүсгэнэ.

Процесс нь браузер тухайн хуудсын HTML контентыг (content) татаж авах (fetch) үед эхэлдэг. Тэр HTML-ийг parse хийж, Document Object Model буюу DOM-ийг үүсгэнэ — энэ нь хуудсын контентыг илэрхийлсэн мод хэлбэрийн (tree-like) бүтэц юм.

Дараах энгийн HTML жишээг авч үзье:

<html>
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to our website.</p>
  </body>
</html>

Браузер үүнийг parse хийж, html нь root болж, дотор нь body, харин body дотор h1 болон p элементүүдийг агуулсан DOM tree үүсгэнэ.

Дараа нь браузер CSS-ийг боловсруулж, CSS Object Model буюу CSSOM-ийг үүсгэнэ. Энэ нь элементүүд хэрхэн загварчлагдахыг (styled) тодорхойлдог өөр нэг мод бүтэц юм.

Дараах CSS-г авч үзье:

h1 {
color: blue;
}
p {
font-size: 16px;
}

CSS Object Model нь h1 элементүүдийг цэнхэр (blue), харин p элементүүдийг 16 пикселийн фонт хэмжээтэй (font size) байхыг тодорхойлно.

Дараа нь браузер DOM болон CSS Object Model-ийг нэгтгэж render tree үүсгэнэ. Энэ мод нь зөвхөн хуудсыг рэндэрлэхэд шаардлагатай node-уудыг агуулж, тус бүр нь өөрийн харгалзах style-тай хослогдсон байдаг.

Манай жишээнд render tree нь h1 болон p элементүүдээс бүрдэж, тохирох style-уудтай байна.

Render tree бэлэн болмогц браузер layout буюу reflow гэж нэрлэгдэх процессоор элемент бүрийн яг байршил (position) болон хэмжээг (size) тооцоолдог. Энэ нь элементүүд дэлгэц дээр зөв байрлал, зөв хэмжээгээр харагдахыг баталгаажуулдаг.

Эцэст нь браузер пикселүүдийг (pixels) дэлгэц дээр зурж (paint), тооцоолсон style болон layout-ийн дагуу элемент бүрийг рэндэрлэдэг. Илүү төвөгтэй хуудсанд энэ нь олон давхарга (layers)-аас бүрдэж, тэдгээрийг нэгтгэн (composited) эцсийн визуал үр дүнг (final visual output) бий болгодог.

Дүгнэж хэлбэл, HTML болон CSS-ийг parse хийхээс эхлээд дэлгэц дээр пикселүүдийг зурах хүртэл, браузерийн рэндэрлэх процесс нь нарийн төвөгтэй (sophisticated) алхмуудын дараалал юм.
