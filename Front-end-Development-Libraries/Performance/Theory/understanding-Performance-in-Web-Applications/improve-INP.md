# INP-ийг хэрхэн хэмжих болон сайжруулах вэ?

Chrome DevTools ашиглан Interaction to Next Paint буюу INP-г хэмжих практик аргыг хамтдаа алхам алхмаар үзье.

Юуны өмнө "F12" товчийг дарах эсвэл хуудас дээр Right-click хийгээд "Inspect" дээр дарж DevTools-ийг нээнэ.

Одоо таны веб хуудсанд search bar байна гэж төсөөлье.

Хэрэглэгч хайлтын текст бичихэд suggestion dropdown гарч ирэх хүртэл 600ms зарцуулагдлаа гэж бодъё. Chrome DevTools энэ өндөр INP утгыг тэмдэглэнэ.

Та heavy JavaScript-ийг defer хийх, урт хугацаатай task-уудыг (long tasks) багасгах, мөн event handling-ийг сайжруулах замаар оптимизаци хийж чадна. Ингэснээр INP 150ms хүртэл буурч, харилцан үйлдэл бараг шууд (instant) мэт санагдана.

Гоё байгаа биз?

Одоо interaction delay-ийг тодорхойлоход туслах, ингэснээр илүү хурдан, жигд (smoother) хэрэглэгчийн туршлага (user experience) бий болгох практик хэмжилтийн аргыг харцгаая!

Эхлээд Chrome DevTools-ийг нээнэ. Үүний тулд Google Chrome-ийг нээж, шинжлэхийг хүссэн веб хуудсандаа орж, "F12" товчийг дарах эсвэл хуудас дээр right-click хийгээд "Inspect" дээр дарна. Одоо "Performance" таб руу оръё — тэнд танд хэрэгтэй бүх зүйл байна.

Одоо бид Interaction to Next Paint буюу INP-г тодорхойлоход бэлэн боллоо. Бид хамгийн урт interaction delay-ийг хайж харна. Сайн INP нь ихэвчлэн 200ms-ээс доош байдаг, харин муу INP нь 500ms-ээс дээш байдаг. Тэгээд л боллоо!
