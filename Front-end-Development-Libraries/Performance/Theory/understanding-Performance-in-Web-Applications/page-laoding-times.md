# Хуудасны ачаалах хугацааг (Page Loading Times) багасгах ямар аргууд байдаг вэ?

Та хэзээ нэгэн вебсайт хэт удаан ачаалагдсанаас болж орхиж байсан уу? Удаан ачаалах нь хэрэглэгчдийг бухимдуулж, bounce rate-ийг нэмэгдүүлдэг. Өнөөдөр бид хуудасны ачаалах хугацааг багасгах практик аргуудыг судалж, хэрэглэгчдээ идэвхтэй (engaged) байлгах талаар үзнэ.

Нэгдүгээрт: медиа нөөцүүдийг (media assets) оновчтой болгох. Том хэмжээтэй зураг болон видео нь удаан ачаалах гол шалтгаануудын нэг байдаг. Эдгээр нөөцүүдийг оновчтой болгосноор та сайтынхаа хурдыг мэдэгдэхүйц нэмэгдүүлэх боломжтой:

```
Зургийг шахах (Compress Images): TinyPNG зэрэг хэрэгслүүдийг ашиглан зургийн файлын хэмжээг чанарыг алдагдуулахгүйгээр багасгах.

Орчин үеийн формат ашиглах (Use Modern Formats): Зурганд WebP, видеонд AV1 зэрэг форматуудыг ашиглаж илүү сайн шахалтын (compression) үр дүнд хүрэх.

Lazy Loading ашиглах: Дэлгэц дээр харагдахгүй (off-screen) зураг болон видеог хэрэгтэй үед нь л ачаалах. HTML дээр media tag-д loading="lazy" attribute нэмнэ.
```

Дараагийнх нь HTTP хүсэлтүүдийг (HTTP requests) багасгах. Вебсайт тань хүсэлт гаргах бүрт ачаалах хугацаа нэмэгддэг. HTTP хүсэлтийн тоог багасгах нь илүү хурдан ачаалах боломжийг бүрдүүлнэ. Та дараах аргуудыг ашиглаж болно:

```
Файлуудыг нэгтгэх (Combine Files): CSS болон JavaScript файлуудыг нэгтгэж хүсэлтийн тоог багасгах.

Чухал CSS-ийг inline хийх (Use Inline Critical CSS): Эхний рэндэрлэлтийг (initial rendering) хурдасгахын тулд чухал CSS-ийг шууд HTML дотор байрлуулах.

CSS Sprites ашиглах: Олон зургийг нэг sprite sheet болгон нэгтгэж хүсэлтийн тоог багасгах.
```

Мөн мэдээж та Browser Caching-ийг ашиглаж болно. Caching нь браузерт вебсайтын зарим хэсгийг локалд хадгалах боломж олгож, дахин орж ирсэн хэрэглэгчдэд ачаалах хугацааг багасгана:

```
Expiry Headers тохируулах: Cache-Control header ашиглан браузер файлуудыг хэр удаан хадгалахыг заах.

Assets-ийн хувилбаржуулалт (Version Assets): Файлуудыг шинэчлэхдээ нэрийг нь өөрчлөх эсвэл query string ашиглаж, браузер шинэ хувилбарыг татахыг баталгаажуулах.
```

Үүнийг та мөн файлуудыг багасгах (minify) болон шахах (compress) аргуудтай хамтад нь хийж болно, ингэснээр файлын хэмжээ багасч, илүү хурдан татагдана:

```
CSS болон JavaScript-ийг minify хийх: Илүүдэл whitespace болон comment-уудыг устгах (JavaScript-д UglifyJS, CSS-д cssnano зэрэг хэрэгслүүд ашиглах).

Шахалт идэвхжүүлэх (Enable Compression): Серверээ Gzip ашиглан дамжуулах файлуудын хэмжээг багасгахаар тохируулах.
```

Мөн та web font-уудыг оновчтой болгож болно. Web font нь дизайныг сайжруулдаг ч буруу ашиглавал сайтыг удаашруулж болно:

```
Font variant-уудыг хязгаарлах: Зөвхөн хэрэгтэй character set болон style-уудыг оруулах.

font-display ашиглах: CSS дотор font-display property-г тохируулж, font ачаалж байх үеийн харагдах байдлыг хянах.
```

Тэгэхээр дүгнэж хэлбэл, эдгээр бүх стратегийг хэрэгжүүлснээр та хуудасны ачаалах хугацааг мэдэгдэхүйц багасгаж, илүү сайн хэрэглэгчийн туршлага (user experience) болон сайтын гүйцэтгэлийг (performance) сайжруулж чадна.
