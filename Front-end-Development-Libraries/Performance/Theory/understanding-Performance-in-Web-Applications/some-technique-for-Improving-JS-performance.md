# JavaScript-ийн гүйцэтгэлийг (Performance) сайжруулах ямар техникүүд байдаг вэ?

JavaScript-ийн гүйцэтгэлийг сайжруулах хамгийн чухал техникүүдийг авч үзье.

Та гайхаж магадгүй, гэхдээ эхний техник бол боломжтой үедээ JavaScript-ийг аль болох бага ашиглах юм.

Хэрэв та энгийн static вебсайт хөгжүүлж байгаа бол JavaScript дээр тулгуурласан framework ашиглахгүйгээр яг ижил үр дүнд хүрэх боломжтой байж магадгүй.

Мөн ашиглагдахгүй JavaScript кодыг устгаж, боломжтой бол илүү энгийн шийдлүүдийг ашиглах хэрэгтэй.

Жишээлбэл, та custom шийдлүүдийн оронд браузерын built-in боломжуудыг ашиглаж болно. Эдгээрт form validation, браузерын өөрийн video player, мөн custom JavaScript animation-уудын оронд CSS animation-ууд орно.

Анимаци (animations)-ийг аль болох бага ашигла.

JavaScript код доторх DOM manipulation-ыг багасгах хэрэгтэй, учир нь энэ нь тооцооллын хувьд (computationally) өндөр өртөгтэй.

JavaScript кодоо critical болон non-critical үүрэгтэй модулиудад хуваах нь мөн тустай. Ингэснээр critical хэсгүүдийг аль болох эрт preload хийж, non-critical хэсгүүдийг defer хийснээр хуудсыг аль болох хурдан рэндэрлэх боломжтой болно.

Файлуудыг preload хийх нь тэдгээрийг ашиглах үед заавал бэлэн байна гэсэн баталгаа өгөхгүй, гэхдээ татаж авах (download) процесс эрт эхлэх тул нийт хүлээлгийн хугацаа богиносох ёстой.

Кодоо олон файл болгон хуваасны дараа build процессын үед автоматаар minify хийж, файлын хэмжээг багасган ачаалах хугацааг оновчтой болгож болно.

Хэрэв event listener шаардлагагүй болсон бол устгах хэрэгтэй:

element.removeEventListener("mousemove", handleMouseMove);

Энэ жишээнд бид mousemove event-ийн event listener-ийг устгаж байгаа тул handleMouseMove функц тухайн event trigger болох үед дуудагдахгүй болно. Хэрэв event listener-ийг хадгалсаар байвал програм шаардлагагүй тооцоолол хийсээр байх бөгөөд энэ нь нийт гүйцэтгэлд нөлөөлнө. Хэрэв нэгэн зэрэг олон идэвхтэй (active) event listener байвал энэ нөлөө илүү их мэдрэгдэнэ.

Мөн event-уудын тухай ярьж байгаа болохоор, боломжтой үедээ event delegation ашиглахыг зөвлөдөг.

Энэ нь child элемент бүр дээр event listener тавих шаардлагагүй гэсэн үг. Та үүнийг parent дээр шууд тавьж, event нь bubble хийгдэн (bubbles up) тухайн event-ийг боловсруулах чадвартай parent хүртэл дамжина.

JavaScript-ийг оновчтой болгох нь хурдан, хариу үйлдэлтэй (responsive) веб аппликейшн хөгжүүлэхэд маш чухал. Бодит төсөл дээр та вебсайтынхаа гүйцэтгэлийг байнга шинжилж, сайжруулах боломжуудыг хайж байх хэрэгтэй.
