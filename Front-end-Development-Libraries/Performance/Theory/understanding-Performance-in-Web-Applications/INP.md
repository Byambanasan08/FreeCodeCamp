# INP гэж юу вэ?

Та хэзээ нэгэн цагт веб хуудсан дээр дарсан ч юу ч болоогүй мэт санагдаж байсан уу? Энэ саатал (delay) нь хэрэглэгчийн туршлагад (user experience) нөлөөлдөг бөгөөд Interaction to Next Paint буюу INP гэсэн хэмжүүрээр хэмжигддэг.

INP нь хэрэглэгчийн харилцан үйлдэл (interaction) — жишээ нь click эсвэл товч дарах (key press) — хийсэн мөчөөс эхлээд браузер (browser) дэлгэцийг дараагийн удаа шинэчлэх (update) хүртэлх хугацааг хэмжих замаар хуудсын нийт хариу үйлдлийн чадварыг (overall responsiveness) үнэлдэг. INP бага байх тусам илүү хурдан хариу үйлдэлтэй (responsive) хуудас гэсэн үг.

Гэхдээ яагаад INP чухал вэ? Өндөр INP нь хэрэглэгчдэд хуудсыг хариу өгөхгүй (unresponsive) мэт мэдрэмж төрүүлж, бухимдал (frustration) үүсгэдэг. INP-г оновчтой болгох нь хэрэглэгчдэд цаг тухайд нь feedback өгөхийг баталгаажуулж, тэдний туршлагыг сайжруулдаг.

Chrome-ийн DevTools болон web performance API зэрэг хэрэгслүүд нь interaction саатлыг (interaction delays) хянах замаар INP-г хэмжихэд тусалдаг.

INP-г сайжруулахын тулд main thread дээрх ажлыг багасгах, event handler-уудыг оновчтой болгох, мөн хэрэглэгчийн харилцан үйлдэл нь цаг тухайд нь визуал шинэчлэлт (visual updates) үүсгэж байгааг баталгаажуулах хэрэгтэй.

INP-г ойлгож, оновчтой болгосноор та хурдан бөгөөд хариу үйлдэлтэй мэт мэдрэгддэг веб туршлагуудыг (web experiences) бүтээж, хэрэглэгчдээ сэтгэл хангалуун байлгаж чадна.
