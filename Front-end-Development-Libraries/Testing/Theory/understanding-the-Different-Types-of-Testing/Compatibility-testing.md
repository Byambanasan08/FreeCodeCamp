# Compatibility Testing гэж юу вэ?

Та веб аппликейшнуудаа (web applications) өргөн хэмжээнд (at scale) хөгжүүлэх үед зөвхөн өөрийн компьютерийн орчноор хязгаарлагдахгүйгээр бодох хэрэгтэй болно. Эдгээр аппликейшнууд янз бүрийн төхөөрөмжүүд (devices), үйлдлийн системүүд (operating systems), болон браузерууд (browsers) дээр хэрхэн ажиллах вэ? Энэ үед compatibility testing хэрэг болно.

Compatibility testing-ийн зорилго нь таны програм хангамж (software) янз бүрийн компьютерийн орчинд (computing environments) хэр сайн ажиллаж байгааг шалгах явдал юм. Энэ нь багт (team) алдаануудыг (bugs) production-д хүрэхээс өмнө илрүүлэх боломж олгож, илүү жигд (smooth) бүтээгдэхүүн нэвтрүүлэхэд (product launch) тусалдаг.

Compatibility testing-ийн төрөлд backwards болон forwards compatibility, hardware testing, operating systems, networks, browsers, болон mobile devices орно.

Эдгээр төрлүүдийг илүү дэлгэрэнгүй авч үзье, эхлээд backwards болон forwards compatibility-оос эхэлье. Backwards compatibility гэдэг нь одоогийн програм хангамж өмнөх хувилбаруудтай (earlier versions) нийцэж (compatible) ажиллахыг хэлнэ. Forwards compatibility гэдэг нь програм хангамж ирээдүйн хувилбаруудтай (future versions) ажиллах боломжтой байхыг хэлнэ.

Дараагийн төрөл бол hardware testing. Hardware compatibility testing нь програм хангамж өөр өөр hardware тохиргоонууд (configurations)-д зөв ажиллаж байгаа эсэхийг шалгадаг. Үүнд янз бүрийн төрлийн процессор (processors), санах ой (memory), хадгалах төхөөрөмж (storage), болон график карт (graphics cards) орно.

Өөр нэг төрөл бол operating system compatibility testing. Та Mac дээр сайн ажилладаг боловч Windows дээр алдаатай, эсвэл эсрэгээрээ ажилладаг програм бүтээхийг хүсэхгүй. Мөн Ubuntu, Fedora зэрэг Linux distribution-уудыг ч авч үзэх хэрэгтэй.

Дараагийн төрөл бол network compatibility testing. Хэрэглэгчид өөр өөр сүлжээний хурд (network speeds), протокол (protocols), аюулгүй байдлын тохиргоо (security settings) гэх мэт нөхцөлд ажиллана. Тиймээс програм хангамжийн баг эдгээр бүх нөхцөлийг тест хийхдээ анхаарах хэрэгтэй.

Мөн browser compatibility testing байдаг. Зах зээл дээр Google Chrome, Safari, Firefox зэрэг түгээмэл браузерууд байдаг. Браузер бүр бага зэрэг өөрөөр ажилладаг тул веб аппликейшнуудаа олон браузер дээр жигд (consistently) ажиллаж байгаа эсэхийг сайтар тестлэх нь чухал.

Сүүлийн төрөл бол mobile testing. Веб хөгжүүлэлтийн эхэн үед ихэнх хөгжүүлэгчид mobile төхөөрөмжүүдийг тестлэх талаар төдийлөн боддоггүй байсан. Учир нь 2000-аад оны эхээр mobile төхөөрөмжүүд өнөөдрийнх шиг хүчирхэг, өргөн хэрэглэгддэг байгаагүй. Харин одоо сая сая хүмүүс вебсайтуудыг гар утас болон таблет дээр ашиглаж байна. Тиймээс таны програм хангамж Android болон iOS-ийн олон төрлийн төхөөрөмж дээр ажиллаж байгаа эсэхийг баталгаажуулах нь маш чухал.

Эдгээр нь аппликейшнуудаа хөгжүүлэхэд авч үзэх ёстой тестийн төрлүүдийн зөвхөн зарим нь юм. Хэдийгээр энэ нь илүү их ажил мэт санагдаж болох ч compatibility testing хийх нь чухал, учир нь таны аппликейшн янз бүрийн компьютерийн орчинд зөв ажиллаж байгаа эсэхийг баталгаажуулах шаардлагатай.
