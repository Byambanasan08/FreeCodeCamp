# Software Development Life Cycle гэж юу вэ, мөн энэ нь тестлэхтэй (Testing) хэрхэн холбоотой вэ?

Програм хангамжийн баг (software teams)-д ажиллах үед шинэ бүтээгдэхүүн (products) болон сервисүүдийг (services) зохион бүтээж (designing), хөгжүүлэх (building) тодорхой процесс байх нь маш чухал. Software Development Life Cycle (SDLC) нь аппликейшнуудыг (applications) бүтээж, нэвтрүүлэхэд (deploy) өргөн ашиглагддаг.

Энэ хичээлд бид Software Development Life Cycle (SDLC)-ийн үе шат бүрийг задлан тайлбарлаж, тэнд тестлэх (testing) ямар үүрэгтэйг харна.

Software Development Life Cycle-д ашиглагддаг стандарт үе шатууд:

```
Planning stage

Design stage

Implementation stage

Testing stage

Deployment stage болон

Maintenance stage
```

Одоо эдгээр үе шатуудыг илүү дэлгэрэнгүй авч үзье.

Planning stage-д хөгжүүлэлтийн баг (development team) санал болгож буй ажлын шаардлагуудыг (requirements) stakeholder-уудаас цуглуулдаг. Stakeholder гэдэг нь тухайн бүтээгдэхүүн эсвэл сервисийн амжилтад шууд ашиг сонирхолтой (vested interest) хүн юм. Жишээлбэл stakeholder нь хөрөнгө оруулагчид (investors), эзэмшигчид (owners), эсвэл бусад ажилчид байж болно. Planning үе шатанд мөн ажлын зардлыг (costs) тооцоолж, ажлыг хэзээ дуусгах хуваарь (schedule) гаргадаг.

Дараагийн үе шат бол Design stage. Энэ үед програм хангамжийн баг шаардлагуудыг задлан, хамгийн тохиромжтой шийдлүүдийг (solutions) тодорхойлдог. Үүнд ямар гадаад (external) хэрэгслүүдийг ашиглах, мөн ажлыг гүйцэтгэхэд ямар дотоод (internal) хэрэгслүүдийг бүтээх хэрэгтэйг шийдэх орно.

Design stage-ийн дараа Implementation stage эхэлнэ. Энэ нь баг шаардлагуудыг удирдахад хялбар (manageable) ажлуудад хувааж, багийн гишүүдэд хуваарилж, хөгжүүлж эхлэх үйл явц юм.

Implementation stage дууссаны дараа баг Testing stage руу шилжинэ. Энэ үед шинэ ажлуудад manual болон automated testing хийгдэнэ. Зарим тохиолдолд баг хөгжүүлэлтийн бүх үе шатанд аппликейшнийг тестэлж, гарч ирэх асуудлуудыг (issues) эрт илрүүлж засдаг.

Дараагийн үе шат бол Deployment stage. Энэ үед баг шинэ өөрчлөлтүүдийг build эсвэл testing environment-д байршуулдаг (deploy). Ингэснээр production руу оруулахаас өмнө аппликейшнийг дахин тестэлж, асуудлуудыг засах боломжтой болдог. Production application гэдэг нь эцсийн хэрэглэгчид (end users) ашиглах аппликейшн юм.

Сүүлийн үе шат бол Maintenance stage. Энэ нь production аппликейшн дээр хэрэглэгчдээс гарч ирэх асуудлуудыг засах үйл явц юм. Мөн системийг хянаж (monitoring), програм хангамжийг сайжруулах боломжуудыг хайх орно.

Эцэст нь, салбарт (industry) Software Development Life Cycle-ийн өөр өөр загварууд (models) байдаг, жишээлбэл Waterfall болон Agile. Waterfall загвар нь дараагийн үе шат эхлэхээс өмнө өмнөх үе шат бүрийг бүрэн дуусгах шаардлагатай байдаг. Харин Agile загвар нь ажлыг sprint-үүдэд хуваан, итератив (iterative) хөгжүүлэлт хийж, багууд хурдан өөрчлөлт хийх болон ажлыг үе шаттайгаар хүргэхэд анхаардаг.

Software Development Life Cycle-ийг дагаж мөрдвөл хөгжүүлэлтийн баг болон stakeholder-ууд шаардлага, зардлын тооцоо, болон хуваарийн талаар нэг ойлголттой (same page) байж чадна. Энэ нь илүү үр ашигтай (efficient) хамтын ажиллагаа (collaboration) болон харилцааг (communication) бий болгодог.
