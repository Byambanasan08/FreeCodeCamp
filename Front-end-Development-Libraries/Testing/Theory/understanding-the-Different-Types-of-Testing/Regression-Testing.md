# Regression Testing гэж юу вэ?

Аппликейшн (application) өсөж хөгжихийн хэрээр шинэ өөрчлөлтүүд (new changes) санамсаргүйгээр (unintentionally) өмнөх функционал (existing functionality)-ыг эвдэх (break) тохиолдол элбэг байдаг. Үүнийг regression гэж нэрлэдэг.

Эдгээр асуудлуудыг урьдчилан илрүүлэхийн тулд regression testing ашиглаж болно. Regression testing гэдэг нь аппликейшний тодорхой хэсгүүд дээр functional test-үүдийг дахин ажиллуулж, бүх зүйл өмнөх шигээ хүлээгдэж буй байдлаар (as expected) ажиллаж байгаа эсэхийг шалгах явдал юм. Хэрэв та bug fix, enhancement, эсвэл configuration шинэчлэлт хийсэн бол regression testing хийх нь зүйтэй.

Regression testing хийхэд ашиглаж болох янз бүрийн хэрэгслүүд байдаг. Үүнд Puppeteer, Playwright, Selenium, болон Cypress орно.

Regression testing-д ашиглагддаг хэд хэдэн техникүүд байдаг. Эхнийх нь unit regression testing. Энэ нь аппликейшнд томоохон өөрчлөлт эсвэл засвар (fixes) хийсэн бүрд шалгах ёстой зүйлсийн жагсаалттай байхыг хэлнэ.

Дараагийн техник нь partial regression testing. Энэ нь шинэ өөрчлөлтүүд аппликейшний тодорхой хэсгүүдийг эвдээгүй эсэхийг зорилтот (targeted) байдлаар шалгах арга юм.

Эцэст нь complete regression testing байдаг. Энэ нь хамгийн их цаг шаарддаг (time-consuming) бөгөөд хамгийн дэлгэрэнгүй арга бөгөөд кодын сан (codebase)-д хийгдсэн сүүлийн өөрчлөлтүүдээс нөлөөлсөн бүх функцуудыг шалгаж тест хийхийг хэлнэ.

Мөн retesting болон regression testing нь өөр гэдгийг анхаарах хэрэгтэй. Retesting нь мэдэгдэж буй (known) асуудлууд засагдсан эсэхийг шалгахад ашиглагддаг бол regression testing нь сүүлийн өөрчлөлтүүдээс үүссэн байж болох мэдэгдээгүй (unknown) асуудлуудыг илрүүлэхэд чиглэдэг.
