# Performance Testing гэж юу вэ?

Хэрэв програм хангамжийн аппликейшн (software application) удаан хариу өгдөг эсвэл тогтворгүй (unstable) байвал энэ нь муу хэрэглэгчийн туршлага (user experience)-д хүргэнэ. Performance testing гэдэг нь аппликейшний хурд (speed), хариу үйлдэл (responsiveness), өргөтгөх чадвар (scalability), болон тогтвортой байдлыг (stability) өөр өөр ачааллын (workloads) нөхцөлд шалгах үйл явц юм. Гол зорилго нь гүйцэтгэлийн bottleneck-уудыг хэрэглэгчдэд хүрэхээс өмнө илрүүлж засах явдал юм.

Энэ хичээлд бид performance testing-ийн хэд хэдэн төрлийг авч үзнэ.

Эхний төрөл нь load testing. Энэ контекстэд load гэдэг нь систем дээр ирж буй ажлын хэмжээ эсвэл шаардлагыг хэлнэ. Load testing нь систем хэвийн (normal) болон оргил (peak) ачааллын үед хэрхэн ажиллаж байгааг тодорхойлдог. Гол зорилго нь response time болон resource usage зэрэг хэмжүүрүүдийг шинжилж, одоогийн software build бэлэн эсэхийг тодорхойлох юм.

Өөр нэг төрөл нь stress testing. Энэ нь аппликейшнийг хэт өндөр ачаалал (extreme loads)-д оруулж, систем өндөр ачаалалд хэрхэн хариу үзүүлж байгааг шалгах явдал юм. Гол зорилго нь системийг хязгаараас нь давуулан (beyond its limits) тестлэж, bottleneck болон сул талуудыг (vulnerabilities) илрүүлэх юм.

Дараагийн төрөл нь soak testing бөгөөд үүнийг endurance testing гэж мөн нэрлэдэг. Энэ нь load testing-ийн нэг төрөл бөгөөд системийг урт хугацаанд (extended period of time) өндөр ачаалалд оруулж тестлэх явдал юм. Гол зорилго нь memory leak болон гүйцэтгэлийн бууралт (performance degradation)-ыг илрүүлэх юм.

Өөр нэг төрөл бол spike testing. Энэ нь ачааллыг гэнэт огцом (dramatically) нэмэгдүүлж, бууруулан, систем эдгээр өөрчлөлтөд хэрхэн хариу үзүүлж байгааг шинжилдэг. Гол зорилго нь ийм огцом ачааллын өөрчлөлтөд систем алдаа гаргах эсэхийг (fail) тодорхойлох юм.

Эцэст нь breakpoint testing буюу capacity testing байдаг. Энэ нь ачааллыг аажмаар (slowly) нэмэгдүүлж, систем эвдрэх (fail) эсвэл гүйцэтгэл нь муудах (degrade) цэг хүртэл хүргэх явдал юм. Гол зорилго нь системийн хязгаар (limits) болон багтаамжийг (capacities) илүү сайн ойлгох юм.

Эдгээр нь таны аппликейшн дээр хийж болох performance testing-ийн зарим төрлүүд юм. Гүйцэтгэл нь програм хангамжийн хөгжүүлэлтэд маш чухал үүрэгтэй тул таны аппликейшн томоохон асуудалгүй, хэрэглэгчдэд жигд (smooth) туршлага өгөхийн тулд эдгээр тестүүдийг хийх нь маш чухал.
