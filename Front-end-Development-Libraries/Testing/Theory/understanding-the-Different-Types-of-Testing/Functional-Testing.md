# Functional Testing гэж юу вэ, мөн энэ нь Unit Testing-ээс юугаараа ялгаатай вэ?

Өмнөх хичээлүүдэд та unit testing ашиглан кодын жижиг нэгжүүдийг (units) хэрхэн тестлэхийг сурсан. Харин unit testing нь functional testing-ийн нэг хэсэг юм. Functional testing нь аппликейшний функц болон боломжууд (features) хүлээгдэж буй байдлаар ажиллаж байгаа эсэхийг шалгадаг.

Энэ хичээлд бид functional болон non-functional testing-ийн ялгааг авч үзнэ.

Functional testing-ийн зорилго нь системийг (system) бүхэлд нь олон төрлийн scenario-уудын дагуу тестлэж, систем хүлээгдэж буй байдлаар ажиллаж байгаа эсэхийг баталгаажуулах юм. Functional testing-ийн нэг жишээ бол smoke testing юм. Smoke testing гэдэг нь илүү нарийвчилсан тест хийхээс өмнө системд үндсэн (basic) эсвэл чухал (critical) алдаа байгаа эсэхийг урьдчилан шалгах явдал юм.

Харин үүний эсрэгээр non-functional testing нь performance болон reliability зэрэг зүйлс дээр төвлөрдөг.

Functional testing-ийн бодит жишээ scenario нь хэрэглэгчийн нэвтрэлт (user login) байж болно. Хэрэглэгч өөрийн username болон password-оор аппликейшнд нэвтрэх боломжтой байх ёстой. Хэрэв амжилттай бол тэд өөрийн account-ийн мэдээлэлд хандах боломжтой байх ёстой. Хэрэв асуудал гарвал тухайн алдааг (error) зөв зохицуулах (handle) хэрэгтэй. Тест бичихдээ та бүх төрлийн амжилттай (success) болон амжилтгүй (failure) scenario-уудыг тооцох хэрэгтэй.

Non-functional testing-ийн жишээ бол таны аппликейшний гүйцэтгэлийг (performance) өөр өөр сүлжээний (network) нөхцөлд тестлэх явдал юм. Дараагийн хичээлд та performance testing-ийн талаар илүү дэлгэрэнгүй судлах болно.

Functional testing чухал байдаг шалтгаанууд нь: та аппликейшнд гарч болох алдаануудыг (bugs) эрт илрүүлэхийг хүсдэг. Мөн таны аппликейшн төлөвлөлтийн (planning phase) эхэнд тодорхойлсон шаардлагуудыг (requirements) хангаж байгаа эсэхийг баталгаажуулахыг хүсдэг. Эцэст нь хэрэглэгчид сайн хэрэглэгчийн туршлага (user experience) авах ёстой. Хэрэглэгчид аппликейшнийг ашиглахад хүндрэлтэй байх ёсгүй, мөн эвдэрсэн функцууд (broken features) болон системийн алдаанаас (system failures) болж саатах ёсгүй.
