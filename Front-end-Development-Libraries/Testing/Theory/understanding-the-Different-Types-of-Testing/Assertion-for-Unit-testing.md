# Unit testing-д assertion-ууд хэрхэн ажилладаг вэ?

Та өмнөх хичээл дээр assertion болон unit testing-тэй анх танилцсан. Сануулахад, unit testing гэдэг нь програмын бүхэл үр дүнг шалгахаас ялгаатайгаар, тус тусдаа функц (functions) эсвэл компонентуудын (components) гаралтыг (output) бие даан шалгахыг хэлнэ. Assertion-ууд нь код хүлээгдэж буй байдлаар (as expected) ажиллаж байгаа эсэхийг шалгахад ашиглагддаг.

Энэ хичээлд бид unit test дэх assertion-уудыг илүү дэлгэрэнгүй авч үзэж, зарим шилдэг туршлагуудыг (best practices) судална.

Түгээмэл ашиглагддаг assertion library бол Chai юм. Энэ нь freeCodeCamp-ийн кодын сан (codebase)-д ашиглагддаг ижил library юм. Та workshop эсвэл lab хийх үед автоматаар ажилладаг test suite таны кодыг шалгаж, зөв ажиллаж байгаа эсэхийг баталгаажуулдаг.

Full-Stack Developer Curriculum-аас хэдэн жишээг харцгаая.

Build a Calculator workshop-д Step 1 дээр addTwoAndSeven нэртэй функцыг шалгах тест байна:

assert.isFunction(addTwoAndSeven);

Энэ assertion нь өгөгдсөн кодонд addTwoAndSeven нэртэй функц байгаа эсэхийг шалгана. Хэрэв хэрэглэгч addTwoAndSeven-д функц биш (жишээ нь array, string эсвэл өөр төрлийн утга) оноовол тест амжилтгүй болно.

Workshop-ийн дараагийн хэсгээс өөр нэг assertion-ийг харцгаая:

assert.equal(addThreeAndFour(), 7);

Энэ assertion нь addThreeAndFour функцийн буцаасан утга (return value) 7-той тэнцүү эсэхийг шалгана. Хэрэв буцаасан утга 7-той тэнцэхгүй бол тест амжилтгүй болно.

Chai-д үүнээс гадна boolean эсэхийг шалгах, объект тодорхой property-той эсэхийг шалгах, эсвэл утга null эсэхийг шалгах зэрэг олон төрлийн assertion-ууд байдаг. JavaScript-ийн бусад түгээмэл assertion library-үүдэд should.js болон expect.js орно.

Та аль assertion library ашиглаж байгаагаас үл хамааран, анхаарах хэд хэдэн шилдэг туршлага байдаг. Эхнийх нь ойлгомжтой assert message бичих. Тодорхой, утгатай assert message нь тестийн зорилгыг (intent) тайлбарлах ёстой. Үгүй бол аль тест алдаж байгааг (failing) олоход хэцүү болно.

Өөр нэг шилдэг туршлага бол тодорхой failure message бичих. Тодорхой failure message нь ямар нөхцөл үнэн (true) болсон талаар тайлбарлах ёстой. Энэ нь яагаад тухайн assertion алдаж байгааг ойлгоход тусална.

Та код бичиж суралцах явцдаа testing-тэй сайн танилцаж, өөрийн төслүүдэд тест бичиж эхлэх нь чухал. Энэ нь таныг анхны developer ажлынхаа бэлтгэлд маш их тусална.
