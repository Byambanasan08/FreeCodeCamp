# Alpha болон Beta Testing гэж юу вэ?

Анхны хөгжүүлэлт (initial development) болон програм хангамжийн тест (software testing) дууссаны дараа аппликейшнийг тестерүүд болон бодит хэрэглэгчдээр тестлүүлэх нь чухал. Энэ үед alpha болон beta testing хэрэг болдог.

Alpha testing нь сонгогдсон (select) тестерүүдийн багахан бүлэг аппликейшнийг шалгаж, зах зээлд гаргахаас өмнө алдаа (bugs) байхгүй эсэхийг баталгаажуулах үйл явц юм. Энэ нь acceptance testing-ийн нэг хэсэг бөгөөд програм хангамж хэрэглэгчдийн хэрэгцээг (needs) хангаж байгаа эсэхийг гаргахаас өмнө шалгадаг.

Alpha testing нь мөн white box болон black box testing техникүүдийг ашигладаг. Black box testing нь зөвхөн аппликейшний хүлээгдэж буй зан төлөвт (expected behavior) анхаардаг бол white box testing нь тестер дотоод компонентуудыг (internal components) мэдэж, тэдгээр дээр тест хийхийг хэлнэ.

Alpha testing-ийн дараа beta testing үе шат ордог. Энэ үед аппликейшнийг бодит хэрэглэгчдэд (real users) нээлттэй болгодог. Жинхэнэ хэрэглэгчид аппликейшнтэй харилцан үйлдэл хийж, feedback өгдөг. Энэ нь user acceptance testing-ийн нэг хэлбэр бөгөөд аппликейшн бизнесийн шаардлага (business requirements) болон хэрэглэгчийн хэрэгцээг хангаж байгаа эсэхийг баталгаажуулдаг.

Хэдийгээр эдгээр хоёр үе шат цаг их шаарддаг (time-consuming) ч alpha болон beta тестийг аппликейшн дээрээ хийх нь маш чухал. Энэ нь аппликейшн зах зээлд гаргахад үнэхээр бэлэн болсон, мөн хэрэглэгчдийн хэрэгцээг хангаж байгаа эсэхийг баталгаажуулдаг.
