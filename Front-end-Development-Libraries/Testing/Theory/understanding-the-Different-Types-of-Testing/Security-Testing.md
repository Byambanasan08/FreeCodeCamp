# Security Testing гэж юу вэ?

Веб нь асар их мэдээлэл (information) болон үйл ажиллагаагаар (activities) дүүрэн хүчирхэг орчин юм. Гэхдээ болгоомжгүй байвал аюултай орчин ч байж болно. Олон хакерууд (hackers) болон кибер гэмт хэрэгтнүүд (cyber criminals) вебсайтуудыг эвдэж (breach), өгөгдлийг (data) алдагдуулах шинэ арга замуудыг хайж байдаг.

Та Meta болон Microsoft зэрэг тэрбум долларын компаниудад нөлөөлсөн өндөр хэмжээний (high-profile) өгөгдлийн алдагдлын (data breaches) талаар сонсож байсан байх. Веб аппликейшн (web applications) дээр ажиллахдаа та эмзэг байдал (vulnerabilities) болон сул талуудыг (weaknesses) илрүүлэхийн тулд нарийн (thorough) security testing хийх хэрэгтэй.

Security audit болон testing хийхэд анхаарах хэд хэдэн зарчим:

```
Confidentiality: Энэ нь эмзэг мэдээллийг (sensitive information) зориулсан хүлээн авагчаас өөр хүмүүст задруулахаас хамгаалдаг.

Integrity: Энэ нь хортой хэрэглэгчид (malicious users) хэрэглэгчийн мэдээллийг өөрчлөхөөс сэргийлэхийг хэлнэ.

Authentication: Энэ нь хэрэглэгчийн хэн болохыг (identity) баталгаажуулж, тухайн системийг ашиглах эрхтэй эсэхийг шалгах үйл явц юм.

Authorization: Энэ нь баталгаажсан (authenticated) хэрэглэгч ямар үйлдэл хийх эрхтэй, эсвэл системийн аль хэсэгт хандах эрхтэйг тодорхойлох процесс юм.

Availability: Энэ нь зөвшөөрөгдсөн (authorized) хэрэглэгчдэд мэдээлэл болон үйлчилгээ шаардлагатай үедээ бэлэн байхыг баталгаажуулдаг.

Non-repudiation: Энэ нь илгээгч болон хүлээн авагч аль аль нь хүргэлт болсон (proof of delivery) болон илгээгчийн identity баталгаажсан нотолгоотой байхыг баталгаажуулдаг. Энэ нь илгээгч мэдээлэл илгээсэн гэдгээ үгүйсгэхээс хамгаалдаг.
```

Одоо security testing яагаад хэрэгтэйг ойлгосон тул түгээмэл аюул заналхийлэл (security threats)-үүдийг харцгаая:

```
Cross-site Scripting (XSS): Та үүнийг өмнөх хичээлд үзсэн. Сануулахад, XSS халдлага нь халдагч (attacker) веб хуудсанд хортой скрипт (malicious scripts) оруулж, бусад хэрэглэгчид үзэх үед тэдгээр скриптүүд ажиллах явдал юм. Эдгээр скриптүүд нь хэрэглэгчийн браузер дотор ажиллаж, cookies болон session өгөгдлийг хулгайлах, эсвэл хэрэглэгчийн мэдлэг, зөвшөөрөлгүйгээр бусад хортой үйлдэл хийх боломжтой.

SQL Injection: SQL нь relational database-д мэдээллийг удирдах, хадгалах хэл юм. SQL injection нь халдагчид database-д хортой код оруулах боломж олгодог. Та SQL, relational database, болон SQL injection-ийн талаар дараагийн модульд илүү дэлгэрэнгүй судлах болно.

Denial-of-Service (DoS) attack: Энэ нь халдагчид вебсайт руу маш их хэмжээний хүсэлт (requests) эсвэл урсгал (traffic) илгээж, серверийг удаашруулж, бүр гацахад хүргэж, сайтыг хэрэглэгчдэд ашиглах боломжгүй болгодог.
```

Аюул заналхийллийн олон төрлүүд байдаг боловч веб аппликейшнуудаа хамгаалж, илүү аюулгүй болгоход туслах олон хэрэгслүүд байдаг. Security testing хэрэгслүүдийг ерөнхийд нь дараах ангилалд хувааж болно:

```
Static application security testing: Эдгээр хэрэгслүүд нь аппликейшний source code-г шинжилж, аюулгүй байдлын эмзэг байдал (security vulnerabilities)-ийг илрүүлдэг.

Dynamic application security testing (DAST): Эдгээр хэрэгслүүд нь аппликейшний front-end-тэй харилцаж, боломжит аюулгүй байдлын сул талуудыг илрүүлдэг. Static application security testing-ээс ялгаатай нь DAST хэрэгслүүд source code-д хандах боломжгүй.
```

Өөр нэг төрлийн security testing бол penetration testing (эсвэл товчоор pentest) юм. Энэ нь аппликейшн дээр хиймэл (simulated) кибер халдлага үүсгэж, системийн эмзэг байдал (vulnerabilities)-ийг илрүүлэх үйл явц юм. Дунд болон том хэмжээний компаниуд ихэвчлэн тусгай кибер аюулгүй байдлын (cybersecurity) багтай байдаг бөгөөд тэд тогтмол pentest хийж, security audit-ынхаа нэг хэсэг болгодог.

Кибер аюулгүй байдлын (cybersecurity) ертөнц маш өргөн, нарийн төвөгтэй. Гэхдээ хөгжүүлэгч бүр веб аппликейшнуудад тулгардаг нийтлэг аюул заналхийлэл болон тэдгээрээс хэрхэн хамгаалах аргуудыг ойлгож байх ёстой.
