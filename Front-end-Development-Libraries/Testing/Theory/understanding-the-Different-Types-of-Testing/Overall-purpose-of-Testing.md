# Аппликейшнуудаа (Applications) тестлэхийн (Testing) ерөнхий зорилго юу вэ?

Одоог хүртэл та HTML, CSS, JavaScript зэрэг хэлүүдээр олон жижиг төслүүд (projects) бүтээсэн. Гэхдээ таны төслүүд зөв ажиллаж байгаа эсэхийг хэрхэн мэдэх вэ?

Нэг боломж бол manual testing гэж нэрлэгддэг аргыг ашиглах. Энэ нь тестер (tester) аппликейшний хэсэг бүрийг шалгаж, бүх функцуудыг туршиж үзэн зөв ажиллаж байгаа эсэхийг баталгаажуулах явдал юм. Хэрэв тест хийх явцад ямар нэг алдаа (bugs) илэрвэл тестер тэдгээрийг software team-д мэдэгдэж, засуулах болно.

Хэдийгээр аппликейшнуудаа гараар (manually) тестлэх нь чухал боловч ихэнх тохиолдолд хангалтгүй байдаг. Бодит амьдрал дээрх аппликейшнууд олон компонент (components) болон сервис (services)-ээс бүрддэг бөгөөд ихэвчлэн нарийн төвөгтэй (complex) байдаг. Тиймээс зөвхөн manual testing-д найдах нь гаднаас харагдахгүй (beneath the surface) олон алдаа үлдэхэд хүргэж, энэ нь аппликейшний ашиглахад (usability) болон засварлахад (maintainability) сөргөөр нөлөөлдөг.

Энд automated testing хэрэг болно. Та аппликейшн тань хүлээгдэж буй байдлаар (as expected) ажиллаж байгаа эсэхийг шалгах тусдаа тест код бичиж, гараар дарж шалгах шаардлагагүй болгоно. Manual болон automated тестүүдийг аль аль нь юу шалгаж байгаагаас хамааран өөр өөр төрлүүдэд ангилж болно.

Дараагийн хэдэн хичээлд бид салбарт (industry) ашиглагддаг янз бүрийн тестийн төрлүүдийг авч үзнэ, тухайлбал unit testing, end-to-end testing, functional testing болон бусад.

Аппликейшндээ тест бичих нь таны код яг хийх ёстой зүйлээ хийж байгааг баталгаажуулахад тусална. Та тестийн суурь ойлголтуудтай танилцаж эхлэх нь маш чухал, учир нь энэ нь хөгжүүлэгч (developer)-ийн ажилд зайлшгүй хэрэгтэй чадвар юм.
