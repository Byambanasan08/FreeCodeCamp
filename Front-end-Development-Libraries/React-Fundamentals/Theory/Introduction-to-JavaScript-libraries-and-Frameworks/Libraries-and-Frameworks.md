## JS Libraries болон Frameworks-ийн үүрэг юу вэ, мөн яагаад тэдгээрийг салбарт (industry) ашигладаг вэ?

JavaScript libraries (сангууд) болон frameworks (хүрээнүүд) нь урьдчилан бичигдсэн код (pre-built code)-ыг өгдөг бөгөөд энэ нь хөгжүүлэлтийн процессыг хялбаршуулдаг (streamlines). Libraries болон frameworks хоёулаа бүтээмжийг (productivity) сайжруулах, код бичих практикийг (coding practices) стандартчилахад (standardize) тусалдаг боловч тэдгээрийн хандлага (approach) болон хөгжүүлэгчид (developers)-д өгдөг хяналтын түвшин (level of control)-д ялгаатай байдаг.

Libraries нь ихэвчлэн тодорхой нэг ажлыг шийдэхэд төвлөрдөг, жишээ нь DOM (Document Object Model)-ийг удирдах (manipulating), events (үйл явдлууд)-ийг зохицуулах (handling), эсвэл AJAX requests-ийг удирдах (managing) зэрэг.

JavaScript library-ийн нэг жишээ бол jQuery юм.

jQuery нь 2010-аад оны эхээр маш их алдартай болсон бөгөөд DOM manipulation, event handling, animations, effects болон бусад олон зүйлийг хялбаршуулахад өргөн хэрэглэгдэж байсан. Энэ library нь мөн plugin-уудын (нэмэлтүүдийн) баялаг экосистемийг (ecosystem) санал болгож байсан бөгөөд ингэснээр date pickers, sliders, modal dialogs зэрэг нийтлэг вэб компонентуудыг (web components) хялбархан бүтээх боломжийг олгодог байсан.

jQuery нь одоо өмнөх шигээ өргөн хэрэглэгддэггүй болсон ч vanilla JavaScript (цэвэр JavaScript)-д төвөгтэй байсан ажлуудыг илүү хялбар хэрэгжүүлэхэд хөгжүүлэгчдэд ихээхэн тусалсан.

Frameworks нь харин эсрэгээрээ аппликейшн (applications) бүтээхэд илүү тодорхой бүтэц (defined structure)-ийг өгдөг. Тэдгээр нь ихэвчлэн хөгжүүлэгчид дагаж мөрдөх шаардлагатай дүрэм (rules) болон conventions (хэвшмэл загварууд)-ын цогцыг агуулдаг.

Жишээ нь Angular нь component-based architecture (компонент дээр суурилсан архитектур)-ийг дэмждэг бөгөөд аппликейшн зохион байгуулах, бүтээхэд зориулсан тодорхой conventions болон tools-ийг өгдөг. Angular нь component-уудыг хэрхэн зохион байгуулах, state (төлөв)-ийг хэрхэн удирдах, routing (замчлал)-ыг хэрхэн зохицуулах, services-тэй хэрхэн харилцах талаар хөгжүүлэгчдэд тодорхой зааварчилгаа өгдөг тул илүү opinionated framework (хатуу байр суурьтай хүрээ) болдог.

Framework-ийн бусад жишээнүүдэд Remix болон Next.js орно.

Үүнтэй харьцуулахад React нь UI library (хэрэглэгчийн интерфэйсийн сан) бөгөөд илүү уян хатан (flexible) бөгөөд ямар нэг тодорхой архитектурын загварыг албаддаггүй. React нь голчлон view layer (харагдах хэсэг)-т анхаардаг бөгөөд аппликейшний дизайн (application design)-ийн олон шийдвэрийг хөгжүүлэгчдэд үлдээдэг.

Libraries болон frameworks нь бүх төрлийн хэмжээтэй төслүүдэд ашиглагддаг боловч аль нэгийг нь сонгох нь ихэвчлэн тухайн төслийн шаардлагаас (requirements) хамаардаг. Libraries нь тодорхой функцүүдэд (functionalities) уян хатан байдал өгдөг бол frameworks нь илүү төвөгтэй аппликейшнуудад (complex applications) зориулсан бүтэцтэй хандлагыг (structured approach) өгдөг.

Салбарт (industry) libraries болон frameworks-ийг олон шалтгааны улмаас өргөн хэрэглэдэг. Тэдгээр нь нийтлэг асуудлуудад хурдан шийдэл (quick solutions) өгснөөр хөгжүүлэлтийг ихээхэн хурдасгадаг.

JavaScript-д түгээмэл тохиолддог нэг асуудал бол dates болон timezones (цагийн бүс)-тэй ажиллах явдал юм. Гэвч date manipulation, time zones, parsing болон formatting of dates-д туслах built-in шийдэлтэй libraries байдаг.

Олон алдартай libraries болон frameworks нь сайн шалгагдсан (well-tested) бөгөөд томоохон community (нийгэмлэг)-уудын дэмжлэгтэйгээр хөгжүүлэгдэж, арчилж (maintained) байдаг. Энэ нь тэдгээрийг бүх зүйлийг эхнээс нь (from scratch) өөрөө бүтээхээс илүү практик (practical) болгодог.

Libraries болон frameworks нь бодит амьдралын нөхцөл байдалд (real life scenarios) үр дүнтэй болох нь батлагдсан best practices болон patterns-ийг дагаж мөрддөг. Энэ нь илүү бат бөх (robust) болон өргөтгөх боломжтой (scalable) аппликейшн бүтээхэд хүргэдэг.

Дүгнэж хэлбэл, libraries болон frameworks нь нийтлэг асуудлуудад хурдан шийдэл өгч, хөгжүүлэлтийг хурдасгаж, best practices-ийг дэмждэг. Libraries болон frameworks-ийн ялгааг ойлгож, тэдгээрийг хэзээ ашиглахыг мэдэх нь аливаа JavaScript хөгжүүлэгчийн хувьд үнэ цэнтэй ур чадвар юм.
