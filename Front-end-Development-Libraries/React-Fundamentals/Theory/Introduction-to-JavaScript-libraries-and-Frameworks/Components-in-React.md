# React-д Components гэж юу вэ, мөн тэд хэрхэн ажилладаг вэ?

Components (компонентууд) нь React аппликейшнуудын барилгын үндсэн хэсгүүд (building blocks) бөгөөд хөгжүүлэгчдэд (developers) төвөгтэй хэрэглэгчийн интерфэйсийг (user interfaces) жижиг, удирдахад хялбар хэсгүүдэд хуваах боломжийг олгодог. Ингэснээр том хэмжээний аппликейшн хөгжүүлэх болон арчлах (maintain) нь илүү хялбар болдог.

React-д component-уудын хоёр төрөл байдаг: functional components (функционал компонентууд) болон class-based components (класс дээр суурилсан компонентууд). Орчин үеийн React-д хөгжүүлэгчид functional components-ийг ашигладаг бөгөөд бидний өнөөдөр үзэх бүх жишээнүүд functional components байх болно.

Дээд түвшинд (higher level) та компонентуудыг UI-г тодорхойлсон элементүүдийг буцаадаг JavaScript функцүүдтэй адилхан гэж ойлгож болно.

Энэ UI нь JSX (JavaScript Syntax Extension)-ээр тодорхойлогддог бөгөөд энэ нь JavaScript-д зориулсан синтаксын өргөтгөл (syntax extension) бөгөөд HTML-тэй төстэй харагддаг ч UI элементүүдийг илүү declarative (зарлан тодорхойлох хэлбэрээр) бичих боломжийг олгодог.

React компонентын жишээг харцгаая:

function Greeting() {
const name = "John"
{/_ The result will be Hello John_/}
return <h1 className="title">Hello {name}</h1>;
}

Энэ жишээнд бид Greeting нэртэй компонент тодорхойлсон. h1 tag-ийн доторх {} (муруй хаалт)-ууд нь JavaScript expression (илэрхийлэл)-ийг оруулах боломжийг олгодог бөгөөд ингэснээр бид h1 элемент дотор name хувьсагчид хандаж байна.

Мөн бид h1 элемент дээр title нэртэй className хэрэглэж байна.

Гэхдээ яагаад бид энгийн HTML элементүүд дээр байдаг шиг class биш, className ашиглаж байна вэ?

Учир нь JavaScript-д class гэдэг нь reserved name (тусгайлан нөөцөлсөн нэр) юм. Тиймээс бид className ашиглах шаардлагатай.

Мөн бид JSX дотор үр дүнг харуулсан comment (тайлбар) ашиглаж байна. Энд анхаарах нэг чухал зүйл бол та ийм маягаар энгийн comment синтакс ашиглаж болох боловч ажиллуулахын тулд заавал муруй хаалтад {} ороосон байх шаардлагатай:

{/_ Block Comments _/}

Та бас нэг зүйлийг анзаарсан байж магадгүй, бид компонентын нэрийн эхэнд том үсэг ашиглаж байна. Гэхдээ яагаад бид бүхэлд нь жижиг үсгээр бичиж болохгүй вэ? ]
[ Энэ нь React нь том үсгээр (capital letter) эхэлсэн компонентуудыг custom components (хэрэглэгчийн тодорхойлсон компонентууд) гэж үздэг бол жижиг үсгээр бичигдсэн элементүүдийг built-in HTML elements (суурь HTML элементүүд) гэж үздэгтэй холбоотой.

React нь <div> эсвэл <span> зэрэг жижиг үсгээр бичигдсэн tag-уудтай таарах үед тэдгээрийг стандарт HTML элемент гэж ойлгодог. Харин компонентын нэр том үсгээр эхэлсэн бол React үүнийг user-defined component (хэрэглэгчийн тодорхойлсон компонент) гэж үзэж, түүнд тохируулан render (дүрслэх) хийнэ. Энэ ялгаа нь React-д native HTML tag-ууд болон таны үүсгэсэн компонентуудыг ялгаж танихад тусалдаг.

Манай аппликейшнд энэ Greeting компонентыг ашиглахын тулд бид дараах байдлаар бичнэ:

<Greeting />

Энэ нь хуудсан дээр Hello John гэсэн тексттэй h1 элементийг render хийнэ. Гэхдээ энд ашигласан синтаксийг анхааралтай харна уу. Бид компонентыг ашиглах үед энэ нь forward slash (/) болон greater than тэмдэг (>)-ээр төгсөж байна.

JSX-тэй ажиллах үед бүх tag болон компонентуудыг заавал хаах (explicitly closed) шаардлагатай. Тиймээс хэрэв компонент эсвэл tag нь ямар нэг children (дотор элемент)-гүй бол та доорх байдлаар заавал хаах хэрэгтэй:

<Greeting /> {/_ /> is required _/}

Одоог хүртэл бид зөвхөн нэг h1 элементийг render хийхийг харсан. Гэхдээ та үнэндээ олон элемент render хийж чадна.

Дараах жишээ кодыг харцгаая:

function Greeting() {
const name = "John";
{/_ This will throw an error _/}
return <h1>Hello {name}</h1>

  <p>Nice to meet you.</p>
}

Бид Nice to meet you гэсэн өөр нэг өгүүлбэр нэмэх гэж оролдож байгаа боловч энэ нь хуудсан дээр зөв render болохгүй байна.

Оронд нь error message (алдааны мессеж) гарч ирж байна. Алдааны мессеж нь Adjacent JSX elements must be wrapped in an enclosing tag гэж байна.

Та энэ алдааг авч байгаа шалтгаан нь олон sibling elements (ах дүү түвшний элементүүд)-ийг parent element (эцэг элемент)-д багтаах шаардлагатай байдагтай холбоотой. Та h1 болон p элементүүдийг энгийн div дотор ороож болох ч, энэ алдааг засах өөр нэг арга байдаг.

React fragments (фрагментууд) нь элементүүдийг бүлэглэхэд (group) ашиглагддаг. Шинэчилсэн жишээ дараах байдлаар харагдана:

function Greeting() {
const name = "John";
return ( <Fragment> <h1>Hello {name}</h1> <p>Nice to meet you.</p> </Fragment>
);
}

Мөн та fragments-ийн богино хэлбэр (shorthand) болгон хоосон JSX tag-уудыг ашиглаж болно:

function Greeting() {
const name = "John";
return (
<> <h1>Hello {name}</h1> <p>Nice to meet you.</p>
</>
);
}

Дараагийн хичээлүүдэд бид компонент болон JSX-тэй хэрхэн ажиллах талаар илүү ихийг сурах болно. Харин одоогоор та компонент ашиглан хэрэглэгчийн интерфэйс бүтээх талаар бат бөх суурь ойлголттой боллоо. ]
