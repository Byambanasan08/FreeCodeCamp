## Нэг Хуудаст Програмууд (Single Page Applications) гэж юу вэ, мөн тэдгээртэй холбоотой ямар асуудлууд байдаг вэ?

Уламжлалт олон хуудаст вэбсайтуудаас (multi-page websites) ялгаатай нь, нэг хуудаст програмууд (SPAs) зөвхөн нэг HTML хуудсыг ачаалж, хэрэглэгч апп-тай харилцах явцад бүх хуудсыг дахин ачаалалгүйгээр (reload) агуулгыг динамикаар шинэчилдэг. Энэ хандлага нь илүү хурдан, илүү хариу үйлдэл сайтай (responsive) програмуудыг бий болгоход хүргэдэг боловч мөн тодорхой бэрхшээлүүд болон анхаарах зүйлсийг дагуулдаг.

SPAs нь програмын төлөвийг (state) удирдах болон агуулгыг дүрслэх (render) зорилгоор JavaScript-ийг их хэмжээгээр ашигладаг. Серверээс шинэ HTML хуудсуудыг хүсэхийн оронд, SPAs нь JavaScript ашиглан DOM-ийг өөрчилж (manipulate) болон өгөгдлийг асинхроноор (asynchronously) татаж авдаг. Үүнийг ихэвчлэн React, Vue, эсвэл Angular зэрэг сангууд (libraries) болон фреймворкууд (frameworks)-ыг ашиглан хийдэг бөгөөд эдгээр нь нарийн төвөгтэй хэрэглэгчийн интерфэйс (user interfaces) бүтээхэд маш сайн хэрэгслүүдийг санал болгодог.

SPAs нь зарим нийтлэг асуудлуудтай. Нэг чухал асуудал нь дэлгэц уншигч (screen readers) нь динамикаар шинэчлэгдсэн агуулгатай ажиллахад хүндрэлтэй байж болно. Агуулга хуудсыг дахин ачаалах/шинэчлэх (reload/refresh)гүйгээр өөрчлөгдөх үед, дэлгэц уншигчид эдгээр өөрчлөлтийг мэдэгдэхгүй байж болох бөгөөд энэ нь хэрэглэгчдэд манай вэб апп дээрх шинэчлэлтүүдийг мэдэх боломжгүй болгодог.

SPAs-тэй холбоотой өөр нэг сорилт нь навигаци (navigation) болон браузерын түүх (browser history)-тэй холбоотой. Хэрэглэгчид буцах (back) болон урагш (forward) товчлуурууд уламжлалт вэбсайтууд дээр ажилладаг шиг ажиллахыг хүлээдэг. Гэвч SPAs дээр энэ нь зөв ажиллахгүй байж болно, учир нь техникийн хувьд вэб апп-ын URL өөрчлөгддөггүй. Вэб апп-ын URL өөрчлөгддөггүй тул хэрэглэгчид тодорхой хуудсыг bookmark хийж чаддаггүй. Хуудсыг дахин ачаалах (refresh) нь одоогийн харагдаж буй төлөвийг хадгалах (maintain) биш, харин програмыг эхний төлөв (initial state) рүү буцааж магадгүй.

SPAs нь SEO оновчлолд (SEO optimization) бэрхшээл үүсгэж болно. Google зэрэг хайлтын системүүд нь динамикаар ачаалагдсан агуулгыг индексжүүлэхэд (indexing) хүндрэлтэй байж болно, учир нь тэд JavaScript-ийг зөв гүйцэтгэж чадахгүй эсвэл анхны HTML хуудсанд багтаагүй агуулгыг алгасаж магадгүй. Энэ нь хуудсууд зөв индексжигдэхгүй байхад хүргэж болно.

Гэсэн хэдий ч орчин үеийн хайлтын системүүд SPAs-ийг мөлхөх (crawl) чадвараа сайжруулсан бөгөөд сервер талын дүрслэлт (server-side rendering, SSR), урьдчилан дүрслэх (pre-rendering), мөн SEO-д ээлтэй URL-ууд (SEO-friendly URLs) ашиглах зэрэг аргууд нь эдгээр асуудлуудыг бууруулахад тусалж, SPAs-ийн SEO-г сайжруулах боломжтой.

Гүйцэтгэл (performance) нь бас нэг анхаарах зүйл юм. SPAs нь бүх програмыг нэг дор ачаалдаг бөгөөд энэ нь ачааллах хугацаа (loading time) уртасвал хэрэглэгчид удаан хугацаанд хоосон дэлгэц харахад хүргэдэг. Одоо интернетийн хурд удаан хэрэглэгчдийн нөхцөл байдлыг бодоод үзээрэй. Ерөнхийдөө хэрэглэгчийн туршлага (user experience) тийм ч таатай байхгүй.

Дүгнэж хэлбэл, Нэг Хуудаст Програмууд (Single Page Applications) олон давуу талтай боловч мөн өвөрмөц сорилтуудыг дагуулдаг. Хөгжүүлэгчид SPAs бүтээхдээ хүртээмж (accessibility), ашиглахад хялбар байдал (usability), SEO, болон гүйцэтгэлийн (performance) асуудлуудыг анхааралтай авч үзэх хэрэгтэй. Эдгээр асуудлуудыг шийдвэрлэснээр хурдан, хариу үйлдэл сайтай, бүх хэрэглэгчдэд хүртээмжтэй SPAs бүтээх боломжтой.
