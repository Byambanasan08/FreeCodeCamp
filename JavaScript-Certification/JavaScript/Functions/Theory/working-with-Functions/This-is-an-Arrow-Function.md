### Arrow функц гэж юу вэ, тэдгээр нь хэрхэн ажилладаг вэ?

Өмнөх хичээлээр та кодыг илүү модульчлагдсан, арчлахад хялбар, үр ашигтай болгоход тусалдаг дахин ашиглагдах кодны хэсэг болох функцтэй ажиллах талаар сурсан. Өмнөх бүх жишээнүүд дээр ердийн функцийн синтакс (бичиглэл) ашиглаж байсан:

```javascript
function greetings(name) {
  console.log("Sain uu, " + name + "!");
}
```

Гэхдээ JavaScript дээр функц бичих өөр нэг арга бол "arrow function" (сумт функц) илэрхийлэл үүсгэх юм. Өмнөх жишээг arrow функцийн синтакс руу хэрхэн хөрвүүлж болохыг энд харуулав:

```javascript
const greetings = (name) => {
  console.log("Sain uu, " + name + "!");
};
```

Энэхүү шинэчилсэн жишээнд бид `greetings` нэртэй `const` хувьсагч үүсгэж, түүнд аноним функцийг оноож байна. `function` гэдэг түлхүүр үг байхгүй болсон, мөн `name` параметр болон функцийн их бие (body) хоёрын хооронд сум (`=>`) нэмэгдсэнээс бусад бичиглэл танд танил харагдах болно. Хэрэв таны параметрийн жагсаалтад зөвхөн нэг л параметр байгаа бол дугуй хаалтыг дараах байдлаар хасаж болно:

```javascript
const greetings = name => {
  console.log("Sain uu, " + name + "!");
};
```

Хэрэв таны arrow функц ямар нэгэн параметргүй бол дугуй хаалтыг заавал дараах байдлаар ашиглах ёстой:

```javascript
const greetings = () => {
  console.log("Sain uu");
};
```



Функцийн талаар анх сурч байхдаа функцийн их биеийг заавал гоёлын хаалтаар (`{}`) хүрээлэх ёстой байсан. Гэхдээ хэрэв таны функцийн их бие зөвхөн нэг мөр кодноос бүрдэж байвал гоёлын хаалтыг дараах байдлаар хасаж болно:

```javascript
const greetings = name => console.log("Sain uu, " + name + "!");
```

Ердийн функцийн бичиглэл дээр дугуй хаалт болон гоёлын хаалтыг хасвал ажиллахгүй гэдгийг анхаарах нь чухал юм. Хэрэв та иймэрхүү зүйл хийхийг оролдвол алдаа заах болно:

```javascript
// Энэ нь бичиглэлийн (syntax) алдаа гаргана
function greetings name console.log("Sain uu, " + name + "!");
```

Энэ төрлийн нэг мөр бүхий функцүүд нь зөвхөн arrow функцийн бичиглэл ашиглаж байгаа үед л ажиллана. Өөр нэг гол ойлголт бол `return` оператор юм. Талбай тооцоолоход arrow функцийн бичиглэл ашигласан жишээг энд харуулав:

```javascript
const calculateArea = (width, height) => {
  const area = width * height;
  return area;
};

console.log(calculateArea(5, 3)); // 15
```

Бид функц дотор `area` нэртэй хувьсагч үүсгэж, дараа нь тэр хувьсагчийг буцааж байна. Гэхдээ бид кодоо арай цэгцтэй болгож, тооцооллыг өөрийг нь шууд буцааж болно:

```javascript
const calculateArea = (width, height) => {
  return width * height;
}; 

console.log(calculateArea(5, 3)); // 15
```

Хэрэв та гоёлын хаалтыг хасаад, тооцооллыг нэг мөрөнд байрлуулахыг оролдвол `Uncaught SyntaxError: Unexpected token 'return'` гэсэн алдааны зурвас авах болно:

```javascript
const calculateArea = (width, height) => return width * height;
```

Энэ алдаа гарч байгаа шалтгаан нь та `return` операторыг хасах шаардлагатай байгаад оршино. Та `return` операторыг хасахад алдаа арилж, функц тооцооллыг далд хэлбэрээр (implicitly) буцаасаар байх болно.

```javascript
const calculateArea = (width, height) => width * height;
```



Тэгвэл arrow функцийн бичиглэлийг хэзээ ашиглах вэ? Энэ нь нөхцөл байдлаас хамаарна. Олон хөгжүүлэгчид үүнийг өөрсдийн хувийн төслүүддээ тогтмол ашигладаг. Гэсэн хэдий ч баг болон хамт олноор ажиллаж байгаа үед сонголт нь ихэвчлэн тухайн кодын санд (codebase) ердийн функц эсвэл arrow функцийг ашиглаж байгаа эсэхээс хамаардаг. Дараагийн хичээлүүдээр бид хэзээ arrow функцийг ашиглах, хэзээ ашиглахаас зайлсхийх талаар үзэх болно.
