Cache болон Service Workers гэж юу вэ, мөн хэрхэн ажилладаг вэ?

Caching гэдэг нь файлуудын хуулбарыг түр хадгалах (temporary storage) байршилд хадгалж, тэдгээрийг илүү хурдан хандах боломжтой болгох үйл явц юм.

Та вэбсайт руу ороход таны browser зарим файлуудыг (зураг, CSS, JavaScript гэх мэт) локал байдлаар хадгалж чадна. Энэ нь дараагийн удаа тухайн сайтыг нээхэд эдгээр файлуудыг серверээс дахин авахын оронд таны төхөөрөмжөөс шууд ачаалж, сайтыг илүү хурдан нээх боломжийг олгодог.

Service worker гэдэг нь таны web page-ээс тусдаа, background-д ажилладаг script юм. Энэ нь network request-уудыг intercept хийх, cache-д хандах, мөн web app-ийг offline ажиллах боломжтой болгох үүрэгтэй. Энэ нь Progressive Web Apps-ийн гол бүрэлдэхүүн (key component) юм.

Тэгвэл cache болон service worker-ууд Progressive Web Apps-д хэрхэн хамтран ажилладаг вэ?

PWA-ууд нь app шиг туршлага (app-like experience) олгодог web app-ууд юм. Эдгээр нь offline ажиллах, push notification илгээх, мөн mobile төхөөрөмж эсвэл компьютерийн home screen дээр суулгах боломжтой.

Хэрэглэгч анх PWA-д нэвтрэхэд service worker чухал файлуудыг cache хийж чадна.

Хэрэглэгчид app-ийг offline үед ч ашиглах боломжтой бөгөөд дахин online болсны дараа хийгдсэн өөрчлөлтүүд сервертэй синхрончлогдоно (sync).

Caching болон service worker-ийн хослол нь муу network нөхцөлд ч web app-уудыг хурдан, найдвартай (fast and reliable) туршлага өгөх боломжийг бүрдүүлдэг.
