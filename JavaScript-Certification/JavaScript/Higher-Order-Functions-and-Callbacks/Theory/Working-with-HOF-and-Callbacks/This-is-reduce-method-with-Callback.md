Reduce арга гэж юу вэ, тэр нь хэрхэн ажилладаг вэ?

`reduce` арга нь JavaScript-д массивыг боловсруулж, түүнийг **ганцхан утга** (single value) болгон нэгтгэх боломжийг олгодог функц юм. Энэхүү ганц утга нь тоо, тэмдэгт мөр (string), объект, эсвэл бүр өөр нэг массив ч байж болно.

Үүнийг `reduce` (хураангуйлах, багасгах) гэж нэрлэсэн шалтгаан нь массивыг ганцхан үр дүн болгон "хураангуйлж" байгаатай холбоотой юм. Эхэндээ жаахан төвөгтэй санагдаж болох ч `reduce`-ийг ойлгож авснаар та олон тохиолдолд кодоо маш ихээр хялбарчлах боломжтой болно.

Үндсэндээ `reduce` нь массивын элемент бүрт дарааллын дагуу функцийг хэрэгжүүлж, тооцоолол бүрийн үр дүнг дараагийнх руу нь дамжуулах замаар ажилладаг. Энэхүү функцийг ихэвчлэн **reducer function** гэж нэрлэдэг.



Reducer функц нь үндсэн хоёр параметр хүлээн авдаг: **accumulator** (хуримтлуулагч) болон **current value** (одоогийн утга). 
* **Accumulator** нь таны хийж буй үйлдлүүдийн явац дахь үр дүнг хадгалах газар юм.
* **Current value** нь тухайн үед боловсруулагдаж буй массивын элемент юм.

`reduce` хэрхэн ажилладгийг тайлбарлах жишээг харна уу:

```javascript
const numbers = [1, 2, 3, 4, 5];
const sum = numbers.reduce(
  (accumulator, currentValue) => accumulator + currentValue,
  0
);

console.log(sum); // 15
```

Энэ жишээнд бид `reduce`-ийг ашиглан массив дахь бүх тоонуудын нийлбэрийг гаргаж авч байна.
Reducer функц нь accumulator-ийг авч (энэ нь `reduce`-ийн хоёр дахь аргументаар зааж өгснөөр 0-ээс эхэлж байна), тоо бүрийг түүн дээр нэмнэ.
Нэмэлт бүрийн үр дүн нь дараагийн алхмын шинэ accumulator болно.



`reduce` арга нь хоёр дахь аргумент болгон **initial value** (анхны утга)-г авч болно. Энэ нь accumulator-ийн эхлэх утга юм. Дээрх жишээнд бид үүнийг 0 гэж тохируулсан.

Хэрэв та анхны утгыг зааж өгөхгүй бол `reduce` нь массивын эхний элементийг анхны accumulator болгон ашиглаж, үйл явцыг хоёр дахь элементээс эхлүүлнэ.

`reduce`-ийн нэг гайхалтай тал нь түүний уян хатан чанар юм. Та reducer функцийг өөрөө тодорхойлдог учраас массивыг хэрхэн боловсруулах, ямар төрлийн үр дүн гаргаж авахаа бүрэн хянах боломжтой. Энэ нь `reduce`-ийг туйлын хүчирхэг болгодог ч, нөгөө талаас анх сурч байгаа хүнд жаахан ойлгоход бэрх болгож магадгүй юм.

Дадлага хийснээр та `reduce` аргатай ажиллах арга барилыг маш сайн эзэмших болно.
