 Method Chaining гэж юу вэ, тэр нь хэрхэн ажилладаг вэ?

**Method chaining** (аргуудын хэлхээ) гэдэг нь хэд хэдэн аргыг (methods) ар араас нь цувуулан дуудах техник юм. Та JavaScript-ийн олон төрлийн утгууд, тухайлбал тэмдэгт мөр (strings), массив (arrays) болон объектууд дээр энэхүү хэлхээг ашиглаж болно. Хэдийгээр тэмдэгт мөрүүд нь "primitive" (анхдагч) утга боловч, таныг string-ийн арга ашиглах үед JavaScript тэднийг түр зуур String объект дотор "боодог" (wrap) юм.



Янз бүрийн string аргуудыг ашигласан жишээг харна уу:

```javascript
const result = "  Hello, World!  "
  .trim()            // Сул зайг арилгах
  .toLowerCase()     // Жижиг үсэг рүү хөрвүүлэх
  .replace("world", "JavaScript"); // Солих

console.log(result); // "hello, JavaScript!"
```

Энэ жишээнд бид нэг тэмдэгт мөрөөс эхэлж, гурван үйлдлийг дараалан гүйцэтгэж байна: сул зайг арилгах, жижиг үсэг рүү хөрвүүлэх, мөн "world" гэдэг үгийг "JavaScript"-ээр солих. Арга бүр шинэ тэмдэгт мөр буцаадаг бөгөөд тэрхүү үр дүн нь дараагийн аргыг дуудах "бай" (target) болдог.

Method chaining нь нарийн төвөгтэй үйлдлүүд дээр ажиллаж байх үед кодын уншигдах чадварыг мэдэгдэхүйц сайжруулдаг.

Жишээлбэл, `filter`, `map` болон `reduce` аргуудыг хэлхэж ашигласан энэ тохиолдлыг авч үзье:

```javascript
const transactions = [
  { amount: 100, type: "credit" },
  { amount: 20, type: "cash" },
  { amount: 150, type: "credit" },
  { amount: 50, type: "cash" },
  { amount: 75, type: "credit" }
];

const totalCreditWithBonus = transactions
  .filter((transaction) => transaction.type === "credit") // 1. Зөвхөн credit-ийг шүүх
  .map((transaction) => transaction.amount * 1.1)         // 2. 10%-ийн бонус нэмэх
  .reduce((sum, amount) => sum + amount, 0);              // 3. Нийлбэрийг гаргах

console.log(totalCreditWithBonus); // 357.5
```



Энэ жишээнд бидэнд гүйлгээний (transactions) массив байгаа бөгөөд объект бүр нь дүн (amount) болон төрөл (credit эсвэл cash)-тэй байна.

Бид эхлээд гүйлгээнүүд дундуур шүүлтүүр (filter) хийж, зөвхөн кредит картын гүйлгээнүүдээс бүрдсэн шинэ массив үүсгэнэ. Дараа нь шүүгдсэн үр дүн дээрээ `map` аргыг хэлхэж, гүйлгээний дүн бүрийг 1.1-ээр үржүүлнэ (энэ нь 10%-ийн бонус нэмж буй хэрэг).

Үүний дараа тэрхүү үр дүн дээрээ `reduce` аргыг хэлхэж, дүн бүрийг нэмснээр 357.5 гэсэн үр дүнд хүрч байна.

Method chaining нь кодыг илүү товчхон бөгөөд уншихад хялбар болгодог ч үүнийг тунгалаг, зөв зохистой ашиглах нь чухал юм.

Маш урт хэлхээнүүд нь дебаг (debug - алдааг олох) хийхэд хүндрэлтэй болж магадгүй, учир нь хэлхээний аль алхамд алдаа гарч байгаа нь шууд тодорхой харагддаггүй. Тиймээс маш урт хэлхээг илүү ойлгомжтой, алдаа шалгахад хялбар болгох үүднээс хэд хэдэн алхам болгон хуваах нь ихэвчлэн зөв туршлага болдог.
