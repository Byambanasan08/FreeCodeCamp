Sort арга хэрхэн ажилладаг вэ?

Програмчлалд өгөгдлийг эрэмбэлэх (эрэмбэ тогтоох) маш олон төрлийн арга байдаг. Энэ хичээлээр бид JavaScript-ийн бэлэн `sort` арга дээр анхаарлаа хандуулах болно.

`sort` арга нь массивын элементүүдийг эрэмбэлэхэд ашиглагддаг бөгөөд эрэмбэлэгдсэн массивын холбоосыг (reference) буцаадаг. Элементүүдийг тухайн байрлалд нь (in place) шууд эрэмбэлдэг тул массивын хуулбар үүсдэггүй.

`sort` аргын үндсэн синтакс:

```javascript
array.sort(compareFunction);
```

`compareFunction` (харьцуулах функц) нь сонголтоор ашиглагддаг параметр бөгөөд эрэмбэлэх дарааллыг тодорхойлдог. Тоог хэрхэн эрэмбэлэх талаар бид дараа нь харьцуулах функцийг хэрхэн ашиглахыг үзэх болно.

Эхний жишээнд бидэнд санамсаргүй дараалалтай тэмдэгт мөрүүдийн (strings) массив байна:

```javascript
const fruits = ["Banana", "Orange", "Apple", "Mango"];
```

Бидний зорилго бол энэ массивыг цагаан толгойн дарааллаар эрэмбэлэх юм. Үүний тулд `fruits` массив дээр `sort` аргыг дуудахад хангалттай.

```javascript
const fruits = ["Banana", "Orange", "Apple", "Mango"];
fruits.sort();

console.log(fruits); // ["Apple", "Banana", "Mango", "Orange"]
```

Үр дүн нь "Apple" жимснээс эхэлсэн цагаан толгойн дарааллаар эрэмбэлэгдсэн массив байх болно.



Дараагийн жишээнд бид дараах тоон массивыг эрэмбэлэхийг хүсэж байна:

```javascript
const numbers = [414, 200, 5, 10, 3];
```

Хэрэв бид энэ тоон массив дээр шууд `sort` аргыг ашиглавал хүлээгдээгүй үр дүн гарна.

```javascript
const numbers = [414, 200, 5, 10, 3];
numbers.sort();

console.log(numbers); // [10, 200, 3, 414, 5]
```

Бид үр дүнг `[3, 5, 10, 200, 414]` гэж гарна гэж хүлээж байсан ч үүний оронд `[10, 200, 3, 414, 5]` гэж гарлаа. Энэ нь `sort` арга элементүүдийг **тэмдэгт мөр (string) рүү хөрвүүлж**, дараа нь тэдгээрийн **UTF-16 кодын утгыг** (code units) харьцуулдагтай холбоотой юм.

UTF-16 кодын утгууд нь тэмдэгт мөр дэх тэмдэгтүүдийг илэрхийлдэг тоон утгууд юм. Жишээ нь 65, 66, 67 гэсэн тоонууд нь харгалзан A, B, C тэмдэгтүүдийг илэрхийлдэг. Иймээс "200" гэсэн тэмдэгт мөрний эхний тэмдэгт "2" нь "3" гэсэн тэмдэгт мөрнөөс өмнө эрэмбэлэгддэг учраас 200 нь 3-ын өмнө бичигдэж байгаа хэрэг юм.



Энэ асуудлыг шийдэх арга бол `sort` аргад **харьцуулах функц** (compare function) өгөх явдал юм. Харьцуулах функц ашиглан тоон массивыг хэрхэн эрэмбэлэх жишээг харна уу:

```javascript
const numbers = [414, 200, 5, 10, 3];

numbers.sort((a, b) => a - b);

console.log(numbers); // [3, 5, 10, 200, 414]
```

Энд `a` болон `b` параметрүүд нь харьцуулагдаж буй хоёр элемент юм. Харьцуулах функц нь:
* Хэрэв `a` нь `b`-ийн өмнө орох ёстой бол **сөрөг утга**,
* Хэрэв `a` нь `b`-ийн дараа орох ёстой бол **эерэг утга**,
* Хэрэв `a` болон `b` тэнцүү бол **тэг** утгыг буцаах ёстой.

Эхний харьцуулалт 414 болон 200-ийн хооронд явагдана. $414 - 200 = 214$ буюу эерэг утга гарч байна. Энэ нь 414 нь 200-ийн дараа орох ёстойг илэрхийлнэ. Дараагийн харьцуулалт 200 болон 5-ын хооронд явагдаж, $200 - 5 = 195$ (эерэг утга) гарна. Энэ нь 200 нь 5-ын дараа орох ёстойг заана. Энэ процесс массивын бүх элементүүд дээр давтагдсанаар тоонууд зөв дарааллаар эрэмбэлэгдэнэ.



Програмчлалд өгөгдөл эрэмбэлэх маш олон арга байдаг ч JavaScript-ийн бэлэн `sort` арга нь массивыг эрэмбэлэх шаардлагатай олон тохиолдолд маш хэрэгцээтэй бөгөөд үр ашигтай байдаг.
