### Null болон Undefined өгөгдлийн төрлүүдийн харьцуулалт хэрхэн ажилладаг вэ?

JavaScript хэлэнд `null` болон `undefined` нь утга байхгүйг илэрхийлдэг хоёр өөр өгөгдлийн төрөл боловч харьцуулалт хийх үед тэдгээрийн ажиллах зарчим өөр өөр байдаг. Эдгээр төрлүүд харьцуулалтын төрөл бүрийн нөхцөлд хэрхэн харилцан үйлчилж байгааг ойлгох нь алдаагүй, найдвартай код бичихэд нэн чухал юм.

`undefined` төрлөөс эхэлцгээе. Хувьсагчийг зарласан боловч утга оноогоогүй тохиолдолд тухайн хувьсагч нь `undefined` байна. Энэ нь анхны утга оноогоогүй (uninitialized) хувьсагчид болон аргумент дамжуулаагүй функцийн параметрийн өгөгдмөл (default) утга юм.

Харин `null` төрөл нь зориудаар "утга байхгүй" гэдгийг илэрхийлж оноодог утга юм. Энэ нь ихэвчлэн хувьсагчийг санаатайгаар ямар ч утгагүй гэдгийг заахад ашиглагддаг.

`null` болон `undefined`-ийг тэнцүүгийн оператор (`==`) ашиглан харьцуулах үед JavaScript төрөл хувиргалт (type coercion) хийдэг. Энэ нь харьцуулалт хийхээсээ өмнө операторуудыг ижил төрөл рүү хөрвүүлэхийг оролддог гэсэн үг юм. Энэ тохиолдолд `null` болон `undefined` нь хоорондоо тэнцүүд тооцогдоно:

```javascript
console.log(null == undefined); // true
```

Гэсэн хэдий ч, утга болон төрлийг хоёуланг нь шалгадаг, төрөл хувиргалт хийдэггүй нарийн тэнцүүгийн операторыг (`===`) ашиглах үед `null` болон `undefined` нь тэнцүү биш байна:

```javascript
console.log(null === undefined); // false
```

Нөхцөлт шалгах оператор бичих эсвэл коддоо тэнцүү эсэхийг шалгах үед энэ ялгааг санаж байх нь чухал юм. `null` эсвэл `undefined`-ийг бусад утгуудтай тэнцүүгийн оператор (`==`) ашиглан харьцуулахад үр дүн нь төсөөлөөгүй байдлаар гарч болно. Жишээ нь:

```javascript
console.log(null == 0);  // false
console.log(null == ''); // false
console.log(undefined == 0); // false
console.log(undefined == ''); // false
```

Тэнцүүгийн оператор ашиглах үед `null` болон `undefined` нь зөвхөн бие биетэйгээ (болон өөрсөдтэйгөө) тэнцүү байдаг тул эдгээр харьцуулалтууд `false` утга буцааж байна. Бусад харьцуулалтууд дээрх `null`-ийн ажиллах зарчим нь нэлээд ээдрээтэй:

```javascript
console.log(null > 0);  // false
console.log(null == 0); // false
console.log(null >= 0); // true
```

Нөгөө талаас, `undefined` нь тоон контекстэд үргэлж `NaN` (Not-a-Number) рүү хөрвүүлэгддэг бөгөөд энэ нь `undefined`-ийг тоотой харьцуулсан бүх харьцуулалтыг `false` болгодог:

```javascript
console.log(undefined > 0);  // false
console.log(undefined < 0);  // false
console.log(undefined == 0); // false
```

Эдгээр нарийн ялгаануудыг харгалзан үзвэл, утгуудыг харьцуулахдаа, ялангуяа `null` болон `undefined`-тэй ажиллах үед нарийн тэнцүүгийн операторыг (`===`) ашиглахыг зөвлөдөг. Энэ хандлага нь гэнэтийн төрөл хувиргалтаас зайлсхийхэд тусалж, кодын ажиллагааг илүү тодорхой болгодог.

Дүгнэж хэлэхэд, `null` болон `undefined` нь хоёулаа утга байхгүйг илэрхийлэхэд ашиглагддаг боловч харьцуулалт хийх үед өөр өөрөөр ажилладаг. Эдгээр ялгааг ойлгох нь ойлгомжтой бөгөөд алдаагүй JavaScript код бичих гол түлхүүр юм.
