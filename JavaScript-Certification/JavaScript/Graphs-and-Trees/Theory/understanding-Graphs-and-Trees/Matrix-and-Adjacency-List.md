Matrix болон Adjacency List хэрхэн ажилладаг вэ?

Graph-ууд нь node-уудын (мөн vertices гэж нэрлэдэг) цогц болон тэдгээрийг холбосон edge-үүдээс бүрдсэн маш хүчирхэг data structure-ууд юм.

Graph-уудыг код дотор хэрэгжүүлэх (implement) хоёр нийтлэг арга байдаг:

* Adjacency matrices
* Adjacency lists

Эдгээрийг илүү дэлгэрэнгүй авч үзээд, давуу болон сул талуудыг нь ойлгоё.
Adjacency Matrices

Эхлээд adjacency matrices-ийг авч үзье.

Adjacency matrix гэдэг нь хоёр хэмжээст array (two-dimensional array) бөгөөд мөр (rows) болон баганууд (columns) нь graph-ийн vertices-ийг илэрхийлдэг.

Matrix доторх утгууд нь node-уудын хоорондын edge эсвэл холболтыг илэрхийлдэг.

Жишээлбэл, хэрэв matrix гэдэг хувьсагчид хадгалагдсан matrix байвал, matrix[i][j] дээр хадгалагдсан утга (энд i нь мөр, j нь багана) нь node i болон node j-ийн хоорондын edge эсвэл холболтыг илэрхийлнэ.

Graph нь жинтэй (weighted) эсвэл жингүй (unweighted) эсэхээс хамаарч эдгээр утгуудын утга өөр өөр байж болно:

* Хэрэв graph нь жингүй (unweighted) бол, 1 гэсэн утга нь эдгээр node-уудын хооронд edge байна гэсэн үг, харин 0 гэсэн утга нь edge байхгүйг илэрхийлнэ.

* Хэрэв graph нь жинтэй (weighted) бол, тухайн утга нь node-уудыг холбож буй edge-ийн жинг (weight) илэрхийлнэ.

Adjacency matrix ашиглахын нэг том давуу тал нь хоёр node-ийн хооронд edge байгаа эсэхийг шалгах нь тогтмол хугацаанд O(1) гүйцэтгэгддэг. Учир нь програм зөвхөн 2D array доторх тухайн утгыг л олоход хангалттай.

Гэхдээ edge-үүдийг олоход ийм үр ашигтай байдал нь нэг сул талтай хамт ирдэг. Adjacency matrix нь O(V²) хэмжээний квадрат (quadratic) санах ойн шаардлагатай байдаг, энд V нь graph-ийн node-уудын тоо юм.

Энэ нь sparse graph-ууд (edge цөөнтэй graph)-д үр ашиггүй. Яагаад вэ? Учир нь graph нь sparse байвал, node-уудын хооронд edge байхгүйг илэрхийлэхийн тулд matrix дотор маш олон 0 утгууд хадгалагдана, эдгээр 0-үүд нь мөн санах ой эзэлсээр байдаг.

Мөн adjacency matrix нь node-ийн хөршүүдийг (neighbors) олоход үр ашиггүй байдаг, учир нь програм бүхэл мөр эсвэл баганаар гүйж (iterate) 0 болон 1 утгуудыг шалгах шаардлагатай болдог. Хамгийн муу тохиолдолд энэ процесс O(V) хугацаа авна, энд V нь graph-ийн node-уудын тоо юм.

Одоо энэ тодорхой graph-д зориулсан adjacency matrix-ийн жишээг харцгаая:

Adjacency matrix-д:

* Мөр бүр (row) нь нэг node-ийг илэрхийлнэ. Эхний мөр нь node A-г, хоёр дахь мөр нь node B-г гэх мэтээр илэрхийлнэ.

* Багана бүр (column) нь мөн node-ийг илэрхийлнэ.

* Matrix доторх утга бүр нь node-уудын хос бүрийн хооронд edge байгаа эсэхийг илэрхийлнэ. 0 гэсэн утга нь эдгээр node-уудын хооронд edge байхгүйг, харин 1 гэсэн утга нь edge байгааг илэрхийлнэ.

Диагональ дээрх утгууд нь node бүр өөртэйгөө холбогдсон (self-loop) эсэхийг илэрхийлдэг, өөрөөр хэлбэл node өөрийгөө холбосон edge байгаа эсэхийг. Манай жишээнд бүгд 0 байгаа, учир нь энэ graph-д self-loop байхгүй.

Энэ нь adjacency matrix-ийн харагдах байдлыг харуулах visual representation бөгөөд мөр болон баганууд нь харгалзах node-уудыг хэрхэн илэрхийлж байгааг үзүүлж байна.

Жишээлбэл, эхний мөр нь [0, 1, 1, 1] байна, учир нь node A нь node B, C, болон D-тэй холбогдсон:

# A  B  C  D

# A   [0, 1, 1, 1],

# B   [1, 0, 0, 1],

# C   [1, 0, 0, 0],

# D   [1, 1, 0, 0]

Мөн энэ нь яг адил adjacency matrix-ийг JavaScript кодоор хэрэгжүүлсэн хувилбар юм:

```javascript
const adjacencyMatrix = [
  [0, 1, 1, 1], // A-ийн хөршүүд нь B, C, болон D
  [1, 0, 0, 1], // B-ийн хөршүүд нь A болон D
  [1, 0, 0, 0], // C-ийн цорын ганц хөрш нь A
  [1, 1, 0, 0]  // D-ийн хөршүүд нь A болон B
];
```

Adjacency Lists

Graph-ийг илэрхийлэх өөр нэг түгээмэл арга бол adjacency lists ашиглах юм.

Adjacency list гэдэг нь node бүрийн бүх хөршүүдийг хадгалдаг array эсвэл object (эсвэл Map) юм.

Adjacency list-ийг хэрэгжүүлэх хоёр нийтлэг арга байдаг:

* Array байдлаар, энд index бүр нь node-ийг илэрхийлж, тухайн index дээр хадгалагдсан array нь түүний хөршүүдийг агуулна.
* Object (эсвэл Map) байдлаар, энд key бүр нь node-ийг илэрхийлж, тухайн key-д холбогдсон утга (array) нь түүний хөршүүдийг агуулна.

Adjacency list нь санах ойн хэрэглээний хувьд adjacency matrix-ээс илүү үр ашигтай. Энэ нь O(V + E) space complexity-тэй байдаг, энд V нь vertices (node)-ийн тоо, E нь edge-үүдийн тоо юм.

Мөн хөрш node-уудыг олоход үр ашигтай, учир нь энэ үйлдэл нь тухайн node-тэй холбогдсон list-ийг шууд авахад л хангалттай.

Гэхдээ энд бас tradeoff бий.

Adjacency list нь хоёр node-ийн хооронд edge байгаа эсэхийг шалгахад adjacency matrix-ээс бага үр ашигтай.

Хайлт хийх процесс нь ихэвчлэн O(deg(v)) хугацаа авдаг (энд deg(v) нь шалгаж буй node-ийн хөршүүдийн тоо), харин хамгийн муу тохиолдолд хэрэв нэг node бүх бусад node-уудтай холбогдсон бол энэ нь O(V) болж болно.

Энэ graph-д зориулсан adjacency list-ийн жишээг харцгаая:

Энэ adjacency list нь object хэлбэрээр хэрэгжсэн байна. Object доторх key бүр нь нэг node-ийг илэрхийлдэг бөгөөд тухайн key-д харгалзах утга нь тухайн node-ийн бүх хөршүүдийг агуулсан array юм:

```javascript
const adjacencyList = {
  A: ["B", "C", "D"],
  B: ["A", "D"],
  C: ["A"],
  D: ["A", "B"]
};
```

Өөрөөр хэлбэл, бид үүнийг 2 хэмжээст array (2D array) байдлаар хэрэгжүүлж болно, энд index бүр нь node-ийг илэрхийлнэ. Жишээлбэл, index 0 нь node A-г, index 1 нь node B-г гэх мэт:

```javascript
const adjacencyList = [
  ["B", "C", "D"], // A-ийн хөршүүд (index 0)
  ["A", "D"],      // B-ийн хөршүүд (index 1)
  ["A"],           // C-ийн хөршүүд (index 2)
  ["A", "B"]       // D-ийн хөршүүд (index 3)
];
```

Энэ 2D array нь adjacency matrix-тэй төстэй харагдаж болох ч, үнэндээ маш ялгаатай гэдгийг анзаараарай.

* Adjacency matrix нь graph доторх edge-үүд эсвэл edge-ийн жинг (weights) илэрхийлэх 0, 1, эсвэл бусад утгуудыг хадгалдаг.

* Adjacency list нь node бүрийн бүх хөршүүдийн бодит жагсаалтыг (actual list) хадгалдаг.

Энэ нь маш чухал ялгаа бөгөөд та үүнийг сайн ойлгосон байх хэрэгтэй.

Adjacency matrix болон adjacency list аль аль нь graph-ийг хэрэгжүүлэхэд маш чухал юм. Аль нэгийг нь сонгох нь graph-ийн хэмжээ болон өгөгдлийг хэрхэн ашиглах хэрэгцээнээс хамаарна. Adjacency matrix нь edge ихтэй (dense) graph-д илүү тохиромжтой бол, adjacency list нь бодит амьдралын нөхцөлд илүү түгээмэл тохиолддог edge цөөнтэй (sparse) graph-д ихэвчлэн илүү тохиромжтой сонголт байдаг.
