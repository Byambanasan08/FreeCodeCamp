Computer Science-д Graph гэж юу вэ?

Graph-ууд нь объектууд эсвэл entity-үүдийн хоорондын холболт (connections) эсвэл харилцааг (relationships) илэрхийлэхэд ашиглагддаг data structure-ууд юм. Эдгээрийг ихэвчлэн network-уудыг моделчлоход ашигладаг.

Graph ашиглан моделчилж болох network-уудын төрөлд social networks, transportation networks, communications networks, мөн recommendation systems хүртэл багтана.

Жишээлбэл, graph-ууд нь social media platform дээрх хэрэглэгчдийн хоорондын холболтыг, эсвэл замын network дахь хотуудын хоорондын холболтыг илэрхийлж чадна.

Эдгээр нь маш уян хатан (versatile) юм.

Graph нь ихэвчлэн хоорондоо шугамаар (lines) холбогдсон цэгүүд (points) эсвэл дугуй (circles)-уудын цуглуулга байдлаар дүрслэгддэг.

Эдгээр дугуй болон шугамууд нь graph-ийн үндсэн хоёр бүрэлдэхүүн хэсгийг илэрхийлдэг: nodes болон edges. Nodes (эсвэл vertices гэж нэрлэдэг) нь graph-аар моделчилж буй network-ийн объектууд эсвэл entity-үүдийг илэрхийлдэг. Эдгээр нь хэрэглэгчид (users), бүтээгдэхүүнүүд (products), станцууд (stations), хотууд (cities), эсвэл загварт (model) багтсан бусад entity-үүд байж болно.

Энэ жишээнд nodes-уудыг дугуйгаар илэрхийлж, тэдгээрийг харагдах байдлаар ялгахын тулд A, B, C, D, болон E үсгүүдээр тэмдэглэсэн.

Edges нь node-уудын хоорондын холболтууд юм. Хэрэв хоёр node edge-ээр холбогдсон байвал тэд network дотор ямар нэгэн байдлаар холбогдсон гэсэн үг.

Энэ жишээнд өөр өөр node-уудын хосуудыг холбосон таван edge байна. Node A нь node B-тэй холбогдсон. Node B нь node A, C, болон D-тэй холбогдсон гэх мэт.

Холболтын тодорхой утга нь context-оос хамаарна. Энэ нь хоёр хотыг холбосон бодит зам (physical) байж болох эсвэл social media platform дээрх хоёр хэрэглэгчийн хоорондын холбоо шиг илүү хийсвэр (abstract) байж болно.

Хэрэв хоёр node edge-ээр шууд холбогдсон байвал, жишээ нь энэ жишээнд A болон B node-ууд, тэдгээрийг adjacent nodes гэж нэрлэдэг.

Графын төрлүүд (Types of Graphs)

Янз бүрийн шинж чанар, хэрэглээтэй олон төрлийн graph байдаг. Тэдгээрийн заримыг нь авч үзье.
Чиглэлгүй графууд (Undirected Graphs)

Чиглэлгүй графууд нь edge-үүд нь тодорхой чиглэлгүй байдаг graph-ууд юм. Энэ төрлийн edge-ийг ихэвчлэн node-уудын хооронд татсан шулуун шугамаар дүрсэлдэг. Энэ нь хэрэв A болон B node-уудыг холбосон edge байвал, уг холболт хоёр чиглэлд хоёуланд нь үйлчилнэ гэсэн үг: node A-аас node B рүү, мөн node B-ээс node A рүү.

Graph-аар моделчилж буй network-оос хамааран энэ холболт өөр өөр утгатай байж болно.

Жишээлбэл, хэрэв та social media platform дээрх хэрэглэгчдийн хоорондын холболтыг моделчилж байгаа бол, энэ нь хэрэглэгч A нь хэрэглэгч B-тэй холбогдсон, мөн хэрэглэгч B нь хэрэглэгч A-тай холбогдсон гэсэн үг. Энэ холболт нь хоёр чиглэлтэй (bidirectional) юм.
Чиглэлтэй графууд (Directed Graphs)

Үүний эсрэгээр, чиглэлтэй графууд нь edge-үүд нь тодорхой чиглэлтэй байдаг graph-ууд юм.

Хэрэв node A-аас node B рүү холболт байвал, энэ нь заавал node B-ээс node A рүү холболт байна гэсэн үг биш.

Чиглэлтэй graph-ийн edge-үүдийг ихэвчлэн төгсгөлдөө сумтай шулуун шугамаар дүрсэлж, чиглэлийг илэрхийлдэг.

Энд нэг жишээ байна. Энэ graph-д та node A-аас node B рүү явж чадна, харин node B-ээс node A рүү явж чадахгүй, учир нь edge-ийн чиглэл тийм байна. Жишээлбэл, хэрэв та замын network моделчилж байгаа бол, энэ нь нэг чигийн замуудыг илэрхийлэхэд ашигтай. Та хот A-аас хот B рүү тухайн замаар явж чадна, харин хот B-ээс хот A рүү явж чадахгүй. Та өөр зам сонгох хэрэгтэй болно.

Хэрэв чиглэлтэй graph-ийн node-уудын хооронд хоёр чиглэлтэй холболт байвал, үүнийг хоёр чиглэлтэй edge ашиглан илэрхийлж болно.

Энд нэг жишээг харж болно. Та node B-ээс node D рүү, мөн node D-ээс node B рүү явж чадна, учир нь тэднийг холбосон хоёр edge байгаа, гэхдээ edge бүр өөрийн чиглэлтэй.

Vertex шошготой графууд (Vertex Labeled Graphs)

Vertex шошготой граф (vertex labeled graph) гэдэг нь node бүр нь өөрийн өгөгдлөөс гадна шошго (label) эсвэл тодорхойлогч (identifier)-той холбогдсон байдаг graph юм. Эдгээр шошгууд нь node-уудыг ялган таних, тэдгээрийг харагдах байдлаар илэрхийлэх, мөн тэдгээрийн талаар нэмэлт мэдээлэл хадгалахад ашиглагддаг.

Жишээлбэл, тээврийн network-ийн graph-д node-ууд нь хотууд байж болох бөгөөд тэдний шошгууд нь хотын нэр, координат, эсвэл тухайн моделийн зорилгод хэрэгтэй бусад шинж чанарууд байж болно.
Циклтэй графууд (Cyclic Graphs)

Циклтэй графууд (cyclic graphs) нь дор хаяж нэг цикл (cycle)-тэй чиглэлтэй графууд (directed graphs) юм.

Цикл (cycle) гэдэг нь graph-ийн edge-үүдийг дагаж явснаар эхэлсэн node руугаа буцаж очих боломжтой зам (path) юм.

Энэ жишээнд бид чиглэлтэй graph байна. Илүү анхааралтай харвал энэ нь цикл агуулж байгааг анзаарах болно. Хэрэв бид node B-ээс эхлээд node C рүү, дараа нь node D рүү явбал, чиглэлтэй edge-үүдээр дамжин дахин node B рүү буцаж чадна.

Энэ нь цикл тул энэ нь cyclic graph юм.
Edge шошготой графууд (Edge Labeled Graphs)

Edge шошготой графууд (edge labeled graphs)-д edge-үүд нь шошготой (labels) холбогдсон байдаг. Эдгээр шошгуудыг ихэвчлэн тухайн edge-ийн хажууд дүрсэлдэг.
Жинтэй графууд (Weighted Graphs)

Жинтэй графууд (weighted graphs) нь edge шошготой графын нэг тусгай төрөл бөгөөд edge дээрх шошгууд нь хооронд нь харьцуулж болох, мөн арифметик үйлдэл хийхэд ашиглаж болох утгуудыг илэрхийлдэг.

Зарим edge-үүд өндөр жинтэй (higher weight), харин зарим нь бага жинтэй (lower weight) байдаг. Эдгээр жин (weights) нь тухайн edge-ийн “өртөг” (cost)-ийг илэрхийлдэг.

Жишээлбэл, эдгээр нь хоёр хотын хоорондын зай (distance), эсвэл нэг хотоос нөгөө хот руу очих хугацаа (time)-г илэрхийлж болно.

Энэ бол жинтэй графын (weighted graph) жишээ юм. Бид жин бүрийг өөрийн харгалзах edge-ийн хажууд бичдэг. Node B-ээс node D рүү явах “өртөг” нь 3 бөгөөд энэ нь чиглэлгүй граф (undirected graph) учраас node D-ээс node B рүү буцахад мөн адил өртөгтэй байна.

Чиглэлтэй циклгүй граф (Directed Acyclic Graph)

Computer science-д маш өргөн хэрэглэгддэг өөр нэг төрлийн graph бол directed acyclic graph бөгөөд энэ нь циклгүй (acyclic) чиглэлтэй граф (directed graph) юм.

Энд нэг жишээ байна. Энэ нь чиглэлтэй граф, учир нь edge бүр тодорхой чиглэлтэй.

Мөн энэ нь циклгүй (acyclic), учир нь ямар ч цикл байхгүй. Яагаад вэ? Анзаарахад, хэрэв та тодорхой нэг node-оос эхэлбэл, edge-үүдийн чиглэлийн улмаас дахин тэр node руу буцаж очих боломжгүй.
Холбогдоогүй граф (Disconnected Graph)

Мөн энэ хичээлд авч үзэх сүүлийн төрлийн graph бол disconnected graph юм.

Disconnected graph гэдэг нь хоёр буюу түүнээс олон бүлэг node-уудтай бөгөөд тэдгээр бүлгүүд нь хоорондоо ямар ч edge-ээр холбогдоогүй байдаг graph юм.

Бодит жишээ нь social media network байж болно, энд хоёр буюу түүнээс олон бүлэг хүмүүс хоорондоо огт танил биш, мөн нийтлэг найзгүй байдаг.

Энэ бол жишээ. Эхний компонент (component) нь A, B, C node-уудтай. Хоёр дахь компонент нь D болон E node-уудтай. Эдгээр компонентуудын хооронд ямар ч edge байхгүй тул энэ нь disconnected graph юм. Та graph-уудыг sets, functions, болон arrays зэрэг олон янзаар хэрэгжүүлж (implement) болно. Та энэ талаар дараагийн хичээлүүдэд илүү ихийг сурах болно.

Graph-ууд болон graph-ийн янз бүрийн төрлүүдийн шинж чанарыг ойлгох нь computer science болон бусад олон салбарт өргөн хүрээний асуудлуудыг шийдвэрлэхэд маш чухал юм.
