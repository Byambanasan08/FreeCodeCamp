Depth First болон Breadth First Search хэрхэн ажилладаг вэ?

Та data structures болон algorithms-той ажиллаж эхлэхэд, хийх шаардлагатай хамгийн түгээмэл үйлдлүүдийн нэг нь node бүрийг очиж үзэх (visiting each node) гэдгийг удахгүй ойлгох болно.

Энэ процессыг data structure-ийг “traversing” хийх гэж нэрлэдэг.

Traversal-ууд нь data structure доторх node бүр дээр ямар нэгэн үйлдэл хийхэд ашиглагддаг, жишээлбэл тэдний утгуудыг хэвлэх, тодорхой утгыг олох, эсвэл node-ууд дээр тодорхой үйлдлүүд хийх.

Node бүрийг системтэйгээр очиж үзсэнээр та ямар ч node алгасагдахгүй гэдгийг баталгаажуулдаг.

Гэхдээ data structure-ийг ямар дарааллаар (order) traversal хийхийг яаж тодорхойлох вэ? Процесс хаанаас эхлэх ёстой вэ, дараагийн node-ийг хэрхэн сонгох вэ?

Хэрэв data structure-ийг traversal хийх тодорхой арга байхгүй бол, энэ нь яг л тодорхой замгүй лабиринт (maze) дотор явахтай адил болно.

Ийм үед breadth-first search (BFS) болон depth-first search (DFS) зэрэг algorithm-ууд маш чухал болдог. Эдгээрийг graph-уудыг traversal хийхэд, мөн хоёр node-ийн хооронд зам (path) олоход өргөн ашигладаг.

Эдгээрийг data structure-ийг traversal хийхэд ашиглах үед, node-уудыг ямар дарааллаар очиж үзэхийг тодорхойлж өгдөг бөгөөд ингэснээр ямар ч node алгасагдахгүй.

Одоо breadth-first search (BFS)-аас эхэлье.
Breadth-First Search (BFS)

Breadth-first search (BFS) нь graph дотор дараагийн түвшин (level) рүү шилжихээс өмнө тухайн node-ийн бүх хөрш (neighbor) node-уудыг эхэлж очиж үздэг algorithm юм.

Энэ нь жингүй (unweighted) graph дотор хоёр node-ийн хооронд хамгийн богино зам (shortest path)-ыг олоход ашиглагдаж болно, учир нь энэ нь түвшин бүр дээрх бүх node-уудыг шалгадаг тул хамгийн цөөн edge-тэй замыг эхэлж олдог.

Энэ algorithm-ыг ихэвчлэн queue data structure ашиглан хэрэгжүүлдэг бөгөөд энэ нь очиж үзсэн node-уудыг хянахад ашиглагддаг. Queue нь FIFO (first in, first out) зарчмыг дагадаг, өөрөөр хэлбэл queue-д хамгийн түрүүнд орсон node нь хамгийн түрүүнд гарна.

Algorithm дараах байдлаар ажиллана:

* Та тодорхой нэг node-оос эхэлнэ.

* Тэр node-ийг “visited” гэж тэмдэглээд queue-д нэмнэ.

* Queue хоосон биш байх хугацаанд, одоогийн node-ийг queue-оос гаргана (dequeued). Дараа нь тухайн node-ийн бүх хөрш node-уудын хувьд, хэрэв хөрш нь өмнө нь очиж үзээгүй бол, түүнийг “visited” гэж тэмдэглээд queue-д нэмнэ.

Анхаарах нэг чухал зүйл нь, breadth-first search (BFS) нь queue-г санах ойд хадгалах шаардлагатай бөгөөд энэ queue нь олон node агуулж болох тул энэ algorithm-ын space requirement (санах ойн хэрэглээ) нэлээд их байж болно. Ялангуяа нэг түвшинд олон node-тай graph-д энэ нь илүү мэдэгдэхүйц байдаг.

Одоо BFS-ийг graph-ийн нэг төрөл болох tree дээр хэрэглэх жишээг харцгаая.

Та дараагийн хичээлүүдэд tree-ийн талаар илүү ихийг сурах болно, гэхдээ tree нь үндсэндээ циклгүй (no cycles) graph бөгөөд node-ууд нь шаталсан бүтэцтэй (hierarchy) байдлаар зохион байгуулагдсан байдаг. Цикл (cycle) гэдэг нь эхэлсэн node дээрээ буцаж ирдэг зам юм.

Одоо энэ tree дээр breadth-first search (BFS) algorithm-ыг хэрэгжүүлье:

Алхам 1:

Бид tree-ийн root болох node A-аас эхэлнэ. Бид A-г queue-д нэмээд шууд “visited” гэж тэмдэглэнэ.

* Queue: [A]

* Visited: {A}

Алхам 2:

Бид node A-г queue-оос гаргана (dequeue). Бид түүний очиж үзээгүй (unvisited) хүүхдүүдийг (node B дараа нь node C) queue-д нэмээд тэднийг “visited” гэж тэмдэглэнэ.

* Queue: [B, C]

* Visited: {A, B, C}

Ижил түвшинд (same level) байгаа node-уудыг queue-д ямар дарааллаар нэмэх нь data structure-ийн implementation болон graph-ийн representation дотор edge-үүд (connections) ямар дарааллаар хадгалагдсанаас хамаарна.

Хэрэв implementation тогтвортой (consistent) байвал, ижил түвшний node-уудыг traversal хийх тодорхой дараалал algorithm-ын зөв байдалд нөлөөлөхгүй. Энэ нь ямар ч байсан бүх node-уудыг түвшин бүрээр нь очиж үзнэ.

Алхам 3:

Бид node B-г queue-оос гаргана. Бид түүний очиж үзээгүй хүүхдүүдийг (node D дараа нь node E) queue-д нэмээд “visited” гэж тэмдэглэнэ.

* Queue: [C, D, E]

* Visited: {A, B, C, D, E}

Алхам 4:

Бид node C-г queue-оос гаргана. Бид түүний очиж үзээгүй хүүхдүүдийг (node F дараа нь node G) queue-д нэмээд “visited” гэж тэмдэглэнэ.

* Queue: [D, E, F, G]

* Visited: {A, B, C, D, E, F, G}

Алхам 5:

Бид node D-г queue-оос гаргана. Энэ node нь очиж үзээгүй хүүхдүүдгүй тул visited set-д ямар ч өөрчлөлт гарахгүй.

* Queue: [E, F, G]

* Visited: {A, B, C, D, E, F, G}

Алхам 6:

Бид node E-г queue-оос гаргана. Энэ node нь очиж үзээгүй хүүхдүүдгүй тул visited set-д ямар ч өөрчлөлт гарахгүй.

* Queue: [F, G]

* Visited: {A, B, C, D, E, F, G}

Алхам 7:

Бид node F-г queue-оос гаргана. Энэ node нь очиж үзээгүй хүүхдүүдгүй тул visited set-д ямар ч өөрчлөлт гарахгүй.

* Queue: [G]

* Visited: {A, B, C, D, E, F, G}

Алхам 8:

Бид node G-г queue-оос гаргана. Энэ node нь очиж үзээгүй хүүхдүүдгүй тул visited set-д ямар ч өөрчлөлт гарахгүй.

* Queue: []

* Visited: {A, B, C, D, E, F, G}

Queue хоосон болсон үед traversal дуусна.

Node-уудыг дараах дарааллаар traversal хийсэн:

A → B → C → D → E → F → G

Algorithm нь node-уудыг түвшин (level)-ээр нь хэрхэн очиж үзэж байгааг анзаараарай.

Бид node A-аас эхэлж, дараа нь дараагийн түвшинд шилжин node B болон C-г очиж үзээд, дараагийн түвшинд node D, E, F, болон G-г очиж үздэг. Энэ нь breadth-first search (BFS)-ийн үндсэн зарчим юм.
Depth-First Search (DFS)

Breadth-first search (BFS) нь эхлээд ижил түвшинд байгаа бүх хөрш node-уудыг очиж үздэг бол, depth-first search (DFS) нь нэг мөчир (branch)-ийг аль болох гүн рүү дагаж, дараа нь буцаж (backtrack) өөр замыг судалдаг.

Та энэ algorithm-ыг лабиринт (maze)-ийг судалж байгаа мэтээр төсөөлж болно: та нэг замыг сонгоод эцэс хүртэл нь дагана, хэрэв мухардал (dead end) тулгарвал буцаж, өөр зам сонгоно.

Depth-first search (DFS)-ийг ихэвчлэн нэг шийдэлтэй бодлогуудыг (puzzles) шийдэх, graph доторх циклүүдийг (cycles) илрүүлэх, мөн холбогдсон компонентуудыг (connected graph components) олоход ашигладаг.

Энэ algorithm-ыг recursion ашиглан эсвэл stack data structure ашиглан хэрэгжүүлж болно, энэ нь очиж үзсэн node-уудыг хянахад ашиглагддаг.

Stack нь LIFO (last in, first out) зарчмыг дагадаг, өөрөөр хэлбэл stack-д хамгийн сүүлд орсон node нь хамгийн түрүүнд гардаг.

Algorithm дараах байдлаар ажиллана:

* Та тодорхой нэг node-оос эхэлнэ.

* Тэр node-ийг “visited” гэж тэмдэглээд stack-д нэмнэ.

* Stack хоосон биш байх хугацаанд, одоогийн node-ийг pop (гаргах) хийнэ. Энэ үед бид түүнийг “visit” буюу боловсруулж байна (жишээ нь, утгыг нь хэвлэх). Дараа нь түүний бүх очиж үзээгүй хөрш node-уудыг “visited” гэж тэмдэглээд stack-д нэмнэ.

Энэ algorithm-ын нэг сул тал нь жингүй (unweighted) graph дотор хоёр node-ийн хооронд хамгийн богино замыг үргэлж олох баталгаа байдаггүй.

Одоо бид Depth-First Search (DFS)-ийг өмнөх tree жишээн дээрээ хэрхэн хэрэглэхийг харцгаая.

Алхам 1:

Бид root node болох A-аас эхэлнэ. Бид үүнийг “visited” гэж тэмдэглээд stack-д нэмнэ.

* Stack: [A]

* Visited: {A}

Алхам 2:

Бид node A-г stack-оос pop хийж гаргана.

Дараа нь түүний очиж үзээгүй хүүхдүүд болох node B болон node C-г stack-д нэмнэ. Бид тэдгээрийг урвуу дарааллаар (reverse order), өөрөөр хэлбэл эхлээд C дараа нь B-г нэмнэ, ингэснээр B нь дээр (top) байрлаж (LIFO) дараагийнх нь болж боловсруулагдана. Мөн бид тэднийг “visited” гэж тэмдэглэнэ.

* Stack: [C, B]

* Visited: {A, B, C}

Алхам 3:

Бид node B-г stack-оос pop хийж гаргана.

Дараа нь түүний очиж үзээгүй хүүхдүүд болох node D болон node E-г stack-д урвуу дарааллаар (E дараа нь D) нэмнэ. Мөн бид тэднийг “visited” гэж тэмдэглэнэ.

* Stack: [C, E, D]

* Visited: {A, B, C, D, E}

Алхам 4:

Бид node D-г stack-оос pop хийж гаргана. Энэ node-д stack-д нэмэх хүүхэд байхгүй.

* Stack: [C, E]

* Visited: {A, B, C, D, E}

Алхам 5:

Бид node E-г stack-оос pop хийж гаргана. Энэ node-д stack-д нэмэх хүүхэд байхгүй.

* Stack: [C]

* Visited: {A, B, C, D, E}

Алхам 6:

Бид node C-г pop хийж гаргана.

Дараа нь түүний хүүхдүүд болох node F болон node G-г stack-д урвуу дарааллаар (node G дараа нь node F) нэмээд “visited” гэж тэмдэглэнэ.

* Stack: [G, F]

* Visited: {A, B, C, D, E, F, G}

Алхам 7:

Бид node F-г stack-оос pop хийж гаргана. Энэ node-д stack-д нэмэх хүүхэд байхгүй.

* Stack: [G]

* Visited: {A, B, C, D, E, F, G}

Алхам 8:

Бид node G-г stack-оос pop хийж гаргана. Энэ node-д stack-д нэмэх хүүхэд байхгүй.

* Stack: []

* Visited: {A, B, C, D, E, F, G}

Stack хоосон болсон үед traversal дуусч, бүх node-ууд очиж үзэгдсэн байна.

Algorithm нь node-уудыг дараах дарааллаар очиж үзсэн:

A → B → D → E → C → F → G

Node A-аас эхэлж, дараа нь бид tree-ийн доош чиглэлд node B, мөн node D болон E хүртэл бүрэн бууж, дараа нь буцаж дээш гарч node C рүү, дараа нь node F болон G рүү шилжиж байгааг анзаараарай. Энэ нь depth-first search (DFS)-ийн үндсэн зарчим бөгөөд бүрэн замуудыг (full paths) эхэлж дагаж, дараа нь буцаж (backtracking) бусад замуудыг хайдаг.

Энэ тохиолдолд бид энэ жишээг stack ашиглан шийдсэн. Үүнээс гадна, depth-first search (DFS)-ийг recursion ашиглан хэрэгжүүлж болно, энд функц нь одоогийн node-ийг боловсруулж, дараа нь түүний очиж үзээгүй хөрш бүр дээр өөрийгөө дахин дуудах (call) байдлаар ажилладаг. Функцийн call stack нь LIFO (last-in, first-out) дарааллыг далд байдлаар удирддаг.

Breadth-first search (BFS) болон depth-first search (DFS) аль аль нь graph болон tree-уудыг traversal хийхэд маш чухал algorithm-ууд юм. Breadth-first search (BFS) нь node-уудыг түвшин бүрээр нь судалдаг бөгөөд энэ нь жингүй (unweighted) graph-д хамгийн богино зам (shortest path)-ыг олоход маш тохиромжтой. Харин depth-first search (DFS) нь нэг мөчир (branch)-ийг аль болох гүн рүү дагаж, дараа нь буцаж бусад замуудыг судалдаг бөгөөд энэ нь лабиринт (maze) шийдэх болон цикл (cycle)-ийг илрүүлэхэд маш тохиромжтой. Тэдгээрийн давуу болон сул талуудыг ойлгох нь тухайн асуудалд хамгийн тохиромжтойг нь сонгоход тусалдаг.
