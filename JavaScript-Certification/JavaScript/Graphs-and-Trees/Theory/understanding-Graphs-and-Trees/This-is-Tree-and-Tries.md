Trees болон Tries гэж юу вэ, мөн тэд хэрхэн ажилладаг вэ?

Tree-үүд нь computer science-ийн ертөнцөд маш чухал юм.

Tree нь graph-ийн нэг тодорхой төрөл юм.

Graph нь tree гэж ангилагдахын тулд дараах нөхцлүүдийг хангах ёстой:

* Loop эсвэл cycle (эхлэл болон төгсгөлийн node ижил байдаг зам)-гүй байх.

* Холбогдсон (connected) байх (node бүрийг бусад node бүрээс хүрч болох байх).

Tree-үүд нь non-linear data structure бөгөөд node-уудыг шаталсан бүтэц (hierarchy)-д зохион байгуулдаг, энд node-ууд нь child, sibling, болон parent node-уудтай байж болно.

Root node нь tree-ийн хамгийн дээд хэсэгт байдаг node юм. Энэ нь parent node-гүй цорын ганц node юм. Энэ нь data structure-ийг бүхэлд нь traversal хийх эхлэл node бөгөөд ихэвчлэн breadth-first search (BFS) эсвэл depth-first search (DFS) зэрэг algorithm-уудыг ашиглан эхэлдэг.

Энэ бол tree-ийн graphical жишээ: Node-ууд шаталсан бүтэцтэй тул тэдгээрийн хооронд тодорхой харилцаа (relationships) байдаг.

Parent node гэдэг нь өөрөөсөө доош байрлах node-уудтай шууд холбогдсон node юм. Диаграмд node A нь node B болон C-ийн parent node юм.

Child node гэдэг нь өөрөөсөө дээр байрлах node-той шууд холбогдсон node юм. Диаграмд node D болон E нь node C-ийн child node-ууд юм.

Node D болон E нь мөн leaf гэж ангилагдана. Leaf гэдэг нь child node-гүй node юм. Та үүнийг tree-ийн “салбар”-ын (branches) төгсгөл гэж ойлгож болно.

Tree node-ууд мөн чухал шинж чанаруудтай:

* Depth: root-оос тухайн node хүртэлх замын урт. Жишээлбэл, диаграмд node D-ийн depth нь 2, учир нь root-оос эхэлбэл түүнд хүрэхийн тулд хоёр edge дамжих шаардлагатай.

* Height: тухайн node-оос доош leaf хүртэлх замын урт. Жишээлбэл, node C-ийн height нь 1, учир нь энэ нь leaf node-уудаас нэг түвшин дээр байрладаг.

* Degree: тухайн node-ийн child node-уудын тоо. Диаграмд node B-ийн degree нь 0, учир нь энэ нь leaf node бөгөөд child node байхгүй. Харин node C-ийн degree нь 2, учир нь хоёр child node-той.

Tree нь мөн өөрийн height-тэй байдаг. Tree-ийн height гэдэг нь root node-ийн height юм.

Tree-үүдийн олон төрөл байдаг, үүнд Binary Trees, Binary Search Trees, AVL trees, Red-Black Trees, болон B-Trees орно.

Binary Trees болон Binary Search Trees

Эдгээр нь tree-үүдийн хамгийн өргөн хэрэглэгддэг төрлүүдийн хоёр нь юм.

Binary tree гэдэг нь node бүр хамгийн ихдээ хоёр child node-той байж болох tree-ийн төрөл бөгөөд эдгээр нь left child node болон right child node юм. Тийм ээ, энэ нь таны өмнө нь харсан жишээ нь binary tree гэсэн үг юм! Binary search tree нь binary tree-ийн илүү тодорхой (specific) хувилбар бөгөөд маш тодорхой эрэмбэлэлтийн (ordering) шинж чанартай байдаг.

Үүнийг ойлгохын тулд эхлээд subtree гэдгийг ойлгох хэрэгтэй. Subtree гэдэг нь өөрөө бие даасан tree болж чаддаг tree-ийн нэг хэсэг юм.

Манай tree-ийн жишээнд node C, D, болон E нь өөрсдөө тусдаа tree үүсгэж байгаа тул тэднийг subtree гэж үзнэ. Binary search tree (BST)-ийн ordering property нь дараахыг тогтоодог: аливаа node-ийн хувьд, түүний left subtree дахь бүх утгууд тухайн node-ийн утгаас бага, харин right subtree дахь бүх утгууд тухайн node-ийн утгаас их байна.

Left болон right subtree-ууд нь өөрсдөө мөн binary search tree байх ёстой.

Энэ төрлийн эрэмбэлэлт нь tree balanced байх үед search, insertion, болон deletion үйлдлүүдийг маш үр ашигтай болгодог.

Balanced tree гэдэг нь аль ч node-ийн left болон right subtree-уудын өндөр (height) нь хоорондоо маш ойролцоо байдаг tree бөгөөд ингэснээр үйлдлүүд үр ашигтай хэвээр үлддэг.

Tries

Одоо та tree болон binary search tree-ийн талаар илүү сайн ойлголттой болсон тул tries руу гүнзгий орж үзье.

Tries нь string-үүдийн олонлогийг хадгалахад ашиглагддаг tree data structure юм.

Tries-ийг мөн prefix tree гэж нэрлэдэг, учир нь тэд string-ийг prefix дээр нь тулгуурлан хайх үйлдлүүдэд маш үр ашигтай байдаг.

Trie дахь node бүр string-ийн нэг тэмдэгтийг (character) илэрхийлдэг.

Root node нь ямар нэг тодорхой тэмдэгтийг илэрхийлдэггүй тул үүнийг хоосон string (empty string)-ийг илэрхийлж байна гэж ойлгож болно.

Та root-оос доош trie-ийг дагаж явах үед тухайн node хүртэлх зам нь тодорхой нэг prefix-ийг тодорхойлдог. Үг (word)-ийг олохын тулд та тэр prefix-ийг дагаж, хайж буй үгийг илэрхийлсэн node-д хүрэх хүртэл явна.

Бүрэн үг (complete word)-ийг илэрхийлж буй node-ууд дээр end-of-word marker тавигддаг.

Энэ нь "top", "tea", болон "ten" гэсэн үгсийг агуулсан trie-ийн жишээ юм.

"tea" болон "ten" үгс нь "te" гэсэн ижил prefix-ийг хуваалцаж байгаа тул data structure нь сүүлийн тэмдэгт хүртэл ижил замаар явдаг бөгөөд сүүлийн тэмдэгт дээр end-of-word marker тавигддаг. Энэ диаграмд үүнийг node-ийн эргэн тойронд улаан хүрээ зурж харуулсан: Search үйлдлийн worst-case time complexity нь O(L) бөгөөд энд L нь таны хайж буй string-ийн урт юм.

Insertion мөн үр ашигтай. Энэ үйлдэл нь trie-д одоогоор байхгүй тэмдэгтүүдэд зориулж шинэ node-ууд үүсгэхийг л шаарддаг.

Энэ data structure-ийн хамгийн том давуу тал нь олон string ижил prefix-тэй байвал тэдний зам давхцдаг тул prefix өөрөө зөвхөн нэг удаа хадгалагддаг.

Энэ үр ашиг нь tries-ийг autocomplete болон spell checker зэрэг функцуудыг хэрэгжүүлэхэд төгс болгодог.

Гэсэн хэдий ч tries нь бүх төрлийн string-ийн олонлогт үр ашигтай биш. Хэрэв string-ийн олонлог нь маш олон давтагдашгүй (unique) тэмдэгттэй байвал үр ашиггүй байж болно. Учир нь энэ тохиолдолд маш олон ялгаатай тэмдэгтийг тус тусад нь node болгон хадгалах шаардлагатай болно. Эдгээр node-уудаар дамжин хайлт хийх нь оновчтой биш байх магадлалтай.

Одоо та tree-ийн янз бүрийн төрлүүд болон тэдгээрийг юунд ашигладгийг ойлгосон тул бодит (real-world) нөхцөлд ашиглаж эхэлж болно. Аль data structure-ийг зөв сонгохыг мэдэх нь өдөр тутмын ажилдаа тулгардаг асуудлуудыг шийдвэрлэхэд маш үнэ цэнэтэй чадвар юм.
