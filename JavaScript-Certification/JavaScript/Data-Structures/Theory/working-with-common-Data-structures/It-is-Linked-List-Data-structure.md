Singly Linked Lists хэрхэн ажилладаг вэ, мөн тэдгээр нь Doubly Linked Lists-ээс юугаараа ялгаатай вэ?

Linked list нь шугаман (linear) data structure бөгөөд node бүр дараагийн node-той холбогдсон байдаг.

Эдгээр холболтууд нь node-уудын гинжин (chain) хэлбэртэй data structure үүсгэдэг бөгөөд node бүр өгөгдөл (data) болон linked list доторх дараагийн node-ийг заах reference хадгалдаг. Бид эдгээр reference-уудыг ашиглан эхний node-оос дараагийн node руу, цааш нь дараалан шилждэг.

Linked list-ууд нь stacks, queues, болон deques зэрэг бусад data structure-уудыг хэрэгжүүлэхэд түгээмэл ашиглагддаг. Мөн эдгээрийг depth-first search болон breadth-first search зэрэг чухал graph алгоритмуудыг хэрэгжүүлэхэд ашиглаж болно.

1. Singly Linked Lists

Singly linked list нь linked list-ийн нэг төрөл бөгөөд node бүр дарааллын дараагийн node-той холбогдсон байдаг.

Node бүр дараагийн node-ийг заасан reference хадгалснаар хоорондоо холбогддог.

Node бүрт байх энэ ганц reference нь linked list-ийг нэг чиглэлээр, эхнээс нь төгсгөл хүртэл traversal хийх боломжийг олгодог.

Search нь зөвхөн урагш (forward) чиглэлд хөдөлж чадна, арагш (backwards) биш.

Энэ жишээнд та head node болох node A-аас эхэлнэ.

Head node нь linked list-ийн эхний node юм.

Singly linked list-д head node нь ихэвчлэн шууд хандах боломжтой цорын ганц node байдаг. Энэ нь тодорхой node-ийг хайж олох үед search процесс эхлэх цэг юм. Процесс node A-аас эхэлж, дараа нь node B, дараа нь node C, эцэст нь tail node болох node D руу шилжинэ. Хэрэв та кодондоо тодорхой логик хэрэгжүүлбэл энэ процесс үүнээс өмнө зогсож ч болно.

Tail node нь хамгийн сүүлийн node юм. Энэ нь процесс linked list-ийн төгсгөлд хүрсэн эсэхийг тодорхойлоход ашиглагддаг.

2. Node оруулах (Inserting Nodes)

Linked list-ийн хамгийн том давуу талуудын нэг нь тэдгээр нь тогтмол хэмжээтэй (fixed size) биш байдаг. Node-уудын хоорондох холболтуудыг шинэчлэх замаар шаардлагатай үед томруулж (expand) эсвэл багасгаж (shrink) болно.

Та linked list-ийн эхэнд (start), дунд (middle), болон төгсгөлд (end) node оруулж болно.

Linked list нь заавал node-уудыг тодорхой дарааллаар хадгалах шаардлагагүй. Дараалал нь node-уудын хоорондох холболтуудаар тодорхойлогдоно.

Гэхдээ хэрэв таны тодорхой хэрэглээ (use case)-д node-уудыг тодорхой дарааллаар хадгалах шаардлагатай бол та кодондоо тэр логикийг хэрэгжүүлж болно, мөн таны тодорхойлсон шалгуур (criteria) нь node эхэнд, дунд, эсвэл төгсгөлд орох эсэхийг шийднэ.

Linked list-ийн эхэнд node оруулахын тулд та шинэ node болон өмнө нь head node байсан node-ийн хооронд холболт үүсгэж, шинэ node-ийг head node болгоход л хангалттай.

Энэ жишээнд бид node E-г эхэнд оруулж, энэ шинэ node-ийг linked list-ийн head node болгож байна. Linked list-ийн эхэнд node оруулах нь тогтмол хугацааны төвөгшилтэй O(1), учир нь зөвхөн head node-ийн reference болон шинэ head node-оос дараагийн node руу холболтыг шинэчлэхэд л хангалттай.

Энэ жишээнд бид node E-г linked list-ийн эхэнд оруулж байна. Энэ нь зөв ажиллана. Гэхдээ хэрэв бид linked list-ийг цагаан толгойн дарааллаар (alphabetical order) эрэмбэлсэн байлгахыг хүсвэл node E-г linked list-ийн төгсгөлд оруулах шаардлагатай болно.

Linked list-ийн төгсгөлд node оруулахын тулд эхлээд төгсгөл хүртэл очоод, дараа нь шинэ node-той холболт үүсгэж, түүнийг шинэ tail node болгоно.

Энэ үйлдэл нь шугаман хугацааны төвөгшилтэй O(n) бөгөөд энд n нь linked list-д хадгалагдаж буй node-уудын тоо юм, учир нь оруулга хийхийн тулд эхлээд linked list-ийн төгсгөл хүртэл хүрэх хэрэгтэй бөгөөд энэ нь нэг node-оос дараагийн node руу дамжин явсаар төгсгөлд хүрэхийг шаарддаг. Хэрэв node-ийг linked list-ийн дунд хэсэгт оруулах шаардлагатай бол node-уудын хоорондох холболтуудыг мөн шинэчлэх хэрэгтэй болно. Дарааллын өмнөх node нь шинэ node-той холбогдох ёстой бөгөөд шинэ node нь дараагийн node-той холбогдох ёстой, дараах зурагт үзүүлсэн шиг. Оруулах (insertion) үйлдлийн санах ойн төвөгшил (space complexity) нь тогтмол O(1) байдаг, учир нь шинэ node оруулах нь зөвхөн тухайн node-ийг үүсгэж, node-уудын хоорондох холболтуудыг шинэчлэхийг л шаарддаг. Энэ үйлдэл нь linked list-ийн хэмжээнээс хамаарахгүй.

3. Node устгах (Removing Nodes)

Node оруулах боломжтой шиг, та мөн linked list-ийн эхэнд (start), дунд (middle), болон төгсгөлд (end) node-уудыг устгаж болно.

Эхнээс node устгахын тулд та head node-ийн reference-ийг шинэчлэх хэрэгтэй бөгөөд энэ нь дарааллын дараагийн node байх ёстой.

Энэ үйлдэл нь тогтмол хугацааны төвөгшилтэй O(1), учир нь зөвхөн linked list-ийн head node-ийн reference-ийг шинэчлэхэд л хангалттай. Linked list-ийн дунд хэсгээс node устгахын тулд та өмнөх node-ийн reference-ийг шинэчилж, дараагийн node-той нь холбож, тэдгээрийн хооронд "гүүр" (bridge) шиг холболт үүсгэх хэрэгтэй, үүнийг та энэ зурагт харж болно. Ингэснээр устгахыг хүссэн node, энэ тохиолдолд node B, холболтын дарааллаас хасагдаж, дараагийн удаа traversal хийх үед хүрэх боломжгүй болно.

Linked list-ийн төгсгөлөөс node устгахын тулд та өмнөх node-ийн холболтыг устгаж, тухайн node-ийг шинэ tail node болгох хэрэгтэй. Ингэснээр linked list шинэ tail node дээр төгсөнө.

Энэ үйлдэл нь шугаман хугацааны төвөгшилтэй O(n), учир нь эхлээд linked list-ийн төгсгөл хүртэл хүрэх шаардлагатай. Устгах (deletion) үйлдлийн санах ойн төвөгшил (space complexity) нь тогтмол O(1), учир нь node устгахад нэмэлт санах ой шаардлагатай биш.
4. Doubly Linked Lists

Одоо та singly linked list-ийн талаар илүү сайн ойлголттой болсон тул doubly linked list-ийн талаар ярилцъя.

Doubly linked list-д node бүр хоёр reference хадгалдаг: дараагийн node руу заах reference болон дарааллын өмнөх node руу заах reference.

Энэ нь doubly linked list-ийг хоёр чиглэлд traversal хийх боломжийг олгодог гэсэн үг. Мөн энэ төрлийн linked list-д шаардлагатай үед төгсгөлөөс (end) traversal эхлүүлэхийн тулд linked list дотор tail node-ийн reference-ийг хадгалах нь түгээмэл байдаг.

Сайн сонсогдож байна, тийм үү? Эдгээр нь singly linked list-ээс илүү уян хатан (flexible) юм.

Гэхдээ doubly linked list нь singly linked list-ээс илүү их санах ой шаарддаг, учир нь node бүр нэг reference биш, хоёр reference хадгалдаг.

Тиймээс та өөрийн төсөлд (project) тохирох data structure-ийг сонгохдоо үүнийг анхаарах хэрэгтэй.

Энд tradeoff бий.

Insertion болон deletion үйлдлүүд яг адилхан ажилладаг. Гэхдээ ялгаа нь одоо node бүрт хоёр reference шинэчлэх шаардлагатай бөгөөд мөн doubly linked list-ийн төгсгөлд элементүүдийг маш үр ашигтай оруулах, шаардлагатай үед ар талаас (back) traversal эхлүүлэхийн тулд tail node-ийн reference-ийг хянах (track) хэрэгтэй болно.

Singly болон doubly linked list-ууд нь computer science-д дараалсан (sequential) байдлаар элементүүдийг хадгалах болон удирдахад ашиглагддаг чухал data structure-ууд юм. Тэдгээрийн ялгааг ойлгох нь тодорхой хэрэглээнд (specific application) зөвийг нь сонгоход маш чухал.
