Stacks болон Queues хэрхэн ажилладаг вэ?

Stacks болон queues нь computer science-д түгээмэл ашиглагддаг data structure-ууд юм.

Эдгээр нь элемент нэмэх болон устгах тодорхой дүрэм (specific rules)-ийг дагадаг шугаман (linear) data structure-ууд юм.
Stacks

Эхлээд Stacks-ийг авч үзье.

Stack нь Last-in, First-out (LIFO) data structure юм.

Энэ нь stack-д хамгийн сүүлд нэмэгдсэн элемент хамгийн түрүүнд устгагдана гэсэн үг.

Stack нь хоёр төгсгөлтэй бөгөөд тэдгээрийг бид top болон bottom гэж нэрлэдэг.

Элементүүд нь stack-ийн top хэсгээс нэмэгдэж, мөн top хэсгээс устгагддаг.

Та stack-ийг аяганы овоо (pile of dishes) гэж төсөөлж болно, та аягыг зөвхөн овооны дээд талд тавьж, мөн зөвхөн дээд талаас нь авч болно. Элемент нэмэх болон устгах эдгээр үйлдлүүд энэ context-д тусгай нэртэй байдаг.

Stack-д элемент нэмэхийг "push" operation гэж нэрлэдэг. Бид элемент-ийг stack-ийн top хэсэгт нэмэх үед “push хийлээ” гэж хэлдэг. Stack-ээс элемент устгахыг "pop" operation гэж нэрлэдэг. Бид stack-ийн top хэсгээс элемент устгах үед “pop хийлээ” гэж хэлдэг. Та stack-ийн bottom хэсэгт бид үнэндээ ямар нэгэн үйлдэл хийдэггүй, харин үүнийг reference болгон хадгалдаг гэдгийг харж болно.

Push болон pop үйлдлүүдийн хугацааны төвөгшил (time complexity) нь ихэвчлэн O(1), өөрөөр хэлбэл тогтмол хугацааны төвөгшилтэй байдаг.

Та элемент-ийг stack дээр push хийхэд, элемент зүгээр л top хэсэгт нэмэгдэнэ.

Та элемент-ийг stack-ээс pop хийхэд, top дээрх элемент устгагдана.

Иймээс эдгээр үйлдлийг гүйцэтгэх хугацаа нь stack-ийн хэмжээ ямар байхаас үл хамааран тогтмол байдаг.

Push болон pop үйлдлүүдийн санах ойн төвөгшил (space complexity) нь ихэвчлэн O(1) байдаг. Энэ нь эдгээр үйлдлийг гүйцэтгэхэд шаардлагатай санах ойн хэмжээ нь stack-ийн хэмжээнээс үл хамааран тогтмол байна гэсэн үг.

Queues

Одоо та stacks-ийн талаар илүү их мэдлэгтэй болсон тул Queues-ийг сурцгаая.

Queue нь First-in First-out (FIFO) шугаман (linear) data structure юм. Энэ нь queue-д хамгийн түрүүнд нэмэгдсэн элемент хамгийн түрүүнд устгагдана гэсэн үг.

Queues нь хоёр төгсгөлтэй: front болон back. Элементүүд queue-ийн back хэсэгт нэмэгдэж, front хэсгээс устгагддаг.

Та queue-ийг супермаркет дээр бараа төлөхөөр (pay for groceries) дараалалд (line) зогсож буй хүмүүсийн эгнээ гэж төсөөлж болно. Эгнээнд хамгийн түрүүнд зогссон хүн хамгийн түрүүнд касс (cash register) руу ордог бол шинэ хүмүүс эгнээнгийн төгсгөлд орж ирдэг.

Элемент нэмэх болон устгах үйлдлүүд queue-ийн context-д тусгай нэртэй байдаг.

Queue-ийн back хэсэгт элемент нэмэхийг "enqueue" operation гэж нэрлэдэг.

Enqueue үйлдлийн үед шинэ элемент queue-ийн төгсгөлд нэмэгдэж, эгнээнгийн сүүлд орно. Queue-ийн front хэсгээс элемент устгахыг "dequeue" operation гэж нэрлэдэг.

Dequeue үйлдлийн үед queue-ийн front дээрх элемент устгагдаж, дараагийн элемент шинэ front болдог. Enqueue болон dequeue үйлдлүүдийн хугацааны төвөгшил (time complexity) нь O(1), өөрөөр хэлбэл тогтмол хугацаатай. Эдгээр үйлдлийг гүйцэтгэх хугацаа нь queue-ийн хэмжээнээс үл хамааран тогтмол байна.

Enqueue болон dequeue үйлдлүүдийн санах ойн төвөгшил (space complexity) нь ихэвчлэн O(1) байдаг. Энэ нь эдгээр үйлдлийг гүйцэтгэхэд шаардлагатай санах ойн хэмжээ нь queue-ийн хэмжээнээс үл хамааран тогтмол байна гэсэн үг.

Stacks болон queues нь элементүүдийг зохион байгуулах (organizing) болон удирдах (managing)-ад ашиглагддаг data structure-ууд юм. Эдгээрийг ойлгох нь төрөл бүрийн програмчлалын хэрэглээнд үр ашигтай алгоритмууд бүтээхэд маш чухал.
