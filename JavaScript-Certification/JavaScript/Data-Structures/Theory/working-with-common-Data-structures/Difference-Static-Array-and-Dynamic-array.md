Dynamic arrays нь static arrays-аас хэрхэн ялгаатай вэ?

Arrays нь computer science дахь суурь (fundamental) data structure юм. Бүх array-ууд өгөгдлийн дараалалтай (ordered) цуглуулгыг хадгалдаг боловч төрөл (type)-өөсөө хамааран дотооддоо (behind the scenes) өөр өөрөөр ажиллаж болно.

Тэдгээрийн дотоод үйл ажиллагаа (underlying behavior) нь програмын үр ашигт (efficiency) чухал нөлөө үзүүлдэг тул dynamic болон static arrays болон тэдгээрийн ялгааг судалж, програмдаа хамгийн үр ашигтайг нь сонгох хэрэгтэй.

Эхлээд static arrays-аас эхэлье.

Static arrays нь тогтмол хэмжээтэй (fixed size) байдаг. Тэд элементүүдийг санах ойн залгаа (adjacent) байршлуудад хадгалдаг.

Static array-ийн хэмжээ нь array-ийг анх үүсгэх (initialize) үед тодорхойлогддог. Нэгэнт тэр санах ойн хэсгийг (memory block) хуваарилсан бол энэ нь тогтмол болж, програм ажиллаж байх үед өөрчлөгдөх боломжгүй. Энэ нь static array-ийн гол онцлог юм.

Элементүүдийг санах ойн залгаа байршлуудад хадгалах нь өгөгдөлд хандах (data retrieval) үйлдлийг илүү үр ашигтай болгодог, учир нь програм нь эхний элементын байршлыг хадгалаад, index-үүдийг ашиглан энгийн тооцоолол хийж бусад элементүүдийн байршлыг олж чаддаг.

Үүний ачаар static array-ийн утгуудад хандах үйлдэл тогтмол хугацаанд O(1) явагддаг бөгөөд энэ нь маш үр ашигтай.

Та static array-ийг хадгалах элементүүдийн тоог урьдчилан мэдэж байгаа үед ашиглаж болно. Мөн утгуудад маш ойр ойрхон хандах шаардлагатай үед ашиглахад тохиромжтой, учир нь access үйлдэл маш хурдан.

Гэхдээ энэ data structure нь томрох (grow) эсвэл багасах (shrink) боломжгүй тул хадгалах элементүүдийн тоо өөрчлөгдөж болох тохиолдолд dynamic array ашиглах хэрэгтэй.

Static array-ийн хэмжээг нэмэгдүүлэхийг оролдвол шинэ array үүсгэж, хуучин array-ийн бүх элементүүдийг шинэ array руу хуулж хийх шаардлагатай бөгөөд энэ нь үр ашиггүй. Ийм нөхцөлд dynamic array илүү тохиромжтой, учир нь энэ процессыг автоматаар зохицуулдаг.

JavaScript-д C эсвэл Java шиг уламжлалт доод түвшний (low-level) static array байдаггүй боловч та built-in Array object ашиглан тогтмол хэмжээтэй array-ийг дуурайлган (simulate) үүсгэж болно:

```js id="c3v9xr"
let numbers = new Array(3); // хэмжээ нь 3
numbers[0] = 10;
numbers[1] = 20;
numbers[2] = 30;
```

JavaScript нь array-ийн хэмжээг өөрчлөхийг зөвшөөрдөг ч ойлголтын хувьд энэ нь тогтмол урттай array-ийг илэрхийлж байна.

JavaScript дахь array-ууд нь анхнаасаа dynamic байдаг тул одоо тэдгээрийг харцгаая.

Dynamic arrays нь илүү уян хатан (flexible), учир нь програм ажиллаж байх үед автоматаар томрох эсвэл багасах боломжтой.

Тэдгээр нь автомат хэмжээ өөрчлөх (automatic resizing) механизмаар ажилладаг бөгөөд анхны array дүүрэх үед элементүүдийг шинэ array руу хуулдаг. Энэ процесс нь үр ашигтай хийгддэг, учир нь шинэ array-ийн хэмжээг ухаалгаар (efficient way) сонгодог тул тооцооллын хувьд үнэтэй (computationally expensive) үйлдлүүдийг бага давтамжтай болгодог.

Dynamic array-ийн элементүүдэд хандах нь тогтмол хугацаанд O(1) явагддаг тул энэ үйлдэл маш үр ашигтай.

Array-ийн дунд элемент оруулах нь шугаман хугацаа O(n) шаарддаг, учир нь дараагийн элементүүдийг шилжүүлэх (relocate) хэрэгтэй болдог.

Array-ийн төгсгөлд элемент нэмэх нь хэрэв хангалттай зай байгаа бол тогтмол хугацаа O(1) авна, харин array дүүрсэн бөгөөд хэмжээг нь өөрчлөх шаардлагатай бол энэ үйлдэл O(n) төвөгшилтэй болдог.

Та хадгалах элементүүдийн тоог урьдчилан мэдэхгүй үед dynamic array ашиглах хэрэгтэй. Мөн элементүүдийг ойр ойрхон нэмэх (insert) болон устгах (delete) шаардлагатай үед ашиглахад тохиромжтой.

JavaScript-д array үүсгэхийн тулд та элементүүдийг дөрвөлжин хаалт (square brackets) дотор, таслалаар тусгаарлан бичнэ:

```js id="q7m2pn"
let numbers = [3, 4, 5, 6];
```

Энэ нь дөрвөн элементтэй dynamic array үүсгэж байна.

Та array доторх элементэд хандахын тулд хувьсагчийн нэрийг бичээд, дараа нь index-ийг агуулсан дөрвөлжин хаалт (square brackets)-ыг ашиглана.

Index-үүд нь эхний элементэд 0-оос эхэлж, дараагийн элемент бүрт 1-ээр нэмэгдэнэ.

```js id="x2m9qn"
numbers[0]; // 3
numbers[1]; // 4
numbers[2]; // 5
numbers[3]; // 6
```

Утгыг шинэчлэхийн тулд та зүгээр л дахин оноох (reassign) хэрэгтэй:

```js id="k8v1rp"
numbers[2] = 16
```

Одоо array нь [3, 4, 16, 6] болно.

Array-ийн төгсгөлд элемент нэмэхийн тулд .push() method-ийг ашиглана:

```js id="n4t7xz"
numbers.push(7)
```

Одоо array нь [3, 4, 16, 6, 7] болно:

Тодорхой index дээр элемент оруулахын тулд JavaScript-д Python шиг .insert() method байхгүй, харин та .splice() method-ийг ашиглаж болно. Синтакс нь дараах байдалтай:

```js id="p9c3vt"
array.splice(index, deleteCount, newElement)
```

Жишээ:

```js id="z6m2kx"
numbers.splice(3, 0, 15);
```

Энэ жишээ нь:

```
Index 3-аас эхэлнэ  
0 элемент устгана  
15-ыг оруулна  
```

Хэрэв array [3, 4, 16, 6, 7] байсан бол одоо [3, 4, 16, 15, 6, 7] болно.

Та мөн тодорхой index дээрх элементийг .splice() method ашиглан устгаж болно:

```js id="t3v8pm"
numbers.splice(2, 1);
```

Энэ нь index 2-оос эхлээд 1 элемент устгана гэсэн үг.

Сүүлийн элементийг устгахын тулд .pop() method-ийг ашиглана:

```js id="q1n5wr"
numbers.pop();
```

JavaScript дахь array-уудтай ажиллахад ашиглаж болох маш олон built-in method-ууд байдаг бөгөөд та тэдгээрийг documentation-оос харж болно.

Энэ бол dynamic array-ийн давуу тал юм.

Ерөнхийдөө, та элементүүдийн тоог урьдчилан мэдэж байгаа бөгөөд тэдгээрт ойр ойрхон хандах шаардлагатай үед static array ашиглах, харин элементүүдийн тоо тодорхойгүй эсвэл хугацааны явцад өөрчлөгдөж байвал dynamic array ашиглах хэрэгтэй.

Та static array-ийн энгийн байдал (simplicity) болон dynamic array-ийн уян хатан байдал (flexibility)-ын хоорондын tradeoff-ийг үргэлж бодолцох хэрэгтэй. Эдгээр нь тус тусдаа тодорхой хэрэглээ (use cases) болон нөхцөлд хэрэгтэй байдаг. Тухайн асуудалд аль нь илүү тохиромжтойг сонгох чадвар нь таны дасгалаар аажмаар хөгжүүлэх асуудал шийдэх чадварын нэг хэсэг юм.
