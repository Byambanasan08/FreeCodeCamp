Сайн асуудал шийдэх (problem-solving) техникүүд болон алгоритмын сорилтуудад (algorithmic challenges) хэрхэн хандах вэ?

Суралцах явцдаа та асуудал шийдэх хүчтэй чадварыг (strong problem-solving skills) хөгжүүлэх дээр ажиллах хэрэгтэй. Эдгээр үндсэн чадварууд нь таны өдөр тутмын ажилд бодит амьдралын (real-world) асуудлуудыг шийдэхэд маш чухал байх болно.

Алгоритмын сорилтуудыг (algorithmic challenges) шийдэх нь дадлага хийх маш сайн арга юм. Энэ нь аналитик сэтгэлгээ (analytical way of thinking), асуудлыг үндсэн бүрэлдэхүүн хэсгүүдэд (core components) задлах чадвар, мөн зөв гаралт (output) үүсгэхийн тулд үр ашигтай (efficiently) шийдэл олох чадварыг шаарддаг.

Гэхдээ хаанаас эхлэх вэ?

Та эдгээр сорилтуудыг шийдэж эхлэхэд ашиглаж болох хэд хэдэн асуудал шийдэх техникүүд байдаг.

Жишээ болгон бид JavaScript ашиглан string-ийг урвуу (reverse) болгоё.

Энэ бол сорилт:

"Given a string, write an algorithm that returns a new string with the characters in reverse order."

Ийм төрлийн асуудалтай тулгарах үед хийх ёстой хамгийн эхний зүйл бол тайлбарыг (description) олон удаа уншиж, яг юу хэлж байгааг нь ойлгосон эсэхээ баталгаажуулах явдал юм. Хэрэв та энэ алхмыг алгасвал эсвэл хэт хурдан уншвал чухал мэдээллийг алдаж болно.

Дараа нь, асуудалтай танилцсаны дараа үүнийг үндсэн бүрэлдэхүүн хэсгүүдэд нь задлах хэрэгтэй.

Өөрөөсөө асуугаарай:

```
"Оролт (input) нь юу вэ?"

"Хүлээгдэж буй гаралт (expected output) нь юу вэ?"

"Би оролтыг хүлээгдэж буй гаралт руу хэрхэн хувиргах вэ?"
```

Энэ асуудалд, сорилт "Given a string…" гэж эхэлж байгаа тул оролт нь string гэдгийг тодорхойлж болно.

Гаралт нь "тэмдэгтүүд нь урвуу дарааллаар байрласан шинэ string" юм.

Тиймээс та эхний string-ийг авч урвуулах хэрэгтэй.

Эхний энэ анализ нь эхэндээ бага зэрэг давтагдсан (repetitive) мэт санагдаж болох ч шаардлагыг бүрэн ойлгож байгаа эсэхээ баталгаажуулахад маш их тустай.

Дараа нь та хөгжүүлэх алгоритм оролтыг гаралт болгон хэрхэн хувиргах талаар бодож эхлэх хэрэгтэй.

Энэ төлөвлөлт болон анализын үед pseudocode ашиглан шаардлагатай алхмуудыг зураглах (map out) нь түгээмэл байдаг.

Pseudocode нь алгоритмын логикийг өндөр түвшинд (high-level) тайлбарласан бөгөөд ерөнхий шинж чанартай, тодорхой нэг програмчлалын хэл дээр суурилаагүй байдаг.

Pseudocode нь жинхэнэ код шиг албан ёсны (formal) биш, учир нь энэ нь зөвхөн хүн уншихад зориулагдсан. Энэ нь нэг хараад ойлгогдохоор энгийн байх ёстой. Үүний гол зорилго нь гүйцэтгэгдэх алхмуудын дарааллыг тодорхой харуулах явдал юм.

Pseudocode нь ихэвчлэн Англи хэл шиг нийтлэг бичгийн хэл болон IF, ELSE, FOR, WHILE зэрэг програмчлалын бүтэц (constructs)-ийн холимог байдаг.

Энэ бол "Reverse a String" сорилтыг шийдэхэд ашиглаж болох pseudocode-ийн жишээ юм.

```
GET original_string

SET reversed_string = ""

FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string

DISPLAY reversed_string
```

Алхамуудыг ойлгоход хялбар байдлаар хэрхэн бичсэнийг анзаараарай. Үг болон бүтэц (constructs)-үүд нь таны дагаж буй стандартуудаас хамаарч өөр байж болно.

Хэрэв та хүсвэл энэ алхмуудыг ижил логикоор JavaScript эсвэл бусад програмчлалын хэл дээр хэрэгжүүлж болно, учир нь pseudocode нь програмчлалын хэлээс хамааралгүй байдаг.

Энэ үед та энэ асуудлыг олон янзын аргаар шийдэж болдгийг аль хэдийн ойлгосон байх магадлалтай. Энэ нь string-ийг урвуулах цорын ганц арга биш.

Гэхдээ зөв алгоритмыг сонгох нь чухал гэдгийг санаарай.

Өмнөх хичээлд та алгоритмын төвөгшил (algorithmic complexity)-ийн талаар болон яагаад хугацаа (time) болон санах ой (space)-н хувьд үр ашигтай алгоритм сонгох нь чухал болохыг сурсан.

Энд хөгжүүлэгч (developer)-ийн хувьд таны үүрэг маш чухал болно. Та сорилтыг шийдэх хамгийн үр ашигтай алгоритмыг сонгох хэрэгтэй.

Боломжит өөр өөр алгоритмуудыг бодож үзэх нь чухал асуудал шийдэх чадвар бөгөөд та үүнийг байнга дасгалжуулах хэрэгтэй. Та өөрөөсөө таны pseudocode-д санал болгож буй шийдэл хамгийн сайн нь мөн үү гэж асуугаарай.

Жишээлбэл, элементүүдийг эрэмбэлэх (sorting) олон алгоритм байдаг ч тэдгээрийн зарим нь бусдаасаа илүү үр ашигтай байдаг. Жишээлбэл, Bubble sort нь том жагсаалтуудыг эрэмбэлэхэд маш үр ашиггүй, харин Quick Sort нь ихэвчлэн илүү үр ашигтай байдаг.

Манай "Reverse a String" сорилтын хувьд, хэрэв бид алгоритмаа JavaScript дээр хэрэгжүүлэхээр төлөвлөж байгаа бол дараах аргуудын аль нэгийг ашиглаж болно:

```
.split(""), .reverse(), болон .join("") зэрэг built-in method-уудыг ашиглан шинэ урвуу string буцаах.

Зүүнээс баруун тийш тэмдэгтүүдээр давтаж, шинэ string-ийн эхэнд шинэ тэмдэгтийг нэмэх.

String-ийг array болгон хувиргаж, дараа нь loop ашиглан гараар урвуулж, буцаан string болгох.
```

Алийг нь ашиглах вэ? Энэ бол таны сонголт.

Эдгээр шийдвэрүүдийг өөрийн мэдлэг болон туршлага дээр үндэслэн гаргах нь таны application-ийн эцсийн гүйцэтгэлд (performance) маш их нөлөө үзүүлж чадна. Өөр өөр аргууд, тэдгээрийн үр ашиг, нөлөөлөл (implications), болон хэрэгжилтийг (implementation) бодолцож үзээрэй.

Өөрөөсөө асуугаарай:

```
"Би энэ асуудалд хэрхэн хандах вэ?"

"Би ямар data structure-ууд ашиглах вэ?"

"Миний сонгосон data structure-ууд энэ асуудалд хамгийн үр ашигтай нь мөн үү?"

"Би бүх боломжит edge case-уудыг хамарч байна уу?"
```

Edge case-ууд гэдэг нь алгоритм зохицуулах ёстой хүрээний (boundaries) онцгой, хүчинтэй (valid) оролтууд эсвэл нөхцлүүд юм.

Жишээлбэл, "Reverse a String" сорилтод хоосон string (empty string)-ийг оролт болгон авах нь edge case болно. Та үүнийг зөв боловсруулж байна уу? Хэрэв үгүй бол энэ edge case-ийг хэрхэн зөв зохицуулах талаар бодож, pseudocode-дээ нэмээрэй.

Дараа нь, төлөвлөгөөндөө сэтгэл хангалуун болсон үед та implementation үе шат руу шилжиж болно. Энэ үе шатанд та алгоритмаа програмчлалын хэл дээр хэрэгжүүлнэ.

Програмынхаа бүтцийг (structure) зохион байгуулахдаа та уншихад хялбар, ойлгомжтой модульчлагдсан (modular) код бичих хэрэгтэй.

Өөрийн одоогийн мэдлэг дээр үндэслэн програмчлалын хэлний хэрэгслүүдийг ашиглаарай. Зарим програмчлалын хэлүүд нийтлэг асуудлуудад зориулсан built-in шийдлүүдтэй байдаг. Боломжтой бол тэдгээрийг ашиглаарай.

Тогтвортой (consistent) байхын тулд JavaScript эсвэл өөрийн сонгосон програмчлалын хэлний best practices-ийг дагаарай.

Кодоо бичих явцдаа тест хийж, edge case-уудыг зөв боловсруулж байгаа эсэхийг шалгаарай.

Шийдлээ хэрэгжүүлсний дараа бүх жишээнүүд дээр зөв ажиллаж байгаа эсэхийг шалгаж, шаардлагатай бол кодоо илүү ойлгомжтой эсвэл энгийн болгохын тулд дахин сайжруулж (refactor) болно.

Өөрийн шийдэл рүү буцаж очих нь маш чухал. Хөгжүүлэлт нь заавал шулуун шугаман (linear), алхам алхмаар процесс биш. Та үргэлж код руугаа буцаж очиж, шүүмжлэлтэй сэтгэлгээг (critical thinking) ашиглан сайжруулж болно.

Эдгээр нь алгоритмын сорилтуудад хандахад ашиглаж болох нийтлэг асуудал шийдэх техникүүд юм. Хэрэв та тогтмол дасгал хийвэл, та аажмаар асуудал шийдэх чадвараа хөгжүүлэх болно.
