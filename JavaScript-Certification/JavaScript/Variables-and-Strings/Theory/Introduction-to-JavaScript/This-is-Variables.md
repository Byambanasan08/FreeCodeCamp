Variable гэж юу вэ, мөн JavaScript-д хувьсагчийг нэрлэх заавар юу вэ?

JavaScript-д хувьсагч (variable) нь таны програмын туршид хандаж, өөрчилж болох өгөгдлийг хадгалах сав шиг үүрэг гүйцэтгэдэг.

Та хувьсагчийг утга хадгалдаг хайрцаг гэж ойлгож болно. Хувьсагчийн тусламжтайгаар та тоо, текст зэрэг мэдээллийг хадгалж, хэрэгтэй үедээ дахин ашиглах боломжтой.

JavaScript-д хувьсагч зарлах нэг арга нь `let` түлхүүр үгийг ашиглах юм. Та `let` болон бусад зарлах аргуудын талаар дараагийн хичээлүүдэд илүү дэлгэрэнгүй судлах болно.

Жишээ нь `age` нэртэй хувьсагчийг `let` ашиглан зарлах:

```
let age;
```

Одоогоор `age` хувьсагчид утга оноогдоогүй байна. Хэрэв та үүнийг ашиглавал `undefined` буцаана, өөрөөр хэлбэл утгагүй гэсэн үг.

Жишээ:

ТАЙЛБАР: `console.log()` нь мэдээллийг console (браузерын debug хийх хэсэг)-д хэвлэдэг функц юм. Та үүнийг дараагийн хичээлүүдэд илүү дэлгэрэнгүй үзнэ. Мөн `//` тэмдэг нь comment (тайлбар) бичихэд ашиглагддаг бөгөөд код ажиллах үед тооцогдохгүй.

```
let age;
console.log(age); // undefined
```

Хувьсагчид утга оноохын тулд assignment operator (=) ашиглана:

```
let age = 25;
```

Одоо `age` хувьсагчийг ашиглахад 25 гэсэн утгыг буцаана:

```
let age = 25;
console.log(age); // 25
```

Assignment operator (=) нь тэнцүү гэсэн утгыг шалгадаг биш, харин утга оноодог. Тэнцүү эсэхийг шалгах операторуудыг та дараагийн хичээлүүдэд үзнэ.

Хувьсагчид утга оноох үйлдлийг initialization (эхлүүлэх) гэж нэрлэдэг.

`let` ашиглан хувьсагч зарлахын нэг давуу тал нь та түүнд дахин утга оноож (reassign) болдог. Програмчлалд reassignment гэдэг нь аль хэдийн утгатай хувьсагчид шинэ утга оноохыг хэлнэ.

Жишээ:

```
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

Одоо `age` хувьсагч 30 гэсэн утгатай болсон байна. Анхаарах зүйл нь `age` аль хэдийн зарлагдсан тул дахин `let` бичих шаардлагагүй.

Reassignment хийхдээ зөвхөн хувьсагчийн нэрийг ашиглахад хангалттай. Энэ нь програм ажиллах явцад утгыг шинэчилж, өөрчлөх боломжийг олгодог. Жишээ нь тоглоомын оноог шинэчлэхэд ашиглаж болно.

Хувьсагчийн нэр өгөх нь энгийн мэт боловч зарим дүрэм, сайн дадлууд байдаг.

Хувьсагчийн нэр нь хадгалж буй өгөгдлийг илэрхийлэх ёстой. Жишээ нь x гэж нэрлэхийн оронд age эсвэл points гэх мэт ойлгомжтой нэр ашиглах нь илүү тохиромжтой.

```
// Муу нэршил
let x = 10;
let y = "John";

// Сайн нэршил
let age = 10;
```

JavaScript-д хувьсагчийн нэр нь үсэг, доогуур зураас (_) эсвэл долларын тэмдэг ($)-аар эхлэх ёстой. Тоогоор эхэлж болохгүй.

```
// Зөв нэршил
let age;
let _score;
let $total;

// Буруу нэршил
let 1stPlace; // тоогоор эхэлсэн
```

Хувьсагчийн нэр нь case-sensitive байдаг. Өөрөөр хэлбэл age болон Age нь өөр өөр хувьсагчид болно.

```
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

Иймээс camelCase гэх мэт нэгэн жигд нэршлийн дүрэм баримтлах нь чухал. camelCase гэдэг нь эхний үг жижиг үсгээр, дараагийн үг бүр том үсгээр эхэлдэг хэлбэр юм.

Жишээ:

```
let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;
```

Мөн JavaScript-д ашиглагддаг зарим түлхүүр үгсийг (жишээ нь let, const, function, return) хувьсагчийн нэр болгон ашиглаж болохгүй.

Түүнчлэн (!) эсвэл (@) зэрэг тусгай тэмдэгтүүдийг хувьсагчийн нэрэнд ашиглахаас зайлсхийх хэрэгтэй. Хамгийн сайн нь үсэг, тоо, доогуур зураас, долларын тэмдэг ашиглах юм.

Эдгээр дүрмийг дагаснаар таны код илүү цэвэр, ойлгомжтой, цаашид өргөжүүлэхэд хялбар болно.
