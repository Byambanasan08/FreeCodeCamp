# JavaScript-д `typeof` оператор хэрхэн ажилладаг вэ, мөн `typeof null`-ийн алдаа гэж юу вэ?

JavaScript-д `typeof` оператор нь хувьсагч эсвэл утгын өгөгдлийн төрлийг шалгах энгийн боловч хүчирхэг хэрэгсэл юм. Энэ нь үргэлж тухайн төрлийг илэрхийлсэн string утга буцаадаг.

Жишээ:

```
let num = 42;
console.log(typeof num); // "number"
```

Энэ жишээнд `num` хувьсагчид 42 гэсэн утга оноосон байна. `typeof` операторыг ашиглахад `"number"` гэсэн string буцаана.

Өөр нэг жишээ:

```
let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"
```

Энд `isUserLoggedIn` хувьсагчид `true` утга оноосон тул `typeof` нь `"boolean"` гэж буцаана.

`typeof` оператор нь код шалгах (debug хийх) болон ямар төрлийн өгөгдөлтэй ажиллаж байгаагаа ойлгоход маш хэрэгтэй.

Гэхдээ `null`-тэй холбоотой нэг сонин асуудал байдаг.

Жишээ:

```
let exampleVariable = null;
console.log(typeof exampleVariable); // "object"
```

Энд `exampleVariable` хувьсагчид `null` утга оноосон боловч `typeof` нь `"object"` гэж буцааж байна.

Энэ нь JavaScript-ийн алдартай “алдаа” (bug) гэж үзэгддэг бөгөөд хэлний анхны хувилбаруудтай холбоотой.

JavaScript анх зохиогдох үед `null` утгыг тусгай төрлийн object гэж дотооддоо төлөөлдөг байсан тул ийм хачирхалтай үр дүн гардаг болсон.

Харамсалтай нь энэ нь хэлний нэг хэсэг болон үлдсэн бөгөөд ойлгомжгүй байж болох ч үүнийг мэдэж байх шаардлагатай.
