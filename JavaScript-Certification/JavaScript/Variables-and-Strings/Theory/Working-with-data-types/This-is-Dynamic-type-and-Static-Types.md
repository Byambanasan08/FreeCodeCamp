# JavaScript-д Dynamic Typing гэж юу вэ, мөн Statically Typed хэлнээс юугаараа ялгаатай вэ?

JavaScript нь dynamically typed хэл бөгөөд энэ нь хувьсагч зарлах үед өгөгдлийн төрлийг заавал тодорхойлох шаардлагагүй гэсэн үг юм. Харин хувьсагчийн төрөл нь програм ажиллах явцад оноосон утгаас хамаарч тодорхойлогддог. Энэ нь нэг хувьсагчийн төрлийг програмын явцад өөрчлөх боломжийг олгодог.

Жишээ:

```
let example = "Hello";
example = 42;
```

Энэ жишээнд `example` хувьсагч эхэндээ string төрөлтэй байсан боловч дараа нь number төрөлтэй болсон байна.

Dynamic typing-ийн уян хатан байдал нь JavaScript-ийг хурдан код бичих, туршихад хялбар болгодог. Гэхдээ програм томрох тусам алдаа илрүүлэхэд хэцүү болох сул талтай.

Statically typed хэлүүд (жишээ нь C#, C++)-д хувьсагч зарлах үед өгөгдлийн төрлийг заавал тодорхойлдог бөгөөд дараа нь тэр төрлийг өөрчилж болдоггүй.

Жишээ (C#):

```
int data = 42; // data үргэлж integer байх ёстой
data = "Hello"; // Энэ нь алдаа үүсгэнэ
```

Dynamic typing ба static typing-ийн ялгаа нь уян хатан байдал болон аюулгүй байдлын хооронд байдаг.

Dynamic typing нь илүү уян хатан боловч runtime үед алдаа гарах магадлалтай.

Statically typed хэлүүд нь илүү хатуу дүрэмтэй тул зарим алдааг програм ажиллахаас өмнө илрүүлэх боломжтой боловч уян хатан байдал бага байдаг.

Өөр нэг жишээ:

```
let data = 100;       // Эхэндээ number
data = "New data";    // Дараа нь string болж өөрчлөгдөнө
```

Statically typed хэлэнд ийм өөрчлөлт хийх боломжгүй, учир нь өгөгдлийн төрөл тогтмол байдаг.

Дүгнэж хэлбэл, JavaScript-ийн dynamic typing нь хувьсагчийн төрлийг чөлөөтэй өөрчлөх боломж олгодог тул уян хатан боловч гүйцэтгэлийн явцад гэнэтийн алдаа гарах эрсдэлтэй.

Харин Java зэрэг statically typed хэлүүд нь хувьсагчийн төрлийг урьдчилан тодорхойлох шаардлагатай байдаг бөгөөд энэ нь алдааг эрт илрүүлэх давуу талтай ч уян хатан байдал багатай байдаг.
