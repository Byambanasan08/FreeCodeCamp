Primitive болон Non-Primitive өгөгдлийн төрлүүдийн ялгаа юу вэ?

JavaScript-д primitive (үндсэн) болон non-primitive (үндсэн бус) өгөгдлийн төрлүүдийн ялгааг ойлгох нь алдаагүй, үр ашигтай код бичихэд маш чухал юм. Эдгээр хоёр төрлийн өгөгдөл нь санах ойд хэрхэн хадгалагдах болон таны програм дотор хэрхэн удирдагдах тал дээр өөр өөрөөр ажилладаг.



Primitive өгөгдлийн төрлүүд нь JavaScript дахь өгөгдлийн хамгийн энгийн хэлбэрүүд юм. Үүнд: `number`, `bigint`, `string`, `boolean`, `null`, `undefined`, болон `symbol` орно. Эдгээр төрлүүд нь ганц утгыг илэрхийлдэг бөгөөд объект биш учраас "primitive" (үндсэн) гэж нэрлэгддэг.

Та primitive өгөгдлийн төрлүүдтэй ажиллахдаа тэдгээрийн утгатай шууд харьцдаг. Жишээлбэл, та primitive утга бүхий хувьсагч үүсгэхэд тухайн утга нь хувьсагч дотор шууд хадгалагддаг.

Primitive утгууд нь хувиршгүй (**immutable**) байдаг бөгөөд энэ нь нэгэнт үүсгэгдсэн бол тэдгээрийн утгыг өөрчлөх боломжгүй гэсэн үг юм. Гэсэн хэдий ч та хувьсагчид шинэ утга дахин оноож болно. Primitive өгөгдлийн төрлүүдтэй ажиллах жишээг энд харуулав:

```javascript
let num1 = 5;
let num2 = num1;
num1 = 10;

console.log(num2); // 5
```

Энэ жишээнд бид `num1`-ээс primitive утгыг (5) `num2`-т оноож байна. Энэ нь тухайн утгын бие даасан хуулбарыг үүсгэдэг. Үүний үр дүнд эх хувьсагчид (`num1`) оруулсан аливаа өөрчлөлт хуулбар хувьсагчид (`num2`) нөлөөлөхгүй.

Нөгөө талаас, Non-primitive өгөгдлийн төрлүүд нь илүү нарийн төвөгтэй байдаг. JavaScript-д эдгээр нь **объектууд** бөгөөд үүнд ердийн объектууд, массивууд болон функцууд орно. Primitive-ээс ялгаатай нь non-primitive төрлүүд нь шинж чанар (properties) эсвэл элемент хэлбэрээр олон утгыг агуулж чаддаг.

Та non-primitive утга бүхий хувьсагч үүсгэхэд хувьсагч дотор хадгалагдаж буй зүйл нь объект өөрөө биш, харин тухайн объект санах ойд хадгалагдаж буй байршлын **лавлах (reference)** байдаг. Энэ нь ажиллах явцад зарим чухал ялгааг бий болгодог. Non-primitive төрлийн жишээг энд харуулав:



```javascript
const originalPerson = { name: "John", age: 30 };
const copiedPerson = originalPerson;

originalPerson.age = 31;

console.log(copiedPerson.age); // 31
```

Энэ жишээнд бид `name` болон `age` гэсэн хоёр шинж чанартай `originalPerson` нэртэй объекттой байна. Дараа нь бид `originalPerson` объектыг `copiedPerson` нэртэй хувьсагчид оноосон.

Үүний дараа бид `originalPerson` объектын `age` (нас) утгыг шинэчилсэн. `copiedPerson` объектын `age` шинж чанарыг консол дээр хэвлэхэд энэ нь шинэчлэгдсэн утгыг харуулж байна.

Гэхдээ яагаад ийм зүйл болж байна вэ? Энэ нь `originalPerson` болон `copiedPerson` хоёулаа санах ой дахь **нэг ижил объект руу** зааж (referencing) байгаатай холбоотой юм.

JavaScript-д объектыг өөр хувьсагчид оноохдоо та объектыг өөрийг нь биш, харин тухайн объект руу очих лавлахыг (reference) хуулж байдаг. Үүнийг **лавлахаар дамжуулан өнгөц хуулбарлах (shallow copying by reference)** гэж нэрлэдэг. Үүний үр дүнд нэг лавлахаар дамжуулан объектод оруулсан аливаа өөрчлөлт тухайн объект руу зааж буй бүх лавлахуудад тусгагддаг.

Та JavaScript-тэй үргэлжлүүлэн ажиллах явцдаа primitive болон non-primitive төрлүүдийн ялгааг ойлгох нь чухал болох олон нөхцөл байдалтай тулгарах болно. Энэ бол хэлний олон талын үндэс суурь болох суурь ойлголт бөгөөд үр ашигтай, зөв код бичих түлхүүр юм.

---
**Primitive утгууд "хуулбарлагддаг" (copied by value), харин объектууд "лавлахаар дамждаг" (passed by reference) гэсэн зарчим ойлгомжтой байна уу?**
