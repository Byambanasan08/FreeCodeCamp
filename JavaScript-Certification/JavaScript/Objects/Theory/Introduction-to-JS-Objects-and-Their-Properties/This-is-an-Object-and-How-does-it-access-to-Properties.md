 JavaScript-д Объект (Object) гэж юу вэ, объектын шинж чанарт (Properties) хэрхэн хандах вэ?

JavaScript-д объект гэдэг нь хоорондоо хамааралтай өгөгдөл болон функцуудыг хадгалах, зохион байгуулах боломжийг олгодог үндсэн өгөгдлийн бүтэц юм. Та объектыг төрөл бүрийн мэдээлэл агуулсан сав гэж төсөөлж болно, яг л бичиг хэргийн шүүгээ өөр өөр хавтас, баримт бичгүүдийг агуулдагтай адил юм.

Эдгээр мэдээллийн хэсгүүдийг **properties** (шинж чанар) гэж нэрлэх бөгөөд тэдгээр нь нэр (эсвэл түлхүүр - key) болон утгаас (value) бүрдэнэ.

```javascript
const exampleObject = {
  propertyName: value,
}
```

Объектууд нь маш олон талын ач холбогдолтой бөгөөд JavaScript-ийн амин сүнс нь болдог. Үнэндээ JavaScript дахь бараг бүх зүйл объект, эсвэл объект шиг ажиллаж чаддаг. Үүнд массивууд, функцууд, тэр байтугай тэмдэгт мөр (string) болон тоо зэрэг үндсэн (primitive) өгөгдлийн төрлүүд тодорхой байдлаар ашиглагдах үедээ объект шиг харьцагддаг.



JavaScript-ийн энэхүү объект төвтэй шинж чанар нь уг хэлийг ийм уян хатан, хүчирхэг болгодог шалтгаануудын нэг юм. Одоо объект хэрхэн үүсгэхийг харцгаая:

```javascript
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};
```

Энэ жишээнд бид `name`, `age`, болон `city` гэсэн гурван шинж чанартай `person` нэртэй объект үүсгэлээ. Шинж чанар бүр нь нэр болон утгатай бөгөөд тэдгээрийг хоёр цэгээр (`:`) тусгаарласан байна.

Одоо эдгээр шинж чанаруудад хэрхэн хандах талаар судалцгаая. JavaScript-д объектын шинж чанарт хандах үндсэн хоёр арга байдаг: **цэгэн бичиглэл** (dot notation) болон **дөрвөлжин хаалтын бичиглэл** (bracket notation).

#### Цэгэн бичиглэл (Dot notation)
Цэгэн бичиглэл нь объектын шинж чанарт хандах хамгийн түгээмэл бөгөөд ойлгомжтой арга юм. Цэгэн бичиглэлийн үндсэн синтакс энд байна:

`objectName.propertyName`

Бидний `person` объект дээр цэгэн бичиглэлийг хэрхэн ашиглахыг энд харуулав:

```javascript
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person.age);   // 30
```

Цэгэн бичиглэл нь товч бөгөөд уншихад хялбар байдаг тул та хандахыг хүссэн шинж чанарынхаа нэрийг яг таг мэдэж байгаа үед, мөн тэрхүү нэр нь JavaScript-ийн хүчинтэй танигч (valid identifier) байх үед (өөрөөр хэлбэл тоогоор эхлээгүй, тусгай тэмдэгт эсвэл хоосон зай агуулаагүй байх) энэ аргыг сонгох нь дээр байдаг.

#### Дөрвөлжин хаалтын бичиглэл (Bracket notation)
Нөгөө талаас, дөрвөлжин хаалтын бичиглэл нь дөрвөлжин хаалт дотор тэмдэгт мөр (string) ашиглан объектын шинж чанарт хандах боломжийг олгодог. Дөрвөлжин хаалтын бичиглэлийг хэрхэн ашиглахыг энд харуулав:

```javascript
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person["name"]); // Alice
console.log(person["age"]); //  30
```

Дөрвөлжин хаалтын бичиглэл нь цэгэн бичиглэлээс илүү уян хатан байдаг, учир нь энэ нь JavaScript-ийн хүчинтэй танигч биш шинж чанарын нэрсийг ашиглах боломжийг олгодог. Жишээлбэл, хэрэв танд хоосон зайтай эсвэл тоогоор эхэлсэн шинж чанарын нэр байгаа бол дөрвөлжин хаалтын бичиглэлийг ашиглах шаардлагатай болно:

```javascript
const oddObject = {
  "1stProperty": "Hello",
  "property with spaces": "World"
};

console.log(oddObject["1stProperty"]);  // Hello
console.log(oddObject["property with spaces"]);  // World
```



Дөрвөлжин хаалтын бичиглэлийн өөр нэг давуу тал нь шинж чанаруудад динамикаар хандахын тулд хувьсагч ашиглах боломжийг олгодог явдал юм:

```javascript
const person = {
  name: "Alice",
  age: 30,
  city: "Wonderland"
};

let propertyName = "city";
console.log(person[propertyName]); // Wonderland
```

Энэхүү уян хатан байдал нь кодыг бичиж байх үедээ шинж чанарын нэрийг яг таг мэдэхгүй байгаа тохиолдолд, эсвэл хэрэглэгчийн оролт эсвэл өөр ямар нэгэн динамик эх сурвалжаас ирсэн шинж чанарын нэрстэй ажиллаж байгаа үед дөрвөлжин хаалтын бичиглэлийг маш хэрэгтэй болгодог.

JavaScript дахь объектууд нь маш хүчирхэг бөгөөд олон талын ач холбогдолтой гэдгийг тэмдэглэх нь зүйтэй. Тэд зөвхөн тэмдэгт мөр, тоо гэх мэт энгийн утгуудыг агуулаад зогсохгүй массивууд эсвэл бусад объектуудыг ч агуулж болно.

Объектууд болон тэдэнтэй хэрхэн ажиллахыг ойлгох нь JavaScript-д маш чухал, учир нь тэдгээр нь уг хэл дээр болон олон JavaScript сан (libraries), фрэймворк (frameworks) дотор маш өргөн хэрэглэгддэг.

Та JavaScript-ийг үргэлжлүүлэн сурч, ажиллах явцдаа объектуудыг эзэмших нь нарийн төвөгтэй бөгөөд хүчирхэг програмуудыг бүтээх асар их боломжийг нээж өгдөг болохыг олж мэдэх болно.
