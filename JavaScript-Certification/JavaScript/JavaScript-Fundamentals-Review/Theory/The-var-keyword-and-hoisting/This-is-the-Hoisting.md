# Hoisting гэж юу вэ?

Hoisting (өргөлт) нь JavaScript-ийн анхан шатныхныг ихэвчлэн төөрөгдүүлдэг зан төлөв боловч үүнийг ойлгох нь таны кодонд гарч болох нарийн төвөгтэй алдаанаас зайлсхийхэд тусална. Энгийнээр хэлбэл, hoisting гэдэг нь JavaScript код ажиллаж эхлэхээс өмнө, хөрвүүлэх (compilation) шатанд зарлалтуудыг (declarations) өөрийнх нь хамаарах хүрээний (scope) хамгийн дээд хэсэгт шилжүүлдэг анхдагч зан төлөв юм.

Hoisting-ийг ойлгохын тулд JavaScript нь хоёр үе шаттай ажилладаг гэдгийг мэдэх нь чухал: **хөрвүүлэх үе (compilation phase)** болон **гүйцэтгэх үе (execution phase)**. Хөрвүүлэх үе шатанд JavaScript engine нь таны кодыг нэг бүрчлэн шалгаж, хувьсагч болон функцүүдэд зориулж санах ойн зайг бэлддэг. Чухам энд л hoisting ажиллаж эхэлдэг.

1. Хувьсагчийн hoisting (Variable Hoisting):
   Та `var` түлхүүр үг ашиглан хувьсагч зарлах үед JavaScript тухайн зарлалтыг (declaration) хүрээнийх нь хамгийн дээд хэсэгт өргөдөг. Гэвч **зөвхөн зарлалт нь өргөгддөг болохоос утга оноолт (initialization) нь өргөгддөггүй** гэдгийг анхаарах нь маш чухал юм. Энэ нь та хувьсагчийг зарлахаасаа өмнө кодондоо ашиглаж болно гэсэн үг боловч таныг утга оноох хүртэл түүний утга нь `undefined` байх болно.

```javascript
console.log(x); // undefined
var x = 5;
console.log(x); // 5
```

Энэ кодонд бид `x`-ийг зарлахаас өмнө ашигласан ч алдаа заахгүй, харин `undefined` гэсэн үр дүн гарч байна. Учир нь JavaScript `var x` гэсэн зарлалтыг дээр нь гаргаж өргөсөн ч `x = 5` гэсэн утга оноолтыг өргөөгүй юм. Энэ нь яг л кодыг дараах байдлаар дахин бичсэнтэй адилхан харагдана:

```javascript
var x;
console.log(x); // undefined
x = 5;
console.log(x); // 5
```

2. Функцийн hoisting (Function Hoisting):
   Функцийн hoisting нь арай өөрөөр ажилладаг. Та "function declaration" (функц зарлах) синтакс ашиглан функц зарлах үед **функцийн нэр болон функцийн бие (body) хоёулаа** өргөгддөг. Энэ нь та функцийг зарлахаасаа өмнө дуудаж болно гэсэн үг юм. Функцийн hoisting-ийн жишээ:

```javascript
sayHello(); // "Hello, World!"

function sayHello() {
  console.log("Hello, World!");
}
```

Энэ тохиолдолд функц бүхэлдээ хүрээнийхээ дээд хэсэгт өргөгдсөн тул бид `sayHello()`-ийг зарлахаас нь өмнө дуудаж чадаж байна.

3. let болон const-ийн hoisting:
   ES6-д нэвтрүүлсэн `let` болон `const` зарлалтууд дээр hoisting арай өөрөөр ажилладаг гэдгийг анхаарах хэрэгтэй.

```javascript
console.log(y); // ReferenceError алдаа заана
let y = 10;
```

Эдгээр зарлалтууд нь өргөгддөг боловч тэдгээр нь утга авч эхлээгүй (not initialized) байдаг бөгөөд та кодондоо зарлахаас нь өмнө тэдэнд хандах боломжгүй. Үүнийг ихэвчлэн **Temporal Dead Zone** (Түр зуурын үхмэл бүс) гэж нэрлэдэг.

Hoisting-ийг ойлгох нь илүү цэвэр, таамаглах боломжтой код бичихэд тусална. Гэсэн хэдий ч hoisting-д хэт найдах нь кодыг уншихад болон арчлахад хэцүү болгодог. Тиймээс хамгийн сайн туршлага (best practice) бол hoisting-оос үл хамааран хувьсагчдаа хүрээнийх нь дээд хэсэгт зарлаж, функцүүдээ ашиглахаасаа өмнө тодорхойлж байх явдал юм. Энэ нь таны кодын зан төлөвийг илүү тодорхой болгож, бусад хүмүүс болон ирээдүйн "та өөрөө" кодыг тань ойлгоход хялбар болгоно.
