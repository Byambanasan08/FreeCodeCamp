# var түлхүүр үг гэж юу вэ, яагаад үүнийг ашиглахыг зөвлөхөө байсан бэ?

JavaScript дахь `var` түлхүүр үг нь хувьсагч зарлах анхны аргуудын нэг юм. Энэ нь хэл үүссэн цагаас эхлэн байсаар ирсэн бөгөөд олон жилийн турш хувьсагч үүсгэх үндсэн арга байсаар ирсэн. Гэсэн хэдий ч JavaScript хөгжиж, хөгжүүлэгчид илүү туршлагажих тусам `var`-ыг ашиглахын тодорхой сул талууд илэрч эхэлсэн бөгөөд энэ нь 2015 онд `let` болон `const`-ыг нэвтрүүлэхэд хүргэсэн юм.

Та `var`-аар хувьсагч зарлах үед энэ нь **function-scoped** (функцийн хүрээтэй) эсвэл **globally-scoped** (глобал хүрээтэй) болдог. Энэ нь хэрэв та функцийг дотор `var`-аар хувьсагч зарлавал тэр нь зөвхөн тухайн функц дотроо хандах боломжтой гэсэн үг. Харин ямар нэгэн функцийн гадна зарлавал таны бүх скрипт даяар хандаж болох глобал хувьсагч болно. Энэ зан төлөв нь заримдаа хүлээгдээгүй үр дүнд хүргэж, кодыг ойлгоход хэцүү болгодог.

`var`-тай холбоотой нэг асуудал бол ижил нэртэй хувьсагчийг ямар нэгэн алдаа заалгүйгээр олон удаа дахин зарлах (redeclare) боломжийг олгодогт оршино. Энэ нь санамсаргүйгээр утгыг дарах (overwrite) болон алдааг олж засварлахад (debugging) хүндрэл учруулдаг.

```javascript
var num = 5;
console.log(num); // 5

// Ижил нэрээр дахин зарлахыг зөвшөөрдөг бөгөөд алдаа заадаггүй
var num = 10;
console.log(num); // 10
```

`var`-ын хамгийн том асуудал бол түүнд **block scoping** (блок хүрээ) байхгүй явдал юм. `if` нөхцөл эсвэл `for` давталт зэрэг "блок" дотор `var`-аар зарласан хувьсагч нь тухайн блокийн гадна хандагдсан хэвээр байдаг.

```javascript
if (true) {
  var num = 5;
}
console.log(num); // 5 (Блокийн гадна байгаа ч хандаж болж байна)
```

Энэхүү зан төлөв нь хувьсагч "алдагдах" (leak) болон кодыг алдаанд өртөмтгий болгоход хүргэдэг. Эдгээр асуудлуудаас болж орчин үеийн JavaScript хөгжүүлэлтэд `var`-аас татгалзаж, `let` болон `const`-ыг илүүд үзэх болсон. Эдгээр түлхүүр үгс нь бусад олон програмчлалын хэлүүдтэй ижил замаар "block scoping"-ийг хангадаг. Мөн тэд нэг хүрээ (scope) дотор дахин зарлахыг зөвшөөрдөггүй нь санамсаргүйгээр утга дарахаас сэргийлдэг.

Хэдийгээр `var` нь JavaScript-ийн нэг хэсэг хэвээр байж, бүх вэб хөтөч дээр ажилладаг ч орчин үеийн хөгжүүлэлтэд `let` болон `const`-ыг ашиглахыг зөвлөдөг. Тэдгээр нь хүрээний тодорхой дүрмүүдийг хангаж, нийтлэг алдаанаас сэргийлж, кодын зан төлөвийг илүү таамаглах боломжтой болгодог.
