# JavaScript Module гэж юу вэ, түүнийг програмдаа хэрхэн импортлох болон экспортлох вэ?

JavaScript-д модуль гэдэг нь хоорондоо хамааралтай функц, класс эсвэл хувьсагчдыг нэгтгэсэн, бие даасан кодын нэгж юм. Модулийг өөрийн аппликейшнийг босгох "барилгын блок" буюу номын нэг бүлэгтэй адилхан гэж ойлгож болно. Модуль бүр тодорхой нэг функц дээр төвлөрдөг бөгөөд энэ нь таны кодыг илүү зохион байгуулалттай, арчлахад хялбар, дахин ашиглах боломжтой болгодог. Модулиуд нь нэрний зөрчлөөс (naming conflicts) сэргийлж, аппликейшнээ салангид боловч хоорондоо холбогдсон хэсгүүдэд хуваах боломжийг олгодог.

JavaScript дахь модулийн ойлголт цаг хугацааны явцад хувьсан өөрчлөгдөж ирсэн боловч өнөөдөр хамгийн өргөн ашиглагддаг бөгөөд дэмжигддэг арга бол **ES6 (ECMAScript 2015)** модулийн систем юм. Энэ систем нь өөр өөр JavaScript орчнуудад модулийг тодорхойлох, ашиглах стандарт аргыг бүрдүүлдэг.

Модуль үүсгэхийн тулд та JavaScript кодоо тусдаа файлд бичнэ. Аппликейшнийхээ бусад хэсэгт ашиглахыг хүссэн ямар ч хувьсагч, функц эсвэл классыг заавал **экспортлох** (export) шаардлагатай. Үүнийг та `export` түлхүүр үгийг ашиглан хийнэ.

Жишээлбэл, танд математикийн функцууд агуулсан `math.js` гэдэг файл байна гэж бодъё:

```javascript
export function add(a, b) {
  return a + b;
}

export function subtract(a, b) {
  return a - b;
}

const PI = 3.14159;
export { PI };
```

Энэ жишээнд бид хоёр функц (`add` ба `subtract`) болон нэг тогтмол утгыг (`PI`) экспортолж байна. Та нэг модулиас хэрэгцээтэй гэж үзсэн олон зүйлээ экспортолж болно.

Эдгээр экспортолсон зүйлсийг аппликейшнийхээ өөр хэсэгт ашиглахын тулд та тэдгээрийг **импортлох** (import) хэрэгтэй. Үүнийг `import` түлхүүр үг ашиглан хийдэг. Та `math.js` доторх функцуудыг `app.js` файлдаа ашиглахыг хүсвэл:

```javascript
import { add, subtract, PI } from "./math.js";

console.log(add(5, 3)); // Үр дүн: 8
console.log(subtract(10, 4)); // Үр дүн: 6
console.log(PI); // Үр дүн: 3.14159
```

Энд бид `math.js` модулиас өөрт хэрэгтэй тодорхой функц болон тогтмол утгуудыг импортолж байна. `'./math.js'` хэсэг нь JavaScript-д тухайн модулийн файлыг одоогийн файлтай харьцуулахад хаана байгааг зааж өгч байна.

Заримдаа та тухайн модулийн экспортолсон бүх зүйлийг нэг дор импортлохыг хүсэж болно. Үүнийг од (\*) тэмдэг ашиглан хийж болно:

```javascript
import * as MathFunctions from "./math.js";

console.log(MathFunctions.add(5, 3)); // Үр дүн: 8
console.log(MathFunctions.subtract(10, 4)); // Үр дүн: 6
console.log(MathFunctions.PI); // Үр дүн: 3.14159
```

Энэ тохиолдолд `math.js`-ээс ирсэн бүх зүйл `MathFunctions` (эсвэл таны өгсөн дурын нэр) гэдэг объектын шинж чанарууд (properties) болон орж ирнэ.

Өөр нэг түгээмэл хэрэглээ бол модульд **"default export"** (анхдагч экспорт) ашиглах явдал юм. Энэ нь ихэвчлэн тухайн модуль ердөө нэг гол функцийг л экспортолж байгаа үед ашиглагддаг. Нэг модульд зөвхөн нэг л анхдагч экспорт байж болно.

Энэ нь ингэж харагдана:

```javascript
// math.js файл дотор
export default function multiply(a, b) {
  return a * b;
}

// app.js файл дотор
import multiply from "./math.js";

console.log(multiply(4, 5)); // Үр дүн: 20
```

"Default export"-ийг импортлох үед танд **их хаалт `{ }` хэрэггүй** бөгөөд импортолж буй зүйлдээ ямар ч нэр өгч болно гэдгийг анхаараарай.

Мөн вэб хөтөч дээр ES6 модулийг ашиглахын тулд `<script>` таг дээрээ `type="module"` гэж зааж өгөх шаардлагатайг санаарай:

```html
<script type="module" src="app.js"></script>
```

Модулиуд нь JavaScript кодыг зохион байгуулах, бүтэцжүүлэх хүчирхэг аргыг олгодог. Тэд танд аппликейшнээ жижиг, удирдахад хялбар хэсгүүдэд хуваах, кодыг дахин ашиглах боломжийг олгож, бүтэц бүрийг хооронд нь цэвэр зааглаж өгдөг. Том хэмжээний аппликейшн бүтээх тусам модулиуд таны JavaScript хөгжүүлэлтийн зэвсэглэлийн зайлшгүй чухал хэрэгсэл болохыг та олж харах болно.

---

**`export { PI }` (нэрлэсэн) болон `export default` (анхдагч) хоёрын импортлох үеийн ялгаа (их хаалттай ба хаалтгүй) ойлгомжтой байна уу? Та өөрийн хийж буй вэб хуудсандаа кодыг ингэж тусдаа файлуудад хувааж үзсэн үү?**
