# Linter болон Formatter гэж юу вэ, тэдгээр нь кодын нэгдмэл байдалд(consistency) хэрхэн тусалдаг вэ?

Програм хангамж хөгжүүлэлтийн ертөнцөд цэвэр, нэгдмэл бөгөөд алдаагүй код бичих нь маш чухал юм. Яг энд л **linter** болон **formatter**-ууд хэрэг болдог. Эдгээр хэрэгслүүд нь хөгжүүлэгчдэд төсөл болон багийн хэмжээнд кодын чанар, нэгдмэл байдлыг хангахад зайлшгүй шаардлагатай.

Эхлээд **linter**-ийн талаар ярилцъя. Linter (линтер) гэдэг нь програмчлалын алдаа, баг (bug), загварын алдаа болон сэжигтэй бүтцүүдийг илрүүлдэг "static code analysis" (кодыг ажиллуулахгүйгээр шинжлэх) хэрэгсэл юм. "Lint" гэдэг нэр томьёо нь анх C хэлний эх кодыг шалгадаг Unix-ийн нэгэн туслах програмаас үүсэлтэй.

Өнөөдөр JavaScript-ийг оролцуулаад ихэнх програмчлалын хэлүүдэд зориулсан линтерүүд байдаг. Линтерүүд хэд хэдэн аргаар тусалдаг. Нэгдүгээрт, тэд кодыг ажиллахаас (runtime) өмнө гарч болзошгүй алдааг барьж авдаг. Жишээлбэл, линтер нь зарлагдаагүй хувьсагч ашиглах эсвэл функцийг буруу тооны аргументтай дуудах зэргийг анхааруулж болно. Хоёрдугаарт, тэд кодын стандарт болон "best practices" (шилдэг туршлага)-ыг мөрдүүлдэг. Үүнд догол мөр (indentation), цэг таслал (semicolon) ашиглах, эсвэл нэг мөрөнд байх текстийн дээд хэмжээ зэрэг дүрмүүд багтаж болно. Эцэст нь, тэд кодын санг (codebase), ялангуяа олон хөгжүүлэгч нэг төсөл дээр ажиллаж байгаа үед нэгдмэл байлгахад тусалдаг.

JavaScript-ийн хамгийн түгээмэл линтер бол **ESLint** юм. ESLint юуг анхааруулж болох энгийн жишээг энд харуулав:

```javascript
function doSomething(x) {
  return x + z;
}
```

Энэ кодонд `z` хувьсагч зарлагдаагүй тул ESLint нь "z is not defined" гэсэн алдааг заана. Мөн хамгийн сүүлийн өгүүлбэрт цэг таслал (semicolon) дутуу байгааг линтер кодын редактор дээр анхааруулж магадгүй юм.

Нөгөө талаас, **formatter** (форматтер) нь таны кодыг тодорхой загварын зааврын дагуу автоматаар хэлбэржүүлдэг хэрэгсэл юм. Линтерүүд зарим асуудлыг автоматаар засаж чаддаг ч форматтерууд нь тусгайлан таны кодыг урьдчилан тохируулсан загварт нийцүүлэн дахин бичих зориулалттай байдаг.

Форматтерууд нь хөгжүүлэгч бүрийн хувийн сонирхлоос үл хамааран төсөл эсвэл багийн хэмжээнд кодын ижил хэв маягийг хангадаг. Мөн тэд кодыг гараар форматлахад зарцуулах цаг хугацаа болон оюуны энергийг хэмнэдэг. Эцэст нь, тэд кодын хэв маягтай холбоотой маргааныг үгүй болгосноор кодын хяналтын (code review) процессыг илүү үр дүнтэй болгодог.

JavaScript-ийн хамгийн түгээмэл форматтер бол **Prettier** юм. Prettier кодыг хэрхэн форматлаж болох жишээг энд харуулав. Форматлахаас өмнө:

```javascript
function longFunction(argument1, argument2, argument3) {
  return argument1 + argument2 + argument3;
}
```

Prettier-ээр форматласны дараа:

```javascript
function longFunction(argument1, argument2, argument3) {
  return argument1 + argument2 + argument3;
}
```

Линтер болон форматтеруудыг хөгжүүлэлтийн явцад (workflow) янз бүрийн аргаар нэгтгэж болно. Тэдгээрийг таны "build process"-д оруулж болох ба эсвэл текст редактор, IDE-дээ нэмэлт (plugin) байдлаар суулгаж, код бичиж байх үедээ бодит цагийн санал хүсэлт авч болно. Линтер болон форматтерыг хамтад нь ашиглах нь кодын чанар болон нэгдмэл байдлыг мэдэгдэхүйц сайжруулдаг. Жишээлбэл, та ESLint-ийг ашиглан гарч болзошгүй алдааг барьж, тодорхой дүрмүүдийг мөрдүүлж, харин Prettier-ийг ашиглан бүх форматлах ажлыг гүйцэтгүүлж болно.

Олон хөгжүүлэлтийн багууд эдгээр хэрэгслүүдийг төслийн тохиргооны нэг хэсэг болгон тохируулж, кодыг систем рүү оруулахаас (commit) өмнө линтер болон форматтерыг заавал ажиллуулдаг "pre-commit hooks"-ыг ашигладаг. Энэ нь репозитори дахь бүх код багийн чанар, загварын стандартад нийцэж байгааг баталгаажуулдаг.

Дүгнэж хэлэхэд, линтер болон форматтерууд нь кодын чанарыг хадгалах, алдааг эрт илрүүлэх, кодын сангийн нэгдмэл байдлыг хангахад тусалдаг хүчирхэг хэрэгслүүд юм. Кодын хяналтын эдгээр хэсгийг автоматжуулснаар хөгжүүлэгчид кодын хэв маягийн талаар маргахаас илүүтэйгээр асуудлыг шийдвэрлэхэд илүү анхаарал хандуулах боломжийг олгодог.
