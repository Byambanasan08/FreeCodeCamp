# Closure гэж юу вэ, тэр нь хэрхэн ажилладаг вэ?

Closure (хумилт) нь JavaScript-ийн хамгийн хүчирхэг бөгөөд ихэвчлэн буруу ойлгогддог боломжуудын нэг юм. Үндсэндээ бол **closure** гэдэг нь гадна талын функц нь ажиллаж дуусаад утгаа буцаасны дараа ч тухайн функцийн хүрээнд (outer lexical scope) байсан хувьсагчдад хандах эрхтэй хэвээр байдаг "дотор талын функц" юм. Энэ нь төвөгтэй сонсогдож болох ч JavaScript-ийн олон дэвшилтэт програмын хэв шинжийг бүрдүүлдэг суурь ойлголт юм.

Closure-ийг ойлгохын тулд дараах жишээнээс эхэлцгээе:

```javascript
function outerFunction(x) {
  let y = 10;
  function innerFunction() {
    console.log(x + y);
  }
  return innerFunction;
}

let closure = outerFunction(5);
closure(); // 15
```

Энэ жишээнд `outerFunction` нь `x` гэсэн параметр хүлээн авч, `y` гэсэн дотоод хувьсагчийг зарлаж байна. Дараа нь `x` болон `y`-ийг хоёуланг нь ашигладаг `innerFunction`-ийг дотор нь тодорхойлсон байна. Эцэст нь `innerFunction`-ийг буцааж байна. Бид `outerFunction(5)`-ыг дуудахад энэ нь `innerFunction`-ийг буцаах ба бид түүнийг `closure` гэдэг хувьсагчид оноож байна. Дараа нь бид `closure()`-ийг дуудахад `outerFunction` ажиллаж дууссан байсан ч `x` болон `y`-д хандсан хэвээр байна. Энэ бол closure-ийн гол мөн чанар юм.

Дотор талын функц нь өөрийн гадна талын орчны (lexical environment) хаягийг хадгалж үлддэг тул гадна талын функц ажиллаж дууссан ч тухайн орчны хувьсагчдад хандах эрхээ хадгалсаар байдаг.

#### Хувийн (Private) хувьсагч үүсгэх:

Closure нь "private" буюу гаднаас хандах боломжгүй хувьсагч болон функц үүсгэхэд маш хэрэгтэй. Дараах жишээг харна уу:

```javascript
function createCounter() {
  let count = 0;
  return function () {
    count++;
    return count;
  };
}

let counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

Энэ тохиолдолд `createCounter` нь `count` хувьсагчийг нэмэгдүүлж, буцаадаг функцийг өөрөө буцааж байна. `count` хувьсагчид `createCounter`-ийн гаднаас шууд хандах боломжгүй боловч буцаагдсан функц (манай closure) түүнд хандах эрхтэй. Биднийг `counter()`-ийг дуудах бүрд `count`-ийг нэмэгдүүлж, утгыг нь буцаасаар байх болно.

Closure нь гадна талын хүрээнээс олон хувьсагчийг "барьж" (capture) авч чадна. Жишээлбэл:

```javascript
function multiply(x) {
  return function (y) {
    return x * y;
  };
}

let double = multiply(2);
console.log(double(5)); // 10
```

Энд дотор талын функц нь `multiply`-ийн `x` параметрыг барьж авсан байна. Бид `multiply(2)`-ыг дуудаж `double`-ийг үүсгэхэд энэ нь аргументыг нь үргэлж 2-оор үржүүлдэг функцийг буцааж байна.

Closure-ийн талаар анхаарах нэг чухал зүйл бол тэд хувьсагчдыг утгаар нь (by value) биш, харин **хаягаар нь (by reference)** барьж авдаг. Энэ нь хэрэв баригдсан хувьсагчийн утга өөрчлөгдвөл closure нь шинэ утгыг харах болно гэсэн үг юм.

```javascript
function createIncrementer() {
  let count = 0;
  return function () {
    count++;
    console.log(count);
  };
}

let increment = createIncrementer();
increment(); // 1
increment(); // 2
```

Биднийг `increment`-ийг дуудах бүрд энэ нь анхны утгын хуулбартай биш, харин яг ижил `count` хувьсагчтай ажиллаж байна. Closure бол JavaScript-ийн хүчирхэг хэрэгсэл юм. Та JavaScript-тэй үргэлжлүүлэн ажиллах тусам closure-ийг үр дүнтэй ашиглаж сурах нь цэвэр, үр ашигтай, хүчирхэг код бичих чадварыг тань ихээхэн сайжруулна гэдгийг анзаарах болно.
