# Memory Management гэж юу вэ, тэр нь JavaScript дээр хэрхэн ажилладаг вэ?

Санах ойн удирдлага (Memory management) нь програмчлалын амин чухал ойлголт боловч анхан шатныханд бага зэрэг ойлгомжгүй байж болох тул үүнийг энгийнээр тайлбарлацгаая.

Та ямар нэгэн програм, түүний дотор вэб хөтөч дээр JavaScript код ажиллуулах үед тухайн програм ажиллаж буй бүх мэдээллээ хадгалахын тулд санах ой (memory) шаарддаг. Үүнд хувьсагчууд, функцууд, объектууд буюу үндсэндээ таны кодын үүсгэж, ашиглаж буй бүх зүйл багтана. Санах ойн удирдлага гэдэг нь энэхүү санах ойг хянах, шаардлагатай үед нь хуваарилах (allocating), ашиглахаа больсон үед нь чөлөөлөх (freeing) үйл явц юм.

Зарим програмчлалын хэл дээр хөгжүүлэгчид санах ойг гараар (manually) удирдах шаардлагатай болдог. Тэд компьютерт хэзээ шинэ зүйлд санах ой хуваарилах, хэзээ ашиглахаа больсон санах ойг чөлөөлөх вэ гэдгийг тодорхой хэлж өгөх хэрэгтэй болдог гэсэн үг. Энэ нь хүчтэй арга боловч санах ойг чөлөөлөхөө мартах нь "memory leaks" (санах ойн алдагдал) үүсгэх аюултай тул нэлээд төвөгтэй байж болно.

Харин JavaScript нь **автомат санах ойн удирдлагыг** ашигладаг. Энэ нь JavaScript (илүү тодорхой хэлбэл таны вэб хөтөч дээрх JavaScript engine) санах ойг хуваарилах болон чөлөөлөх ажлыг таны өмнөөс хариуцна гэсэн үг юм. Та кодондоо санах ойг заавал гараар чөлөөлөх шаардлагагүй. Энэхүү автомат процессыг ихэвчлэн **Garbage Collection** (Хог цуглуулах) гэж нэрлэдэг.

Энэ нь энгийнээр ингэж ажилладаг:

1. **Allocation (Хуваарилалт):** Та JavaScript кодондоо хувьсагч, объект эсвэл функц үүсгэх үед тэдгээрийг хадгалах санах ой автоматаар хуваарилагдана.
2. **Use (Ашиглалт):** Дараа нь та эдгээр хувьсагч, объект, функцуудтай ажиллахдаа хуваарилагдсан санах ойг ашигладаг.
3. **Release (Чөлөөлөлт):** JavaScript engine нь санах ой дахь ямар нэгэн зүйл хэзээ хэрэггүй болохыг тогтоох ухаалаг аргуудтай. Ерөнхийдөө, хэрэв таны програмд тухайн өгөгдөлд хандах эсвэл ашиглах ямар ч боломжгүй болсон бол түүнийг "хэрэггүй болсон" гэж үзнэ. Тодорхой хугацааны давтамжтайгаар "garbage collector" ажиллаж, хэрэггүй болсон санах ойг олж чөлөөлөн, дараагийн хэрэглээнд бэлэн болгодог.

Энэ процесс автоматаар явагддаг нь маш сайн хэрэг, учир нь та санах ойг өөрөө удирдах гэж санаа зовох шаардлагагүй. Гэсэн хэдий ч энэ ойлголтыг мэдэх нь чухал, учир нь та заримдаа хэрэггүй болсон зүйлсийнхээ хаягийг (reference) санамсаргүйгээр хадгалж үлдсэнээр "garbage collector"-ийг тухайн санах ойг чөлөөлөхөд нь саад болж магадгүй юм. Жишээ нь:

```javascript
function createLargeArray() {
  let largeArray = new Array(1000000);
  return function () {
    console.log(largeArray.length);
  };
}

let printArrayLength = createLargeArray();
printArrayLength();
```

Энэ кодонд `createLargeArray` ажиллаж дууссаны дараа ч `largeArray`-г "garbage collect" хийж (санах ойг чөлөөлж) болохгүй, учир нь буцаагдсан функц (printArrayLength) түүнд хандах эрхтэй хэвээр байна. Энэ бол **closure** (хумилт) бөгөөд "closure" нь маш хэрэгтэй боловч заримдаа таны төсөөлснөөс илүү их санах ой ашиглахад хүргэж болно. Та "closure"-ийн талаар ирээдүйн хичээлүүдээр илүү дэлгэрэнгүй үзэх болно.

Анхан шатныхны хувьд санах ойн удирдлагын нарийн ширийнийг хэт их анхаарах шаардлагагүй. JavaScript-ийн автомат "garbage collection" ихэнх зүйлийг таны өмнөөс зохицуулна. Гэвч JavaScript-ийн аялалдаа ахих тусам эдгээр ойлголтыг мэдэх нь илүү үр ашигтай код бичихэд, ялангуяа том хэмжээний аппликейшн эсвэл хязгаарлагдмал нөөцтэй орчинд ажиллахад тань тусална.

Санаж яваарай: Боломжтой бол глобал хувьсагчаас зайлсхийх, функцүүд тань юуг "close over" хийж (дотроо хадгалж) байгааг анхаарах зэрэг кодын сайн туршлага нь JavaScript engine-д санах ойг илүү үр дүнтэй удирдахад тусалдаг.
