Хувьсагч болон функцийг нэрлэх түгээмэл аргууд юу вэ?

Хувьсагч болон функцийг зөв нэрлэх нь цэвэр, уншихад хялбар, арчлахад дөхөм код бичихэд маш чухал аспект юм. Нэрлэх сайн туршлага нь таны кодыг "өөрийгөө баримтжуулдаг" (self-documenting) болгодог бөгөөд ингэснээр их хэмжээний тайлбар (comments) бичих шаардлагыг багасгаж, бусад хөгжүүлэгчид болон ирээдүйн "та өөрөө" кодыг тань ойлгоход илүү хялбар болгодог.

JavaScript дахь ерөнхий нэрлэх дүрмүүдээс эхэлцгээе. Өмнөх хичээлүүд дээр та хувьсагчийн нэрэнд **camelCase** (тэмээ хэлбэртэй бичиглэл) ашиглах талаар сурсан.

Boolean (логик) хувьсагчдын хувьд **is**, **has**, эсвэл **can** гэх мэт угтваруудыг (prefixes) ашиглах нь түгээмэл туршлага юм. Энэ нь уншигчид тухайн хувьсагчийг шууд boolean төрөл болохыг хэлж өгдөг:

```javascript
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```



Функцуудын хувьд нэр нь тухайн функц юу хийдгийг тодорхой илэрхийлэх ёстой. Үйл үгээр эхлэх нь ихэвчлэн тустай байдаг:

```javascript
function getUserData(){
  /* ... */
}
function calculateTotal(){
  /* ... */
}
function validateInput(){
  /* ... */
}
```

Boolean утга буцаадаг функцуудыг ихэвчлэн "predicates" гэж нэрлэдэг бөгөөд тэдгээрт мөн адил **is**, **has**, эсвэл **can** угтваруудыг ашиглаж болно:

```javascript
function isValidEmail(email) {
  /* ... */
}
function hasRequiredFields(form) {
  /* ... */
}
```

Өгөгдөл хүлээн авч (retrieve) буй функцуудыг **get** гэдэг үгээр эхлүүлэх нь түгээмэл:

```javascript
function getProductDetails(productId) {
  /* ... */
}
function getUserProfile(userId) {
  /* ... */
}
```

Өгөгдлийг оноож (set) буй функцуудыг **set** гэдэг үгээр эхлүүлэх нь түгээмэл:

```javascript
function setUserPreferences(preferences) {
  /* ... */
}
function setPageTitle(title) {
  /* ... */
}
```

Үйл явдал боловсруулах (event handler) функцуудын хувьд та **handle** угтварыг эсвэл **handler** дагаврыг (suffix) ашиглаж болно:

```javascript
function handleClick(){
  /* ... */
}
function onSubmit(){
  /* ... */
}
function keyPressHandler(){
  /* ... */
}
```

"Event handler" гэдэг нь товчлуур дээр дарах зэрэг ямар нэгэн үйл явдал (event) болсны дараа хийгдэх үйлдэл юм. Та энэ талаар ирээдүйн хичээлүүдээр үзэх болно.

Давталтын итератор хувьсагчдыг нэрлэхдээ **i**, **j**, эсвэл **k** гэх мэт ганц үсэг ашиглах нь түгээмэл боловч үүрлэсэн (nested) давталтууд эсвэл илүү төвөгтэй итерациудын хувьд илүү тодорхой нэр ашиглах нь тустай байж болно:

```javascript
for (let i = 0; i < array.length; i++) {
  /* ... */
}
for (let studentIndex = 0; studentIndex < students.length; studentIndex++) {
  /* ... */
}
```

Массивын нэрэнд тухайн хувьсагч олон зүйл агуулж байгааг илэрхийлэхийн тулд нэр үгийг олон тоон (plural nouns) дээр ашиглахыг анхаараарай:

```javascript
const colors = ['red', 'green', 'blue'];
const userNames = ['Alice', 'Bob', 'Charlie'];
```



Гол зорилго нь таны кодыг аль болох "өөрийгөө тайлбарладаг" (self-explanatory) болгоход оршино гэдгийг санаарай. Хэрэв танд ямар нэгэн хувьсагч эсвэл функц юу хийдгийг тайлбарлахын тулд заавал "comment" (тайлбар) нэмэх шаардлага гарч байвал, тухайн нэрийг илүү тодорхой нэрээр солих талаар бодож үзэх нь зүйтэй.

Эцэст нь, өөрийн кодын сан (codebase) эсвэл багтайгаа нэгдмэл байх хэрэгтэй. Хэрэв танай баг нэрлэх тогтсон дүрмүүдтэй бол тэдгээрийг дагаж мөрдөөрэй. Тогтвортой байдал (consistency) нь кодыг оролцож буй хүн бүрт уншихад хялбар, арчлахад дөхөм болгодог.
