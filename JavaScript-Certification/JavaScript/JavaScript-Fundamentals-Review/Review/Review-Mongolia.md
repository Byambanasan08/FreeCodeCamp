# JavaScript-ийн үндсэн ойлголтуудын тойм (Full Text)

1. String байгуулагч (Constructor) болон toString() арга

1.1 Тодорхойлолт: String объект нь тэмдэгтүүдийн дарааллыг илэрхийлэхэд ашиглагддаг. String объектууд нь `String` байгуулагч функцийг ашиглан үүсгэгддэг бөгөөд энэ нь "primitive" (анхдагч) утгыг объект дотор боож (wrap) өгдөг.

```javascript
const greetingObject = new String("Hello, world!");
console.log(typeof greetingObject); // "object"
```

1.2 toString() арга: Энэ арга нь утгыг тэмдэгт мөр (string) хэлбэрт хөрвүүлдэг. Энэ бол та тоо, boolean, массив болон объектууд дээр ашиглаж болох арга юм.

```javascript
const num = 10;
console.log(num.toString()); // "10"

const arr = [1, 2, 3];
console.log(arr.toString()); // "1,2,3"
```

Энэ арга нь 2-оос 36 хүртэлх тоо байж болох нэмэлт **radix** (суурь тоо) параметрийг хүлээн авдаг. Энэхүү radix нь тооны системийг илэрхийлдэг бөгөөд жишээ нь хоёртын системд 2, наймтын системд 8 гэх мэт. Хэрэв radix-ийг зааж өгөхгүй бол анхдагчаар 10-тын систем (decimal) ашиглана.

```javascript
const num = 10;
console.log(num.toString(2)); // "1010" (binary - хоёртын систем)
```

2. Number байгуулагч (Constructor)

2.1 Тодорхойлолт: `Number` байгуулагч нь тоон объект үүсгэхэд ашиглагддаг. Тоон объект нь `isNaN` болон `toFixed` зэрэг хэд хэдэн хэрэгцээтэй шинж чанар (properties) болон аргуудыг (methods) агуулдаг. Ихэнх тохиолдолд та `Number` байгуулагчийг бусад төрлийн өгөгдлийг тоон төрөл рүү хөрвүүлэхэд ашиглах болно.

```javascript
const myNum = new Number("34");
console.log(typeof myNum); // "object"

const num = Number("100");
console.log(num); // 100
console.log(typeof num); // number
```

3. Хувьсагч болон функцийг нэрлэх шилдэг туршлага

**3.1 camelCasing:** Тогтсон ёсоор JavaScript хөгжүүлэгчид хувьсагч болон функцийг нэрлэхдээ **camel casing** ашигладаг. Camel casing гэдэг нь эхний үг бүхэлдээ жижиг үсгээр, дараагийн үгнүүд нь том үсгээр эхлэхийг хэлнэ. Жишээ нь: `isLoading`.

**3.2 Boolean нэрлэх:** Boolean (үнэн/худал) хувьсагчдад `is`, `has`, эсвэл `can` гэсэн угтваруудыг ашиглах нь нийтлэг туршлага юм.

```javascript
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

**3.3 Функц нэрлэх:** Функцийн нэр нь тухайн функц юу хийдэг болохыг тодорхой илэрхийлэх ёстой. Boolean утга буцаадаг функцүүдэд (үүнийг ихэвчлэн "predicates" гэдэг) дээрхтэй адил `is`, `has`, `can` угтваруудыг ашиглаж болно. Өгөгдөл татаж авдаг функцүүд ихэвчлэн `get` үгээр, өгөгдөл оноодог функцүүд `set` үгээр эхэлдэг. Үйлдэл боловсруулах (event handler) функцүүдэд `handle` угтвар эсвэл `Handler` дагавар ашиглаж болно.

```javascript
function getUserData() {
  /* ... */
}
function isValidEmail(email) {
  /* ... */
}
function getProductDetails(productId) {
  /* ... */
}
function setUserPreferences(preferences) {
  /* ... */
}
function handleClick() {
  /* ... */
}
```

**3.4 Давталт доторх хувьсагч нэрлэх:** Давталт (loop) доторх итератор хувьсагчдыг нэрлэхдээ `i`, `j`, эсвэл `k` гэх мэт ганц үсэг ашиглах нь түгээмэл байдаг.

### 4. Sparse Arrays (Сийрэг массив)

**4.1 Тодорхойлолт:** Массивт хоосон зайнууд (empty slots) байх боломжтой. Хоосон зай гэдгийг дотор нь юу ч байхгүй зай гэж тодорхойлно. Энэ нь `undefined` утгатай массивын зайнаас өөр ойлголт юм. Ийм төрлийн массивуудыг **sparse arrays** гэж нэрлэдэг.

```javascript
const sparseArray = [1, , , 4];
console.log(sparseArray.length); // 4
```

### 5. Linters болон Formatters

**5.1 Linters:** Линтер гэдэг нь програмын алдаа, согог, стилийн алдаа болон сэжигтэй бүтцүүдийг тэмдэглэдэг статик код шинжилгээний хэрэгсэл юм. Нийтлэг линтер хэрэгслийн жишээ бол **ESLint** юм.

**5.2 Formatters:** Форматтер гэдэг нь таны кодыг тодорхой загварын удирдамжийн (style guide) дагуу автоматаар хэлбэржүүлдэг хэрэгсэл юм. Нийтлэг форматтер хэрэгслийн жишээ бол **Prettier** юм.

### 6. Memory Management (Санах ойн удирдлага)

**6.1 Тодорхойлолт:** Санах ойн удирдлага гэдэг нь санах ойг хянах, шаардлагатай үед нь хуваарилах, ашиглахаа больсон үед нь чөлөөлөх үйл явц юм. JavaScript нь **автомат санах ойн удирдлагыг** ашигладаг. Энэ нь JavaScript (илүү тодорхой хэлбэл таны вэб хөтөч дээрх JavaScript engine) санах ойг хуваарилах болон чөлөөлөх ажлыг таны өмнөөс хариуцна гэсэн үг юм. Та кодондоо санах ойг заавал гараар чөлөөлөх шаардлагагүй. Энэхүү автомат процессыг ихэвчлэн **"garbage collection"** (хог цуглуулах) гэж нэрлэдэг.

### 7. Closures (Хумилт)

**7.1 Тодорхойлолт:** Closure гэдэг нь гадна талын функц нь ажиллаж дуусаад утгаа буцаасны дараа ч тухайн функцийн хүрээнд (outer lexical scope) байсан хувьсагчдад хандах эрхтэй хэвээр байдаг функц юм.

```javascript
function outerFunction(x) {
  let y = 10;
  function innerFunction() {
    console.log(x + y);
  }
  return innerFunction;
}

let closure = outerFunction(5);
closure(); // 15
```

### 8. var түлхүүр үг болон Hoisting

**8.1 Тодорхойлолт:** `var` нь 2015 оноос өмнө хувьсагч зарлах анхны арга байсан. Гэвч `var`-д scope (хүрээ), дахин зарлах болон бусадтай холбоотой зарим асуудлууд байсан. Тийм ч учраас орчин үеийн JavaScript програмчлалд үүний оронд `let` болон `const`-ыг ашигладаг.

**8.2 var ашиглан хувьсагчийг дахин зарлах:** Хэрэв та `let` ашиглан хувьсагчийг дахин зарлахыг оролдвол `SyntaxError` алдаа авна. Харин `var`-ын тусламжтайгаар хувьсагчийг дахин зарлахыг зөвшөөрдөг.

```javascript
// Алдаа заана: SyntaxError: Identifier 'num' has already been declared
let num = 19;
let num = 18;

var myNum = 5;
var myNum = 10; // Ижил нэрээр зарлахыг зөвшөөрнө, алдаа заахгүй
console.log(myNum); // 10
```

**8.3 var ба Scope (Хүрээ):** `if` нөхцөл эсвэл `for` давталт зэрэг блок дотор `var`-аар зарласан хувьсагч нь тухайн блокийн гадна хандагдсан хэвээр байдаг.

```javascript
if (true) {
  var num = 5;
}
console.log(num); // 5
```

**8.4 Hoisting (Өргөлт):** Энэ нь JavaScript код ажиллаж эхлэхээс өмнө, хөрвүүлэх (compilation) шатанд зарлалтуудыг өөрийнх нь хамаарах хүрээний хамгийн дээд хэсэгт шилжүүлдэг анхдагч зан төлөв юм. Та `var` түлхүүр үг ашиглан хувьсагч зарлах үед JavaScript зарлалтыг хүрээнийх нь дээд хэсэгт өргөдөг.

```javascript
console.log(num); // undefined
var num = 5;
console.log(num); // 5
```

Та "function declaration" синтакс ашиглан функц зарлах үед функцийн нэр болон функцийн бие хоёулаа өргөгддөг. Энэ нь та функцийг кодондоо зарлахаасаа өмнө дуудаж болно гэсэн үг юм.

```javascript
sayHello(); // "Hello, World!"

function sayHello() {
  console.log("Hello, World!");
}
```

`let` эсвэл `const`-оор хийсэн хувьсагчийн зарлалтууд өргөгддөг боловч тэдгээр нь утга аваагүй (not initialized) байдаг бөгөөд та кодондоо зарлахаас нь өмнө тэдэнд хандах боломжгүй. Энэ зан төлөвийг ихэвчлэн **"temporal dead zone"** гэж нэрлэдэг.

```javascript
console.log(num); // ReferenceError алдаа заана
let num = 10;
```

### 9. Import, Export болон Модультай ажиллах

**9.1 Module:** Энэ нь холбогдох функц, класс эсвэл хувьсагчдыг нэгтгэсэн бие даасан кодын нэгж юм. Модуль үүсгэхийн тулд та JavaScript кодоо тусдаа файлд бичнэ.

**9.2 Exports:** Аппликейшнийхээ бусад хэсэгт ашиглахыг хүссэн ямар ч хувьсагч, функц эсвэл классыг `export` түлхүүр үг ашиглан ил тод экспортолсон байх шаардлагатай. Экспортын хоёр төрөл байдаг: **named export** (нэрлэсэн экспорт) болон **default export** (анхдагч экспорт).

**9.3 Imports:** Экспортолсон зүйлсийг аппликейшнийхээ өөр хэсэгт ашиглахын тулд та тэдгээрийг `import` түлхүүр үг ашиглан импортлох хэрэгтэй. Импортын төрлүүд нь **named import**, **default import** болон **namespace import** байж болно.

```javascript
// math.js нэртэй файл дотор бид дараах функцуудыг экспортолж байна:

// Named export (Нэрлэсэн экспорт)
export function add(num1, num2) {
  return num1 + num2;
}

// Default export (Анхдагч экспорт)
export default function subtract(num1, num2) {
  return num1 - num2;
}
```

```javascript
// Өөр нэг файл дотор бид math.js-ээс функцуудыг импортолж болно.

// Named import - Энэ мөр add функцийг импортолж байна.
// Функцийн нэр нь math.js-ээс экспортолсон нэртэй яг таарах ёстой.
import { add } from "./math.js";

// Default import - Энэ мөр subtract функцийг импортолж байна.
// Функцийн нэр нь юу ч байж болно.
import subtractFunc from "./math.js";

// Namespace import - Энэ мөр тухайн файлаас бүх зүйлийг импортолж байна.
import * as Math from "./math.js";

console.log(add(5, 3)); // 8
console.log(subtractFunc(5, 3)); // 2
console.log(Math.add(5, 3)); // 8
console.log(Math.subtract(5, 3)); // 2
```
