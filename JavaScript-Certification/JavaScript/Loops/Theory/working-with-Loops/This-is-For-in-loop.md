 For...in давталт гэж юу вэ, түүнийг хэзээ ашиглах ёстой вэ?

**For...in** давталтыг объектын шинж чанаруудыг (properties) нэг бүрчлэн дамжиж давтахад ашиглах нь хамгийн тохиромжтой байдаг. Энэхүү давталт нь объектын бүх тоологдох (enumerable) шинж чанарууд, түүний дотор удамшсан (inherited) болон тоон бус шинж чанаруудыг дамждаг.

* **Inherited property (Удамшсан шинж чанар):** Энэ нь тухайн объектын прототайп хэлхээнээс (prototype chain) удамшиж ирсэн шинж чанар юм.
* **Non-numeric property (Тоон бус шинж чанар):** Энэ нь тоо биш, эсвэл тоо руу хөрвүүлж болохгүй тэмдэгт мөр бүхий шинж чанар юм.

`for...in` давталтын үндсэн бичиглэл (syntax) энд байна:

```javascript
for (variable in object) {
  // ажиллах кодын блок
}
```



Дээрх жишээний `variable` (хувьсагч) нь тухайн үед давтагдаж буй объектын "одоогийн шинж чанар буюу түлхүүрийг" (property/key) төлөөлнө.

`for...in` давталт хэрхэн ажилладагийг илүү сайн ойлгохын тулд хэдэн жишээ харцгаая.

Эхний жишээнд бид `fruit` (жимс) объекттой бөгөөд шинж чанар бүрээр нь дамжиж, утгыг нь консол дээр хэвлэхийг хүсэж байна:

```javascript
const fruit = {
  name: 'apple',
  color: 'red',
  price: 0.99
};

for (const prop in fruit) {
  console.log(fruit[prop]);
}
```

Энд `prop` хувьсагч нь объектын одоогийн шинж чанарыг (түлхүүрийг) төлөөлж байна. Шинж чанар бүрийн утгад хандахын тулд `fruit[prop]` (bracket notation) ашиглаж байна. Эхний итерацид `prop` нь `name`, хоёр дахь итерацид `color` гэх мэтээр үргэлжилнэ. Консол дээр "apple", "red", "0.99" гэсэн үр дүнгүүд хэвлэгдэнэ.

Хоёр дахь жишээнд бид үүрлэсэн (nested) объекттой бөгөөд шинж чанар бүрийн утгыг консол дээр хэвлэхийг хүсэж байна:

```javascript
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

for (const prop in person) {
  console.log(person[prop]);
}
```

`address` шинж чанар нь өөрөө объект юм. `for...in` давталт нь `person` объектын шинж чанаруудыг дамжихдаа `address` объектыг бүхэлд нь консол дээр хэвлэх болно. Консол дээрх үр дүн иймэрхүү харагдана:

```text
John
30
{ street: '123 Main St', city: 'Anytown', state: 'CA' }
```

Хэрэв та `address` объектын доторх шинж чанаруудыг мөн салгаж давтахыг хүсвэл эхний давталтын дотор өөр нэг `for...in` давталтыг үүрүүлж (nest) болно:

```javascript
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

// Утга нь объект мөн эсэхийг шалгах функц
function isObject(obj) {
  return typeof obj === 'object' && !Array.isArray(obj) && obj !== null;
}

for (const prop in person) {
  if (isObject(person[prop])) {
    // Хэрэв утга нь объект байвал доторхыг нь дахин давтана
    for (const nestedProp in person[prop]) {
      console.log(person[prop][nestedProp]);
    }
  } else {
    console.log(person[prop]);
  }
}
```



Энэ жишээнд бид утга нь объект мөн эсэхийг шалгадаг `isObject` гэсэн туслах функц ашигласан.
* `Array.isArray` аргыг ашиглан утга нь массив мөн эсэхийг шалгаж байна. Өмнө нь `!` (NOT) оператор тавьснаар "массив биш" эсэхийг шалгаж байгаа хэрэг юм.
* Зөвхөн `typeof === 'object'` гэж шалгаж болохгүй шалтгаан нь JavaScript-д **массив** мөн объект гэж тооцогддогтой холбоотой. Бид массивыг энэ шалгалтаас хасахыг хүссэн.
* Мөн JavaScript-ийн түүхэн алдаанаас (bug) болоод `typeof null` нь "object" гэж буцаадаг тул `null` утгыг мөн хасаж байна.

Хэрэв нөхцөл үнэн байвал (өөрөөр хэлбэл утга нь объект байвал) бид үүрлэсэн объектын шинж чанаруудыг давтах өөр нэг `for...in` давталтыг ажиллуулна. `nestedProp` хувьсагч нь үүрлэсэн объектын одоогийн шинж чанарыг төлөөлнө. Өөрчлөгдсөн үр дүн консол дээр ийм харагдана:
"John", 30, "123 Main St", "Anytown", "CA".

`for...in` давталт нь объектын шинж чанаруудаар дамжихад маш хэрэгтэй. Гэхдээ **массивын элементүүдийг давтахад `for...in` ашиглахыг зөвлөдөггүй.** Үүний оронд `for...of` давталт эсвэл массивын `forEach`, `map`, `filter`, `reduce` зэрэг аргуудыг ашиглаарай. Тэдгээрийг та ирээдүйн хичээлүүдээр үзэх болно.
