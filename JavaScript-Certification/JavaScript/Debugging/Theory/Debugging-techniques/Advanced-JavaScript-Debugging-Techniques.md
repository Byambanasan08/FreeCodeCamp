What Are Some Examples of Using Advanced JavaScript Debugging Techniques?

JavaScript-ийн ахисан түвшний debugging техникүүдийг ашиглах жишээнүүд юу вэ?

---

JavaScript програмуудаа debug хийх нь зөвхөн console.log() ашиглахаас илүү өргөн хүрээтэй ойлголт юм. Илүү ахисан техникүүд байдаг бөгөөд тэдгээрт дасвал debugging хийх нь маш амархан болдог. Эдгээр ахисан техникүүдийг авч үзье, ингэснээр та алдаа гарсан үед JavaScript программаа дуртайяа debug хийх болно.

Бидний эхлээд авч үзэх ойлголт бол breakpoints-тэй ажиллах юм. Breakpoint нь таны кодын тодорхой мөр дээр execution-ийг түр зогсоох боломжийг олгодог. Түр зогссоны дараа та хувьсагчдыг шалгах, expression-уудыг үнэлэх, мөн call stack-ийг ажиглах боломжтой.

Chrome browser дээр кодын дурын мөр дээр breakpoint нэмэхийн тулд developer tools-ийг нээж Sources tab руу орно, debug хийх JavaScript файлаа нээж, тухайн мөрийн дугаарт дарж breakpoint тохируулна.

Execution breakpoint дээр хүрч зогссоны дараа баруун дээд буланд байрлах icon-уудыг ашиглан кодыг алхам алхмаар гүйцэтгэж болно. Мөн та тухайн мөр дээр right-click хийж "Add conditional breakpoint…" сонгосноор нөхцөлтэй breakpoint нэмж болно.

Одоо watch expressions руу шилжье. Watch expressions нь код ажиллаж байх явцад, тухайн хувьсагч эсвэл expression одоогийн scope-д байхгүй байсан ч утгыг нь хянах боломжийг олгодог.

Watch expression нэмэхийн тулд developer tools-ийн Sources tab руу орж, баруун талд байрлах watch panel-ийг олж, plus (+) icon дээр дарж нэмнэ.

Дараагийн авч үзэх ойлголт бол profiling юм. Profiling нь performance-ийн саад (bottleneck)-ыг илрүүлэхэд тусалдаг бөгөөд CPU ашиглалт, функцийн дуудалт, execution хугацаа зэргийг бичиж авч, screenshot хүртэл авах боломжтой.

Үүнийг хийхийн тулд Performance tab-ийг нээж, record дарж, profiling хийхийг хүссэн үйлдлээ гүйцэтгэнэ. Дараа нь бичлэгийг зогсоож үр дүнг шинжилнэ.

Хэрэв таны аппликейшн тодорхой үйлдлүүд дээр гацаж байвал энэ нь их хэрэгтэй. Profiler ашигласнаар аль функц эсвэл кодын аль хэсэг хамгийн их нөөц ашиглаж байгааг харж болно.

Одоо network request-уудыг шалгах талаар авч үзье. Network request шалгах нь API request-тай холбоотой алдаануудыг (жишээ нь parameter алдаа, хаягийн алдаа, эсвэл server-ийн алдаа) debug хийхэд тусалдаг.

Network tab ашиглахын тулд developer tools-ийг нээж Network tab руу орно, дараа нь request бүр дээр дарж header, response, payload зэрэг дэлгэрэнгүй мэдээллийг харж болно.

Энэ хичээлийн сүүлчийн хэсэгт console.table болон console.dir-ийг авч үзнэ.

console.table() нь хүснэгтэн өгөгдлийг console дээр хүснэгт хэлбэрээр харуулдаг. Энэ нь нэг заавал шаардлагатай аргумент авна (энэ нь array эсвэл object байх ёстой), мөн харуулах property эсвэл багануудыг заах optional аргумент авч болно.

console.dir() нь тодорхой JavaScript object-ийн property-уудыг интерактив жагсаалт хэлбэрээр харуулах боломж олгодог. Энэ нь nested property-уудыг хүртэл дэлгэж харах боломжтой hierarchical жагсаалт үүсгэнэ.

Одоо та debugging техникүүдийн өргөтгөсөн хэрэгслийн багцтай боллоо. Эдгээр шинэ чадваруудаа удахгүй хийх JavaScript төслүүд дээрээ туршиж үзээрэй. Эдгээр стратегийг ашигласнаар та асуудлуудыг илүү хурдан олж, засах боломжтой болж, илүү цэвэр, найдвартай код бичих болно.
