What Is the Fetch API, and What Are Common Types of Resources That Are Fetched from the Network?

Fetch API нь вэб аппликейшнуудыг сүлжээний хүсэлт (network requests) хийх боломжтой болгодог бөгөөд ихэвчлэн серверээс өгөгдөл татах эсвэл сервер рүү өгөгдөл илгээхэд ашиглагддаг. Энэ API нь fetch() нэртэй method-ийг өгдөг бөгөөд та үүнийг ашиглан эдгээр хүсэлтүүдийг хийж чадна. Fetch-ийг ашиглах энгийн жишээг харъя:

```javascript id="l6q8ch"
fetch('https://api.example.com/data')
```

Энэ жишээнд бид [https://api.example.com/data](https://api.example.com/data) руу GET хүсэлт (GET request) хийж байна. Үүний үр дүнд бидэнд өгөгдөл буцаж ирэх бөгөөд үүнийг JSON формат руу хөрвүүлээд (convert) хаана ч ашиглаж болно.

Анхдагч байдлаар (default), Fetch API нь өгөгдөл татахдаа GET method ашигладаг. Энэ болон бусад түгээмэл HTTP request method-уудыг дараагийн хичээлд авч үзнэ.

Одоо сүлжээнээс татагддаг (fetched) зарим түгээмэл төрлийн нөөцүүд (resources)-ийн талаар ярилцъя.

Манай вэб аппликейшнуудад цаг агаарын мэдээлэл (weather data), мэргэжлийн жагсаалт (professions list), улсын нэрсийн жагсаалт (country names list), улсын код (country code) эсвэл улсын туг (country flag) зэрэг нийтлэг өгөгдлүүд хэрэгтэй байдаг. Эдгээр өгөгдлүүдийг ашигласнаар бид аппликейшнаа илүү мэдээлэлтэй, интерактив болгож чадна. Fetch API-ийн ачаар бид эдгээр нөөцүүдийг сүлжээнээс авч чадна.

Зураг (images) нь хамгийн их татагддаг нөөцүүдийн нэг юм. Та fetch-ийг ашиглан зургуудыг статик эсвэл хэрэглэгчийн үйлдлээс хамааран динамикаар ачаалж, вэб аппликейшндаа харуулах боломжтой.

Текст файлууд (text files) нь мөн ихэвчлэн татагддаг нөөцүүдийн нэг төрөл юм. Үүнд тохиргооны файлууд (configuration files), лог файлууд (log files), эсвэл бүр бүхэл баримт бичгүүд (documents) багтаж болно, эдгээрийг та вэб хуудсандаа харуулахыг хүсэж байж болно.

Зарим тохиолдолд та аудио эсвэл видео файлуудыг татаж авч болно. Fetch API нь эдгээр төрлийн нөөцүүдийг мөн дэмждэг тул та олон төрлийн өгөгдөлтэй ажиллах боломжтой.
