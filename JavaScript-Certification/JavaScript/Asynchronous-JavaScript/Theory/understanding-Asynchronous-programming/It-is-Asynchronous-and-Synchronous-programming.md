# What Is Asynchronous JavaScript, and How Does It Differ from Synchronous JavaScript?

Asynchronous JavaScript-ийн талаар суралцъя. Asynchronous үйл явдлууд нь өөр өөр цаг хугацаанд, бие даан явагддаг.

Software development-ийн хүрээнд "asynchronous" гэдэг нь програмын үндсэн урсгалаас (main flow) хамааралгүйгээр, ард (background) ажилладаг даалгавруудыг хэлдэг. Asynchronous процессуудын гол давуу тал нь тэд үндсэн програмын гүйцэтгэлийг (execution) блоклодоггүй.

Энэ нь удаан хугацаа шаардаж болох даалгавруудад маш хэрэгтэй, жишээ нь алсын серверээс өгөгдөл татах (fetching data from a remote server), том файлууд боловсруулах (processing large files), хэрэглэгчийн оролтыг боловсруулах (handling user input), болон төвөгтэй тооцоолол хийх (performing complex calculations). Үүнийг бид asynchronous programming гэж нэрлэдэг.

Энэ арга нь таны өмнө нь ашиглаж байсан synchronous programming техниктэй эсрэгцдэг. Та олон мөр код бичих үед юу болох, хэзээ болохыг ихэвчлэн урьдчилан таамаглаж чаддаг. Эхний мөр ажиллана, дараа нь хоёр дахь мөр, тэгээд бичсэн дарааллаараа үргэлжилнэ.

Дараах жишээнд хувьсагч тодорхойлсон эхний мөр эхэлж ажиллана, дараа нь хоёр дахь мөр, эцэст нь гурав дахь мөр болох console дээр string хэвлэх мөр ажиллана:

```javascript
const topic = "JavaScript";
const learning = `I'm learning ${topic}!`;
console.log(learning);
```

Бүх зүйл дараалсан (sequential) бөгөөд урьдчилан таамаглах боломжтой. Дараагийн мөр эхлэхээс өмнө өмнөх мөр бүрэн дуусдаг. Ийм төрлийн JavaScript програмыг single-threaded гэж нэрлэдэг.

Thread ойлголт маш чухал. Thread гэдэг нь процесс доторх гүйцэтгэлийн нэгж (unit of execution) юм. Энэ нь програм доторх тусдаа хяналтын урсгал (flow of control) гэж ойлгож болно.

Synchronous programming-д thread-ууд дараалсан байдлаар, нэгнийхээ араас нэг нь ажилладаг. Хэрэв thread блоклогдвол (жишээ нь хэрэглэгчийн оролтыг хүлээж байвал), тэр thread дуусах хүртэл бүх процесс блоклогдоно.

Asynchronous programming-д thread-ууд зэрэг (concurrently) ажиллаж чаддаг, өөрөөр хэлбэл олон thread-ийг нэгэн зэрэг ажиллуулах боломжтой. Ингэснээр нэг thread блоклогдсон ч програм үндсэндээ ажилласаар байж, хэрэглэгчид хариу өгөх чадвар (responsiveness)-аа алдахгүй.

Asynchronous programming нь ихэвчлэн callbacks, promises, эсвэл async/await ашиглан non-blocking үйлдлүүдийг (operations) зохицуулдаг.

Өмнөх хичээлүүдэд үзсэнээр callback гэдэг нь дараа нь дуудагдах function-ийг өөр function-д дамжуулахыг хэлдэг. Event handlers нь callback-ийн нэг төрөл бөгөөд та өмнө нь ашиглаж байсан. Өмнө нь JavaScript-д asynchronous function хэрэгжүүлэх хамгийн түгээмэл арга байсан. Гэвч хэрэв callback function дотор дахин callback function ашиглавал код болон логик маш хурдан төвөгтэй болдог.

Одоогийн байдлаар JavaScript-д asynchronous programming хэрэгжүүлэх хамгийн түгээмэл арга нь promise юм. Promise гэдэг нь asynchronous процессын ирээдүйн дуусалт (completion) эсвэл алдаа (failure), мөн түүний утгыг илэрхийлдэг объект юм.

Promise үүсэх үед түүний утга тодорхой байдаггүй. Энэ нь зөвхөн asynchronous процесс дууссаны дараа тодорхой болдог. Жишээ нь таны вэб аппликейшнд алсын API-гаас өгөгдөл татах процесс хэсэг хугацаа шаардаж болно. Энэ хооронд та хэрэглэгчдэд сайн хэрэглэгчийн туршлага (user experience) өгөхийг хүснэ, тийм үү?

Үүнийг хэрэгжүүлэхийн тулд та promise үүсгэж, asynchronous процесс ажиллаж байх хугацаанд user interface-ийг идэвхтэй (active) болон интерактив байлгаж болно. Процесс дууссаны дараа promise нь API-гаас буцаж ирсэн өгөгдлийг агуулна, ингэснээр аппликейшн тухайн өгөгдлийг ашиглах эсвэл зөв харуулах (render) боломжтой болно.

Asynchronous programming нь үр ашигтай JavaScript аппликейшн бүтээхэд маш хүчтэй хэрэгсэл юм. Synchronous болон asynchronous programming-ийн ялгааг ойлгосноор та өөрийн аппликейшнд тохирсон зөв хандлагыг сонгож, илүү үр ашигтай код бичих боломжтой болно.
