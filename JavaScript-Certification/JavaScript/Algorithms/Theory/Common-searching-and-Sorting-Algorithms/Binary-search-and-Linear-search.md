# Binary Search гэж юу вэ, мөн энэ нь Linear Search-ээс юугаараа ялгаатай вэ?

Жагсаалт (list)-аас элемент хайх нь computer science-д маш түгээмэл тохиолддог зүйл юм. Хайлт хийхтэй холбоотой хоёр чухал алгоритм байдаг: linear search болон binary search.

Linear search нь жагсаалтын эхнээс эхэлж, хайж буй зорилтот утга (target value)-ыг олох хүртэл элемент бүрийг дарааллаар шалгадаг.

Хэрэв зорилтот утга олдвол, жагсаалт дахь тухайн index-ийг буцаана. Хэрэв зорилтот утга олдохгүй бол -1 буцаана. -1-ийг буцаадаг шалтгаан нь ихэнх програмчлалын хэлэнд энэ нь хүчинтэй (valid) index биш юм.

Linear search-ийн код дараах байдалтай:

```js id="a1x9kp"
function linearSearch(arr, target) {
  for (let i = 0; i < arr.length; i++) {
    if (arr[i] === target) {
      return i;
    }
  }
  return -1;
}
```

Хэрэв хайх жагсаалт [13, 4, 7, 9, 10] бөгөөд зорилтот утга 9 байвал, функц 3-ыг буцаана, учир нь 9 нь index 3 дээр байна.

Хэрэв зорилтот утгыг 5 болгож өөрчилбөл, функц -1 буцаана, учир нь 5 жагсаалтанд байхгүй.

Энэ нь харьцангуй энгийн алгоритм боловч хамгийн үр ашигтай нь биш. Хэрэв танд том хэмжээний жагсаалт байвал, linear search нь зорилтот утгыг олоход удаан хугацаа шаардаж болно.

Linear search-ийн хугацааны төвөгшил (time complexity) нь O(n), учир нь жагсаалтыг хайх хугацаа нь жагсаалтын хэмжээтэй шугаман (linearly) өсдөг.

Linear search-ийн санах ойн төвөгшил (space complexity) нь O(1), учир нь хайлт хийхэд нэмэлт санах ой шаарддаггүй.

Binary search нь том хэмжээний жагсаалтаас хайлт хийхэд илүү үр ашигтай алгоритм юм. Гэхдээ энд нэг нөхцөл бий: жагсаалт нь өсөх дарааллаар (ascending order) эрэмбэлэгдсэн байх ёстой.

Binary search нь жагсаалтыг хоёр хувааж, зорилтот утга дунд хэсэгт (middle) байгаа эсэхийг шалгах замаар ажилладаг. Хэрэв зорилтот утга дунд байвал, тухайн index-ийг буцаана. Үгүй бол алгоритм зорилтот утга жагсаалтын зүүн (left) эсвэл баруун (right) талд байгаа эсэхийг шалгана.

Дараа нь үлдсэн хэсгийг дахин хоёр хуваасаар, зорилтот утгыг олох хүртэл үргэлжлүүлнэ. Хэрэв зорилтот утга жагсаалтанд байхгүй бол -1 буцаана.

Binary search-ийн код дараах байдалтай:

```js id="q4n7vz"
function binarySearch(arr, target) {
  let low = 0;
  let high = arr.length - 1;

  while (low <= high) {
    const mid = Math.floor((low + high) / 2);

    if (arr[mid] === target) {
      return mid;
    } else if (arr[mid] < target) {
      low = mid + 1;
    } else {
      high = mid - 1;
    }
  }

  return -1;
}
```

Бид эхлээд low болон high index-үүдийг тодорхойлдог. Энэ нь хайлт хийх жагсаалтын хүрээг илэрхийлдэг.

Дараа нь low нь high-ээс бага эсвэл тэнцүү эсэхийг шалгана. Хэрэв low нь high-ээс их бол бид бүх жагсаалтыг шалгасан гэсэн үг бөгөөд зорилтот утга олдоогүй байна. Энэ тохиолдолд хайлтыг зогсоож -1 буцаана.

Хэрэв low нь high-ээс бага эсвэл тэнцүү байвал бид жагсаалтын дундах index болох mid-ийг тооцоолдог. Дараа нь зорилтот утга дунд index дээр байгаа эсэхийг шалгана. Хэрэв байвал mid-ийг буцаана.

Үгүй бол, дундах утга зорилтот утгаас бага эсэхийг шалгана. Хэрэв тийм бол low index-ийг mid + 1 болгож шинэчилнэ. Энэ нь жагсаалтын баруун хагасыг хайна гэсэн үг.

Эцэст нь, дээрх нөхцлүүдийн аль нь ч үнэн биш бол high index-ийг mid - 1 болгож шинэчилнэ. Энэ нь жагсаалтын зүүн хагасыг хайна гэсэн үг.

Бид энэ процессыг зорилтот утгыг олох эсвэл жагсаалтанд байхгүйг тогтоох хүртэл давтана.

Binary search-ийн хугацааны төвөгшил (time complexity) нь O(log n), учир нь жагсаалтыг хайх хугацаа нь жагсаалтын хэмжээтэй логарифм (logarithmically) байдлаар өсдөг.

Binary search-ийн санах ойн төвөгшил (space complexity) нь O(1), учир нь хайлт хийхэд нэмэлт санах ой шаарддаггүй.

Binary search болон linear search нь computer science-д тулгарах олон төрлийн асуудлуудад ашиглагдаж болно. Эдгээр хоёр алгоритмын ялгааг ойлгож, аль үед аль нь тохиромжтойг мэдэх нь маш чухал.
