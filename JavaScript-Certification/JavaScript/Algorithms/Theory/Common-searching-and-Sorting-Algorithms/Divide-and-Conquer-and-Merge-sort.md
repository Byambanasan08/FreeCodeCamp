## Divide and Conquer гэж юу вэ, мөн Merge Sort хэрхэн ажилладаг вэ?

Computer science-д divide and conquer paradigm нь асуудлыг рекурсив (recursively) байдлаар жижиг дэд асуудлууд (sub-problems) болгон задлах техник юм. Энэ техникийн гол шинжүүдийн нэг нь recursion бөгөөд функц өөрийгөө дахин дахин дуудаж, base case-д хүрэх хүртэл үргэлжилдэг. Энэ хичээлд бид merge sort алгоритмыг ашиглан divide and conquer техник хэрхэн ажилладгийг илүү сайн ойлгоно.

Дараах тоонуудын жагсаалт байна гэж төсөөлье:

42 37 53 17

Зорилго нь merge sort алгоритмыг ашиглан энэ жагсаалтыг багаас их рүү (smallest to largest) эрэмбэлэх явдал юм. Эхний алхам бол жагсаалтыг хоёр хуваах:

42 37 | 53 17

Дараа нь жагсаалтын зүүн талыг авч үзье:

42 37

Бид энэ дэд жагсаалтыг дахин хоёр хуваасаар, эцэст нь дэд жагсаалт бүр зөвхөн нэг элементтэй болох хүртэл үргэлжлүүлнэ:

42 | 37

Зөвхөн нэг элементтэй жагсаалт нь анхнаасаа эрэмбэлэгдсэн (sorted) байдаг. Дараа нь бид эдгээр нэг элементтэй дэд жагсаалтуудыг нийлүүлж (merge), эрэмбэлэгдсэн жагсаалт үүсгэнэ:

37 42

Дараа нь бид анхны жагсаалтын баруун талд ижил процессыг давтана:

// анхны жагсаалтын баруун тал
53 17

// жагсаалтыг хоёр хуваах
53 | 17

// жагсаалтыг эрэмбэлж нийлүүлэх
17 53

Одоо анхны жагсаалтын хоёр тал хоёулаа эрэмбэлэгдсэн тул бид эдгээрийг нийлүүлж, бүх элементүүдийг эрэмбэлнэ:

17 37 42 53

Алгоритмын код дараах байдалтай:

```js id="m8x4pt"
function mergeSort(arr) {
  if (arr.length <= 1) {
    return arr;
  }

  const mid = Math.floor(arr.length / 2);
  const left = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));

  const sorted = [];
  let i = 0;
  let j = 0;

  while (i < left.length && j < right.length) {
    if (left[i] <= right[j]) {
      sorted.push(left[i]);
      i += 1;
    } else {
      sorted.push(right[j]);
      j += 1;
    }
  }

  return sorted.concat(left.slice(i)).concat(right.slice(j));
}
```

Merge sort-ийн хугацааны төвөгшил (time complexity) нь O(n log n), учир нь жагсаалтыг байнга хоёр хуваадаг (log n) бөгөөд дараа нь нийлүүлдэг (O(n)). Bubble sort зэрэг бусад алгоритмуудаас ялгаатай нь merge sort нь in-place байдлаар эрэмбэлдэггүй бөгөөд санах ойн төвөгшил (space complexity) нь O(n) байдаг.
