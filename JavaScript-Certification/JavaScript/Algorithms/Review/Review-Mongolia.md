# Searching and Sorting Algorithms тойм (Review)

1. Searching Algorithms

Searching алгоритмууд нь тодорхой жагсаалт (list)-аас зорилтот утгыг (target) хайх боломжийг олгодог.

Computer science-д та мэдэх ёстой хоёр төрлийн хайлтын алгоритм байдаг. Тэдгээр нь linear search болон binary search алгоритмууд юм. Эдгээр хоёр алгоритмын ялгаа болон аль нөхцөлд аль нь тохиромжтойг ойлгох нь чухал.

2. Linear Search

   Linear search нь массив (array)-ын элемент бүрийг эхнээс нь эхлэн, зорилтот элемент олдох хүртэл дарааллаар шалгадаг.
   Хэрэв зорилтот элемент олдвол, массив дахь тухайн index-ийг буцаана.
   Хэрэв зорилтот элемент олдохгүй бол -1 буцаана, энэ нь JavaScript-д хүчинтэй биш index гэсэн үг.
   Linear search нь зорилтот утгыг олох хүртэл элемент бүрийг шалгадаг тул том хэмжээний массив дээр үр ашиггүй.
   Linear search-ийн хугацааны төвөгшил (time complexity) нь O(n), учир нь хайлтын хугацаа нь массивын хэмжээтэй шугаман (linearly) өсдөг.
   Linear search-ийн санах ойн төвөгшил (space complexity) нь O(1), учир нь хайлт хийхэд нэмэлт санах ой шаарддаггүй.

3. Binary Search

   Binary search нь массивыг хоёр хувааж, зорилтот утга массивын дунд байгаа эсэхийг шалгах замаар ажилладаг.
   Binary search ажиллах нөхцөл нь массивын элементүүд өсөх (ascending) эсвэл буурах (descending) дарааллаар эрэмбэлэгдсэн байх ёстой.
   Binary search нь том хэмжээний массив дээр хайлт хийхэд илүү үр ашигтай алгоритм, учир нь массивыг хоёр хувааж, зорилтот утга байхгүй талыг бүрэн үл тоомсорлодог.
   Хэрэв зорилтот элемент массивын дунд олдвол, тухайн index-ийг буцаана.
   Хэрэв элемент олдохгүй бол алгоритм зорилтот элемент зүүн эсвэл баруун талд байгаа эсэхийг шалгана.
   Дараа нь массивын үлдсэн хэсгийг дахин хоёр хуваасаар, зорилтот элемент олдох хүртэл үргэлжлүүлнэ.
   Хэрэв зорилтот элемент массив дотор байхгүй бол -1 буцаана.
   Binary search-ийн хугацааны төвөгшил (time complexity) нь O(log n), учир нь хайлтын хугацаа массивын хэмжээтэй логарифм (logarithmically) байдлаар өсдөг.
   Binary search-ийн санах ойн төвөгшил (space complexity) нь O(1), учир нь хайлт хийхэд нэмэлт санах ой шаарддаггүй.

4. Linear Search болон Binary Search-ийн ялгаа

   Binary search нь linear search-ээс илүү том хэмжээний массив дээр тохиромжтой.
   Linear search-ийн хугацааны төвөгшил (time complexity) нь O(n), учир нь хайлтын хугацаа массивын хэмжээтэй шугаман өсдөг.
   Binary search-ийн хугацааны төвөгшил (time complexity) нь O(log n), учир нь хайлтын хугацаа массивын хэмжээтэй логарифм байдлаар өсдөг.

5. Sorting Algorithms болон Divide-and-Conquer

Computer science-д divide-and-conquer нь асуудлыг жижиг дэд асуудлууд (sub-problems) болгон задлаж, илүү хялбар шийдэх техник юм. Recursion нь divide-and-conquer-д ихэвчлэн ашиглагддаг техник бөгөөд divide-and-conquer нь merge sort зэрэг олон үр ашигтай sorting алгоритмуудыг хэрэгжүүлэх хүчирхэг стратеги юм.

6. Merge Sort

   Merge sort нь divide-and-conquer аргыг дагадаг sorting алгоритм юм.
   Энэ нь массивыг рекурсив (recursively) байдлаар жижиг дэд массивууд болгон хувааж, эцэст нь дэд массив бүр зөвхөн нэг элементтэй болох хүртэл үргэлжилдэг.
   Дараа нь эдгээр дэд массивуудыг дахин нийлүүлж (merge), эрэмбэлсэн дарааллаар нэгтгэдэг.
   Merge sort-ийн хугацааны төвөгшил (time complexity) нь O(n log n), учир нь массивыг байнга хоёр хуваадаг (log n) бөгөөд дараа нь нийлүүлдэг (O(n)).
   Merge sort-ийн санах ойн төвөгшил (space complexity) нь O(n), учир нь энэ нь in-place sorting алгоритм биш.
