DOM зангилаанууд (Nodes) DOM модонд бие биенийхээ хувьд хэрхэн оршдог вэ?

DOM-ийн зангилаанууд болон тэдгээрийн DOM мод (tree) доторх харилцан хамаарлын талаар суралцацгаая.

Яг л бодит мод том, жижиг мөчрүүдээс бүрдсэн шаталсан бүтэцтэй байдаг шиг, DOM-ийн зангилаанууд ч мөн бие биетэйгээ шууд болон шууд бус харилцан хамааралтай байдаг. Эдгээр харилцааг тайлбарлахын тулд бид дараах жишээг ашиглах болно:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>DOM Tree Example</title>
  </head>
  <body>
    <h1>Heading 1</h1>
    <p>Paragraph 1</p>
    <ul>
      <li>List item 1</li>
      <li>List item 2</li>
    </ul>
  </body>
</html>
```



Хамгийн дээрээс нь эхэлцгээе. DOM модны язгуур (root) нь `html` элемент юм. Энэ нь HTML баримт бичгийн бүх контентыг агуулдаг хамгийн дээд түвшний сав (container) юм. Бусад бүх зангилаанууд нь энэхүү язгуур зангилааны үр удам (descendants) юм.

Язгуур зангилааны доор бид шатлал дахь бусад зангилаануудыг олж харна. **Эцэг зангилаа** (parent node) гэдэг нь бусад элементүүдийг дотроо агуулж байгаа элементийг хэлнэ. **Хүүхэд зангилаа** (child node) гэдэг нь өөр нэг элемент дотор агуулагдаж байгаа элементийг хэлнэ.

Энэ жишээнд `body` элемент нь параграф (`p`) элементийн эцэг болж байгаа бол параграф элемент нь `body`-ийн хүүхэд юм. Эдгээр элементүүд нь DOM модонд яг ийм харилцаатай зангилаанууд хэлбэрээр дүрслэгддэг.



Бидэнд эцэг болон хүүхэд зангилаа байдгийн адилаар **ах дүү зангилаанууд** (sibling nodes) бас бий. Ах дүү зангилаанууд гэдэг нь нэг ижил эцэгтэй элементүүдийг хэлнэ. Бидний жишээн дээр жагсаалтын хоёр элемент (`li`) хоёулаа нэг ижил эцэг болох эрэмбэгүй жагсаалт (`ul`) дотор байгаа тул тэд ах дүүс юм.

Үүнтэй адилаар `h1` болон параграф (`p`) элементүүд нь хоёулаа `body` гэсэн нэг эцэгтэй тул ах дүүс болно.

Мөн шатлалын өөр өөр түвшний хоорондох шууд бус харилцаанууд байдаг. **Үр удам зангилаанууд** (descendant nodes) гэдэг нь өөр нэг элемент дотор шууд болон шууд бус байдлаар агуулагдаж буй элементүүд юм. Хэрэв нэг элементээс DOM модоор доошоо явсаар нөгөө элементэд хүрч болохоор байвал түүнийг тухайн "өвөг"-ийн үр удам гэж үзнэ.

Бидний жишээн дээр жагсаалтын `li` элементүүд нь `body` элемент дотор шууд бус байдлаар агуулагдаж байгаа тул `body`-ийн үр удам юм.

**Өвөг зангилаа** (ancestor node) гэдэг нь DOM модны шатлалд өөр нэг элементээс илүү дээгүүр байрлаж буй элементийг хэлнэ. Манай жишээнд `body` элемент нь жагсаалтын `li` элементүүдийн өвөг юм.

JavaScript ашиглан DOM модыг удирдах болон дотор нь шилжилт хийхэд эдгээр харилцан хамаарлыг ойлгох нь нэн чухал юм.

**Эцэг, хүүхэд, ах дүү гэсэн энэ "гэр бүлийн мод" шиг логик ойлгомжтой байна уу? DOM дээр ажиллах үед бид ихэвчлэн "энэ элементийн ахыг нь ол" эсвэл "энэ элементийн хүүхдүүдийг нь ав" гэх мэтээр харилцааг нь ашиглаж код бичдэг юм шүү.**
