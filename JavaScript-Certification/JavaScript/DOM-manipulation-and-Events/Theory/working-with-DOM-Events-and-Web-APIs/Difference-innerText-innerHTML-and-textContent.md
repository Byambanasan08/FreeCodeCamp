innerText, textContent, болон innerHTML-ийн ялгаа юу вэ?

`innerText`, `textContent`, болон `innerHTML`-ийн талаар суралцацгаая. Эдгээр нь HTML элементийн агуулгыг авах эсвэл өөрчлөхийн тулд JavaScript-д хандаж болох шинж чанарууд (properties) юм. Хэдийгээр тэдгээр нь эхний харахад маш төстэй мэт санагдавч, үндсэн гол ялгаануудтай. Зөвийг нь сонгох нь таны тусгай хэрэгцээнээс хамаарах тул дэлгэрүүлэн үзэцгээе.

#### 1. innerText

Эхлээд `innerText`-ээс эхэлье.
`innerText` нь HTML элемент болон түүний үр удам (descendants) зангилаануудын **харагдах (visible)** текст агуулгыг илэрхийлдэг. Энэхүү шинж чанар нь нуугдсан текст эсвэл HTML таг-уудыг агуулдаггүй, зөвхөн хөтөч дээр дүрслэгдсэн (rendered) текстийг л авдаг.



Жишээлбэл, энд хоёр параграф агуулсан `div` элемент байна:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Хэрэв бид JavaScript кодондоо `getElementById()` ашиглан энэ HTML элементийн холбоосыг авбал, бид энэ элементийн `innerText` шинж чанарт хандаж болно:

```javascript
const container = document.getElementById("container");
console.log(container.innerText);
```

Энэ элементийн доторх текст (inner text) нь:
```text
Hello, World!
I'm learning JavaScript
```

Энэ шинж чанар нь элемент дотор агуулагдаж буй текстийг (түүний үр удам зангилаануудын текстийг оролцуулан) тэмдэгт мөр (string) болгон буцаадаг.

Таны мэдэх ёстой нэг зүйл бол `innerText` нь зөвхөн тухайн текстийг хүсэж буй **яг тэр мөчид харагдаж байгаа** текстийг л буцаадаг. Хэрэв хүүхэд элемент нь нуугдсан байвал түүний текст харагдахгүй (буцаагдахгүй).

Хоёр дахь параграф нь нуугдсан (hidden) байгаа жишээг харна уу:

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

Хэрэв бид дахин `innerText`-ийг консол дээр хэвлэхийг оролдвол, одоо гаралт дээр хоёр дахь параграфын текст харагдахгүй:

```javascript
const container = document.getElementById("container");
console.log(container.innerText); // Гаралт: Hello, World!
```

Та HTML элементийн `innerText`-ийг дараах байдлаар тохируулж болно, гэхдээ энэ нь одоо байгаа бүх текстийг сольж, мөр шилжилт (line break) болгонд `br` элементийг нэмэх болно:

```javascript
const container = document.getElementById("container");
container.innerText = "JavaScript is awesome!";
```

`innerText` нь харагдах байдлыг (visibility) тооцдог тул түүний утгыг авах үед вэбсайт дээрх зарим элементүүдийн байршлыг дахин тооцоолдог **"reflow"** гэж нэрлэгддэг процессыг идэвхжүүлдэг. Энэ процесс нь тооцооллын хувьд их нөөц шаардаж болох тул боломжтой бол үүнийг өдөөхөөс зайлсхийх хэрэгтэй.

---

#### 2. textContent

Одоо `textContent`-ийн талаар ярилцъя.
`textContent` нь тухайн элемент болон түүний үр удам доторх бүх текстийг **энгийн текст (plain text)** хэлбэрээр буцаадаг.

`innerText` болон `textContent`-ийн хамгийн чухал ялгаа нь: `textContent` нь тухайн HTML элементийн текст харагдаж байгаа эсвэл нуугдсан байгаагаас үл хамааран **үргэлж бүтэн текст агуулгыг** буцаадаг.



HTML дээрх ижил жишээг авч үзье:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Хэрэв та энэ шинж чанарт хандахыг оролдвол, гаралт дээр тухайн элемент болон түүний үр удам зангилаануудын текстийг, догол мөр болон сул зайг (indentation and spacing) хэвээр нь харах болно:

```javascript
const container = document.getElementById("container");
console.log(container.textContent);
```

Хэрэв HTML элемент харагдахгүй байсан ч (жишээ нь хоёр дахь параграф нуугдсан үед), түүний текст энэ шинж чанарт багтсан хэвээр байх болно:

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

```javascript
const container = document.getElementById("container");
console.log(container.textContent); // Нуугдсан текст багтсан хэвээр байна.
```

`textContent` нь мөн `script` болон `style` элементүүдийн агуулгыг багтаадаг.
Хэрэв та ямар нэгэн зангилааны `textContent`-ийн утгыг солихыг оролдвол, энэ нь түүний бүх хүүхэд зангилаануудыг устгаж, тэдгээрийг шинэ тэмдэгт мөр бүхий нэг текст зангилаагаар солих болно:

```javascript
const container = document.getElementById("container");
container.textContent = "Шинэ агуулга";
```

---

#### 3. innerHTML-ээс ялгагдах ялгаа

Эцэст нь, `textContent` болон `innerText` нь `innerHTML`-ээс хэрхэн ялгаатай талаар ярилцъя.
`innerHTML` ашиглан та элементийн **дотоод HTML агуулгыг** (таг-уудыг оролцуулан) тохируулж болдог гэдгийг санаарай. Энэ нь DOM руу динамикаар шинэ HTML оруулахад тустай.

Гэсэн хэдий ч, хэрэв танд тухайн тэмдэгт мөр дээр бүрэн хяналт байхгүй бол (жишээ нь хэрэглэгчийн оруулсан өгөгдөл бүхий string) энэ нь **аюулгүй байдлын эрсдэл (security risk)** үүсгэдэг гэдгийг санаарай. Хэрэв тэр өгөгдөл хортой код агуулж байвал аюулгүй байдлын ноцтой асуудалд хүргэж болзошгүй.



Үүнээс зайлсхийхийн тулд зөвхөн энгийн текст оруулах тохиолдолд `textContent` шинж чанарыг ашиглахыг зөвлөдөг.

### Дүгнэлт

JavaScript дахь `innerText`, `textContent`, болон `innerHTML` шинж чанарууд нь HTML элементийн агуулгад хандах болон удирдах өөр өөр арга замуудыг олгодог. Хэрэв таны зорилго JavaScript дээр HTML агуулгатай үр ашигтай ажиллах бол эдгээр шинж чанаруудын ялгааг ойлгох нь чухал юм.
