### Вэб аппликейшнийг keyboard-оор ашиглах боломжтой болгох ямар аргууд байдаг вэ?

Олон хэрэглэгч mouse-ийн оронд **keyboard** ашигладаг. Үүнд:

* биеийн хөгжлийн бэрхшээлтэй хүмүүс
* давтагдсан хөдөлгөөнөөс үүссэн гэмтэл (repetitive strain injury)-тэй хүмүүс
* эсвэл хувийн сонголтоороо keyboard ашигладаг хүмүүс
  орно.

Мөн screen reader ашигладаг хэрэглэгчид болон mouse нь ажиллахгүй байгаа хэрэглэгчид ч энд хамаарна. Keyboard accessibility нь эдгээр хэрэглэгчид вэб аппликейшнийг **саадгүй, үр дүнтэй ашиглах** боломжийг хангадаг.

Одоо вэб аппликейшнийг keyboard-д ээлтэй болгох **практик аргуудыг** авч үзье.

---

Олон хэрэглэгч вэб хуудсан дээрх интерактив элементүүдээр шилжихдээ **Tab товч**-ийг ашигладаг. Анхдагчаар браузерууд:

* линк
* товч
* формын талбарууд
  зэрэг элементүүдээр **HTML дээрх дарааллын дагуу** tab хийж шилжих боломж олгодог. Үүнийг **natural tab order** гэж нэрлэдэг.

Зарим тохиолдолд:

* аль элемент focus авч болох
* focus-ийн дарааллыг өөрчлөх
  шаардлага гарч болно.

Үүнийг хийхэд **`tabindex` атрибут** ашиглагддаг.

---

### tabindex-ийн үндсэн синтакс

```html
<element tabindex="number">Element Text</element>
```

`tabindex`-ийн утга нь тухайн элемент keyboard навигацид **яаж оролцохыг** тодорхойлдог.

---

### tabindex="0"

`tabindex="0"` нь элементийг **байгалийн tab дараалалд** оруулна.

Tab дарахад:

* button
* дараа нь div
* дараа нь link
  гэсэн дарааллаар (HTML дээрх дарааллын дагуу) focus шилжинэ.

Preview цонхны хоосон хэсэг дээр дарж, дараа нь **Tab** товч дарж focus хэрхэн шилжиж байгааг ажиглаарай. Жишээг ажиллуулахын тулд интерактив редакторыг идэвхжүүлэх шаардлагатай.

```html
<button>First</button>
<div tabindex="0">Second</div>
<a href="#">Third</a>
```

---

### tabindex="-1"

`tabindex="-1"` нь элементийг **programmatically focus авах боломжтой** болгодог.

Энэ нь дараах элементүүдэд хэрэгтэй:

* анхнаасаа focus авдаггүй элементүүд
* гарчиг (heading)
* контейнер
* dialog
* алдааны мессеж

```html
<p tabindex="-1">Sorry, there was an error with your submission.</p>
```

Энэ жишээнд:

* уг paragraph нь байгалийн tab дараалалд **орохгүй**
* хэрэглэгч Tab дарж очих боломжгүй
* харин JavaScript-ээр focus тохируулбал хэрэглэгчийн анхааралд шууд орно

Энэ аргын талаар JavaScript хичээлүүдэд илүү дэлгэрэнгүй үзнэ.

---

### tabindex > 0 (эерэг утгууд)

`tabindex`-ийн утга **0-ээс их** байвал **custom tab order** үүсгэнэ.

Энэ тохиолдолд:

* бага тоо → түрүүлж focus авна

Жишээ:

```html
<input tabindex="2">
<input tabindex="1">
<input tabindex="3">
```

Энд Tab дарахад:

* эхлээд tabindex="1"
* дараа нь tabindex="2"
* дараа нь tabindex="3"
  гэсэн дарааллаар focus авна (HTML дарааллаас үл хамааран).

Эерэг утгуудыг заримдаа:

* toolbar
* нарийн бүтэцтэй widget
  дотор ашигладаг.

Гэхдээ энэ аргыг **ихэвчлэн зөвлөдөггүй**, учир нь:

* навигаци ойлгомжгүй болдог
* хуудас томорч, өөрчлөгдөх тусам
* засварлах, арчлахад хүндрэлтэй болдог

---

### accesskey атрибут

Keyboard accessibility сайжруулах өөр нэг арга бол **`accesskey`** атрибут юм. Энэ нь тодорхой элементийг:

* focus авах
* эсвэл идэвхжүүлэх
  тусгай товчоор холбох боломж олгодог.

Дараах интерактив жишээг туршиж үзэж болно (интерактив редакторыг идэвхжүүлнэ үү):

* `accesskey="s"` → Save товч

  * Windows: **ALT + S**
  * Mac: **CTRL + Option + S**
* `accesskey="c"` → Cancel товч

  * Windows: **ALT + C**
  * Mac: **CTRL + Option + C**
* `accesskey="h"` → Home линк

  * Windows: **ALT + H**
  * Mac: **CTRL + Option + H**

> Анхаарах: accesskey идэвхжүүлэх товчны хослол нь браузер болон үйлдлийн системээс хамааран өөр байж болно. Ихэвчлэн:
>
> * Windows: ALT + тодорхойлсон товч
> * Mac: CTRL + Option + тодорхойлсон товч

```html
<button accesskey="s">Save</button>
<button accesskey="c">Cancel</button>
<a href="index.html" accesskey="h">Home</a>
```

---

### Focus indicator (focus заагч)

Вэб аппликейшнийг keyboard-д ээлтэй болгохын тулд **тод focus indicator** заавал байх ёстой.

Хэрэв:

* браузерийн анхдагч focus заагч хангалтгүй санагдвал
  → та focus төлвийг өөрөө style хийж болно.

Жишээ:

```html
<link href="styles.css" rel="stylesheet">

<button>Example button</button>
```

```css
button:focus {
  outline: 2px solid #005fcc;
}
```

Энд:

* `outline` property нь элементийг тойрсон хүрээг тодорхойлж байна
* focus авсан элемент **ялгарахуйц, тод харагдах** ёстой

Хүртээмжийн шаардлагын дагуу:

* focus indicator нь
* дэвсгэр өнгөтэй харьцуулахад
* **хамгийн багадаа 3:1 өнгөний ялгарал**-тай байх ёстой

---

### Keyboard trap-оос зайлсхийх

Мөн **keyboard trap**-оос зайлсхийх хэрэгтэй. Keyboard trap гэдэг нь:

* хэрэглэгч нэг элемент дээр focus орсны дараа
* түүнээс гарч чадахгүй болох
  нөхцөл юм.

Энэ нь ихэвчлэн:

* modal
* popup
  дотор буруу хэрэгжилтээс болж үүсдэг.

---

### Дүгнэлт

Keyboard accessibility-г хангах нь:

* screen reader ашигладаг хүмүүс
* mouse ашиглах боломжгүй хүмүүс
* keyboard-ийг илүүд үздэг хэрэглэгчид
  гэх мэт **олон төрлийн хэрэглэгчдэд** вэб аппликейшнийг саадгүй ашиглах боломж олгодог.

`tabindex`, `accesskey`, focus indicator, keyboard trap-оос зайлсхийх зэрэг аргуудыг зөв ашигласнаар та **хүртээмжтэй, inclusive** вэб аппликейшн бүтээж чадна.
