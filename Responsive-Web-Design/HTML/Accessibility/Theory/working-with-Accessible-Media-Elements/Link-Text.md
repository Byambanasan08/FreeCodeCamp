### Сайн link text (холбоосын текст)-ийн хүртээмжийн давуу талууд юу вэ, мөн сайн link text-ийн жишээнүүд юу вэ?

Одоо хүртээмж (accessibility)-ийн үүднээс **сайн link text бичихийн ашиг тус** болон **сайн link text-ийн жишээнүүдийг** авч үзье.

Сайн link text-ийн хамгийн анхны, харагдах давуу тал нь **хүн бүр мэдээллийг хурдан олоход тусалдаг** явдал юм. Тодорхой, тайлбарлагдсан холбоосууд нь хэрэглэгчид хаашаа очих, ямар мэдээлэлд хандах гэж байгаагаа урьдчилан ойлгох боломж олгодог. Ингэснээр хэрэглэгч төөрөхгүй, нийт хэрэглэгчийн туршлага (user experience) сайжирна.

Screen reader ашигладаг хүмүүсийн хувьд **тодорхой, тайлбар сайтай link text зайлшгүй шаардлагатай**. Учир нь screen reader нь холбоосын текстийг чангаар уншдаг. Тиймээс **“Read our accessibility guide”** гэсэн текст нь **“Click here”**-ээс хавьгүй илүү ойлгомжтой байдаг.

Link text-ийг тодорхой, тайлбарлагдсан байдлаар бичих нь зөвхөн харааны бэрхшээлтэй хүмүүст ашигтай биш. Энэ нь мөн **танин мэдэхүйн (cognitive) бэрхшээлтэй хүмүүст** ч тодорхой контекст өгч, ойлгоход тусалдаг.

---

### Link text бичихдээ баримтлах шилдэг туршлагууд

* Холбоосуудыг **харагдацын хувьд тод ялгарах** байдлаар (доогуур зураас, өнгө гэх мэт) харуулж, хэрэглэгчид амархан таньж, навигаци хийх боломжтой болго
* **“here”**, **“click here”**, **“more-info”** зэрэг ерөнхий, утга багатай link text-ээс зайлсхий
* Холбоосын зорилгыг илэрхийлсэн, **богино бөгөөд тодорхой** link text бичихийг зорь (ихэвчлэн 2–5 үг)
* Хэрэглэгч ойлгохгүй байж болох **мэргэжлийн үг хэллэг, товчлол**-оос зайлсхий
* Үйлдэлд бус, **очих газар (destination)**-т анхаар. Жишээ нь:
  “click here to read more” биш, харин **“user behavior results”**
* **Өөр өөр хуудас руу зааж байгаа холбоосуудад ижил link text бүү ашигла**
* Холбоосыг **ойр орчмын тексттэйгээ утга зүйн хувьд зохицсон** байдлаар байрлуул.
  Жишээ нь: “for more details, visit our events page”
  гэж бичихээс
  “Click here for more” гэж бичих нь дээр

---

### Жишээнүүд: Сайн ба муу link text

#### Жишээ 1: Event-ийн дэлгэрэнгүй хуудас

Муу жишээ:

```html
<a href="webinar-details-link">Details</a>
```

`Details` гэдэг нь:

* хэт ерөнхий
* дарвал яг ямар мэдээлэл харахыг ойлгуулахгүй

Хэрэглэгч:

* webinar-ын тухай юу
* бүтээгдэхүүн үү
* бодлого юу
  гэдгийг нэмэлт контекстгүйгээр мэдэх боломжгүй.

Сайн жишээ:

```html
<a href="webinar-details-link">
  Get details about our upcoming webinar
</a>
```

Энэ link text нь:

* хэрэглэгчид ямар контент хүлээж байгааг
* тодорхой ойлгуулж
* webinar-тэй холбоотой гэдгийг шууд мэдэгдэж байна

---

#### Жишээ 2: Blog пост руу холбоос

Муу жишээ:

```html
<a href="/blog-post-link">Read more</a>
```

`Read more` нь хүртээмжийн хувьд:

* контекстгүй
* screen reader ашигладаг хүмүүст ойлгомж муутай

Сайн жишээ:

```html
<a href="/blog-post-link">
   Read our latest blog post on web accessibility
</a>
```

Энэ link text нь:

* юуны тухай пост болохыг
* яагаад дарах ёстойг
  тодорхой харуулж байна

---

#### Жишээ 3: Нэмэлт мэдээлэл өгөх холбоос

Муу жишээ:

```html
<a href="/link-to-topic">More info</a>
```

`More info` нь:

* маш ерөнхий
* очих газрын талаар тодорхой мэдээлэл өгдөггүй
* screen reader хэрэглэгчдэд төөрөгдөл үүсгэж болзошгүй

Сайн жишээ:

```html
<a href="/link-to-topic">
   Learn more about our accessibility efforts
</a>
```

Энэ link text нь:

* дарвал **ямар мэдээлэл** харахыг
* шууд ойлгуулахаар
  тодорхой бичигдсэн байна

---

### Дүгнэлт

Сайн link text нь:

* хүн бүрт мэдээлэл олоход хялбар болгодог
* screen reader болон бусад туслах технологи ашигладаг хүмүүст онцгой чухал
* төөрөгдлийг багасгаж
* нийт хэрэглэгчийн туршлагыг сайжруулдаг

Тодорхой, тайлбар сайтай link text бичих нь **хүртээмжтэй, inclusive вэб контент** бүтээх үндсэн элементүүдийн нэг юм.
