aria-hidden атрибут гэж юу вэ, мөн хэрхэн ажилладаг вэ?

Хэрэв та контентыг **хуудсан дээр харуулсан хэвээр үлдээгээд**, харин screen reader зэрэг **туслах технологи ашигладаг хэрэглэгчдээс нуух** шаардлагатай бол `aria-hidden` атрибутыг ашиглаж болно.

Үүний тулд нуухыг хүссэн HTML элемент дээр `aria-hidden="true"` гэж нэмэхэд л хангалттай.

---

### aria-hidden хэрхэн ажилладаг вэ?

`aria-hidden="true"` атрибут нь тухайн элемент болон түүний **бүх дэд элементүүдийг accessibility tree-ээс хасдаг**.
Гэхдээ элемент нь:

* хуудсан дээр **харагдсан хэвээр**
* DOM-оос **устгагдахгүй**

байна.

---

### Түгээмэл ашиглах тохиолдлууд

`aria-hidden`-ийг ихэвчлэн дараах нөхцөлүүдэд ашигладаг:

* **Зөвхөн чимэглэлийн зориулалттай (decorative)** icon, зураг
* **Давхардсан контент** (icon + текст зэрэг)

---

### Чухал анхаарах зүйл

`aria-hidden` нь **зөвхөн туслах технологиос** (жишээ нь screen reader) контентыг нуудаг.
Хэрэв тухайн контент:

* **бүх хэрэглэгчдэд** (mouse, keyboard, screen reader) харагдах ёсгүй бол
  → `aria-hidden` ашиглах **ёсгүй**.

Жишээлбэл:

* Одоогоор хаалттай (collapsed) байгаа **hamburger menu**

  * keyboard хэрэглэгчдэд ч хүрэх боломжгүй байх ёстой
  * ийм тохиолдолд `aria-hidden` биш, харин

    ```css
    display: none;
    ```

    ашиглан DOM-оос бүрэн салгах хэрэгтэй

---

### Keyboard focus-той элемент дээр хэзээ ч ашиглаж болохгүй

**Keyboard-оор focus авч чаддаг элемент дээр `aria-hidden` хэрэглэхийг хатуу хориглоно.**

Учир нь:

* `aria-hidden` нь элементийг accessibility tree-ээс л хасна
* харин DOM-оос хасахгүй

Иймээс:

* keyboard хэрэглэгч `Tab`-аар тухайн элемент рүү очно
* харин screen reader нь түүнийг **зарлахгүй**
* хэрэглэгч “юу ч биш” дээр focus авсан мэт болно

Энэ нь маш муу хэрэглэгчийн туршлага юм.

---

### Жишээ: Icon-ийг accessibility tree-ээс нуух

Доорх жишээнд бид icon-ийг screen reader-ээс нууж, **зөвхөн текстийг** туслах технологид ил болгож байна. Ингэснээр icon + текст давхардаж, төөрөгдөл үүсгэхээс сэргийлнэ.

> **ТАЙЛБАР:** Энэ интерактив жишээнд Font Awesome CDN ашигласан тул preview дээр gear icon харагдана. Preview харахын тулд интерактив редакторыг идэвхжүүлэх шаардлагатай.

```html
<head>
  <!-- Font Awesome CDN -->
  <link
    rel="stylesheet"
    href="https://cdn.jsdelivr.net/npm/@fortawesome/fontawesome-free@6.5.0/css/all.min.css"
  />
</head>

<button>
  <i class="fa-solid fa-gear" aria-hidden="true"></i>
  <span class="label">Settings</span>
</button>
```

Энд:

* icon (`<i>`) нь `aria-hidden="true"` тул screen reader-д харагдахгүй
* харин `Settings` гэсэн текст нь screen reader-д уншигдана

---

### Дараах тохиолдлуудад aria-hidden хэрэглэх шаардлагагүй

Дараах нөхцөлүүдэд элемент аль хэдийн accessibility tree-ээс хасагдсан байдаг тул `aria-hidden` нэмэх хэрэггүй:

* HTML элементэд `hidden` атрибут аль хэдийн байгаа
* Элемент эсвэл түүний эцэг элемент `display: none`-той
* Элемент эсвэл түүний эцэг элемент `visibility: hidden`-тэй

Эдгээр тохиолдолд элемент DOM-оос хасагдсан тул screen reader-д ч харагдахгүй.

---

### Нэмэлт анхаарах зүйл

* Хэрэв эцэг элемент нь `aria-hidden="true"` байвал
  → тухайн элемент дээр `aria-hidden="false"` тавьсан ч
  → туслах технологид **ил болохгүй**

---

### Дүгнэлт

`aria-hidden` атрибут нь:

* screen reader зэрэг туслах технологи ашигладаг хэрэглэгчдээс
* тодорхой элементүүдийг нуухад

ашиглагддаг.

Энэ нь:

* зөвхөн **чимэглэлийн**
* эсвэл **давхардсан** контентыг нуухад

тустай байж болох ч **маш болгоомжтой**, **хязгаарлагдмал** хүрээнд ашиглах ёстой.

Ерөнхий зарчим бол:

> Хуудсан дээр байгаа бүх контент, функциональ байдал нь туслах технологи ашигладаг хүмүүст ч мөн адил хүртээмжтэй байх ёстой.

`aria-hidden`-ийг:

* screen reader хэрэглэгчдэд “сонирхолгүй” гэж таамагласан контентыг нуухад
  ашиглаж **огт болохгүй**.

Screen reader хэрэглэгчид ч мөн:

* хуудасны **бүх мэдээлэлд**
* бүрэн **хандах эрхтэй**

Эдгээр шилдэг туршлагуудыг дагаж, туслах технологиор заавал тест хийснээр та **хүн бүрт хүртээмжтэй, inclusive** онлайн туршлагыг бүтээж чадна.
