### aria-describedby атрибут гэж юу вэ, мөн хэрхэн ажилладаг вэ?

**aria-describedby** атрибут нь хуудас дээр аль хэдийн байгаа контентыг зааж өгснөөр тухайн элементийн талаар **нэмэлт мэдээллийг** screen reader ашигладаг хэрэглэгчдэд хүргэхэд ашиглагддаг. Энэ нь элемент болон тайлбарлаж буй контентын хооронд **програмын түвшний холбоо** (албан ёсоор *accessible description* гэж нэрлэдэг) үүсгэдэг бөгөөд screen reader-үүд хэрэглэгч тухайн элементтэй харилцах үед уг нэмэлт мэдээллийг дамжуулж чаддаг.

aria-describedby-ийн **хамгийн түгээмэл хэрэглээ** нь:

* формын input-уудтай **заавар (instructions)**
* болон **алдааны мессежүүдийг (error messages)**
  холбох явдал юм.

Screen reader хэрэглэгчид хуудсаар навигаци хийхдээ янз бүрийн арга ашигладаг тул эдгээр мессежийг анзааралгүй өнгөрөөх тохиолдол бий. aria-describedby ашигласнаар тэдгээрийг сонсох боломжийг нэмэгдүүлдэг.

Одоо хэрхэн ажилладгийг жишээгээр харцгаая.

---

### Жишээ 1: Нууц үгийн (password) input

Эхний жишээнд бид нууц үг авах формын элементтэй.

Интерактив редакторыг идэвхжүүлээд password input-д хэдэн тэмдэгт оруулаарай. Урьдчилсан харагдац дээр нууц үг масклагдсан (нуугдсан) байгааг харна. Мөн 8-аас доош тэмдэгт оруулсан үед password-help текст улаан хэвээр, 8 буюу түүнээс дээш тэмдэгт оруулахад ногоон болж өөрчлөгдөхийг анзаарна.

> **ТАЙЛБАР:** Энэ интерактив жишээ нь CSS болон JavaScript ашиглан password-help текстийн өнгийг динамикаар өөрчилж байна. JavaScript кодыг яг одоо ойлгох гэж санаа зовох хэрэггүй. Та JavaScript-ийг дараагийн модулиудад сурах болно.

```html
<link rel="stylesheet" href="styles.css">

<form>
  <label for="password">Password:</label>
  <input type="password" id="password" aria-describedby="password-help" />
  <p id="password-help">Your password must be at least 8 characters long.</p>
</form>

<script src="index.js"></script>
```

```css
#password-help {
  color: red;
}
```

```javascript
const passwordEl = document.getElementById("password");
const passwordHelpText = document.getElementById("password-help");

passwordEl.addEventListener("input", (e) => { 
  const userInput = e.target.value;
  passwordHelpText.style.color = userInput.length >= 8 ? "green" : "red";
});
```

Энд:

* `label` элемент нь **Password** текстийг password input-тэй холбож байна
* Нууц үгийн шаардлагыг тайлбарласан `p` элемент байна
* `aria-describedby` атрибут нь password input-ийг уг тайлбар тексттэй холбож өгч байна

Screen reader хэрэглэгч энэ input-тэй харилцах үед:

* эхлээд input-ийн нэрийг (*Password*)
* дараа нь нууц үгийн шаардлагыг
  зарлаж магадгүй.

Гэхдээ энэ нь **заавал 100% баталгаатай биш**. Зарим screen reader-үүд нэмэлт контентыг автоматаар зарлахгүй байж болно, эсвэл зөвхөн тодорхой нөхцөлд л зарладаг. Энэ нь хэвийн үзэгдэл бөгөөд screen reader бүр ARIA атрибутуудыг өөр өөрөөр боловсруулдаг.

Гэсэн ч энэ нь aria-describedby ашиглах шаардлагагүй гэсэн үг биш. Ерөнхийдөө энэ атрибут нь screen reader хэрэглэгчдэд **ихээхэн ашигтай**.

### Жишээ 2: Delete товч

aria-describedby-ийн өөр нэг сайн хэрэглээ бол **устгах (delete) товч** юм.

Доорх жишээнд delete товчны дараа тухайн үйлдэл ямар үр дагавартайг тайлбарласан мессеж байна:

```html
<button aria-describedby="delete-message">Delete</button>

<p id="delete-message">Warning! All deletions are permanent.</p>
```

Энд:

* `aria-describedby` атрибут нь delete товчийг анхааруулгын мессежтэй холбож байна
* `id` атрибутын утга болон `aria-describedby`-ийн утга **яг таарч байх ёстой**

Screen reader хэрэглэгч товч дээр очих үед уг анхааруулгыг сонсох боломжтой болно.

---

### Дүгнэлт

**aria-describedby** атрибут нь тухайн элементийн талаархи **нэмэлт мэдээллийг** screen reader хэрэглэгчдэд хүргэхэд ашиглагддаг хүчирхэг атрибут юм.

Энэ нь ялангуяа:

* формын input-уудтай холбоотой
* заавар
* алдааны мессежүүдийг
  холбох үед хамгийн өргөн хэрэглэгддэг.

Ингэснээр screen reader хэрэглэгчид формоор навигаци хийх явцдаа эдгээр чухал мэдээллийг алгасах магадлалыг **багасгаж**, илүү ойлгомжтой, хүртээмжтэй хэрэглэгчийн туршлагыг бий болгодог.
