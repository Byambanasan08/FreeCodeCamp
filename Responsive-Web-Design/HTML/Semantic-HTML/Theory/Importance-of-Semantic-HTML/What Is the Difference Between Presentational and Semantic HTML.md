Presentational HTML болон Semantic HTML хоёрын ялгаа нь юу вэ?

1. Presentational HTML нь агуулгын *харагдац*, *стиль*, *дүрслэлд* төвлөрдөг.
HTML-ийн эхэн үед хөгжүүлэгчид `center`, `big`, `font` зэрэг элементүүдийг өргөн ашигладаг байсан.
Гэвч орчин үеийн вэб хөгжүүлэлтэд эдгээрийг ашиглах ёсгүй, учир нь:

* боломж нь хязгаарлагдмал
* accessibility буюу хандалттай байдалд сөрөг
* кодын засвар, өргөтгөх боломжийг муутгадаг

Ийм presentational элементүүдийн ихэнх нь **deprecated**, өөрөөр хэлбэл хуучирсан, хэрэглэхийг зөвлөдөггүй болсон.

Гэсэн хэдий ч тэдгээрийн оршин байдаг тухай мэдэх нь хэрэгтэй тул зарим жишээг авч үзье.

---

### **font элемент — текстийн өнгө, хэмжээ тодорхойлдог deprecated элемент**

```html
<font size="5" color="blue">This text is blue and large.</font>
```

Энд:

* `color="blue"` → текстийн өнгийг цэнхэр болгож байна
* `size="5"` → фонтын хэмжээг 1–7 хоорондох утгаар тохируулдаг (5 бол том, 7 хамгийн том)
* Хэрэв size заагаагүй бол default нь 3

Энэ элемент ажилласаар байгаа ч орчин үед ашиглах ёсгүй.
**Фонтын хэмжээ ба өнгө нь HTML биш CSS-ээр заагдах ёстой.**

---

### **center элемент — контентыг хэвтээ тэнхлэгт төвд байрлуулах**

Deprecated болсон presentational элемент.

```html
<center>
  This text is centered.
  <p>HTML is awesome.</p>
</center>

<p>Another example text.</p>
```

Энэ элемент мөн ажиллаж байгаа ч **төвд байрлуулах ажлыг CSS (`text-align: center`) ашиглан хийх ёстой**.

---

### **big элемент — текстийг нэг түвшнээр томруулах**

Deprecated element.

```html
<p>
  This text has a normal font size.
  <big>This text is larger.</big>
  Some other text.
</p>
```

`big` элемент нь хүрээлсэн текстийг нэг түвшнээр томруулдаг байсан.
Гэвч энэ үүргийг орчин үед **CSS-ийн font-size** ашиглан гүйцэтгэнэ.

→ Тиймээс ашиглаж болохгүй.

---

2. Тэгвэл оронд нь юу ашиглах ёстой вэ? — Semantic HTML

Semantic HTML нь elementos-ийг *агуулга болон утга учрын* дагуу ашигладаг.
Энэ нь HTML-ийг:

* уншихад ойлгомжтой
* тайлбарлахад хялбар
* засварлахад амар
* өргөтгөхөд үр ашигтай

болгодог.

Semantic HTML ашиглах нь:

1. SEO сайжруулдаг

Хайлтын системүүд семантик бүтэцтэй HTML-ийг илүү сайн ойлгож, илүү зөв индексэлж чадна.

2. Accessibility сайжруулдаг

Screen reader зэрэг туслах технологиудын хувьд семантик бүтэц нь хуудасны агуулгыг зөв тодорхойлоход амин чухал.

---

Semantic HTML-ийн жишээнүүд

* `header` → баримтын толгой хэсгийг тодорхойлно
* `nav` → навигацийн холбоосуудын бүлэг
* `section` → холбогдолтой контентыг бүлэглэх
* `figure` → зураг, диаграмм зэрэг дүрслэл

Доорх жишээнд header дотор nav байрлаж байна:

```html
<header>
  <nav>
    <a href="#">Home</a>
    <a href="#">About</a>
    <!--Add the products link here-->
    <a href="#">Contact</a>
  </nav>
</header>
```

Эдгээр семантик элементүүд нь HTML дотор ямар үүрэг гүйцэтгэж байгаагаа *өөрсдөө* тодорхой харуулдаг.

Semantic элемент маш олон тул таны ямар ч төсөлд тохирох элемент заавал олдоно.

---

## **Дүгнэлт**

Presentational HTML → харагдацад төвлөрсөн (HTML дотор стиль хийхээс бүрддэг, хуучирсан арга)
Semantic HTML → контентын утга, бүтэц, зориулалтад төвлөрсөн (зөвлөмжлөсөн орчин үеийн арга)

**Semantic HTML нь илүү:**

* ойлгомжтой
* хандалттай
* SEO-д ашигтай
* засах, арчлахад хялбар

Тиймээс presentational HTML-ээс бүрэн татгалзаж, орчин үеийн semantic HTML ашиглах нь хамгийн зөв практик юм.
