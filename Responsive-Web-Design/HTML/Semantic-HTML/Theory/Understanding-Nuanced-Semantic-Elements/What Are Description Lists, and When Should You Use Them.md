Тайлбарын жагсаалт (Description Lists) гэж юу вэ, хэзээ ашиглах вэ?

Тайлбарын жагсаалт нь нэр томъёо болон тэдгээрийн тодорхойлолтыг эмх цэгцтэй, уншихад хялбар байдлаар харуулахад маш тохиромжтой. Жишээлбэл, толь бичиг эсвэл glossary-д үг бүрийн тайлбар нь хамт байрладагтай адил.

Дараах нь HTML дээрх хоёр нэр томъёо болон тэдгээрийн харгалзах дэлгэрэнгүй тайлбарыг агуулсан тайлбарын жагсаалтын жишээ юм.

Интерактив editor-ийг идэвхжүүлээд кодын comment хийсэн хэсгийг uncomment хийж, шинэ detail item preview цонхонд хэрхэн гарч ирж байгааг үзээрэй.

```html
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
  <!-- <dt>JS</dt>
  <dd>JavaScript</dd> -->
</dl>
```

Энэ тохиолдолд нэр томъёонууд нь HTML болон CSS гэсэн товчлолууд бөгөөд дэлгэрэнгүй тайлбарууд нь тэдгээрийн бүтэн нэршлүүд юм. Эдгээр дэлгэрэнгүй хэсгүүд нь тодорхойлолт байж болохоос гадна тухайн нэр томъёотой холбоотой бусад мэдээлэл байж ч болно.

Тайлбарын жагсаалтыг тодорхойлохын тулд танд гурван HTML элемент хэрэгтэй.

Нэгдүгээрт, бүх жагсаалтыг багтаасан container болох тайлбарын жагсаалтын элемент `dl` байна. Жишээн дээр харахад энэ элемент нь тайлбарын жагсаалтын бусад бүх элементүүдийг хамарч байна.

Дараа нь, нэр томъёо бүрт зориулсан тайлбарын нэр томъёоны элемент `dt` байна. Энэ жишээнд тайлбарын жагсаалт HTML болон CSS гэсэн хоёр нэр томъёотой тул `dt` элемент хоёр байна.

Эцэст нь, нэр томъёо бүрийн дараа тухайн нэр томъёоны тайлбар буюу дэлгэрэнгүй мэдээллийг агуулсан `dd` элемент байна. Энэ жишээнд эдгээр нь HyperText Markup Language болон Cascading Style Sheets юм.

Хөтөч (browser) дээр харахад нэр томъёо бүрийн дараа түүнд харгалзах тайлбар нь харагдана. Анхдагч байдлаар (by default) тайлбарууд нь харагдах байдлаар ялгарахын тулд баруун тийш бага зэрэг догол мөртэй байрладаг.

Гэхдээ тайлбарын жагсаалт нь зөвхөн нэр томъёо болон тодорхойлолтоор хязгаарлагдахгүй. Энэ нь үүнээс 훨씬 уян хатан хэрэглээтэй. Дараах жишээнд хоёр орцтой жор байна.

Интерактив editor-ийг идэвхжүүлээд кодын comment хийсэн хэсгийг uncomment хийж, шинэ detail item preview цонхонд хэрхэн гарч ирж байгааг үзээрэй.

```html
<dl>
  <dt>Flour</dt>
  <dd>2 cups</dd>
  <dt>Sugar</dt>
  <dd>1/2 cup</dd>
  <!-- <dt>Vegetable Oil</dt>
  <dd>2 tablespoons</dd> -->
</dl>
```

Бүх тайлбарын жагсаалт нь `dl` элемент дотор байна. Эхний орц болох Flour нь тайлбарын нэр томъёоны элемент болох `dt` дотор байна. Дараа нь тухайн орцны хэмжээг харуулсан 2 cups гэсэн мэдээлэл нь түүнд харгалзах `dd` элементэд шууд байрлаж байна.

Sugar-ийн хувьд ч мөн адил бүтэц давтагдана. Энэ жишээнд жор нь зөвхөн хоёр орцтой байгаа боловч хэрвээ илүү олон орцтой байсан бол ижил бүтэц тайлбарын жагсаалтын туршид давтагдах боломжтой.

Хөтөч дээр харахад орцнууд зүүн талд тэгшлэгдэн байрлаж, хэмжээнүүд нь харагдах байдлаар ялгарахын тулд догол мөртэй харагдана.

Тайлбарын жагсаалтыг ашиглах бусад жишээ хэрэглээнүүдэд бүтээгдэхүүний техникийн үзүүлэлтүүд (product specifications), түгээмэл асуултууд (frequently asked questions), холбоо барих мэдээлэл (contact information), мөн metadata орно. Ерөнхийдөө, хоорондоо холбоотой хоёр мэдээлэл key-value хос хэлбэртэй байж, нэг нь шошго буюу key-ийн үүрэг гүйцэтгэж, нөгөө нь түүнтэй холбоотой нэмэлт мэдээлэл буюу value-ийн үүрэг гүйцэтгэж байвал тайлбарын жагсаалтыг ашиглахад тохиромжтой.
