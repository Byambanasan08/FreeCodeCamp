1. HTML нь Вэб дээр ямар үүрэгтэй вэ?

HTML буюу **Hypertext Markup Language** нь вэб хуудас бүтээхэд ашиглагддаг *markup* хэл юм. Та ямар нэг вэбсайт руу ороод догол мөр, гарчиг, холбоос, зураг, видео зэрэг контентуудыг харж байвал — тэр нь HTML юм.

Энд HTML элемент ашигласан жижиг жишээ байна. Editor-ийн зарим текстийг өөрчлөөд preview цонхонд хэрхэн шинэчлэгдэж байгааг хараарай.

```
<h1>Main heading element</h1>

<p>I am a paragraph element.</p>
```

HTML нь вэб хуудасны "контент ба бүтэц(content and structure)"-ийг элементүүдийн тусламжтайгаар илэрхийлдэг. Ихэнх элемент нь *opening tag* болон *closing tag*—тай байдаг. Заримдаа эдгээрийг *start tag* болон *end tag* гэж нэрлэдэг. Эдгээр хоёр tag-ийн хооронд контент оршино. Энэ контент нь текст эсвэл бусад HTML элемент байж болно.

Энд догол мөрийн элемент буюу paragraph-ийн өөр нэг жишээ байна. Editor дахь текстийг “I love coding!” гэж солиод preview цонхонд гарах өөрчлөлтийг хараарай.

```
<p>I am a paragraph element.</p>
```

Opening ба closing tag хоёр хоёулаа **зүүн өнцөгт хаалт (<)(left angle bracket)**-аар эхэлж, **баруун өнцөгт хаалт (>)(right angle bracket)**-аар төгсөнө. Tag-ийн нэр нь эдгээр хаалтны дунд байрлана. HTML-ийн tag нэрнүүд *case-insensitive* буюу том жижиг үсгийн ялгаа баримталдаггүй ч практикт **жижиг үсгээр бичих нь хамгийн түгээмэл бөгөөд зөв хэрэглээ** юм.

Paragraph-ийн зөвхөн opening болон closing tag-ийг ойроос харвал:

```
<p>
</p>
```

Opening tag болон closing tag-ийг ялгахад хамгийн гол зүйл нь closing tag-ын **зүүн өнцөгт хаалтны дараах шууд байрлах налуу зураас (/)(forward slash)** юм. Зарим HTML элементүүд closing tag-гүй байдаг. Эдгээрийг **void elements** гэж нэрлэнэ.

Энд void element болох image элемент жишээтэй:

```
<img>
```

Анзаарах зүйл нь image элемент closing tag-гүй бөгөөд ямар ч контент агуулахгүй. Void elements нь контент агуулах боломжгүй бөгөөд зөвхөн *start tag* л байдаг.

Заримдаа void element-уудын “>”-ийн өмнө **/** хэрэглэсэн байдлыг харна, жишээ нь:

```
<img />
```

Prettier зэрэг олон code formatter-ууд void элементэд `/` нэмж бичихийг сонгодог боловч HTML-ийн албан ёсны spec-д “**энэ / нь start tag-ийг self-closing болгож байгаа хэрэг биш, ямар ч нөлөөгүй, шаардлагагүй тэмдэг**” гэж заасан байдаг.

Бодит хөгжүүлэлт дээр та хоёуланг нь харж таарах тул хоёуланг нь танилцаж мэдэх нь чухал.

Хэрэв та зураг харуулахыг хүсвэл image элемент дотроо хэд хэдэн attribute оруулах шаардлагатай. **Attribute** гэдэг нь HTML элементэд нэмэлт тохируулга хийхэд ашиглагддаг тусгай утга.

Энд **src attribute**-тай image элемент жишээ байна. src-ийн утгыг дараах линк рүү өөрчлөөд зураг хэрхэн өөрчлөгдөж байгааг хараарай:

```
https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg
```

```
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg" />
```

**src attribute** нь тухайн зургийн байрлаж буй хаягийг(address) заадаг. Image элементэд **alt attribute** нэмэх нь бас сайн практик. Alt attribute нь тухайн зургаас богино, тодорхой тайлбар текст өгөхөд ашиглагдана.

Энд src болон alt attribute-тай image элемент жишээтэй. src-ийн утгыг дараах буруу линк рүү өөрчлөөд үзээрэй — зураг алга болж alt текст л харагдах болно:

```
https://.freecodecamp.org/curriculum/cat-photo-app/cats.jpg
```

```
<img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/cats.jpg" alt="Two tabby kittens sleeping together on a couch." />
```

Тэгвэл танд “зөвхөн HTML байхад вэбсайт бүтээхэд хангалттай юу?” гэсэн асуулт төрж болох юм. Хариулт нь: **хамаарна**.

Хэрэв та зөвхөн текст ба зураг үзүүлэх жижиг дадлага төсөл хийж байгаа бол HTML ганцаараа хангалттай байж магадгүй. Харин орчин үеийн, мэргэжлийн вэбсайт хийх гэж байгаа бол HTML, CSS, JavaScript — гурвууланг нь хэрэглэх шаардлагатай.

* **HTML** → контент ба бүтэц
* **CSS** → харагдац, загвар
* **JavaScript** → вэб хуудсыг интерактив болгох

Энийг барилга барихтай зүйрлэж болно.

* **HTML** → ханын тоосго, бетон, төмөр — барилгын суурь, бүтцийг бүрдүүлдэг зүйлс
* **CSS** → дотор, гадна засал — байшинг гоё харагдуулдаг дизайн
* **JavaScript** → цахилгаан, усны систем — бүх зүйл ажиллагаатай, амьд байхад шаардлагатай дэд бүтэц
