3. Замын (path) бичлэгт slash, ганц цэг (.), хоёр цэг (..) ямар ялгаатай вэ?**

Та өмнө нь `/public/logo.png`, `./script.js`, эсвэл `../styles.css` зэрэг холбоосуудыг харж байсан байх. Эдгээр нь тусгай утгатай **файлын замууд** юм. Зам бичихдээ гурван үндсэн тэмдэглэгээ ашиглагддаг:

1. **Slash (/ эсвэл )**
2. **Single dot (.)**
3. **Double dot (..)**

---

### **1. Slash — “path separator”**

Slash нь *замыг тусгаарлагч* бөгөөд сан болон файлын нэрсийн хоорондох заагийг илэрхийлдэг.

Жишээ:

* `naomis-files/` → *naomis-files* нэртэй сан
* `naomis/files/` → *naomis* сангийн доторх *files* сан

Windows ихэвчлэн `\` (backslash) ашигладаг бол macOS болон Linux `/` (forward slash) ашигладаг.

---

### **2. Single dot (.) — Одоогийн директор**

Ганц цэг нь **одоогийн хавтас**-ыг заана.

Мөн single dot ашиглагддаг гол шалтгаан:

* Замыг **relative path** гэж ойлгуулах

Өмнөх хичээлд үзсэнээр, relative path нь тухайн файлын байрлалтай харьцуулж зам заадаг.

Жишээ:
`./script.js` → *одоогийн хавтас доторх script.js*

---

### **3. Double dot (..) — Parent директор**

Хоёр цэг нь **эх (parent) хавтас** руу шилжихийг заана.

Энэ нь одоогийн ажлын хавтсаас гаднах файл руу хандах хамгийн түгээмэл арга.

---

### **Жишээ файл бүтэц**

```
my-app/
├─ public/
│  ├─ favicon.ico
│  ├─ index.html
├─ src/
│  ├─ index.css
│  ├─ index.js
```

### **index.html → favicon.ico татах**

`public/index.html` файл `public` хавтсанд байгаа тул:

```
./favicon.ico
```

* `.` → одоогийн public хавтас
* favicon.ico → тэр хавтсан дахь файл

---

### **index.html → src/index.css татах**

Энэ файл `public` хавтаснаас гадна байгаа тул эхлээд parent руу гарна:

```
../src/index.css
```

* `..` → my-app (parent хавтас)
* `/src/index.css` → src хавтсанд орж index.css файлыг авах

---
