DOM Inspector болон DevTools-ийг ашиглан төслөө debug хийх, хөгжүүлэх нь

Та төслөө хөгжүүлж байх явцад програм тань **хүссэнээр ажиллахгүй байх** тохиолдолтой байнга тулгарна.

Программистууд ийм асуудлуудыг ихэвчлэн **bug** гэж нэрлэдэг. Эдгээр bug-ийг олж илрүүлэх, засах үйл явцыг **debugging** гэж нэрлэнэ.

Кодоо debug хийхийн тулд браузерээс олгодог зарим **тусгай хэрэгслүүдийг** ашиглах шаардлагатай.

Тэдгээрээс хамгийн чухал хоёр хэрэгсэл нь **DOM Inspector** болон **Developer Tools (DevTools)** юм.

**DOM Inspector** нь таны одоогийн хуудсан дээрх **HTML бүтцийг** шалгаж харах боломж олгодог.

**DOM** гэдэг нь **Document Object Model** гэсэн үг бөгөөд энэ нь хуудсан дээрх элементүүдийг илэрхийлсэн **мод хэлбэрийн (tree-like) бүтэц** юм. Та DOM-ийн талаар дараагийн модулиудад илүү дэлгэрэнгүй сурах болно.

**Developer Tools** нь таны хуудсан дээрх **HTML, CSS, JavaScript**-ийг шалгах боломжийг олгодог.

Одоо anchor элемент дотор жижиг алдаа агуулсан HTML жишээг авч үзье:

Жишээ код

```html
<a href="https://www.freecodecamp.org/larn/">freeCodeCamp curriculum</a>
```

Энэ холбоос дээр дарахад **404 хуудас** руу орно. **404 хуудас** гэдэг нь хэрэглэгч сервер дээр байхгүй вэб хуудсанд хандах үед гарч ирдэг **алдааны хуудас** юм.

Энэхүү холбоосын зорилго нь **freeCodeCamp curriculum** руу чиглэх ёстой.

Асуудал яг юу болохыг харахын тулд **developer tools** ашиглаж болно.

Браузер дээр developer tools нээхийн тулд хуудсан дээр **right-click** хийгээд **Inspect**-ийг сонгоно.

Мөн:

* **PC** дээр: `Control + Shift + I`
* **Mac** дээр: `Command + Option + I`

товчлолуудыг ашиглаж болно.

Google Chrome дээр developer tools-ийг нээхэд хэд хэдэн tab харагдана. Эхний tab нь **Elements** tab бөгөөд энэ нь тухайн хуудсан дээрх **HTML бүтцийг** харуулдаг.

Хоёр дахь tab нь **Console** tab бөгөөд энэ нь хуудсан дээр гарч буй **алдаанууд (errors)**-ыг харуулдаг.

Эвдэрсэн (broken) холбоосын үед Console tab-ийг шалгавал тухайн холбоостой холбоотой алдааны мессежүүдийг харж болно. Ихэвчлэн **404 error** давтагдан гарч ирнэ. 404 error нь тухайн хуудас **олдоогүй** гэдгийг илэрхийлдэг.

Энэ нь асуудал **anchor элементийн URL-тэй холбоотой** гэдгийг бидэнд ойлгуулна. Href атрибутын утгыг шалгахад **бичгийн алдаа (typo)** байгааг харна.

Одоогийн Console мессеж дээр `/larn` гэсэн зам 404 алдаатай гарч байгаа боловч зөв URL нь `/learn` байх ёстой. Холбоосыг зөв болгосны дараа хэвийн ажиллана.

Та developer tools-тэй ажиллах талаар энэхүү сертификатын явцад илүү дэлгэрэнгүй сурах болно. Одоогоор энэ нь developer tools таны кодыг **debug хийхэд хэрхэн тусалдгийн** богино жишээ юм.
