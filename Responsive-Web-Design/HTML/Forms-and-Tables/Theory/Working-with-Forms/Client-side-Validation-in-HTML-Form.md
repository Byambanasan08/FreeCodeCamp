HTML Form-д Client-Side Validation гэж юу вэ, мөн ямар жишээнүүд байдаг вэ?

Хэрэглэгч таны вэбсайт дээр form бөглөх үед шаардлагатай бүх мэдээллийг **зөв форматтайгаар** бөглөсөн эсэхийг шалгах нь чухал. HTML form control-ууд (жишээ нь input элементүүд) нь буруу өгөгдлийг шалгах боломжтой **built-in validation**-уудыг агуулдаг. Энэ нь мэдээлэл сервер рүү илгээгдэхээс өмнө хэрэглэгч алдаагаа засах боломжийг олгодог.

1. Client-side гэдэг нэр томьёо нь хэрэглэгчийн компьютер эсвэл төхөөрөмж дээр явагдаж буй бүх зүйлийг хэлдэг. Үүнд хэрэглэгч шууд харьцаж буй вэбсайт эсвэл аппын хэсгүүд болох layout, дизайн, интерактив боломжууд багтана.

2. Server-side гэдэг нь вэбсайт эсвэл апп-ыг байршуулж буй сервер дээр явагдаж буй бүх процессыг хэлнэ. Үүнд өгөгдөл боловсруулах, аппликейшн ажиллуулах, хэрэглэгчийн төхөөрөмжөөс ирсэн хүсэлтийг боловсруулах зэрэг орно.

Client-side validation нь чухал боловч **server-side validation**-ийг мөн заавал ашиглах хэрэгтэй. Учир нь муу санаатай хэрэглэгчид client-side шалгалтыг тойрч гарах боломжтой тул сервер талын бат бөх хамгаалалт зайлшгүй шаардлагатай. Үүнийг дараагийн модульд дэлгэрэнгүй судлах болно. Одоохондоо client-side form validation-ийн жишээнүүдийг авч үзье.

Client-side validation-ийн түгээмэл жишээний нэг нь input элемент дээр **required атрибут** ашиглах явдал юм. Required атрибут нь form илгээгдэхээс өмнө тухайн талбарыг заавал бөглөх ёстойг заана. Доорх нь email input дээр required атрибут ашигласан жишээ юм.

Имэйл хаяг оруулахгүйгээр **Submit Form** товчийг дарвал, талбарыг бөглөхийг сануулсан мессеж гарч ирнэ.

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

Браузер бүр энэхүү анхааруулгын мессежийг харуулах өөрийн гэсэн стильтэй байдаг.

Email input ашиглахын бас нэг давуу тал нь зөв форматтай имэйл хаяг эсэхийг шалгах **үндсэн validation**-ийг автоматаар хийдэг явдал юм. Жишээ нь, санамсаргүй үсэг бичээд submit дарвал браузер `@` тэмдэг дутуу байна гэсэн анхааруулга харуулна.

Email талбарт `abc` гэж бичээд submit товчийг дарна уу. Энэ нь зөв имэйл хаяг биш тул анхааруулга гарч ирэх ёстой.

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input required type="email" name="email" id="email" />
  <button type="submit">Submit Form</button>
</form>
```

Браузерууд зөвхөн **стандарт имэйл хаягийн суурь шалгалт** хийдгийг анхаарах хэрэгтэй. Илүү нарийн validation нэмэх нь таны үүрэг бөгөөд үүнийг дараагийн модулиудад судална.

Email input-д ашиглаж болох өөр validation-ууд бол **minlength** болон **maxlength** атрибутууд юм. Доорх нь нэмэлт validation ашигласан жишээ юм.

Талбарт `b@m` гэж бичээд submit товчийг дарна уу. Энэ нь хамгийн бага уртын шаардлагыг хангахгүй тул анхааруулга гарч ирнэ.

```html
<form action="">
  <label for="email">Email Address (Required field):</label>
  <input
    required
    type="email"
    name="email"
    id="email"
    minlength="4"
    maxlength="64"
  />
  <button type="submit">Submit Form</button>
</form>
```

**Minlength** болон **maxlength** атрибутууд нь email input-д оруулах тэмдэгтийн хамгийн бага болон хамгийн их уртыг зааж өгдөг. Хэрэв хамгийн бага уртыг хангахгүй эсвэл хамгийн их уртаас хэтэрвэл браузер анхааруулгын мессеж харуулна.
