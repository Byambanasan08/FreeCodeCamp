DevTools шалгах хэрэгсэл болон CSS баталгаажуулагчийг ашиглан CSS-ээ хэрхэн засварлах вэ?

Хөгжүүлэгчийн хэрэгслүүд, шалгах хэрэгслүүд болон CSS баталгаажуулагчид нь CSS-ийн алдааг засах, таны stylesheet-ийг алдаагүй, оновчтой байлгахад зайлшгүй шаардлагатай хэрэгслүүд юм.

Эдгээр хэрэгслүүд нь таны CSS хэрхэн хэрэгжиж байгааг ойлгоход үнэтэй мэдээлэл өгч, боломжит асуудлуудыг илрүүлэхэд тусалдаг.

Эхлээд browser-ийн хөгжүүлэгчийн хэрэгсэл буюу DevTools-оос эхэлье. Chrome, Firefox, Safari зэрэг ихэнх орчин үеийн browser-ууд DevTools-ийг дотроо агуулсан байдаг. Үүнд хандахын тулд веб хуудсан дээрх элементийг баруун товшоод "Inspect" гэж сонгох эсвэл F12, Cmd+Option+I (macOS дээр) зэрэг keyboard shortcut ашиглаж болно.

DevTools нь таны CSS-ийг бодит цаг дээр шалгах болон өөрчлөх боломжийг олгодог. Styles хэсэг нь сонгогдсон элементэд хэрэглэгдсэн бүх CSS дүрмүүдийг, мөн удамшсан (inherited) стилийг харуулна.

Та тус бүр property-г асаах/унтраах, утгыг нь засах, бүр browser дотор шууд шинэ дүрэм нэмэх боломжтой. Энэхүү шууд feedback нь эх кодоо өөрчлөхгүйгээр өөр өөр стилийг туршихад маш хэрэгтэй.

DevTools-ийн нэг хэсэг болох шалгах хэрэгсэл нь хуудсан дээрх элементүүд дээр cursor-оо аваачихад тэдгээрийн box model-ийг (margin, border, padding, content хэсэг) харуулдаг. Энэ нь layout-ийн асуудлыг оношлох эсвэл элементүүд яагаад тодорхой байрлалд байгааг ойлгоход онцгой тустай.

CSS баталгаажуулагч нь бас нэг чухал хэрэгсэл юм. W3C CSS Validator нь түгээмэл сонголт бөгөөд таны CSS-ийг албан ёсны стандарттай тулгаж, алдаа болон анхааруулгуудыг мэдээлдэг. Үүнийг ашиглахдаа CSS файлаа upload хийх, шууд CSS код оруулах эсвэл URL өгөх боломжтой.

Жишээ нь, дараах CSS код дээр ажиллаж байна гэж үзье:

.container {
width: 100%;
height: 200px
background-color: #F0F0F0;
}

Баталгаажуулагч нь height property-ийн дараа цэг таслал (semicolon) дутуу байгааг зааж өгнө. Ийм төрлийн алдаа амархан анзаарагдахгүй ч stylesheet-д ноцтой асуудал үүсгэж болно.

Responsive дизайн засварлах үед DevTools дахь device emulation функц маш хэрэгтэй. Энэ нь таны сайтыг янз бүрийн дэлгэцийн хэмжээ болон төхөөрөмж дээр хэрхэн харагдахыг дуурайлган харуулдаг. Ингэснээр breakpoint-ийн алдаа эсвэл viewport-ийн хэмжээнээс хамааран зөв ажиллахгүй байгаа стилийг илрүүлэх боломжтой.

Санах хэрэгтэй зүйл бол үр дүнтэй CSS засварлалт нь ихэвчлэн эдгээр хэрэгслүүдийг хослуулан ашиглах явдал юм. Та эхлээд баталгаажуулагч ашиглан синтаксийн алдааг илрүүлж, дараа нь DevTools ашиглан тодорхой элементүүдийг шалгаж, өөрчлөлт туршиж болно. Дараа нь device emulation ашиглан таны стиль өөр өөр дэлгэцийн хэмжээнд зөв ажиллаж байгаа эсэхийг шалгана.

Эдгээр хэрэгслүүдийг сайн эзэмшсэнээр та CSS засварлах үйл явцаа мэдэгдэхүйц хурдасгаж, илүү найдвартай, алдаагүй stylesheet үүсгэж чадна.

Эдгээр аргуудыг тогтмол ашиглах нь зөвхөн тухайн үеийн асуудлыг шийдэхэд туслахаас гадна CSS болон HTML-ийн харилцан үйлчлэлийн талаарх таны ерөнхий ойлголтыг сайжруулна.
