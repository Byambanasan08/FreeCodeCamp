**Агуулгыг хүртээмжгүй болгохгүйгээр хэрхэн нуух вэ?**

Вэб хөгжүүлэлтэд вэб хуудсан дээрх агуулгыг нуух нь түгээмэл практик байдаг. Гэхдээ үүнийг хүртээмжийг (accessibility) алдагдуулахгүйгээр хийх нь маш чухал юм. Агуулгыг нуух өөр өөр аргууд нь туслах технологиуд (assistive technologies) тухайн агуулгыг хэрхэн тайлбарлаж, хэрэглэгчдэд хэрхэн харуулахад өөр өөр нөлөө үзүүлдэг. Одоо хүртээмжийг хадгалан агуулгыг нуух зарим шилдэг практикуудыг авч үзье.

Агуулгыг нуух хамгийн түгээмэл аргуудын нэг нь **display: none** ашиглах явдал юм. Доорх нь жишээ:

```html id="c9m1o1"
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css id="3z0n9c"
.hidden {
  display: none;
}
```

Энэ нь агуулгыг харагдах байдлаар бүрэн нуудаг. Гэхдээ мөн **accessibility tree**-ээс бүрэн устгадаг. Accessibility tree нь screen reader зэрэг туслах технологиуд вэб хуудасны агуулгыг ойлгож, түүнтэй харилцахад ашигладаг бүтэц юм. Энэ нь вэб хуудасны агуулга болон түүний семантик утгыг туслах технологи ойлгож, хэрэглэгчид дамжуулах боломжтой хэлбэрээр илэрхийлдэг.

display: none ашиглах үед screen reader болон бусад туслах технологиуд энэ агуулгад хандаж чадахгүй. Учир нь энэ нь accessibility tree-д багтаагүй байдаг. Тиймээс энэ аргыг зөвхөн агуулгыг **харагдах байдлаас болон хүртээмжээс бүрэн устгах** шаардлагатай үед ашиглах хэрэгтэй.

Агуулгыг нуух өөр нэг арга бол **visibility: hidden** ашиглах явдал юм:

```html id="ix7n44"
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css id="lnt3w2"
.hidden {
  visibility: hidden;
}
```

visibility: hidden нь агуулгыг харагдах байдлаар нуудаг боловч document flow-д үлдээдэг. Энэ нь тухайн элемент хуудсан дээрх зайг эзэлсэн хэвээр байна гэсэн үг юм. display: none-тэй адил visibility: hidden нь мөн агуулгыг accessibility tree-ээс устгадаг. Энэ нь screen reader зэрэг туслах технологиуд энэ агуулгад хандаж чадахгүй гэсэн үг юм. Тиймээс visibility: hidden-ийг **хүн бүрээс, тэр дундаа туслах технологи ашигладаг хүмүүсээс агуулгыг нуух** шаардлагатай үед ашиглах хэрэгтэй.

Хэрэв агуулгыг **харагдах байдлаар нуух боловч screen reader-д хүртээмжтэй байлгах** шаардлагатай бол “visually hidden” эсвэл “screen reader only” гэж нэрлэгддэг техникийг ашиглаж болно. Доорх нь **.sr-only** CSS class ашигласан жишээ бөгөөд энэ нь screen reader-д хүртээмжтэй хэвээр байлгаж, харагдах байдлаар нуух түгээмэл арга юм:

```html id="u3nb7p"
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css id="6i5gfa"
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

Энэ жишээнд бид **position**, **clip**, болон **white-space** зэрэг property-уудыг ашиглаж байна. Та эдгээрийг дараагийн хичээлүүдээр илүү дэлгэрэнгүй сурах болно. Одоохондоо энэ CSS rule нь агуулгыг харагдах байдлаар нуудаг боловч screen reader-д хүртээмжтэй хэвээр үлдээдэг гэдгийг ойлгоход хангалттай. Энэ нь харагдах layout-д нөлөөлөхгүйгээр screen reader ашигладаг хэрэглэгчдэд нэмэлт мэдээлэл өгөхөд хэрэгтэй байдаг.

Агуулгын харагдах байдлыг **toggle** хийхийн тулд **hidden attribute** ашиглаж болно:

```html id="nhu8uf"
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

hidden attribute нь орчин үеийн ихэнх browser-ууд дээр дэмжигддэг бөгөөд агуулгыг **харагдах байдлаас болон accessibility tree-ээс** хоёулангаас нь нууж өгдөг. Үүнийг JavaScript ашиглан амархан идэвхжүүлж эсвэл идэвхгүй болгож болно.

Эцэст нь хэлэхэд чухал агуулгыг нуухдаа болгоомжтой байх хэрэгтэй. Хэрэв мэдээлэл нь вэбсайтыг ойлгох эсвэл ашиглахад чухал бол энэ нь **бүх хэрэглэгчдэд харагдах болон хүртээмжтэй** байх ёстой. Агуулгыг зөвхөн хэрэглэгчийн туршлагыг үнэхээр сайжруулах үед л нуух хэрэгтэй. Эдгээр шилдэг практикуудыг дагаснаар таны агуулга вэбсайттай хэрхэн харилцаж байгаагаас үл хамааран бүх хэрэглэгчдэд хүртээмжтэй хэвээр байх боломжтой.
