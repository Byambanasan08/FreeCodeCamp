# Background-д хамаарах Accessibility-ийн зарим анхаарах зүйлс юу вэ?

Вэб дизайнд background нь веб хуудасны ерөнхий харагдах байдал болон мэдрэмжийг тодорхойлоход чухал үүрэгтэй.

Гэвч background ашиглан дизайн хийхдээ accessibility-г заавал анхаарах шаардлагатай. Ингэснээр таны контент бүх хэрэглэгчдэд, тэр дундаа харааны бэрхшээлтэй хэрэглэгчдэд ч ашиглахад тохиромжтой, уншихад ойлгомжтой байна.

Background-тэй холбоотой accessibility-ийн гол асуудлуудын нэг нь **background болон текстийн хооронд хангалттай contrast байх** явдал юм.

Хэрэв contrast хангалтгүй байвал харааны бэрхшээлтэй хэрэглэгчид, тухайлбал хараа муутай эсвэл өнгө ялгах чадварын алдагдалтай хүмүүс таны хуудасны контентыг уншихад хүндрэлтэй байж болно.

**Contrast** гэдэг нь хоёр өнгийн хоорондох **гэрэл болон харанхуйгийн ялгаа** юм. Background өнгө болон текстийн өнгийн хооронд хангалттай contrast байх нь уншигдах чадварт маш чухал.

**Web Content Accessibility Guidelines (WCAG)** нь дараах хамгийн бага contrast ratio-г зөвлөдөг.

* **Ердийн текстийн хувьд: 4.5:1**
* **Том хэмжээний текстийн хувьд: 3:1**

Жишээлбэл, цайвар саарал background дээр цагаан текст байрлуулах нь муу contrast үүсгэж, текстийг уншихад хэцүү болгодог. Харин хар хөх background дээр цагаан текст байрлуулах нь сайн contrast үүсгэж, бүх хэрэглэгчдэд уншихад илүү ойлгомжтой болгодог.

Доорх нь **муу contrast**-ийн жишээ юм.

```html id="3f5tq4"
<p style="color: lightgray; background-color: whitesmoke;">
   This is an example of poor contrast.
</p>
```

Одоо **сайн contrast**-ийн жишээг харцгаая.

```html id="rqmtqf"
<p style="color: white; background-color: darkslategray;">
  This is an example of good contrast.
</p>
```

Өөр нэг анхаарах зүйл бол текстийг **хэт төвөгтэй эсвэл завгүй background** дээр байрлуулахаас зайлсхийх явдал юм. Жишээлбэл, олон өнгөтэй зураг эсвэл gradient дээр текст байрлуулах нь текстийг background-оос ялгаж харахад хүндрэл учруулж болзошгүй.

Background дизайн хийхдээ мэдээллийг дамжуулахдаа **өнгийг цорын ганц арга болгон ашиглахаас зайлсхийх** хэрэгтэй. Жишээлбэл, алдаа эсвэл амжилтын мессежийг зөвхөн өнгөөр илэрхийлэх (алдааг улаан, амжилтыг ногоон өнгөөр) нь өнгө ялгах бэрхшээлтэй хэрэглэгчдэд асуудал үүсгэж болно.

Өнгөөс гадна та **тэмдэгт эсвэл текст** ашиглан мэдээллийг дамжуулах хэрэгтэй. Жишээлбэл, улаан өнгийн алдааны мессежийн хажууд **икон эсвэл тод (bold) текст** ашиглан алдаа гарсныг илүү тодорхой болгох боломжтой.

Хэдийгээр түгээмэл биш боловч **background аудио эсвэл видео** мөн accessibility-д нөлөөлж болно.

Background хөгжим эсвэл автоматаар тоглогддог видео нь зарим хэрэглэгчдэд, ялангуяа танин мэдэхүйн бэрхшээлтэй хүмүүст анхаарал сарниулах нөлөө үзүүлж болно. Хэрэв та background аудио ашиглаж байгаа бол хэрэглэгчид дууг **mute хийх эсвэл pause хийх боломж** заавал өгөх хэрэгтэй.

Эдгээр accessibility-ийн анхаарах зүйлсийг харгалзан үзсэнээр та ямар ч чадвартай хэрэглэгчдэд **уншихад ойлгомжтой, ашиглахад тохиромжтой, илүү хүртээмжтэй дизайн** бүтээх боломжтой болно.
