**Inline, Internal, External CSS гэж юу вэ, мөн аль тохиолдолд аль нэгийг нь ашиглах вэ?**

CSS-ийг вэб хуудсанд **гурван үндсэн аргаар** хэрэгжүүлж болдог: **inline**, **internal**, эсвэл **external**.

Арга тус бүр өөрийн гэсэн **ашиглах нөхцөл (use case)**, **давуу тал**, **хязгаарлалт**-тай байдаг. Аль аргыг хэзээ хэрэглэхээ зөв ойлгох нь **цэвэр, үр ашигтай, арчлахад хялбар (maintainable)** код бичихэд маш чухал.

Одоо CSS-ийн эдгээр гурван төрлийг задлан тайлбарлаж, хэзээ ашиглах ёстойг нь авч үзье.

---

### **Inline CSS**

**Inline CSS** нь HTML элементийн дотор шууд, `style` атрибут ашиглан бичигддэг. Энэ нь **тухайн нэг элементэд** л загвар хэрэгжүүлнэ.

Inline CSS ашигласан жишээ:

```html
<p style="color: green;">This is an inline-styled paragraph.</p>
```

Энэ жишээнд `style` атрибутыг ашиглан paragraph-ийн текстийн өнгийг **ногоон** болгож байна.

Inline CSS-ийг ихэвчлэн:

* хурдан, нэг удаагийн (one-off) загвар хэрэглэх,
* эсвэл тодорхой нэг элемент дээр бусад загваруудыг **override** хийх
  тохиолдолд ашигладаг.

Гэхдээ ихэнх тохиолдолд inline CSS-ийг **зайлсхийх хэрэгтэй**, учир нь:

* HTML кодыг замбараагүй болгодог,
* кодыг арчлах, өөрчлөхөд илүү төвөгтэй болгодог.

Ихэнх үед **internal** эсвэл **external CSS** ашиглах нь загваруудыг илүү эмх цэгцтэй, арчлахад хялбар байлгадаг.

---

### **Internal CSS**

**Internal CSS** нь HTML баримтын **`head` хэсэгт**, `style` тагийн дотор бичигддэг. Энэ нь **тухайн хуудсын бүх элементүүдэд** загвар хэрэгжүүлнэ.

Internal CSS-ийн жишээ:

```html
<head>
  <style>
    p {
      color: blue;
    }
  </style>
</head>
<body>
  <p>This paragraph is styled using internal CSS.</p>
</body>
```

Дээрх жишээнд internal CSS нь тухайн хуудсан дахь **бүх `p` элементүүдийн** текстийг цэнхэр өнгөтэй болгож байна.

Internal CSS-ийг:

* зөвхөн **нэг HTML хуудас** дээр загвар хэрэглэх шаардлагатай үед,
* нэг хуудастай вэб сайт (single-page website),
* эсвэл өөр газар дахин ашиглах шаардлагагүй загваруудын үед
  ашиглахад тохиромжтой.

Гэхдээ сул талууд бий:

* олон хуудас хооронд **дахин ашиглах боломжгүй**,
* inline CSS-тэй адил HTML ба CSS-ийг хольж бичдэг тул
* томоохон төслүүдэд кодыг арчлахад **хэцүү** болдог.

---

### **External CSS**

**External CSS** нь тусдаа `.css` файлд бичигдэж, HTML баримтад `head` хэсэгт байрлах `link` элементийг ашиглан холбогддог.

Энэ нь **олон хуудсанд ижил загвар** хэрэгжүүлэх боломж олгодог бөгөөд **мэргэжлийн вэб хөгжүүлэлтэд хамгийн түгээмэл, зөвлөмжтэй арга** юм.

Paragraph элементүүдийг загварчлах жишээ:

```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <p>This paragraph is styled using external CSS.</p>
</body>
```

```css
p {
  color: red;
}
```

Өмнөх хичээлүүдэд та `link` элементийн:

* `rel` атрибут нь одоогийн баримт болон холбогдож буй нөөц (resource)-ийн хоорондын **харилцааг** заадаг (жишээ нь stylesheet),
* `href` атрибут нь холбогдож буй нөөцийн **URL**, өөрөөр хэлбэл файлыг хаанаас ачаалахыг заадаг
  гэдгийг сурсан.

External CSS нь:

* **томоохон төслүүдэд**,
* олон хуудсанд **нэгэн жигд загвар** хэрэгжүүлэх үед
  хамгийн тохиромжтой.

Энэ арга нь:

* **separation of concerns** зарчмыг баримталж (HTML нь бүтэц, CSS нь загвар),
* кодыг илүү **арчлахад хялбар**, **өргөтгөх боломжтой (scalable)** болгодог.

---

### **Дүгнэлт**

CSS-ийн аль төрлийг хэзээ ашиглахаа ойлгох нь **үр ашигтай, зөв вэб хөгжүүлэлтэд** маш чухал.

Ихэнх тохиолдолд, ялангуяа **том, төвөгтэй төслүүдэд**, **external CSS** нь таны хамгийн зөв сонголт байх ёстой.
