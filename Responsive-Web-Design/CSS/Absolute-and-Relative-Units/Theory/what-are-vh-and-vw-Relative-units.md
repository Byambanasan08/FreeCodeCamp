**vh болон vw Units гэж юу вэ, мөн хэзээ ашиглах ёстой вэ?**

CSS-д **vh** болон **vw** нь **viewport-д суурилсан relative unit**-ууд бөгөөд элементүүдийн хэмжээг **browser-ийн цонхны хэмжээнээс хамааруулан** тодорхойлох боломжийг олгодог. Эдгээр unit-ууд нь өөр өөр дэлгэцийн хэмжээнд дасан зохицдог **responsive дизайн** бүтээхэд онцгой хэрэгтэй.

**vh** нь **“viewport height”** гэсэн утгатай бөгөөд **1vh** нь viewport-ийн өндрийн **1%**-тай тэнцүү байдаг.

Үүнтэй адил **vw** нь **“viewport width”** гэсэн утгатай бөгөөд **1vw** нь viewport-ийн өргөний **1%**-тай тэнцүү байдаг.

Энэ нь хэрэв та нэг элементийн **height**-ийг **100vh** гэж тохируулбал тухайн элемент нь төхөөрөмжийн бодит pixel хэмжээнээс үл хамааран **viewport-ийн бүх өндрийг** эзэлнэ гэсэн үг юм.

Эдгээр unit-ууд нь **full-screen layout** бүтээх эсвэл дэлгэцийн тодорхой пропорцыг хадгалдаг элементүүд хийх үед маш хэрэгтэй байдаг.

Жишээлбэл, та дэлгэцийг бүхэлд нь дүүргэдэг **hero section** бүтээхдээ эдгээр unit-уудыг ашиглаж болно (жишээг туршихын тулд interactive editor-ийг идэвхжүүлэх шаардлагатай).

<link rel="stylesheet" href="styles.css" />

<section class="hero">
  <h1>100vh / 100vw Example</h1>
  <p>This section fills the entire browser window.</p>
</section>

---css---

body {
  margin: 0;
  font-family: sans-serif;
  border: 5px dashed #333;
}

.hero {
  height: 100vh;
  width: 100vw;
  background-color: #add8e6; 
  padding: 2em;
}

.hero h1 {
  font-size: 2em;
  margin-bottom: 0.5em;
}

.hero p {
  font-size: 1em;
}


Энэхүү CSS нь **hero section** нь ямар төхөөрөмж ашиглаж байгаагаас үл хамааран **viewport-ийн яг хэмжээтэй** үргэлж таарч байхыг баталгаажуулдаг.

**vh** болон **vw** нэгжүүдийг **typography**-д ашиглаж, дэлгэцийн хэмжээнээс хамааран өөрчлөгддөг **responsive текстийн хэмжээ** үүсгэж болно.

Interactive editor-ийг идэвхжүүлээд preview цонхны хэмжээг өөрчилж үзээрэй. Ингэснээр текст **viewport-ийн хэмжээнээс хамааран хэрхэн масштаблагдаж байгааг** харах боломжтой.

<link rel="stylesheet" href="styles.css" />
<h1>Responsive Heading</h1>
<p>The h1 heading scales with the viewport width.</p>

---css---
h1 {
  font-size: 5vw;
}

Энэ нь **h1** элементүүдийн **font size**-ийг **viewport-ийн өргөний 5%**-тай тэнцүү болгоно. Ингэснээр текст нь browser-ийн цонхны хэмжээ өөрчлөгдөхөд **жигд масштаблагдана**.

**vh** болон **vw** нэгжүүдийн нэг давуу тал нь viewport-ийн хэмжээ өөрчлөгдөхөд **бодит цаг дээр хариу үйлдэл үзүүлдэг** явдал юм. Өөрөөр хэлбэл хэрэглэгч browser-ийн цонхны хэмжээг өөрчилбөл эдгээр нэгжээр хэмжээг нь тохируулсан элементүүд **хуудсыг дахин ачаалах шаардлагагүйгээр** автоматаар өөрчлөгдөнө. Гэхдээ эдгээр нэгжүүдийг **болгоомжтой ашиглах** хэрэгтэй. Жишээлбэл **font size**-ийг зөвхөн **vw** нэгжээр тохируулбал нарийн дэлгэц дээр текст **хэт жижиг**, өргөн дэлгэц дээр **хэт том** болох магадлалтай.

Өөр нэг анхаарах зүйл бол **mobile төхөөрөмжүүд** дээр browser-ийн **address bar** гарч ирэх эсвэл алга болох үед **viewport-ийн өндөр өөрчлөгдөж** болдог. Хэрэв та **vh** нэгжийг их хэмжээгээр ашиглаж байгаа бол энэ нь **layout гэнэт шилжих** зэрэг санаанд оромгүй үр дүн үүсгэж болно.

Дүгнэж хэлбэл, **vh** болон **vw** нэгжүүд нь viewport-ийн хэмжээнд дасан зохицдог **responsive layout** болон элементүүд бүтээхэд маш хүчтэй хэрэгсэл юм. Эдгээр нь ялангуяа **full-screen section**, **aspect ratio хадгалах**, болон **жигд масштаблагддаг дизайн** хийхэд маш тохиромжтой. Гэсэн хэдий ч хамгийн сайн хэрэглэгчийн туршлагыг бий болгохын тулд эдгээрийг **болгоомжтой ашиглаж**, ихэвчлэн бусад CSS аргачлалуудтай **хослуулан хэрэглэх** шаардлагатай.
