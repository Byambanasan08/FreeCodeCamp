### CSS Reset гэж юу вэ, мөн түгээмэл жишээнүүд юу байдаг вэ?

**CSS reset** гэдэг нь веб браузерууд HTML элементүүдэд автоматаар хэрэглэдэг **анхдагч форматчлалыг (default formatting)** бүхэлд нь эсвэл хэсэгчлэн устгадаг stylesheet юм. Жишээлбэл, та зарим элементүүд, тухайлбал **paragraph (`p`)** болон **heading (`h1`, `h2` гэх мэт)** элементүүдэд өөрийн CSS-д тодорхой заагаагүй байсан ч **анхдагч margin** аль хэдийн байдаг гэдгийг анзаарсан байж магадгүй.

Мөн энэ нь HTML элементүүдийн олон property дээр ажиглагддаг. Түүнчлэн өөр өөр браузерууд HTML элементүүдэд **өөр өөр анхдагч стиль** хэрэглэж болдог. Жишээлбэл, **Google Chrome** дээрх анхдагч стиль нь **Mozilla Firefox** эсвэл **Microsoft Edge** дээрхтэй яг ижил байх албагүй. Энэ нь браузер бүр дээр **өөр өөр харагдах стиль** үүсгэж болзошгүй бөгөөд үүнийг аль болохоор **зайлсхийх хэрэгтэй**.

Үүнийг шийдэхийн тулд **CSS reset** ашиглан анхдагч стильүүдийг устгадаг. Анхдагч стильүүдийг бүхэлд нь эсвэл хэсэгчлэн устгаснаар дизайны хувьд **нэгэн жигд эхлэл (consistent baseline)** бий болгож, браузер болон төхөөрөмжүүдийн хооронд үүсэх **зөрүүг багасгах** боломжтой болдог. Анхдагч стильүүдийг устгах нь стиль бичих процессыг хялбар болгодог. Учир нь та зөвхөн өөрийн stylesheet-д **ил тодорхой заасан стильүүдийг** л харах болно.

CSS reset хийх **хоёр үндсэн арга** байдаг:

* өөрийн **custom CSS reset** бичих
* **third-party CSS reset** ашиглах

---

### Custom CSS Reset

**Custom CSS reset** гэдэг нь таны төслийн хэрэгцээнд тохируулан **эхнээс нь өөрөө бичсэн stylesheet** юм. Ингэснээр ямар стиль reset хийхийг **бүрэн хянах боломжтой** байдаг. Гэхдээ үүнийг хөгжүүлэх болон засварлахад **цаг зарцуулах шаардлагатай**.

Доорх нь бүх HTML элементүүдийн **margin болон padding-ийг reset хийх** маш түгээмэл CSS дүрмийн жишээ юм.

```html id="9u0u0g"
<link rel="stylesheet" href="styles.css">
<h1>Example Heading</h1>
<p>This is a paragraph.</p>
```

```css id="m8r4k2"
* {
  margin: 0;
  padding: 0;
}
```

Энэ дүрмийг ихэвчлэн CSS stylesheet-ийн **дээд хэсэгт** бичдэг. `*` selector нь **wildcard selector** бөгөөд бүх HTML элементүүдтэй таардаг. Ингэснээр бүх элементүүдийн **margin болон padding дөрвөн талдаа 0** болно.

Энэ нь дизайныг эхлэх **анхны суурь** болдог бөгөөд дараа нь stylesheet-ийн доод хэсэгт илүү **нарийвчилсан selector-ууд** ашиглан шаардлагатай стильүүдийг тохируулж болно.

Та энэ аргыг ашиглан дурын HTML элементийг сонгож, түүний анхдагч стильүүдийг reset хийж болно. Үүний тулд custom stylesheet үүсгээд тохирох **CSS selector** ашиглан элементүүдийг сонгож стильүүдийг тохируулна.

---

### Third-party CSS Reset

Гэхдээ энэ процесс нэлээд **цаг шаардсан ажил** байж болно. Хэрэв та цаг хэмнэхийг хүсвэл **third-party CSS reset** ашиглаж болно. Эдгээр stylesheet-үүд нь аль хэдийн **бэлэн бүтээгдсэн** байдаг тул та тэдгээрийг татаж аваад төслдөө шууд нэмэх боломжтой.

Third-party CSS reset-ийн нэг сайн жишээ бол **Normalize.css** юм. Энэ stylesheet нь HTML элементүүдийн стильүүдийг **нэгэн жигд болгох (normalize)** үүрэгтэй боловч **ашигтай анхдагч стильүүдийг** бүрэн устгахгүйгээр хадгалдаг. Ялангуяа **accessibility-д чухал стильүүдийг** хадгалж үлдээдэг. Мөн түгээмэл тохиолддог **browser bug болон стиль зөрүүг** засдаг.

Өөр нэг сонголт бол **sanitize.css** юм. Энэ нь орчин үеийн бүх том браузерууд дээр **анхдагч стильүүдийг нэгэн жигд болгох** зорилготой CSS library юм. Энэ library нь **Normalize.css**-тэй хамт хөгжүүлэгддэг тул тэд хамтдаа шинэчлэгдэж байдаг. Мөн **forms**, **typography** зэрэг тодорхой зорилгод зориулсан тусдаа stylesheet-үүдийг татаж авах боломжтой.

---

Олон сонголт байдаг боловч та өөрийн төслийн хэрэгцээнд хамгийн тохирохыг нь сонгох хэрэгтэй. Мөн **third-party CSS reset** болон **custom reset**-ийг хамтад нь ашиглах боломжтой.

CSS reset ашиглах үед **accessibility**-ийг мөн анхаарах нь чухал. Таны веб аппликейшн **бүх хэрэглэгчдэд хүртээмжтэй** байх ёстой. Тиймээс **screen reader** болон бусад **assistive technology**-д хэрэгтэй байж болох стильүүдийг reset хийхээс зайлсхийх хэрэгтэй.

Мөн эдгээр нэмэлт stylesheet-үүд нь таны аппликейшний **performance**-д хэрхэн нөлөөлж болохыг бодолцох хэрэгтэй. Учир нь custom стильүүд хэрэгжихээс өмнө тэдгээр stylesheet-үүд **татагдаж (download)** дуусах шаардлагатай байдаг.

Анхдагч стильүүдийг устгаснаар CSS reset нь дизайнаа хэрэгжүүлэх **цэвэр эхлэл (blank starting point)** өгдөг. Ингэснээр браузер болон төхөөрөмжүүдийн хооронд **илүү нэгэн жигд, тогтвортой хэрэглэгчийн туршлага** бий болдог.
