### Формыг загварчлах тойм

### Input элементүүдийг загварчлах шилдэг арга барилууд

**Styling Inputs:** Бүх текст элементүүдийн адил текст input-уудад хэрэглэх хэв загварууд **хүртээмжтэй (accessible)** байх ёстой. Энэ нь фонтын хэмжээ хангалттай том байх, мөн текстийн өнгө нь арын дэвсгэртэй **хангалттай контрасттай** байх шаардлагатай гэсэн үг юм. Input элементүүд мөн **focus авах боломжтой** байдаг. Та хэв загвараа засварлаж байх үед элемент focus авсан үед харагдах **илэрхий тэмдэглэгээг** хадгалж үлдэхэд анхаарах хэрэгтэй. Жишээлбэл, тод хүрээ (bold border) зэрэг нь ийм тэмдэглэгээ болж чадна.

---

### Input элементүүдэд `appearance: none` ашиглах

**appearance: none:** Браузерууд олон HTML элементүүдэд анхдагч хэв загвар (default styling)-ыг автоматаар хэрэглэдэг. `appearance: none` CSS property нь танд загварчлал дээр **бүрэн хяналт** олгодог боловч зарим **хязгаарлалттай** байдаг. Input элементүүдэд зориулсан custom хэв загвар бүтээх үед **focus болон error indicator-ууд** хэвээр хадгалагдаж байгаа эсэхийг заавал шалгах шаардлагатай.

---

### `datetime-local` болон `color` input-уудыг загварчлах үед гардаг нийтлэг асуудлууд

**Common Issues:** Эдгээр тусгай төрлийн input-ууд нь **date picker** болон **color picker** зэрэг интерфэйсийг бий болгохын тулд **нарийн бүтэцтэй pseudo-element-үүдэд** тулгуурладаг. Энэ нь эдгээр input-уудыг загварчлахад нэлээд том хүндрэл үүсгэдэг. Үүний нэг асуудал нь анхдагч хэв загвар (default styling) нь **бүрэн браузераас хамааралтай** байдагт оршино. Тиймээс picker-ийг хүссэнээр харагдуулахын тулд бичсэн CSS нь нэг браузер дээр зөв харагдаж болох ч өөр браузер дээр **огт өөр үр дүн** гаргах боломжтой.
