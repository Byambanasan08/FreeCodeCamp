# CSS тойм

1. CSS-ийн үндэс

```
1.1 CSS гэж юу вэ?: Cascading Style Sheets (CSS) нь HTML элементүүдэд стиль хэрэглэхэд ашиглагддаг тэмдэглэгээний(markup) хэл юм. CSS нь өнгө, background зураг, layout зэрэг олон зүйлийг тодорхойлоход хэрэглэгддэг.
1.2 CSS дүрмийн үндсэн бүтэц: CSS дүрэм нь хоёр үндсэн хэсгээс бүрдэнэ: selector болон declaration block. Selector нь тодорхой HTML элементүүдийг сонгож стиль хэрэглэхэд ашиглагддаг. Declaration block нь тухайн selector-д хамаарах стилүүдийн багцыг тодорхойлдог.
1.3 meta name="viewport" элемент: Энэ meta элемент нь browser-д хуудсын хэмжээ болон масштаблалыг янз бүрийн төхөөрөмж (ялангуяа гар утас, таблет)-д хэрхэн тохируулахыг зааж өгдөг.
1.4 Browser-ийн анхдагч стиль: HTML элемент бүр browser-аас өгөгдсөн анхдагч стильтэй байдаг. Үүнд ихэвчлэн default margin болон padding орно.
```

2. Inline, Internal, External CSS

```
2.1 Inline CSS: HTML элемент дотор style attribute ашиглан шууд бичигддэг. Ихэнх тохиолдолд separation of concerns зарчмын улмаас inline CSS ашиглахгүй.
2.2 Internal CSS: HTML баримтын head хэсгийн <style> tag дотор бичигддэг. Жижиг жишээ кодод ашиглахад тохиромжтой боловч ихэнхдээ ашиглагддаггүй.
2.3 External CSS: Тусдаа CSS файлд бичигдэж, HTML баримттай link элементээр холбогддог. Ихэнх төслүүдэд external CSS ашиглах нь зөв сонголт байдаг.
```

3. width болон height property-тэй ажиллах

```
3.1 width property: Элементийн өргөнийг тодорхойлно. Хэрэв width заагаагүй бол default утга нь auto байдаг. Энэ нь browser тухайн элементийн өргөнийг контент, эцэг элемент, display төрлөөс хамааран тооцоолно гэсэн үг.
3.2 min-width property: Элементийн хамгийн бага өргөнийг тодорхойлно.
3.3 max-width property: Элементийн хамгийн их өргөнийг тодорхойлно.
3.4 height property: Элементийн өндрийг тодорхойлно. Default утга нь мөн auto бөгөөд доторх контентоос хамааран өөрчлөгдөнө.
3.5 min-height property: Элементийн хамгийн бага өндрийг тодорхойлно.
3.6 max-height property: Элементийн хамгийн их өндрийг тодорхойлно.
```

4. CSS combinator-уудын төрөл

```
4.1 Descendant combinator (зай): Энэ нь тодорхой parent элементийн доторх бүх descendant элементүүдийг сонгоход ашиглагдана.

4.2 Child combinator (>): Энэ нь зөвхөн шууд child элементүүдийг сонгоно.

4.3 Next-sibling combinator (+): Энэ нь тодорхой элементээс шууд дараагийн sibling элементийг сонгоно.

4.4 Subsequent-sibling combinator (~): Энэ нь тухайн элементээс хойших бүх sibling элементүүдийг сонгоно.
```

5. Inline, Block, Inline-block элементүүд

```
5.1 Inline элементүүд: Зөвхөн хэрэгтэй хэмжээгээрээ өргөн эзэлж, шинэ мөр эхлүүлдэггүй. Контент дотор урсаж байрладаг. Жишээ нь span, anchor, img элементүүд.
5.2 Block элементүүд: Default-аар боломжит бүх өргөнийг эзэлж, шинэ мөрнөөс эхэлдэг. Жишээ нь div, p, section элементүүд.
5.3 Inline-block элементүүд: display property ашиглан inline-block болгож болно. Inline шиг мөрөнд байрлах боловч block шиг width, height тохируулах боломжтой.
```

6. Margin ба Padding

```
6.1 margin property: Элементийн гаднах зайг (border болон бусад элементүүдийн хоорондох зай) тодорхойлно.
6.2 margin shorthand: 1–4 утгаар бүх талын margin-ийг тохируулж болно. 1 утга — бүх талд; 2 утга — дээд/доод, баруун/зүүн; 3 утга — дээд, хажуу (баруун/зүүн), доод; 4 утга — дээд, баруун, доод, зүүн.
6.3 padding property: Элементийн доторх зайг (контент болон border-ийн хооронд) тодорхойлно.
6.4 padding shorthand: 1–4 утгаар бүх талын padding-ийг тохируулж болно. 1 утга — бүх талд; 2 утга — дээд/доод, баруун/зүүн; 3 утга — дээд, хажуу (баруун/зүүн), доод; 4 утга — дээд, баруун, доод, зүүн.
```

7. CSS Specificity

```
7.1 Inline CSS Specificity: Inline CSS нь хамгийн өндөр specificity-тэй байдаг, учир нь энэ нь элементэд шууд хэрэглэгддэг. Энэ нь internal болон external CSS-ийг дарж үйлчилнэ. Inline style-ийн specificity утга нь (1, 0, 0, 0) байна.
7.2 Internal CSS Specificity: Internal CSS нь HTML баримтын head хэсэг доторх style элементэд тодорхойлогддог. Энэ нь inline style-аас бага specificity-тэй. Internal болон external CSS нь ижил specificity түвшинтэй бөгөөд энэ нь байршлаасаа биш selector-оосоо хамаардаг.
7.3 External CSS Specificity: External CSS нь head хэсэгт link элементээр холбогдож, тусдаа .css файлд бичигддэг. Internal CSS-тэй адил specificity нь selector-оор тодорхойлогдоно. Хэрэв хоёр дүрэм ижил specificity-тэй бол source order буюу дарааллаар шийдэгдэнэ — сүүлд бичигдсэн дүрэм давуу эрхтэй. External CSS нь томоохон төслүүдэд хамгийн сайн maintainability олгодог.
7.4 Universal Selector (*): Энэ нь баримт дахь бүх элементтэй таарах selector юм. Ихэвчлэн бүх элементэд стиль хэрэглэхэд ашиглагддаг бөгөөд browser-уудын хооронд стилүүдийг reset эсвэл normalize хийхэд хэрэгтэй. Universal selector нь хамгийн бага specificity-тэй бөгөөд (0, 0, 0, 0) утгатай.
7.5 Type Selectors: Эдгээр selector-ууд нь tag нэрээр элементүүдийг сонгодог. Type selector нь харьцангуй бага specificity-тэй бөгөөд (0, 0, 0, 1) утгатай.
7.6 Class Selectors: Эдгээр selector-ууд нь цэг (.) болон class нэрээр тодорхойлогдоно. Specificity нь (0, 0, 1, 0) байна. Энэ нь class selector нь type selector-оос давуу боловч ID selector болон inline style-д дарагдана гэсэн үг.
7.7 ID Selectors: ID selector нь (#) тэмдэг болон ID нэрээр тодорхойлогдоно. Энэ нь type болон class selector-оос өндөр specificity-тэй боловч inline style-аас бага. Specificity нь (0, 1, 0, 0).
7.8 !important түлхүүр үг: Энэ нь style дүрэмд хамгийн өндөр давуу эрх өгөхөд ашиглагддаг. Ашигласан тохиолдолд бусад selector-ийн specificity-ээс үл хамааран тухайн style хэрэгжинэ. Гэхдээ !important-ийг болгоомжтой ашиглах хэрэгтэй, учир нь CSS-ийг засварлах, ойлгоход хүндрэл учруулж болзошгүй.
7.9 Cascade Algorithm: Нэг элементэд олон style давхар хэрэглэгдсэн үед аль дүрмийг хэрэгжүүлэхийг шийдэх алгоритм. Энэ нь тодорхой дүрмүүдийн дагуу хамгийн тохиромжтой style-ийг сонгож хэрэглэнэ.
7.10 CSS Inheritance: Parent элементээс child элементүүд рүү стиль дамжих процесс. Энэ нь нэг удаа дээд түвшинд стиль тодорхойлоод олон элементэд давхар заахгүйгээр хэрэглэх боломжийг олгодог.
```

8. Дизайны нэр томьёо

```
8.1 Layout: Хуудас эсвэл дэлгэц дээрх визуал элементүүдийг (текст, зураг, хоосон зай гэх мэт) хэрхэн байрлуулахыг хэлнэ.
8.2 Alignment: Элементүүдийг хооронд нь хэрхэн зэрэгцүүлэн байрлуулахыг хэлнэ. Зөв alignment нь дизайныг цэвэр, эмх цэгцтэй харагдуулна.
8.3 Composition: Элементүүдийг зохион байгуулж уялдаа холбоотой дизайн бүтээх үйл явц.
8.4 Balance: Дизайн доторх визуал жинг хэрхэн хуваарилж байгааг хэлнэ. Симметрик эсвэл асимметрик байдлаар тэнцвэрийг бий болгодог.
8.5 Scale: Нэг элементийн хэмжээг нөгөөтэй нь харьцуулах ойлголт.
8.6 Hierarchy: Дизайн дахь элементүүдийн чухал байдлын дарааллыг тогтооно. Хамгийн чухал мэдээлэл эхэнд анзаарагдах ёстой.
8.7 Contrast: Элементүүдийн хооронд тод ялгаа гаргах үйл явц (өнгө, хэмжээ, хэлбэр гэх мэт). Сайн contrast нь уншихад хялбар болгодог.
8.8 White Space (Negative Space): Дизайн дахь хоосон зай, элементүүдийг хүрээлсэн хэсэг.
8.9 UI (User Interface): Хэрэглэгчийн дэлгэц дээр харагдах бүх визуал болон интерактив элементүүд (icon, зураг, текст, menu, холбоос, button).
8.10 UX (User Experience): Хэрэглэгч бүтээгдэхүүн эсвэл үйлчилгээг ашиглах үед ямар мэдрэмж авч байгааг хэлнэ. Сайн UX нь ойлгомжтой, ашиглахад хялбар, үр ашигтай, хүртээмжтэй, таатай байдаг.
8.11 Design Brief: Төслийн зорилго, шаардлага, чиглэлийг тодорхойлсон баримт бичиг.
8.12 Vector Based Design: Шугам, дүрс, өнгийг математик томьёогоор тодорхойлж дижитал зураг үүсгэх арга.
8.13 Prototyping: Бүтээгдэхүүн эсвэл UI-ийн интерактив загвар (model) бүтээх үйл явц.
```

9. UI Design-ийн үндэс

```
9.1 Background ба foreground өнгөний сайн contrast: Текст уншихад хялбар байлгахын тулд өнгөнүүдийн ялгаа хангалттай байх ёстой. Web Content Accessibility Guidelines (WCAG) нь энгийн текстэд хамгийн багадаа 4.5:1, том текстэд 3:1 харьцааг зөвлөдөг.
9.2 Сайн visual hierarchy: Нүд мэдээллийг зөв дарааллаар хүлээн авах боломжтой байдлаар элементүүдийг зохион байгуулах.
9.3 Responsive зураг: Дэлгэцийн хэмжээнээс хамааран өөрчлөгдөж, бүх төхөөрөмж дээр зөв харагдах зураг.
9.4 Progressive Enhancement: Бүх хэрэглэгчид (browser, төхөөрөмжөөс үл хамааран) үндсэн контент болон функцийг ашиглах боломжтой байхаар дизайн хийх арга.
9.5 User-centered design: Хэрэглэгчийн хэрэгцээ, сонголт, хязгаарлалтад тулгуурласан дизайн. Зорилго нь ойлгомжтой, үр ашигтай, хэрэглэхэд таатай веб хуудас бүтээх.
9.6 User Research: Хэрэглэгчдийн хэрэгцээ, зан төлөв, асуудлыг судлах системтэй процесс.
9.7 User Testing: Хэрэглэгчид аппликейшнийг ашиглах үед өгөгдөл цуглуулах үйл явц.
9.8 A/B Testing: Шинэ feature-ийг хэрэглэгчдийн хэсэгт туршиж, үр дүнг analytics-аар үнэлэх арга.
9.9 User Requirements: Аппликейшн дагах шаардлага, хэрэглэгчийн хэрэгцээ. Судалгаа, стандарт эсвэл stakeholder-уудаас тодорхойлогдож болно.
9.10 Progressive Disclosure: Хэрэглэгчид зөвхөн тухайн үед хэрэгтэй мэдээллийг харуулж, бусдыг нуух дизайн арга.
9.11 Deferred/Lazy Registration: Хэрэглэгчийг бүртгүүлэхгүйгээр эхлээд аппликейшнийг ашиглах боломж олгодог UI загвар.
```

10. Дизайны шилдэг практикууд

```
10.1 Dark Mode: Энэ нь веб аппликейшнд default цайвар өнгөний схемийг бараан өнгөний схемд шилжүүлэх боломж олгодог онцлог юм. Dark mode-д ханалт багатай (desaturated) өнгө ашиглах хэрэгтэй. Эдгээр нь илүү зөөлөн, бага тод өнгөнүүд байдаг.
10.2 Breadcrumbs: Энэ нь хэрэглэгч сайтын бүтэц дотор хаана байгааг харуулдаг navigation хэрэгсэл юм. Breadcrumb-ийг хуудасны дээд хэсэгт байрлуулах нь тохиромжтой. Мөн уншихад хялбар хэмжээтэй байх ёстой ч хэт том байж зай их эзлэхгүй байх хэрэгтэй.
10.3 Card Component: Card компонент нь энгийн, эмх цэгцтэй дизайнтай байх ёстой бөгөөд хэт их мэдээлэл харуулахгүй байх хэрэгтэй. Media ашиглахдаа өндөр чанартай зураг, видео сонгож хэрэглэгчийн туршлагыг сайжруулах хэрэгтэй.
10.4 Infinite Scroll: Энэ нь хэрэглэгч доош гүйлгэхэд илүү их контент ачаалдаг дизайн загвар юм. "Load more" товч ашиглахыг бодолцох хэрэгтэй, учир нь энэ нь хэрэглэгчид контентыг хэзээ үргэлжлүүлж үзэхээ өөрсдөө шийдэх боломж олгодог. Мөн "back" товч нэмснээр хэрэглэгч өмнөх хэсэг рүү буцахдаа дахин дээш гүйлгэх шаардлагагүй болно.
10.5 Modal Dialog: Энэ нь одоогийн хуудасны контентын дээр гарч ирдэг popup төрөл юм. Ихэвчлэн арын контент бүдгэрсэн overlay-тэй байдаг бөгөөд ингэснээр хэрэглэгч modal дээр анхаарлаа төвлөрүүлдэг. Мөн modal-оос гадна дарж хаах боломж олгох нь зүйтэй. HTML dialog элемент ашиглавал олон функц болон accessibility давуу талууд автоматаар хангагдана.
10.6 Form бүртгэлийн progress indication: Энэ нь хэрэглэгч ямар шатанд явж байгааг харуулдаг. Form, бүртгэл, тохиргооны процессуудад ашиглагддаг. Дизайн нь энгийн, амархан харагдахуйц бөгөөд өмнөх алхам руу буцах боломжтой байх ёстой.
10.7 Shopping Cart: Энэ нь хэрэглэгч e-commerce платформ дээр сонгосон бараагаа харах хэсэг юм. Cart нь үргэлж харагдаж байх, нийтлэг icon (сагс, уут гэх мэт) ашиглах, мөн checkout руу шилжих тодорхой call-to-action товчтой байх ёстой.
```

11. Түгээмэл дизайны хэрэгслүүд

```
11.1 Figma: Энэ нь cloud-д суурилсан UI/UX дизайны хэрэгсэл юм. Багууд хаанаас ч хамтран ажиллах боломжтой бөгөөд vector design, auto layout, сэтгэгдэл болон feedback систем зэрэг олон боломжтой.
11.2 Sketch: Хялбар, ойлгомжтой интерфэйстэй тул prototype хурдан хийхэд тохиромжтой хэрэгсэл. UI, icon, веб layout зэрэгт өргөн ашиглагддаг.

```

12. Absolute нэгжүүд

```
12.1 px (Pixels): CSS-д ашиглагддаг тогтмол хэмжээний нэгж бөгөөд нарийн хяналт хийх боломжтой. 1px нь 1/96 инчтэй тэнцүү.
12.2 in (Inch): 96px-тэй тэнцүү.
12.3 cm (Centimeters): 1 инчийн 25.2/64-тэй тэнцүү.
12.4 mm (Millimeters): 1 см-ийн 1/10-тэй тэнцүү.
12.5 q (Quarter-Millimeters): 1 см-ийн 1/40-тэй тэнцүү.
12.6 pc (Picas): 1 инчийн 1/6-тэй тэнцүү.
12.7 pt (Points): 1 инчийн 1/72-тэй тэнцүү.
```

13. Relative нэгжүүд

```
1.3.1 Percent (%): Эцэг элементтэй харьцуулсан хувь хэмжээ. Жишээ нь width: 50% бол эцэг элементийн өргөний тал болно.
13.2 em нэгж: Элементийн font-size-тэй харьцангуй. Font-size-д ашиглавал parent элементийн хэмжээнээс хамаарна.
13.3 rem нэгж: Root элемент (html)-ийн font-size-тэй харьцангуй.
13.4 vh нэгж: Viewport height буюу дэлгэцийн өндрийн 1% нь 1vh.
13.5 vw нэгж: Viewport width буюу дэлгэцийн өргөний 1% нь 1vw.
```

14. calc функц

```
14.1 calc() функц: Энэ функц нь CSS дотор шууд тооцоолол хийх боломжийг олгодог. Ингэснээр viewport хэмжээ эсвэл бусад элементүүдэд тулгуурлан уян хатан, responsive интерфэйс үүсгэж болно.
```

15. Хэрэглэгчийн үйлдэлд суурилсан pseudo-class-ууд

```
15.1 Pseudo-class тодорхойлолт: Эдгээр нь CSS-ийн тусгай түлхүүр үгс бөгөөд элементүүдийг тодорхой төлөв эсвэл байрлалаар нь сонгох боломжийг олгодог.
15.2 Хэрэглэгчийн үйлдлийн pseudo-class( User Action Pseudo-classes)-ууд: Эдгээр нь хэрэглэгчийн үйлдлээс хамааран элементүүдийн харагдах байдлыг өөрчлөхөд ашиглагддаг бөгөөд хэрэглэгчийн туршлагыг сайжруулдаг.
15.3 :active pseudo-class: Элемент идэвхтэй (жишээ нь товч дээр дарах үед) байгаа төлөвийг сонгоно.
15.4 :hover pseudo-class: Элемент дээр cursor аваачсан үеийн төлөвийг тодорхойлно.
15.5 :focus pseudo-class: Элемент focus авсан үед (ихэвчлэн keyboard ашиглах эсвэл input дээр дарах үед) стиль хэрэглэнэ.
15.6 :focus-within pseudo-class: Элемент өөрөө эсвэл түүний доторх аль нэг child элемент focus авсан үед стиль хэрэглэнэ.
```

16. Input pseudo-class-ууд

```
16.1 Input pseudo-class-ууд: Эдгээр нь HTML input элементүүдийг хэрэглэгчийн үйлдлийн өмнө болон дараах төлөвөөр нь сонгоход ашиглагддаг.
16.2 :enabled pseudo-class: Одоогоор идэвхтэй (ашиглах боломжтой) form элементүүдийг сонгоно.
16.3 :disabled pseudo-class: Идэвхгүй (ашиглах боломжгүй) элементүүдэд стиль хэрэглэнэ.
16.4 :checked pseudo-class: Сонгогдсон (checked) төлөвийг илэрхийлнэ.
16.5 :valid pseudo-class: Validation шаардлагыг хангаж буй input талбаруудыг сонгоно.
16.6 :invalid pseudo-class: Validation шаардлага хангаагүй input талбаруудыг сонгоно.
16.7 :in-range болон :out-of-range pseudo-class-ууд: Утга нь заасан хүрээнд багтаж байгаа эсэхээс хамааран элементүүдэд стиль хэрэглэнэ.
16.8 :required pseudo-class: required attribute-тэй input элементүүдийг сонгоно. Энэ нь хэрэглэгчид заавал бөглөх шаардлагатайг илтгэнэ.
16.9 :optional pseudo-class: Заавал бөглөх шаардлагагүй input элементүүдэд стиль хэрэглэнэ.
16.10 :autofill pseudo-class: Browser автоматаар бөглөсөн input талбаруудад стиль хэрэглэнэ.
```

17. Location pseudo-class-ууд

```
17.1 Location pseudo-class-ууд: Эдгээр нь холбоос болон одоогийн баримт доторх target элементүүдийг стильдэхэд ашиглагддаг.
17.2 :any-link pseudo-class: :link болон :visited pseudo-class-уудын нийлбэр бөгөөд href attribute-тэй бүх anchor элементүүдийг сонгоно.
17.3 :link pseudo-class: Зочлоогүй (unvisited) холбоосуудыг сонгоно.
17.4 :local-link pseudo-class: Одоогийн баримт руу чиглэсэн (internal) холбоосуудыг сонгоно.
17.5 :visited pseudo-class: Хэрэглэгчийн өмнө нь орж байсан холбоосуудыг сонгоно.
17.6 :target pseudo-class: URL fragment-аар чиглэгдсэн элементийн стильд хэрэглэнэ.
```

18. Tree-structure pseudo-class-ууд

```
18.1 Tree-structure pseudo-class-ууд: Эдгээр нь элементүүдийг document tree дахь байрлалаар нь сонгож стильддэг.
18.2 :root pseudo-class: Ихэвчлэн хамгийн дээд html элементийг илэрхийлнэ. Бүх баримтад нийтлэг стиль хэрэглэхэд ашиглагдана.
18.3 :empty pseudo-class: Child элементгүй (зөвхөн хоосон зай байж болно) элементүүдийг сонгоно.
18.4 :nth-child(n) pseudo-class: Parent доторх байрлалаар нь элементүүдийг сонгоно.
18.5 :nth-last-child(n) pseudo-class: Сүүлээс нь тоолж элемент сонгоно.
18.6 :first-child pseudo-class: Parent доторх эхний элементийг сонгоно.
18.7 :last-child pseudo-class: Parent доторх сүүлийн элементийг сонгоно.
18.8 :only-child pseudo-class: Parent дотор ганцхан байгаа элементийг сонгоно.
18.9 :first-of-type pseudo-class: Тодорхой төрлийн эхний элементийг сонгоно.
18.10 :last-of-type pseudo-class: Тодорхой төрлийн сүүлийн элементийг сонгоно.
18.11 :nth-of-type(n) pseudo-class: Ижил төрлийн sibling-үүдийн дундаас байрлалаар нь сонгоно.
18.12 :only-of-type pseudo-class: Тухайн төрлийн ганц элемент байвал сонгоно.
```

19. Functional pseudo-class-ууд

```
19.1 Functional pseudo-class-ууд: Илүү нарийн нөхцөл болон хамаарлаар элементүүдийг сонгох боломж олгодог. Эдгээр нь аргумент хүлээн авдгаараа онцлог.
19.2 :is() pseudo-class: Selector-уудын жагсаалтаас аль нэгэнд нь тохирох элементийг сонгоно.
19.3 :where() pseudo-class: :is()-тэй төстэй боловч specificity нь 0 байдаг.
19.4 :has() pseudo-class: "parent selector" гэж нэрлэгддэг бөгөөд тодорхой child агуулсан элементүүдийг сонгох боломж олгодог.
```

20. Pseudo-element-үүд

```
20.1 ::before pseudo-element: content property ашиглан элементээс өмнө нэмэлт (жишээ нь icon) контент оруулна.
20.2 ::after pseudo-element: content property ашиглан элементээс дараа нэмэлт контент оруулна.
20.3 ::first-letter pseudo-element: Элементийн эхний үсгийг сонгож стиль хэрэглэнэ.
20.4 ::marker pseudo-element: Жагсаалтын тэмдэг (bullet эсвэл дугаарлалт)-ийг стильдэх боломж олгодог.
```

21. Өнгөний онол

```
21.1 Өнгөний онолын тодорхойлолт: Энэ нь өнгөнүүд хоорондоо хэрхэн харилцан үйлчилдэг болон хүний хүлээн авалтад хэрхэн нөлөөлдгийг судалдаг. Үүнд өнгөний хамаарал, зохицол (harmony), сэтгэлзүйн нөлөө багтана.
21.2 Үндсэн өнгөнүүд (Primary Colors): Шар, цэнхэр, улаан өнгөнүүд бөгөөд бусад бүх өнгө эдгээрээс үүсдэг.
21.3 Хоёрдогч өнгөнүүд (Secondary Colors): Хоёр үндсэн өнгийг тэнцүү хэмжээгээр хольж гаргана. Ногоон, улбар шар, ягаан (purple) зэрэг нь жишээ.
21.4 Гуравдагч өнгөнүүд (Tertiary Colors): Үндсэн өнгө болон ойролцоох хоёрдогч өнгийг хослуулж гаргана. Шар-ногоон, цэнхэр-ногоон, цэнхэр-ягаан зэрэг орно.
21.5 Дулаан өнгөнүүд (Warm Colors): Улаан, улбар шар, шар зэрэг өнгөнүүд бөгөөд дулаан, тав тухтай мэдрэмж төрүүлдэг.
21.6 Хүйтэн өнгөнүүд (Cool Colors): Цэнхэр, ногоон, ягаан өнгөнүүд бөгөөд тайван, амгалан, мэргэжлийн мэдрэмж төрүүлдэг.
21.7 Өнгөний дугуй (Color Wheel): Өнгөнүүдийн хоорондын хамаарлыг харуулсан тойрог диаграмм. Дизайнерууд өнгөний зохицлыг сонгоход ашигладаг.
21.8 Аналог өнгөний схем (Analogous): Өнгөний дугуйн дээр зэрэгцээ байрлах өнгөнүүдээс бүрдэж, эвтэйхэн, зөөлөн мэдрэмж төрүүлдэг.
21.9 Эсрэг өнгөний схем (Complementary): Өнгөний дугуйн эсрэг талд байрлах өнгөнүүдийг ашиглаж, өндөр contrast болон хүчтэй визуал(visual) нөлөө үзүүлдэг.
21.10 Гурвалсан схем (Triadic): Өнгөний дугуйн дээр ойролцоогоор тэнцүү зайд байрлах гурван өнгөнөөс бүрддэг. Холбовол тэгш өнцөгт гурвалжин үүсгэнэ.
21.11 Нэг өнгийн схем (Monochromatic): Нэг үндсэн өнгийн тод, бараан, ханалт өөр өөр хувилбаруудаас бүрддэг. Нэгдмэл, зохицолтой мэдрэмж өгдөг.
```

22. CSS-д өнгөтэй ажиллах аргууд

```
22.1 Нэрлэсэн өнгөнүүд (Named Colors): Browser-д танигдсан бэлэн нэртэй өнгөнүүд. Жишээ нь: blue, darkred, lightgreen.
22.2 rgb() функц: RGB нь Red, Green, Blue гэсэн гэрлийн үндсэн өнгөнүүд. Эдгээрийг янз бүрийн хэмжээгээр хослуулж өнгө үүсгэнэ.
22.3 rgba() функц: rgb дээр alpha утга нэмснээр тунгалаг байдлыг (transparency) тохируулна.
22.4 hsl() функц: Hue (өнгөний төрөл), Saturation (ханалт), Lightness (гэрэл) гэсэн гурван үзүүлэлтээр өнгийг тодорхойлно.
22.5 hsla() функц: hsl дээр alpha утга нэмсэн хувилбар.
22.6 Hexadecimal: RGB өнгийг илэрхийлэх 6 тэмдэгттэй код. 16-тын тооллын систем (0–9, A–F) ашиглана.
```

23. box-shadow property

```
23.1 Тодорхойлолт: box-shadow нь элементын эргэн тойронд нэг эсвэл хэд хэдэн сүүдэр үүсгэнэ.
23.2 Offset утгууд: Хэвтээ (offset-x) болон босоо (offset-y) утгууд заавал өгнө. Эерэг x баруун тийш, сөрөг x зүүн тийш; эерэг y доош, сөрөг y дээш шилжүүлнэ. 0 утгад нэгж бичих шаардлагагүй.
23.3 Blur radius: Сүүдрийн бүдгэрэлтийн хэмжээг тодорхойлно. Өгөгдөөгүй бол 0 бөгөөд хурц ирмэгтэй байна. Ихсэх тусам илүү зөөлөн болно.
23.4 Spread radius: Сүүдрийн тархалтыг (томсох/жижгэрэх) тодорхойлно. Өгөгдөөгүй бол 0.
23.5 Сүүдрийн өнгө: Named color, hex, rgb(), rgba(), hsl(), hsla() ашиглан тодорхойлж болно.
23.6 inset түлхүүр үг: inset нэмбэл сүүдэр элементийн дотор байрлана.
23.7 Олон сүүдэр: Таслалаар салгаж олон сүүдэр давхарлаж болно. Давхарга нь урдаас хойш байрлана.
```

```
24.1 Linear gradient: Шулуун шугам дагуу өнгөнүүдийг аажмаар шилжүүлдэг. Чиглэл болон өнгөнүүдийг тохируулж болно.
24.2 Radial gradient: Төв цэгээс гадагш тархсан дугуй эсвэл эллипс хэлбэрийн gradient үүсгэнэ.
```

25. Input стийл хийх шилдэг практикууд

```
25.1 Input стийл хийх: Бусад текст элементүүдийн адил input талбарын стиль нь accessibility шаардлага хангах ёстой. Үүнд font хангалттай хэмжээтэй байх, өнгө нь background-той хангалттай contrast-тай байх шаардлагатай. Input элементүүд focus авах боломжтой тул focus үед тод харагдах indicator (жишээ нь тод border) хадгалах хэрэгтэй.
```

26. appearance: none ашиглах

```
26.1 appearance: none: Browser-ууд олон элементэд default стиль өгдөг. appearance: none property нь бүх стийлд бүрэн хяналт өгөх боловч зарим сул талтай. Custom input стиль хийх үед focus болон алдааны (error) indicator-уудыг заавал хадгалах шаардлагатай.
```

27. datetime-local болон color input стийлдэхэд тулгардаг асуудлууд

```
27.1 Түгээмэл асуудлууд: Эдгээр input төрлүүд нь date picker, color picker зэрэг функцуудыг бий болгохын тулд нарийн төвөгтэй pseudo-element ашигладаг. Энэ нь стийл хийхэд хүндрэл үүсгэдэг. Мөн default стиль нь browser бүрээс хамаарч өөр байдаг тул нэг browser-д бичсэн CSS нөгөөд нь өөрөөр харагдаж болно.
```

28. CSS Overflow property

```
28.1 Тодорхойлолт: Overflow нь элемент доторх контент тухайн элементийн хэмжээнээс хэтэрсэн үед хэрхэн харуулахыг тодорхойлно. Overflow нь хоёр чиглэлтэй.
28.2 overflow-x: Хэвтээ (x тэнхлэг) дагуух overflow-ийг тодорхойлно.
28.3 overflow-y: Босоо (y тэнхлэг) дагуух overflow-ийг тодорхойлно.
```

29. CSS Transform property

```
Тодорхойлолт: Энэ property нь элементэд эргүүлэх (rotate), хэмжээг өөрчлөх (scale), гажуудуулах (skew), шилжүүлэх (translate) зэрэг 2D болон 3D хувиргалтуудыг хийх боломж олгодог.
29.1 translate() функц: Элементийг одоогийн байрлалаас нь шилжүүлэхэд ашиглагдана.
29.2 scale() функц: Элементийн хэмжээг өөрчлөх боломж олгодог.
29.3 Accessibility ба transform: Хэрэв transform ашиглан контентыг нуух эсвэл харуулах бол screen reader болон keyboard navigation-д хандах боломж хэвээр байгаа эсэхийг шалгах хэрэгтэй. Контентыг зөв нуухын тулд display: none эсвэл visibility: hidden ашиглах хэрэгтэй, зүгээр харагдахгүй байрлал руу зөөх нь хангалтгүй.
```

30. Box model

```
Тодорхойлолт: CSS box model-д элемент бүр хайрцаг (box)-аар хүрээлэгддэг. Энэ нь content, padding, border, margin гэсэн дөрвөн хэсгээс бүрдэнэ.
30.1 Content: Хамгийн дотор хэсэг бөгөөд текст, зураг зэрэг бодит контент байрлана.
30.2 padding: Content болон border-ийн хоорондох зай.
30.3 border: Элементийн гадна хүрээ.
30.4 margin: Border-ийн гаднах зай бөгөөд бусад элементүүдтэй зайг тодорхойлно.
```

31. Margin collapse

```
Тодорхойлолт: Хоёр элементын босоо margin (top, bottom) давхцах үед илүү их утгатай margin нь үлдэж, нэг margin мэт харагддаг үзэгдэл. Энэ нь зөвхөн босоо чиглэлд үйлчилнэ.
```

32. content-box ба border-box

```
32.1 box-sizing property: Элементийн өргөн, өндрийг хэрхэн тооцоолохыг тодорхойлно.
32.2 content-box: width, height нь зөвхөн content хэсгийг илэрхийлнэ. padding, border, margin орохгүй.
32.3 border-box: width, height нь content + padding + border-г багтаана. margin орохгүй.
```

33. CSS Reset

```
Тодорхойлолт: Browser-ийн default стильүүдийг бүхэлд нь эсвэл хэсэгчлэн устгах stylesheet. Түгээмэл жишээ нь sanitize.css болон normalize.css.
```

34. CSS Filter property

```
Тодорхойлолт: Blur, өнгө өөрчлөх, contrast тохируулах зэрэг эффектүүдийг үүсгэнэ.
34.1 blur() функц: Gaussian blur эффект өгнө. Pixel-ээр тодорхойлогдоно.
34.2 brightness() функц: Гэрэлтүүлгийг тохируулна. 0% бол бүрэн хар, 100%-аас дээш бол илүү тод.
34.3 grayscale() функц: Саарал өнгө рүү шилжүүлнэ. 100% бол бүрэн grayscale.
34.4 sepia() функц: Sepia өнгөний эффект өгнө.
34.5 hue-rotate() функц: Өнгөний тойрог дагуу градусаар эргүүлж өнгийг өөрчилнө.
```

35. CSS Flexbox болон Flex Model-ийн танилцуулга

```
Тодорхойлолт: CSS flexbox нь нэг хэмжээст(one dimensional) layout model бөгөөд элементүүдийг container дотор мөр (row) эсвэл багана (column)-аар зохион байгуулах боломжийг олгодог.
35.1 Flex Model: Энэ модель нь flex item-уудыг flex container дотор хэрхэн байрлуулахыг тодорхойлдог. Flex container бүр хоёр тэнхлэгтэй: main axis болон cross axis.
```

36. flex-direction property

```
Тодорхойлолт: Энэ property нь main axis-ийн чиглэлийг тогтооно. Default утга нь row бөгөөд элементүүдийг нэг мөрөнд (зүүнээс баруун эсвэл баруунаас зүүн) байрлуулна.
36.1 flex-direction: row-reverse;: Элементүүдийн дарааллыг эсрэг чиглэлд эргүүлнэ.
36.2 flex-direction: column;: Элементүүдийг хэвтээ биш босоо чиглэлд байрлуулна.
36.3 flex-direction: column-reverse;: Элементүүдийн босоо дарааллыг урвуу болгоно.
```

37. flex-wrap property

```
Тодорхойлолт: Flex item-ууд container дотор багтахын тулд хэрхэн шинэ мөрөнд шилжихийг тодорхойлно. flex-wrap нь nowrap, wrap, wrap-reverse гэсэн утгуудтай.
37.1 flex-wrap: nowrap;: Default утга. Элементүүд container-оос хэтэрсэн ч шинэ мөр рүү шилжихгүй.
37.2 flex-wrap: wrap;: Элементүүд container-ийн өргөнийг хэтрэх үед дараагийн мөр рүү шилжинэ.
37.3 flex-wrap: wrap-reverse;: Элементүүдийг урвуу дарааллаар шинэ мөр рүү шилжүүлнэ.
37.4 flex-flow property: flex-direction болон flex-wrap-ийн товчилсон хэлбэр.
```

```
Тодорхойлолт: Энэ property нь flex container-ийн main axis дагуу child элементүүдийг хэрхэн зэрэгцүүлэхийг тодорхойлно.
38.1 justify-content: flex-start;: Элементүүдийг main axis-ийн эхлэлд байрлуулна.
38.2 justify-content: flex-end;: Элементүүдийг main axis-ийн төгсгөлд байрлуулна.
```

40. Typography-ийн танилцуулга

```
Тодорхойлолт: Typography нь текстийг харагдахуйц гоё, уншихад хялбар болгохын тулд зөв фонт болон форматыг сонгох урлаг юм. "Type" нь тэмдэгтүүдийг хэрхэн зохион байгуулж, дизайн хийснийг илэрхийлдэг.

40.1 Typeface тодорхойлолт: Typeface нь тэмдэгт, тоо, тэмдэглэгээний ерөнхий дизайн, хэв маяг юм. Энэ нь фонтуудын “загварын суурь” гэж ойлгож болно.

40.2 Font тодорхойлолт: Font нь typeface-ийн тодорхой хувилбар бөгөөд хэмжээ, жин (weight), хэв маяг (style), өргөн (width) зэрэг шинж чанартай.

40.3 Serif Typeface: Энэ төрлийн фонт нь үсгийн төгсгөлд жижиг зураастай (serif) байдаг уламжлалт хэв маягтай. Ихэвчлэн ном зэрэг хэвлэмэл материалд ашиглагддаг.

40.4 Sans Serif Typeface: Энэ нь үсгийн төгсгөлд зураасгүй, илүү орчин үеийн харагддаг. Дэлгэц дээр уншихад хялбар тул digital дизайн-д өргөн хэрэглэгддэг. Жишээ нь Helvetica, Arial, Roboto.

40.5 Font Weight: Үсгийн зузааныг илэрхийлнэ (light, regular, bold, black гэх мэт).

40.6 Font Style: Үсгийн хазайлт, хэлбэр (italic, oblique гэх мэт).

40.7 Font Width: Үсгийн хэвтээ зай (condensed, expanded гэх мэт).

40.8 Baseline: Ихэнх тэмдэгтүүд байрладаг төсөөллийн шулуун.

40.9 Cap Height: Том үсгийн өндөр (baseline-ээс дээш хэмжигдэнэ).

40.10 X-height: Жижиг үсгүүдийн дундаж өндөр (ascender, descender-ийг тооцохгүй).

40.11 Ascenders: Жижиг үсгийн дээд тал руу сунгасан хэсэг (h, b, d, f гэх мэт).

40.12 Descenders: Жижиг үсгийн доош сунгасан хэсэг (y, g, p, q гэх мэт).

40.13 Kerning: Тодорхой үсгийн хосуудын хоорондын зайг тохируулах (жишээ нь A ба V).

40.14 Tracking: Текстийн бүх үсгүүдийн хоорондын зайг тохируулах.

40.15 Leading: Мөрүүдийн хоорондох босоо зай (baseline-ээс baseline хүртэл).

40.16 Typography ашиглах шилдэг практик: Уншихад хялбар, ойлгомжтой фонт сонгох хэрэгтэй. Веб сайтын үндсэн текстэд энэ нь маш чухал. Хэрэглэгчид уншихад амархан тексттэй илүү харилцдаг. Ихэвчлэн 2–3 фонт ашиглах нь тохиромжтой. Хэт олон фонт хэрэглэвэл уншихад хэцүү болж, брэндийн танигдах байдал сулрах магадлалтай. Мөн сайт ачаалах хугацаанд сөргөөр нөлөөлж болно. Font size ашиглан visual hierarchy үүсгэх хэрэгтэй. Жишээ нь гарчиг хамгийн том, дэд гарчиг арай жижиг, текст бүр жижиг байх гэх мэт.
```

41. CSS font-family property

```
Тодорхойлолт: Font family нь ижил дизайнтай фонтуудын бүлэг юм. Нэг family доторх фонтууд нь ижил суурь typeface-тэй боловч style, weight, width-ээрээ ялгаатай байж болно. Та олон font family-г таслалаар салгаж, давуу эрхийн дарааллаар бичиж болно. Эдгээр нь fallback фонтууд болдог. Эцэст нь заавал generic font family оруулах хэрэгтэй.
41.1 Түгээмэл font family: serif, sans-serif, monospace, cursive, fantasy
```

```
Тодорхойлолт: Ихэнх төхөөрөмж дээр суусан байх магадлал өндөр фонтууд. Browser фонт харуулахдаа хэрэглэгчийн төхөөрөмж дээр байгаа эсэхийг шалгадаг. Хэрэв байхгүй бол default фонтыг ашиглана.

42.1 Web safe font-ууд:
    Sans-serif фонтууд нь дэлгэц дээр уншихад хялбар тул веб-д өргөн ашиглагддаг. Жишээ: Arial, Verdana, Trebuchet MS.
    Serif фонтууд нь төгсгөлд жижиг зураастай тул ихэвчлэн хэвлэмэл материалд ашиглагддаг. Жишээ: Times New Roman, Georgia.
```

43. At-rule болон @font-face

```
Тодорхойлолт: At-rule нь browser-д заавар өгдөг CSS мэдэгдэл юм. Media query, keyframes, font face зэрэгт ашиглагдана.
43.1 @font-face: Custom фонт тодорхойлох боломж олгодог. Үүнд фонтын файл, формат, weight, style зэргийг заана. src property заавал байх ёстой бөгөөд фонтын эх сурвалжийг заана.
43.2 Фонтын формат: Browser-д фонтын форматыг зааж өгч болно (заавал биш). Хэрэв format байхгүй бол татсаны дараа автоматаар тодорхойлогдоно. Буруу формат байвал татагдахгүй. Форматууд: collection, embedded-opentype, opentype, svg, truetype, woff, woff2.

43.4 OpenType: Microsoft болон Adobe хөгжүүлсэн scalable фонт формат бөгөөд нэмэлт боломжуудыг дэмждэг.
43.5 tech(): Фонтын технологийг заахад ашиглагддаг (заавал биш).
```

47. Accessibility tree

Accessibility tree нь screen reader зэрэг туслах технологиуд веб хуудасны контентыг ойлгож, харилцахад ашигладаг бүтэц юм.

48. max() функц

max() функц нь өгөгдсөн утгуудын дундаас хамгийн ихийг буцаана:

img {
width: max(250px, 25vw);
}

Дээрх жишээнд viewport-ийн өргөн 1000px-ээс бага бол зураг 250px өргөнтэй байна (учир нь 250px нь 25%-аас их). Харин 1000px-ээс их бол 25vw (viewport-ийн 25%) утгыг авна.

49. CSS ба accessibility-ийн шилдэг практик

```
49.1 display: none; : Энэ нь контентыг accessibility tree-ээс бүрэн хасдаг тул screen reader ашиглаж буй хэрэглэгчид харах боломжгүй болно. Иймээс бүрэн нуух шаардлагатай үед л ашиглах хэрэгтэй.
49.2 visibility: hidden; : Контентыг харагдахгүй болгодог ч document flow-д үлдээнэ. Гэхдээ accessibility tree-ээс мөн хасагдана.
49.3 .sr-only класс: Контентыг харагдахгүй болгож, screen reader-д харагдах боломжтой байлгах нийтлэг арга.
```

.sr-only {
position: absolute;
width: 1px;
height: 1px;
padding: 0;
margin: -1px;
overflow: hidden;
clip: rect(0, 0, 0, 0);
white-space: nowrap;
border: 0;
}

```
49.4 scroll-behavior: smooth;: Smooth scrolling идэвхжүүлнэ.
49.5 prefers-reduced-motion: Хэрэглэгч animation-ийг багасгах тохиргоо хийсэн эсэхийг шалгана.
```

@media (prefers-reduced-motion: no-preference) {

- {
  scroll-behavior: smooth;
  }
  }

Дээрх жишээнд хэрэглэгч animation-ийг хязгаарлаагүй тохиолдолд smooth scroll идэвхжинэ.

```

50.2 hidden: Контентыг визуал болон accessibility tree аль алинаас нь нуух attribute.
```

66. CSS Animation-ийн үндэс

```
Тодорхойлолт: CSS animation нь JavaScript эсвэл нарийн төвөгтэй програмчлал шаардалгүйгээр веб хуудсанд динамик, харагдахуйц сонирхолтой эффект үүсгэх боломжийг олгодог. Энэ нь элементүүдийг тодорхой хугацааны туршид өөр өөр стилүүдийн хооронд жигд шилжүүлэх арга юм.
@keyframes дүрэм: Animation-ийн үе шат болон стилүүдийг тодорхойлно. Animation явцад элемент ямар стильтэй байхыг зааж өгдөг.
animation property: Animation хэрэгжүүлэхэд ашиглагддаг товчилсон property.
animation-name: Ашиглах @keyframes дүрмийн нэрийг заана.
animation-duration: Animation дуусах хугацааг тодорхойлно.
animation-timing-function: Animation цаг хугацааны явцад хэрхэн өөрчлөгдөхийг тодорхойлно (жишээ нь: ease, linear, ease-in-out).
animation-delay: Animation эхлэхээс өмнөх саатлыг заана.
animation-iteration-count: Animation хэдэн удаа давтагдахыг тодорхойлно.
animation-direction: Animation урагш, хойш эсвэл ээлжлэн тоглох эсэхийг тодорхойлно.
animation-fill-mode: Animation эхлэхийн өмнө болон дууссаны дараа элемент ямар стильтэй байхыг тодорхойлно.
animation-play-state: Animation-ийг түр зогсоох, үргэлжлүүлэх боломж олгодог.
```

Accessibility ба prefers-reduced-motion Media Query

```
prefers-reduced-motion Media Query: Animation-тэй холбоотой accessibility-ийн гол асуудлын нэг нь зарим хэрэглэгчдэд таагүй мэдрэмж эсвэл бүр бие махбодын сөрөг нөлөө үзүүлэх явдал юм. Vestibular disorder эсвэл хөдөлгөөнд мэдрэг хүмүүс дэлгэц дээрх хөдөлгөөнөөс болж толгой эргэх, дотор муухайрах, толгой өвдөх зэрэг шинж тэмдэг мэдэрч болно. prefers-reduced-motion media query нь хэрэглэгч системийн түвшинд animation эсвэл хөдөлгөөнийг багасгах тохиргоо хийсэн эсэхийг илрүүлэх боломжийг хөгжүүлэгчдэд олгодог.
```
