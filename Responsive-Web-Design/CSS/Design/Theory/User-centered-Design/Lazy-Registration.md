**Deferred болон Lazy Registration гэж юу вэ?**

Lazy registration нь хэрэглэгчдэд заавал бүртгүүлэх шаардлагагүйгээр таны аппликейшнийг үзэж, түүнтэй харилцах боломж олгодог UI design pattern юм. Үүний сайн жишээ нь `e-commerce` сайт байж болно. Хэрэглэгчид бүтээгдэхүүнүүдийг чөлөөтэй үзэж, хэд хэдэн барааг сагсандаа нэмэх боломжтой байх ёстой. Харин худалдан авах сонирхолтой болсон үедээ л тэд бүртгүүлэх шаардлагатай болно.

Үүний шалтгаан нь хэрэглэгчид өөрийн мэдээллийг өгч бүртгүүлэхээс өмнө таны сайт ямар үнэ цэн санал болгож байгааг харах шаардлагатай байдаг. Аппликейшнээ зохион бүтээхдээ хэрэглэгчид тухайн аппликейшний үнэ цэнийг ойлгож, мөн өөрийн мэдээллийг өгөхөд аюулгүй гэсэн мэдрэмж төрөх боломжийг бүрдүүлэх ёстой. Үгүй бол тэд бүртгүүлэх хүсэлгүй байх бөгөөд ингэснээр та боломжит хэрэглэгчдээ алдах болно.

Та хэрэглэгчийн эмзэг мэдээлэл хамгаалагдсан, аюулгүй хадгалагдана гэдгийг тодорхой илэрхийлж мэдээлэх хэрэгтэй. Дараагийн модулиудад бид таны аппликейшнийг хэрхэн аюулгүй болгох, мөн хэрэглэгчдийн мэдээллийг хэрхэн хамгаалах талаар хэлэлцэх болно.

Lazy registration-ийн өөр нэг сайн жишээ бол YouTube юм. YouTube нь технологи, поп соёл, тоглоом зэрэг олон төрлийн сэдвийг хамарсан сая сая видеотой видео хуваалцах платформ юм. Хэрэв та YouTube-д зочилбол хэрэглэгчид нэвтрэх эсвэл бүртгүүлэх шаардлагагүйгээр хүссэн хэмжээгээрээ видео үзэж чадна. Гэвч хэрэв тэд видеонд like дарах, comment бичих эсвэл ямар нэг сувгийг subscribe хийхийг хүсвэл бүртгүүлэх шаардлагатай болно.

Хэрэв хэрэглэгч үзэж буй контентдоо дуртай болсон эсвэл хэлэлцүүлэгт оролцох хүсэлтэй байвал тэд бүртгүүлэхэд илүү бэлэн байдаг. Lazy registration нь хэрэглэгчдэд өөрийн мэдээллийг өгөхөөс өмнө таны аппликейшний үнэ цэнийг ойлгох боломж олгодог ашигтай design pattern юм.

Дараагийн удаа аппликейшн зохион бүтээхдээ хэрэглэгчийн оролцоо болон хэрэглэгчийг хадгалах түвшинг нэмэгдүүлэхийн тулд lazy registration-ийг ашиглах талаар бодож үзээрэй.
