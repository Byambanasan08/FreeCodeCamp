**Design Fundamentals Review**

1.Design Terminology**

1.1. Layout: Энэ нь мессеж дамжуулах зорилгоор хуудсан дээр эсвэл дэлгэц дээр визуал элементүүдийг хэрхэн байрлуулж зохион байгуулж байгааг хэлнэ. Эдгээр элементүүдэд текст, зураг, болон white space багтаж болно.

1.2. Alignment: Энэ нь элементүүд хоорондоо хэрхэн байрлаж байгааг илэрхийлнэ. Alignment-ийг зөв ашиглах нь дизайныг цэвэр, эмх цэгцтэй харагдуулахад тусалдаг.

1.3. Composition: Энэ нь элементүүдийг зохион байгуулж уялдаа бүхий дизайн бүтээх үйл явц юм. Энэ нь зураг, текст, дүрс зэрэг элементүүд хоорондоо хэрхэн холбогдож, дизайнд уран сайхны байдлаар хувь нэмэр оруулж байгааг тодорхойлдог.

1.3.1 Balance:  Энэ нь композици дотор визуал жин хэрхэн хуваарилагдаж байгааг хэлнэ. Дизайнерууд симметрик эсвэл асимметрик зохион байгуулалтаар тэнцвэрт байдлыг бий болгохыг зорьдог.

1.4. Scale:  Энэ нь нэг элементийн хэмжээ эсвэл хэмжээсийг өөр элементтэй харьцуулж тодорхойлохыг хэлнэ.

1.5. Hierarchy: Энэ нь дизайнд байгаа элементүүдийн ач холбогдлын дарааллыг тогтоодог. Гол зорилго нь хамгийн чухал мэдээлэл эхэлж анзаарагдах нөхцөлийг бүрдүүлэх явдал юм.

1.6. Contrast:  Энэ нь элементүүдийн хооронд тод ялгарал бий болгох үйл явц юм. Үүнийг өнгө, хэмжээ, хэлбэр, текстур эсвэл бусад визуал шинж чанарын ялгаагаар бий болгож болно. Хүчтэй contrast нь унших чадварыг сайжруулахад мөн тустай.

1.7. White Space (negative space):  Энэ нь дизайн доторх хоосон орон зай юм. Өөрөөр хэлбэл элементүүдийг хүрээлж буй сул зайг хэлнэ.

1.8. UI (User Interface):  UI нь хэрэглэгч дэлгэц дээрээ харж, харилцаж болох визуал болон интерактив элементүүдийг хэлнэ. Үүнд icon, зураг, текст, menu, link, болон button зэрэг орно.

1.9. UX (User Experience): UX нь хэрэглэгч бүтээгдэхүүн эсвэл үйлчилгээг ашиглах үедээ ямар мэдрэмж авч байгаатай холбоотой ойлголт юм. Сайн UX дизайнтай аппликейшн нь ойлгомжтой, ашиглахад хялбар, үр ашигтай, хүртээмжтэй, мөн таатай байдаг.

1.10. Design Brief: Энэ нь төслийн зорилго, зорилт, болон шаардлагуудыг тодорхойлсон баримт бичиг юм. Энэ нь дизайны процессыг чиглүүлэх замын зураг бөгөөд эцсийн бүтээгдэхүүн хэрэглэгчийн хэрэгцээг хангаж байгаа эсэхийг баталгаажуулдаг.

1.11. Vector Based Design: Энэ нь шугам, дүрс, болон өнгийг математик томьёогоор тодорхойлон дижитал дүрслэл бүтээхийг хэлнэ.

1.12. Prototyping: Энэ нь бүтээгдэхүүн эсвэл хэрэглэгчийн интерфэйсийн интерактив загвар бүтээх үйл явцыг хэлнэ.

2. UI Design Fundamentals

2.1 Good Contrast for Background and Foreground Colors: Текстийг уншихад ойлгомжтой байлгахын тулд background болон foreground өнгүүдийн хооронд сайн contrast байх нь чухал. Web Content Accessibility Guidelines (WCAG) нь энгийн текстэд хамгийн багадаа **4.5:1**, том текстэд **3:1** contrast ratio байхыг зөвлөдөг.

2.2 Good Visual Hierarchy in Design: Хүчтэй visual hierarchy нь нүд мэдээллийг ямар дарааллаар дагаж харахыг тодорхой чиглүүлдэг. Ингэснээр таны дамжуулахыг хүссэн мэдээлэл төлөвлөсөн дарааллаар хэрэглэгчид хүрдэг.

2.3 Responsive Images:  Responsive images гэдэг нь тухайн зураг харагдаж буй дэлгэцийн хэмжээнд автоматаар таарч өөрчлөгддөг зургуудыг хэлнэ. Энэ нь desktop-оос эхлээд mobile төхөөрөмж хүртэл бүх төхөөрөмж дээр зураг зөв харагдахыг баталгаажуулдаг.

2.4 Progressive Enhancement: Энэ нь ямар browser эсвэл төхөөрөмж ашиглаж байгаагаас үл хамааран бүх хэрэглэгч аппликейшний үндсэн контент болон үндсэн функцуудыг ашиглах боломжтой байхаар зохион бүтээх дизайн аргачлал юм.

2.5 User-centered Design: Энэ нь хэрэглэгчийг нэн тэргүүнд тавьдаг хандлага юм. Үүнд хэрэглэгчийн хэрэгцээ, сонголт, хязгаарлалт зэргийг анхаарч үздэг. User-centered design-ийн зорилго нь ойлгомжтой, ашиглахад үр ашигтай, мөн хэрэглэгчдэд таатай вэб хуудсыг бүтээх явдал юм.

2.6 User Research: Энэ нь таны бүтээгдэхүүнийг ашигладаг хүмүүсийг системтэйгээр судлах үйл явц юм. Үүний зорилго нь хэрэглэгчийн хэрэгцээ, зан төлөв, болон тулгарч буй хүндрэлүүдийг хэмжих явдал юм.

2.7 Exit Interviews: Энэ нь хэрэглэгч аккаунтаа цуцлах үед өгч болох судалгаа юм. Энэ нь хэрэглэгчид яагаад платформыг орхиж байгааг ойлгоход тусалж, churn-ийг бууруулахын тулд юу хийх шаардлагатайг тодорхойлоход тусалдаг.

2.8 User Testing: Энэ нь хэрэглэгчид таны аппликейшнтэй харилцаж байх үед тэднээс өгөгдөл цуглуулах практик үйл ажиллагааг хэлнэ.

2.9 A/B Testing: Энэ нь шинэ функцийг хэрэглэгчдийн санамсаргүй сонгогдсон хэсэгт нэвтрүүлж турших үйл явц юм. Дараа нь analytics өгөгдлийг ашиглан тухайн функц үр ашигтай эсэхийг тодорхойлдог.

2.10 User Requirements: Энэ нь аппликейшн дагаж мөрдөх ёстой хэрэглэгчийн түүхүүд эсвэл шалгуур үзүүлэлтүүдийг хэлнэ. User requirements нь user research, салбарын стандарт, эсвэл stakeholder-уудын санал бодлоор тодорхойлогдож болно.

2.11 Progressive Disclosure: Энэ нь хэрэглэгчийн тухайн мөчид хийж буй үйл ажиллагаанд хамаарах мэдээллийг л харуулж, бусад мэдээллийг нуух дизайн pattern юм. Үүнийг cognitive load-ийг бууруулах болон хэрэглэгчийн туршлагыг илүү ойлгомжтой болгохын тулд ашигладаг.

2.12 Deferred / Lazy Registration: Энэ нь хэрэглэгчдэд заавал бүртгүүлэх шаардлагагүйгээр аппликейшнийг үзэж, түүнтэй харилцах боломж олгодог UI design pattern юм.

3. Design Best Practices

3.1 Dark Mode: Энэ нь веб аппликейшнд байдаг тусгай боломж бөгөөд анхны light өнгийн схемийг dark өнгийн схем рүү өөрчлөх боломж олгодог. Dark mode ашиглах үед desaturated colors хэрэглэх хэрэгтэй. Desaturated colors гэдэг нь өнгийн хүчтэй байдлыг багасгасан, saturation түвшин багатай өнгүүдийг хэлнэ.

3.2 Breadcrumbs: Энэ нь хэрэглэгч тухайн сайтын бүтэц дотор яг хаана байрлаж байгааг харуулдаг navigation хэрэгсэл юм. Breadcrumbs-ийг хуудасны дээд хэсэгт байрлуулах нь хамгийн тохиромжтой бөгөөд ингэснээр хэрэглэгчид амархан олж харна. Мөн breadcrumbs нь уншихад хангалттай том байх ёстой боловч хэт том байж хуудасны их зайг эзлэх ёсгүй.

3.3 Card Component: Card component нь энгийн дизайнтай байх ёстой бөгөөд хэт их мэдээлэл харуулж, харагдах байдлыг төвөгтэй болгох ёсгүй. Media элементүүдийн хувьд хэрэглэгчийн туршлагыг сайжруулахын тулд өндөр чанартай зураг болон видеог сонгох хэрэгтэй.

3.4 Infinite Scroll: Энэ нь хэрэглэгч хуудас доош гүйлгэх үед илүү олон контентыг автоматаар ачаалдаг design pattern юм. Load more button ашиглахыг мөн бодолцох хэрэгтэй, учир нь энэ нь хэрэглэгчдэд хэзээ илүү контент харахаа өөрсдөө шийдэх боломж олгодог. Мөн хэрэглэгчид өмнөх хуудас руу буцах боломжтой байхын тулд back button нэмэх боломжтой, ингэснээр тэд бүхэлд нь дээш гүйлгэх шаардлагагүй болно.

3.5 Modal Dialog: Энэ нь одоогийн хуудасны контентын дээр гарч ирдэг pop-up төрлийн интерфэйс юм. Ихэвчлэн арын контент дээр бүдэг (dim) өнгийн давхарга тавигддаг бөгөөд ингэснээр хэрэглэгч modal доторх контент дээр илүү төвлөрөх боломжтой болдог. Мөн modal-ийг хаахын тулд хэрэглэгчид modal-ийн гадна хэсэг дээр дарах боломжийг олгох нь үргэлж зөв санаа байдаг. HTML dialog element ашиглах үед олон функц болон accessibility давуу талууд нь анхнаасаа дэмжигдсэн байдаг.

3.6 Progress Indication for Form Registration: Энэ нь хэрэглэгч ямар нэг процессын аль шатанд явж байгааг харуулах арга юм. Үүнийг form, registration, болон setup процессуудад ашиглаж болно. Дизайн нь энгийн, амархан олддог байх ёстой бөгөөд өмнөх алхмууд руу буцах боломжтой байх шаардлагатай.

3.7 Shopping Cart: Cart нь e-commerce платформ дээр хэрэглэгч аль хэдийн сонгосон бараануудыг харах боломжтой хэсэг юм. Cart нь хэрэглэгчид үргэлж харагдаж байх ёстой, cart, bag эсвэл basket гэх мэт нийтлэг icon ашиглах хэрэгтэй, мөн checkout руу шилжихэд зориулсан тодорхой call-to-action button байх ёстой.

4. Common Design Tools

4.1 Figma: Энэ нь cloud-based хэрэгсэл бөгөөд User Interface болон User Experience (UI/UX) дизайнд төвлөрдөг. Энэ нь дизайн болон хөгжүүлэлтийн багуудад хаанаас ч хамтран ажиллах боломж олгодог бөгөөд vector-based design, automatic layout, commenting болон feedback систем зэрэг дотоод боломжуудыг санал болгодог.

4.2 Sketch: Энэ нь ойлгомжтой интерфэйс болон энгийн байдлаараа алдартай дизайны хэрэгсэл бөгөөд хурдан prototype бүтээхийг хүсдэг developer-уудад тохиромжтой. Мөн дизайнерууд UI, icon, болон web layout бүтээх зэрэг ажлуудад өргөн ашигладаг.

4.3 Adobe XD: Энэ нь UI/UX дизайнд ашиглагддаг vector-based дизайн болон prototyping хэрэгсэл бөгөөд Photoshop, Illustrator, After Effects зэрэг бусад Adobe аппликейшнүүдтэй маш сайн уялдаатай ажилладгаараа танигдсан.

4.4 Canva: Энэ хэрэгсэл нь poster, cover зураг, presentation, богино видео болон бусад олон төрлийн визуал контент бүтээх боломж олгодог. Хэрэглэхэд хялбар, энгийн дизайнтай тул анхан шатны хэрэглэгчдэд маш тохиромжтой.
