Animation ашиглахтай холбоотой accessibility асуудлууд юу вэ, мөн prefers-reduced-motion хэрхэн тусалдаг вэ?

Animation нь веб сайтын харагдах байдал болон хэрэглэгчийн туршлагыг сайжруулах боломжтой. Гэхдээ энэ нь зарим хэрэглэгчдэд accessibility-ийн ноцтой асуудал үүсгэж болдог. Иймээс эдгээр асуудлыг ойлгож, веб сайтаа бүх хэрэглэгчдэд хүртээмжтэй байлгах шийдлүүдийг хэрэгжүүлэх нь чухал юм.

Animation-тэй холбоотой хамгийн гол accessibility асуудлын нэг нь зарим хэрэглэгчдэд таагүй мэдрэмж эсвэл бүр бие махбодын сөрөг нөлөө үзүүлэх явдал юм. Vestibular disorder эсвэл хөдөлгөөнд мэдрэг хүмүүс дэлгэц дээрх зарим төрлийн хөдөлгөөнийг харахад толгой эргэх, дотор муухайрах, толгой өвдөх зэрэг шинж тэмдэг илэрч болно.

Мөн animation нь танин мэдэхүйн бэрхшээлтэй эсвэл анхаарал төвлөрөлтийн эмгэгтэй хэрэглэгчдэд анхаарал сарниулах шалтгаан болдог. Ялангуяа хурдан анивчих (flashing) эсвэл строб эффект нь маш аюултай. Энэ нь photosensitive epilepsy-тэй хүмүүсийн таталтыг өдөөж болдог. Ерөнхий дүрмээр секундэд гурваас илүү удаа анивчих контентоос зайлсхийх хэрэгтэй.

Өөр нэг асуудал нь animation нь зарим хэрэглэгчдэд контент дээр төвлөрөх эсвэл уншихад хүндрэл үүсгэдэг. Ялангуяа хараа султай эсвэл уншихад бэрхшээлтэй хэрэглэгчид хөдөлж буй текст эсвэл байрлал нь өөрчлөгдөж буй layout-ийг дагаж ойлгоход хэцүү байдаг.

Эдгээр асуудлыг шийдэхийн тулд CSS нь prefers-reduced-motion media query-г санал болгодог. Энэ боломж нь хэрэглэгч системийн түвшинд animation эсвэл хөдөлгөөнийг багасгах тохиргоо хийсэн эсэхийг илрүүлэх боломжийг хөгжүүлэгчдэд олгодог.

prefers-reduced-motion-ийг дараах байдлаар ашиглаж болно:

@media (prefers-reduced-motion: reduce) {

* {
  animation-duration: 0.01ms !important;
  animation-iteration-count: 1 !important;
  transition-duration: 0.01ms !important;
  scroll-behavior: auto !important;
  }
  }

Энэ CSS код нь хөдөлгөөнийг багасгах сонголт хийсэн хэрэглэгчдэд ихэнх animation болон transition-ийг үндсэндээ идэвхгүй болгодог. Үүнийг тайлбарлавал:

@media query нь хэрэглэгч хөдөлгөөнийг багасгахыг хүссэн эсэхийг шалгана. Хэрэв тийм бол доторх стиль хэрэгжинэ.

Media query дотор бүх элементүүдийг (*) сонгон авч animation болон transition property-уудыг дарж (override) байна.

animation-duration болон transition-duration-ийг маш бага утгад (0.01ms) тохируулснаар animation бараг харагдахгүй болно. Гэхдээ бүрэн таслахгүй тул зарим функц хэвийн ажиллах боломжтой.

animation-iteration-count: 1 нь давтагддаг animation-ийг зөвхөн нэг удаа ажиллуулна.

scroll-behavior: auto нь smooth scrolling-ийг унтраана.

!important нь эдгээр дүрмүүд бусад animation style-уудаас давуу хэрэгжихийг баталгаажуулна.

Анхаарах зүйл нь энэ арга нь нийтлэг (blanket) шийдэл юм. Илүү нарийн хяналт хийхийг хүсвэл тодорхой animation-уудын хувьд тусгай reduced-motion хувилбаруудыг тодорхойлж болно.

Жишээ нь:

.animated-element {
transition: transform 0.3s ease-in-out;
}

@media (prefers-reduced-motion: reduce) {
.animated-element {
transition: none;
}
}

Энэ тохиолдолд reduced motion тохиргоо идэвхтэй үед зөвхөн тухайн элемент дээрх transition-ийг унтрааж байна. Ингэснээр хөдөлгөөнд мэдрэг хэрэглэгчдэд илүү тохиромжтой хувилбарыг санал болгож чадна.

Санах хэрэгтэй зүйл нь бүх хөдөлгөөнийг бүрэн устгах биш, харин бүх хэрэглэгчид тав тухтайгаар харилцах боломж олгох явдал юм. Зарим хөдөлгөөн нь хэрэглээний ойлгомжтой байдал болон feedback-д тустай хэвээр байж болно.

Animation хэрэгжүүлэхдээ зүгээр нэг гоёлын зориулалтаар биш, утга учиртайгаар ашиглах хэрэгтэй. Гэнэтийн том хөдөлгөөнөөс зайлсхийж, animation-ийг түр зогсоох, зогсоох, эсвэл нуух боломжийг хэрэглэгчдэд олгох хэрэгтэй. Мөн prefers-reduced-motion query-г ашиглан хөдөлгөөн багатай хувилбар санал болгох нь маш чухал.

Эдгээр accessibility асуудлуудыг харгалзан үзэж, prefers-reduced-motion зэрэг хэрэгслийг ашигласнаар та сонирхолтой animation бүхий боловч бүх хэрэглэгчдэд хүртээмжтэй веб туршлага бий болгож чадна.
