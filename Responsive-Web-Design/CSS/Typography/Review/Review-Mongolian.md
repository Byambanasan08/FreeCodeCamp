# Typography-ийн танилцуулга

**Тодорхойлолт:**
Typography нь текстийг харагдах байдлаар гоёмсог бөгөөд уншихад хялбар болгохын тулд зөв фонт болон форматыг сонгох урлаг юм. “Type” гэдэг нь тус бүрийн тэмдэгтүүд хэрхэн зохион бүтээгдэж, байрлуулж зохион байгуулагдсаныг илэрхийлдэг.

**Typeface-ийн тодорхойлолт:**
Typeface нь тэмдэгтүүд, тоонууд, болон тэмдэглэгээнүүдийн иж бүрдлийн ерөнхий дизайн болон хэв маяг юм. Энэ нь фонтын гэр бүлийн зураг төсөл (blueprint) шиг зүйл юм.

**Font-ийн тодорхойлолт:**
Font нь typeface-ийн тодорхой хувилбар бөгөөд хэмжээ, жин, хэв маяг, болон өргөн зэрэг тодорхой шинж чанаруудтай байдаг.

**Serif Typeface:**
Энэ typeface нь тэмдэгтийн төгсгөл хэсэгт жижиг шугамтай уламжлалт хэв маягтай байдаг. Serif typeface-уудыг ном зэрэг хэвлэмэл материалд өргөн хэрэглэдэг.

**Sans Serif Typeface:**
Энэ typeface нь тэмдэгтийн төгсгөл хэсэгт жижиг шугамгүй, илүү орчин үеийн харагдах байдалтай байдаг. Sans Serif typeface-уудыг дэлгэц дээр уншихад хялбар байдаг тул дижитал дизайнд өргөн ашигладаг. Жишээ нь: Helvetica, Arial, Roboto.

**Font Weight:**
Тэмдэгтийн зузаан бөгөөд light, regular, bold, black зэрэг төрлүүд орно.

**Font Style:**
Тэмдэгтийн налуу байдал болон чиглэл, жишээлбэл italic болон oblique.

**Font Width:**
Тэмдэгтүүдийн эзэлж буй хэвтээ зай, жишээлбэл condensed болон expanded.

**Baseline:**
Ихэнх тэмдэгтүүд байрладаг төсөөллийн шугам.

**Cap Height:**
Том үсгүүдийн өндөр бөгөөд baseline-ээс үсгийн дээд хэсэг хүртэл хэмжигдэнэ.

**X-height:**
Жижиг үсгүүдийн дундаж өндөр бөгөөд ascender болон descender хэсгийг оруулдаггүй.

**Ascenders:**
X-height-ээс дээш гарсан жижиг үсгийн хэсгүүд. Жишээ нь: **h, b, d, f** үсгүүдийн дээд хэсэг.

**Descenders:**
Baseline-ээс доош гарсан жижиг үсгийн хэсгүүд. Жишээ нь: **y, g, p, q** үсгүүдийн сүүл хэсэг.

**Kerning:**
Тодорхой хоёр тэмдэгтийн хоорондох зайг тохируулж уншигдах чадвар болон гоо зүйг сайжруулах арга. Жишээлбэл A болон V үсгүүдийн хоорондох зайг багасгах.

**Tracking:**
Текстийн блок доторх бүх тэмдэгтүүдийн хоорондох зайг тохируулах. Энэ нь тэмдэгтүүдийн хоорондын зай бөгөөд текст хэр нягт эсвэл задгай харагдахыг тодорхойлдог.

**Leading:**
Текстийн мөрүүдийн хоорондох босоо зай. Энэ нь нэг мөрийн baseline-ээс дараагийн мөрийн baseline хүртэл хэмжигдэнэ.

**Typography ашиглах шилдэг практикууд:**
Та дизайнаа ойлгоход хялбар болгохын тулд тодорхой бөгөөд энгийн фонтуудыг сонгох хэрэгтэй. Энэ нь ялангуяа вэбсайтын үндсэн текстэд маш чухал. Хэрэв фонт уншихад хялбар байвал хэрэглэгчид таны контенттой илүү их харилцах магадлалтай. Харааны нэгдмэл байдлыг бий болгохын тулд хамгийн ихдээ хоёр эсвэл гурван фонт ашиглах хэрэгтэй. Хэт олон фонт ашиглах нь текстийг уншихад төвөгтэй болгож, брэндийн таних чанарыг сулруулж болно. Мөн энэ нь вэбсайтын ачааллах хугацааг нэмэгдүүлж хэрэглэгчийн туршлагад нөлөөлж болно. Та гарчиг, дэд гарчиг, догол мөр зэрэг элементүүдэд харагдах шаталсан бүтцийг бий болгохын тулд фонтын хэмжээг ашиглаж болно. Жишээлбэл, вэб хуудасны үндсэн гарчиг нь илүү том фонттой байх ёстой бөгөөд дараа нь жижиг фонттой дэд гарчгууд байрлана. Ингэснээр хэрэглэгчид бүтцийг харааны хувьд ойлгоход илүү хялбар болно.

# CSS font-family Property

**Тодорхойлолт:**
Font family нь ижил дизайнтай фонтуудын бүлэг юм. Нэг гэр бүлд хамаарах бүх фонтууд нь ижил үндсэн typeface дээр суурилдаг боловч хэв маяг, жин, өргөний хувьд өөр өөр хувилбаруудтай байдаг. Та олон font family-г хамгийн өндөрөөс хамгийн бага хүртэл давуу эрхийн дарааллаар, таслалаар тусгаарлан зааж өгч болно. Эдгээр нь fallback фонтууд болж ажиллана. Font-family жагсаалтын төгсгөлд үргэлж generic font family оруулах хэрэгтэй.

```css
font-family: Arial, Lato;
```

**Түгээмэл Font Family-ууд:**

* serif
* sans-serif
* monospace
* cursive
* fantasy

# Web Safe Fonts

**Тодорхойлолт:**
Эдгээр фонтууд нь компьютер эсвэл төхөөрөмж дээр суусан байх магадлал маш өндөр байдаг фонтуудын дэд бүлэг юм. Browser фонт дүрслэх шаардлагатай үед хэрэглэгчийн систем дээр тухайн фонтын файлыг хайдаг. Хэрэв фонт олдохгүй бол системийн default фонт руу fallback хийнэ.

**Web Safe Fonts:**

Sans-serif фонтууд нь тэмдэгтийн төгсгөлд жижиг шугамгүй байдаг тул дэлгэц дээр уншихад хялбар бөгөөд вэб хөгжүүлэлтэд түгээмэл ашиглагддаг.

Жишээ:

* Arial
* Verdana
* Trebuchet MS

Serif фонтууд нь тэмдэгтийн төгсгөлд жижиг “хөл”-тэй байдаг тул уламжлалт хэвлэмэл материалд өргөн хэрэглэгддэг.

Web safe serif фонтууд:

* Times New Roman
* Georgia

# At-rules болон @font-face At-rule

**Тодорхойлолт:**
At-rules нь browser-д заавар өгдөг statement-ууд юм. Та media queries, keyframes, font faces зэрэг stylesheet-ийн янз бүрийн хэсгийг тодорхойлоход ашиглаж болно.

**@font-face:**
Энэ нь фонтын файл, формат, мөн жин болон хэв маяг зэрэг шинжүүдийг зааж өгч custom фонт тодорхойлох боломж олгодог. @font-face rule хүчинтэй байхын тулд src property-г заавал зааж өгөх хэрэгтэй. Энэ property нь фонтын resource-уудын reference-уудыг агуулдаг.

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2"),
       url("path/to/font.woff"),
       url("path/to/font.otf");
}
```

**Font Formats:**
Фонтын resource бүрийн форматыг зааж өгч болно. Энэ нь optional бөгөөд browser-д фонтын форматын талаар hint өгдөг. Хэрэв формат заагаагүй бол resource татагдсаны дараа формат тодорхойлогдоно. Хэрэв формат буруу байвал resource татагдахгүй.

Боломжит форматууд:

* collection
* embedded-opentype
* opentype
* svg
* truetype
* woff
* woff2

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
       url("path/to/font.otf") format("opentype"),
       url("path/to/font.woff") format("woff");
}
```

**WOFF болон WOFF2:**
WOFF нь “Web Open Font Format” гэсэн утгатай. Энэ нь Mozilla болон бусад байгууллагуудын хамтран боловсруулсан өргөн хэрэглэгддэг фонтын формат юм. WOFF болон WOFF2-ын ялгаа нь өгөгдлийг шахах алгоритм юм.

**OpenType:**
Microsoft болон Adobe-ийн боловсруулсан scalable компьютерийн фонтын формат бөгөөд фонтын нэмэлт боломжуудыг ашиглах боломж олгодог. Ихэнх үйлдлийн системүүд дээр өргөн хэрэглэгддэг.

**tech():**
Фонтын resource-ийн технологийг заахад ашиглагддаг. Энэ нь optional.

```css
@font-face {
  font-family: "MyCustomFont"; 
  src: url("path/to/font.woff2") format("woff2"),
       url("path/to/font.otf") format("opentype") tech(color-COLRv1),
       url("path/to/font.woff") format("woff");
}
```

# External Fonts-той ажиллах

**Тодорхойлолт:**
External font нь таны төслийн файлууд дотор шууд багтаагүй фонтын файл юм. Ихэвчлэн тусдаа сервер дээр байршуулсан байдаг. Эдгээр фонтуудыг төслдөө оруулахын тулд link элемент эсвэл CSS дотор @import ашиглаж болно.

**Google Fonts:**
Google-ийн үйлчилгээ бөгөөд вэб хөгжүүлэлтэд зориулсан олон төрлийн фонтуудын цуглуулгыг санал болгодог.

Link element ашигласан жишээ:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Roboto:ital,wght@0,100;0,300;0,400;0,500;0,700;0,900;1,100;1,300;1,400;1,500;1,700;1,900&display=swap" rel="stylesheet">
```

```css
.roboto-thin {
  font-family: "Roboto", sans-serif;
  font-weight: 100;
  font-style: normal;
}
```

@import ашигласан жишээ:

```css
@import url('https://fonts.googleapis.com/css2?family=Roboto:ital,wght@0,100;0,300;0,400;0,500;0,700;0,900;1,100;1,300;1,400;1,500;1,700;1,900&display=swap');
```

**Font Squirrel:**
Custom external фонтуудыг төслүүдэд нэмэхэд ашиглаж болох өөр нэг түгээмэл эх сурвалж.

# text-shadow Property

**Тодорхойлолт:**
Энэ property нь текст дээр сүүдэр нэмэхэд ашиглагддаг. Та X болон Y offset-ийг зааж өгөх хэрэгтэй. Эдгээр нь текстээс сүүдрийн хэвтээ болон босоо зайг илэрхийлдэг. Эдгээр утгууд заавал байх ёстой. X болон Y offset-ийн эерэг утгууд нь сүүдрийг баруун болон доош хөдөлгөнө, харин сөрөг утгууд нь зүүн болон дээш хөдөлгөнө.

```html
<link rel="stylesheet" href="styles.css">
<p>Typography Shadow</p>
```

```css
p {
  text-shadow: 3px 2px;
}
```

**Shadow Color:**
Сүүдрийн өнгийг offset-ийн өмнө эсвэл дараа зааж өгч болно. Хэрэв өнгө заагаагүй бол browser өөрөө өнгийг тодорхойлно. Ихэнх тохиолдолд өнгийг тодорхой зааж өгөх нь илүү зөв байдаг.

```css
p {
  text-shadow: 3px 2px #00ffc3;
}
```

**Blur Radius:**
Blur radius нь optional бөгөөд сүүдрийг илүү зөөлөн болгодог. Анхдагч утга нь 0. Утга их байх тусам blur ихэсч, сүүдэр илүү бүдэг болно.

```css
p {
  text-shadow: 3px 2px 3px #00ffc3;
}
```

**Олон Text Shadow ашиглах:**
Текст нь нэгээс олон сүүдэртэй байж болно. Эдгээрийг таслалаар тусгаарлаж бичнэ. Сүүдрүүд layer байдлаар, урдаас хойш хэрэглэгдэнэ.

```css
p {
  text-shadow:
    3px 2px 3px #00ffc3,
    -3px -2px 3px #0077ff,
    5px 4px 3px #dee7e5;
}
```
