### Мэргэжлийн хөгжүүлэгчид ямар төрлийн хэрэгслүүд ашигладаг вэ?

Мэргэжлийн хөгжүүлэгчид **бүтээмжээ нэмэгдүүлэх**, **кодын чанарыг сайжруулах** зорилгоор олон төрлийн хэрэгслүүдийг ашигладаг. Одоо хамгийн ойлгомжтой мэт санагддаг хэрэгслүүдийг хүртэл багтаан эдгээрийг авч үзье.

---

Эдгээр хэрэгслүүдийн хамгийн эхнийх нь **компьютер** юм. Компьютер бол хөгжүүлэлтийн **үндсэн ажлын орчин** юм. Энэ нь:

* хүчин чадал өндөртэй суурин компьютер (desktop) байж болно
* эсвэл зөөврийн компьютер (laptop) байж болно

Үйлдлийн системийн хувьд:

* **Windows**
* **macOS**
* **Linux**
  аль нэгийг нь ашигладаг.

Мэргэжлийн хөгжүүлэгчид ихэвчлэн:

* **өндөр боловсруулах чадалтай (fast processing power)**
* **их хэмжээний RAM-тай**
  компьютер сонгодог. Учир нь хөгжүүлэлтийн олон даалгавар **их нөөц шаарддаг**.

---

Компьютерийн дараагийн чухал хэрэгсэл бол тухайн компьютер дээр **код бичих, засварлах програм** юм. Үүнийг:

* **code editor**
* эсвэл **integrated development environment (IDE)**
  гэж нэрлэдэг.

---

**Visual Studio**, **IntelliJ IDEA**, **JetBrains**, **PyCharm** зэрэг IDE-үүд нь:

* код автоматаар гүйцээх (code completion)
* debugging
* integrated terminal
  зэрэг **хүчирхэг боломжуудыг** санал болгодог.

---

**Visual Studio Code (VS Code)** нь үндсэндээ code editor боловч:

* өргөн хүрээтэй **extension library**-ийн ачаар
* IDE-тэй төстэй олон боломжуудыг хангадаг.

---

Бусад түгээмэл code editor-ууд:

* **Sublime Text**
* **Atom**
* **Notepad++**
* **Brackets**

---

Code editor болон IDE ашиглан код бичих явцад хийсэн "өөрчлөлтүүдийг хянах" шаардлагатай болдог. Үүнийг хийхэд ашиглагддаг хэрэгсэл бол "version control system" юм.

Хөгжүүлэлтийн салбарт хамгийн өргөн хэрэглэгддэг version control system бол "Git" юм.

---

**GitHub**, **GitLab**, **Bitbucket** зэрэг платформууд нь:

* Git repository-үүдийг **cloud дээр байрлуулах**
* бусад хөгжүүлэгчидтэй **хамтран ажиллах**
* олон **branch** дээр зэрэг ажиллах
* кодын өөрчлөлтүүдийг **merge хийх**
  боломжийг олгодог.

---

Дараагийн чухал хэрэгсэл бол "package manager" юм. Package manager-ууд нь:

- сангууд (libraries)
- төслийн хамаарлууд (dependencies)
  нэмэх, шинэчлэх, устгах процессыг "хялбаршуулдаг".

Түгээмэл package manager-ууд:

1. JavaScript:
  - NPM
  - Yarn
  - PNPM
2. Python:
  - PIP
3. PHP:
  - Composer
4. Java:
  - Maven

---

Янз бүрийн хэрэгслээр код бичсэний дараа хөгжүүлэгчид тухайн код "зөв ажиллаж байгаа эсэхийг шалгах" хэрэгтэй болдог.

Үүний тулд:

1. Cypress
2. Playwright
3. Selenium
  зэрэг testing framework-үүдийг ашигладаг.

Мөн хэл тус бүрт зориулсан testing framework-үүд байдаг. Жишээ нь:

  1. JavaScript:
  - Jest
  2. Python:
   - pytest
  3. Java:
   - JUnit
  4. PHP:
  -PHPUnit

---

Код зөв ажиллаж байгаа эсэхийг шалгахаас гадна, хөгжүүлэгчид тухайн код "хэрэглэгчдэд харагдах байдал" ямар байгааг мөн шалгах шаардлагатай. Үүний тулд **вэб браузер** ашигладаг.

---

**Chrome**, **Firefox**, **Edge**, **Safari** зэрэг орчин үеийн браузерууд нь:

* HTML
* CSS
* JavaScript
  кодыг шалгах зориулалттай **developer tools**-тэй.

Мөн:

* debugging
* performance profiling
  хийх хэрэгслүүд ч багтсан байдаг.

---

Эдгээр хэрэгслүүд нь хөгжүүлэгчдэд:

* вэб аппликейшнээ
* өөр өөр браузер дээр
* хэрэглэгчдэд зориулан
  **шалгах, оновчлох (optimize)** боломжийг олгодог.
