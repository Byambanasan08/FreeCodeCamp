### Вэб аппликейшнд файл / фолдер зохион байгуулах шилдэг туршлагууд (Best Practices)

Вэб аппликейшн хөгжүүлэх үедээ **файл болон фолдерын зохион байгуулалт**-д анхаарах нь маш чухал. Энэ нь:

* кодыг илүү **цэгцтэй**
* **арчилж, засварлахад (maintenance) хялбар**
  болгоход тусалдаг.

Одоо HTML болон CSS-ийн энгийн төслийн **жишээ фолдер бүтэц**-ийг харцгаая.

```
.
├── /assets
│   ├── /images
│   │   ├── logo.png
│   │   ├── banner.jpg
│   │   └── icons.svg
│   ├── /fonts
│   │   ├── custom-font.woff
│   │   └── custom-font.woff2
├── /css
│   ├── main.css
│   ├── about.css
│   └── contact.css
├── index.html
├── about.html
├── contact.html
└── README.md
```

---

### Root directory (үндсэн хавтас)

Жишээний хамгийн дээр байгаа **нэг цэг (.)** нь **root directory** буюу **үндсэн хавтас**-ыг илэрхийлнэ.

Root directory гэдэг нь:

* файлын системийн **хамгийн дээд түвшний хавтас**
* бүх файл, фолдерын зам (file path) эндээс эхэлдэг
* бусад бүх хавтас, файлуудыг агуулдаг

**Directory** гэдэг нь өөрөөр хэлбэл **folder (хавтас)** гэсэн үг юм.

---

### Assets directory

Root directory доторх эхний хавтас нь **assets** юм.

**assets** гэдэг нь:

* тухайн төсөлд ашиглагдаж буй бүх **нөөц файлууд**
* жишээ нь: зураг, фонт, бусад ресурс файлууд

Энэ жишээнд assets хавтас дотор:

* **images** — зураг хадгалах
* **fonts** — фонт хадгалах
  гэсэн хоёр дэд хавтас байна.

---

### CSS directory

Дараагийн хавтас нь **css** юм.

* Заримдаа үүнийг **stylesheets directory** гэж нэрлэдэг
* Энд таны бүх **CSS файлууд** хадгалагдана

Жишээ нь:

* `main.css`
* `about.css`
* `contact.css`

---

### HTML файлууд

**css** хавтасны доор HTML файлууд байрлаж байна.

* **index.html** — вэбсайтын **нүүр хуудас (homepage)**
* **about.html** — About хуудас
* **contact.html** — Contact хуудас

---

### README.md файл

Сүүлд нь **README.md** файл байна.

README файл нь:

* төслийн талаар **товч бөгөөд хэрэгтэй мэдээлэл** өгөхөд ашиглагддаг
* жишээ нь:

  * төсөл юу хийдэг вэ
  * хэрхэн ашиглах вэ
  * бусад чухал тайлбарууд

**.md** өргөтгөл нь **Markdown** гэсэн үг бөгөөд:

* хөнгөн жинтэй markup хэл
* ихэвчлэн **баримт бичиг (documentation)** бичихэд ашиглагддаг

---

### Зохион байгуулалтын давуу тал

Төслийг:

* **assets**
* **CSS**
* **HTML**
  гэсэн бүтэцтэйгээр зохион байгуулснаар:
* хэрэгтэй файлаа хурдан олох
* бусад хөгжүүлэгчид төслийг амархан ойлгох
* хамтран ажиллахад илүү хялбар болдог

---

### Чухал тэмдэглэл

Энд үзүүлсэн бүтэц нь:

* **зөвхөн нэг жишээ**
* файл, фолдер зохион байгуулах **олон өөр арга** байдаг

Төслийн хэмжээ, технологи, багийн дүрмээс хамааран:

* өөр бүтэц сонгож болно

Гэхдээ ямар ч тохиолдолд:

* **логиктой**
* **тогтвортой**
* **ойлгомжтой**
  байх нь хамгийн чухал юм.
