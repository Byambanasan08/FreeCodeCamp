### Вэб аппликейшнд файл нэрлэх шилдэг туршлагууд (Best Practices)

Өөрийн вэб аппликейшнуудыг хийж эхлэх үедээ **файлуудаа хэрхэн нэрлэж байгаадаа** анхаарах хэрэгтэй.

Файлын нэр нь:

* **ойлгомжтой**
* **арчилж, засварлахад хялбар**
  байхаар сонгогдсон байх ёстой.

Одоо HTML файлын **сайн** болон **муу** нэршлийн жишээнүүдийг авч үзье. Эдгээр жишээнүүд дээрх **.html** өргөтгөл нь тухайн файл HTML файл гэдгийг илэрхийлж байна.

---

### Муу файл нэрийн жишээ

```html
index1234.html
```

Зөвхөн нэрийг нь харахад:

* энэ файл **юуны тухай вэ** гэдэг нь ойлгогдохгүй
* бүтээгдэхүүний хуудас уу, блог уу, эсвэл өөр төрлийн хуудас уу гэдэг нь тодорхойгүй

---

### Илүү сайн файл нэрийн жишээ

```html
about-us.html
```

**about-us.html** гэх мэт:

* илүү **тодорхой, тайлбарлагдсан** нэр ашигласнаар
* файлын зорилго шууд ойлгогдоно

Мөн анзаарах нэг зүйл:

* энд **зай (space)** ашиглаагүй
* үг хооронд **hyphen (-)** ашигласан байна

---

### `index.html` файлын тухай

Заримдаа та **index.html** гэдэг файлтай тааралдана. Энэ нь:

* вэбсайтын **үндсэн хуудас (main page)**-ийг илэрхийлдэг **тусгай файл нэр**
* хэрэглэгч вэбсайт руу ороход хамгийн түрүүнд **index.html** файл ачаалагддаг

---

### Тусгай тэмдэгт ашиглах талаар

Файл нэрлэхэд анхаарах бас нэг чухал зүйл бол **тусгай тэмдэгтүүд (special characters)** юм.

Тусгай тэмдэгт, тоо, үсгийг хэт хольж ашиглах нь:

* файлын нэрийг ойлгоход хэцүү болгодог

#### Муу жишээ

```html
file-1!@.html
```

Энэ нэр нь:

* хэрэггүйгээр хэт төвөгтэй
* файлын агуулгын талаар **ямар ч мэдээлэл өгөхгүй**

Тиймээс:

* файл нэрлэхдээ **зөвхөн үсэг (letters)** болон
* **hyphen (-)** ашиглах нь хамгийн тохиромжтой

---

### Хэл бүрийн нэршлийн конвенц (conventions)

Цаашдаа өөр програмчлалын хэлүүдийг сурч эхлэх үед:

* тухайн хэл бүрт **файл нэрлэх нийтлэг дүрэм, конвенц** байдаг гэдгийг анзаарах болно

Ийм тохиолдолд:

* тухайн хэлний **албан ёсны баримт бичиг (official documentation)**
* эсвэл найдвартай эх сурвалжуудыг
  ашиглан файл нэрлэх **шилдэг туршлагыг** судлах нь хамгийн зөв

---

### Багийн орчинд ажиллах үед

Та багийн орчинд ажиллах үед:

* тухайн багийн **тогтоосон файл нэрлэх дүрэм**-ийг дагах шаардлагатай

Иймээс:

* багийн **style guide**-ийг үргэлж шалгаж
* өөрийн файл нэршил бусад гишүүдийнхтэй **нийцтэй** байгаа эсэхийг баталгаажуулах хэрэгтэй

---

### Дүгнэлт

Зөв файл нэршил нь:

* төслийг ойлгомжтой болгох
* засвар үйлчилгээ (maintenance)-ийг хялбаршуулах
* багийн хамтын ажиллагааг сайжруулах
  чухал хүчин зүйл юм.

Тиймээс файл нэрлэхдээ **энгийн, ойлгомжтой, тогтвортой** байдлыг үргэлж баримтлаарай.
